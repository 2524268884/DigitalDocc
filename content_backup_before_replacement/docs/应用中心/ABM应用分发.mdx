## 概述

<ImageZoom
  src="/images/应用中心/ABM应用分发/overview.png"
  alt="ABM应用分发概览截图"
  width={1200}
  height={720}
/>

ABM 应用分发（Apple Business Manager 应用分发）是专为苹果设备设计的企业应用分发解决方案，通过与 Apple Business Manager 集成，使企业能够集中管理和分发 iOS 和 macOS 应用程序到员工设备。

**支持的分发方式：**
- **VPP Code 方式**：通过批量采购计划（Volume Purchase Program）购买的应用授权码分发
- **Managed ID 方式**：通过 Apple Business Manager 的托管 Apple ID 进行应用分发

<Callout type="info">
企业应用分发和 ABM 应用分发共用同一个分发下载页面，根据应用类型自动显示对应的下载选项。
</Callout>

## 前提条件

- 拥有专属集成平台管理员账号
- 拥有 Apple Business Manager 管理员账号
- 已获取 Apple Business Manager 令牌文件（.p7m 格式）
- 已购买 VPP 授权码（如需使用 VPP Code 分发方式）

## 分发下载页面配置

分发应用时，用户将收到下载链接，访问链接后将打开自定义的下载页面。

| 参数 | 说明 |
|------|------|
| 页面 Logo | 自定义应用图标 |
| 页面背景图 | 自定义背景图片 |
| 按钮颜色 | 管理员自定义颜色 |
| 标题文案 | 页面显示的标题，中英文用 `\` 分隔 |
| 底部提示文案 | 页面底部提示，中英文用 `\` 分隔 |
| 下载说明 | 页面显示的应用说明，中英文用 `\` 分隔 |
| 验证方式 | 不验证 / 提取码 / 短信 |

<Callout type="info">
下载页面支持中英文版本。移动端扫码后，根据设备和系统自动显示对应的系统应用选项。
</Callout>

## 最佳实践

**应用授权管理建议**
- **按需购买授权**：根据实际需求购买 VPP 授权码，避免过度采购
- **分组分发策略**：根据部门或角色创建分发组，精确控制应用分发范围
- **授权回收机制**：建立员工离职应用授权回收流程，确保授权有效利用
- **授权监控**：定期审查授权使用情况，及时调整采购计划

**ABM 集成最佳实践**
- **令牌定期更新**：设置令牌过期提醒，确保令牌始终有效
- **多级管理权限**：为不同管理员设置不同的 ABM 管理权限
- **同步计划**：建立定期同步计划，保持应用信息最新
- **备份配置**：定期备份 ABM 配置信息，防止意外丢失

**安全建议**
- **限制访问权限**：严格控制 ABM 令牌和授权码的访问权限
- **加密存储**：确保授权码信息安全加密存储
- **分发审核**：建立应用分发的审核流程，防止未授权分发
- **日志审计**：定期审查分发日志，发现潜在异常

## 常见问题

<Accordions>
<Accordion title="VPP 授权码导入失败">
**可能原因：**
- 授权码文件格式错误
- 授权码已过期
- 与 Apple 服务器连接失败

**解决方法：**
1. 检查授权码文件格式是否正确（应为 Apple 规定的格式）
2. 验证授权码是否在有效期内
3. 检查网络连接，确保可以访问 Apple 服务器
</Accordion>

<Accordion title="令牌上传后应用无法选择">
在 ABM 分发时，令牌虽已成功上传，但在应用列表中无法选择待分发的应用，无法识别应用名称和描述。

**解决方法：**
1. 使用浏览器访问 Apple Business Manager 并登录苹果账号
2. 导航到**偏好设置 > 组织设置**
3. 开启**自定义 APP** 功能选项
4. 进入**自定义应用**，对应用许可进行适当配置
5. 完成后重新下载令牌并上传到专属集成平台
</Accordion>

<Accordion title="ABM 令牌下载问题">
**解决方法：**
1. 通过浏览器访问 Apple Business Manager 并登录苹果账号
2. 点击页面顶部的**偏好设置**
3. 选择**付款和账单信息**选项
4. 在该页面找到下载令牌的选项

下载的令牌文件（.p7m 格式）即可用于 ABM 应用分发功能的配置。
</Accordion>

<Accordion title="应用无法推送到设备">
**可能原因：**
- 设备未注册到 ABM
- MDM 配置文件安装失败
- 应用与设备不兼容
- 没有足够的授权码

**解决方法：**
1. 确认设备已正确注册到 ABM
2. 重新安装 MDM 配置文件
3. 检查应用与设备的兼容性
4. 确认有足够的授权码可用于分发
</Accordion>

<Accordion title="用户无法下载或安装应用">
**解决方法：**
1. 确认用户的 Apple ID 是否正确配置
2. 验证用户设备是否符合应用的系统要求
3. 检查用户网络连接是否正常
4. 确认授权码是否已被正确分配给用户
5. 指导用户在设备上信任企业开发者证书（如适用）
</Accordion>
</Accordions>
