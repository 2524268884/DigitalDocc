接口调用权限配置是专属集成平台的核心安全功能，支持对下游系统（M2M 调用）进行精细化的字段级权限控制。管理员可以针对用户、部门、角色等主体，配置不同资源类型（如用户、部门）的可读/可写字段清单，确保接口调用时仅返回或允许操作被授权的字段信息，提升数据安全合规性。

该功能主要解决企业级应用中不同下游系统对数据访问权限的差异化需求，避免敏感信息泄露，同时支持灵活的权限策略配置和管理。

## 适用场景

- **数据隔离**：不同下游系统只能访问其职责范围内的字段
- **敏感信息保护**：如工资、身份证号等敏感字段仅对授权系统开放
- **权限最小化**：遵循最小权限原则，仅授予必要的字段访问权限
- **合规审计**：满足数据安全合规要求，记录所有字段访问

## 界面概览

接口调用权限配置界面位于**应用配置模块**中，通过**「M2M 字段权限」**标签页提供访问。界面采用分层设计：

- **策略列表区域**：展示当前配置的所有权限策略
- **操作按钮区**：提供新增、编辑、删除策略的快捷入口
- **配置面板**：支持资源类型、操作类型、字段选择的组合配置

## 详细操作

### 新增字段权限策略

创建新的权限策略，用于为指定资源类型配置字段级访问权限。

- 进入**应用配置页面**，点击**「M2M字段权限」**标签页
1. 点击**「添加策略」**按钮，弹出配置对话框
- 在**「选择资源」**下拉列表中选择目标资源类型（用户、部门）
1. 在**「选择操作类型」**中选择操作类型：
   - **查询**：允许下游系统查询该资源类型的记录
   - **新增**：允许下游系统创建该资源类型的新记录
   - **修改**：允许下游系统修改该资源类型的现有记录
1. 在**「选择允许的字段」**多选框中勾选可操作的字段：
   - 若选择**「查询」**操作：显示该资源类型在属性管理中的全部字段
   - 若选择**「新增」**操作：显示非必填字段，必填字段默认选中且不可删除
   - 若选择**「修改」**操作：显示非必填字段，必填字段默认选中且不可删除
1. 点击**「保存」**按钮完成策略创建

### 编辑字段权限策略

修改已有策略的配置：

- 在策略列表中找到需要修改的策略
1. 点击该策略的**「编辑」**按钮
- 修改资源类型、操作类型或字段选择
1. 点击**「保存」**按钮确认修改

### 删除字段权限策略

删除不再需要的策略：

- 在策略列表中找到需要删除的策略
1. 点击该策略的**「删除」**按钮
- 在确认对话框中点击**「确定」**完成删除

**注意**：删除策略后，下游系统将恢复对该资源类型全部字段的默认访问权限。

## 字段权限生效后的 API 行为

### 查询操作

当字段权限策略配置了**查询**操作的允许字段时：

| 场景 | API 返回结果 |
|------|-------------|
| 请求字段在允许列表中 | 正常返回字段值 |
| 请求字段不在允许列表中 | 该字段不返回（而非报错） |
| 请求所有字段 | 仅返回已授权的字段 |

**示例**：配置了用户资源的查询权限，仅允许访问 `username`、`email` 字段。

```json
// API 请求：GET /api/users/123
// API 响应：
{
  "id": "123",
  "username": "zhangsan",
  "email": "zhangsan@company.com"
  // phone_number, address 等未授权字段不会返回
}
```

### 新增操作

当字段权限策略配置了**新增**操作的允许字段时：

| 场景 | API 行为 |
|------|---------|
| 提交字段在允许列表中 | 正常处理 |
| 提交字段不在允许列表中 | 字段被忽略，不写入 |
| 提交必填字段但不在允许列表中 | 返回错误：缺少必填字段 |

**示例**：配置了用户资源的新增权限，仅允许写入 `username`、`email`。

```json
// API 请求：POST /api/users
{
  "username": "newuser",
  "email": "newuser@company.com",
  "phone_number": "13800138000"  // 未授权，将被忽略
}
// API 响应：成功创建用户，但 phone_number 未被写入
```

### 修改操作

当字段权限策略配置了**修改**操作的允许字段时：

| 场景 | API 行为 |
|------|---------|
| 修改字段在允许列表中 | 正常更新 |
| 修改字段不在允许列表中 | 字段保持不变（静默忽略） |
| 尝试修改只读字段 | 返回错误：字段不可修改 |

## 注意事项

- **必填字段**：必填字段在新增和修改操作中不可移除，如需调整请在**属性管理**中修改字段属性
- **字段搜索**：字段选择支持搜索功能，可在下拉框顶部输入字段名称快速定位
- **审计记录**：所有策略配置操作将被审计记录，包括修改人和操作时间
- **权限叠加**：同一资源类型可以配置多条策略，权限取并集

## 常见问题

**Q1：为什么查询返回的字段比预期的少？**

A1：可能原因：
- 配置了查询字段权限策略，但请求的字段不在允许列表中
- 检查策略是否正确配置并启用

**Q2：新增数据时某些字段没有写入成功？**

A2：可能原因：
- 该字段未被配置在新增操作的允许字段列表中
- 检查字段权限策略配置

**Q3：如何快速查找需要配置权限的字段？**

A3：使用字段选择框的搜索功能，输入字段名称关键词即可快速定位。

**Q4：删除策略后会有影响吗？**

A4：删除策略后，下游系统将恢复对该资源类型全部字段的默认访问权限。请确认不再需要该策略后再删除。
