## 概述

<ImageZoom
  src="/images/应用中心/应用密码代填/overview.png"
  alt="应用密码代填概览截图"
  width={1200}
  height={720}
/>

应用密码代填功能允许用户通过专属集成平台访问不支持标准单点登录的传统应用，实现一键自动填充账号密码并登录。

**适用场景：**
- 传统系统集成：无法改造支持标准 SSO 协议的旧系统
- 第三方应用接入：外部供应商应用不提供单点登录接口
- 临时性应用：短期使用的系统，无需复杂 SSO 配置
- 混合身份认证环境：同时存在多种认证机制的企业 IT 环境

## 实现原理

密码代填功能通过浏览器插件与平台配置的账密信息协同工作：

1. 用户点击平台中已配置密码代填的应用
2. 系统自动跳转到目标应用的登录页面
3. 插件自动识别登录表单并填充预设的账号密码
4. 插件触发提交操作，完成自动登录
5. 用户直接进入应用系统，无需手动输入凭证

## 前提条件

- 用户浏览器必须是 **Chrome 浏览器**
- 已离线安装**密码代填插件**
- 管理员已在后台完成密码代填应用配置
- 用户启用插件（插件安装后默认开启）

### 插件安装指南

**安装插件：**

1. 打开 Chrome 浏览器 → **管理扩展程序**
2. 点击**加载已解压的扩展程序**
3. 选择插件存放的目录

**卸载插件：**
在扩展程序页面点击**移除**，确认后即可卸载。

## 特殊环境配置

### 飞书端内跳转到本机浏览器

如需在飞书端内打开应用并使用密码代填：

1. 修改默认浏览器为 Chrome
2. 在 Portal 中点击**独立窗口打开**

### 多浏览器环境

密码代填插件目前仅支持 Chrome 浏览器。在多浏览器环境中，建议：

- 将 Chrome 设置为处理密码代填应用的默认浏览器
- 为用户提供明确的浏览器使用指南
- 考虑将关键应用升级为支持标准 SSO 协议的版本

## 安全最佳实践

- **定期更新密码**：管理员应定期更新密码代填应用的账号密码
- **最小化授权范围**：仅向必要的用户授予密码代填应用的访问权限
- **启用多因子认证**：为平台账号配置 MFA，增强安全性
- **审计日志监控**：定期检查应用访问日志，发现异常访问行为
- **敏感应用特殊处理**：对于敏感系统，实施更严格的访问控制
- **密码复杂度策略**：使用高复杂度密码，用户无需记忆
- **定期安全审查**：对密码代填配置进行定期安全审查
- **插件更新管理**：及时更新密码代填插件到最新版本
- **浏览器隔离**：考虑为密码代填功能使用专用的浏览器配置文件

## 常见问题

<Accordions>
<Accordion title="插件无法自动填充密码">
**可能原因：**
- 插件未正确安装或未启用
- 应用登录页面结构与配置不匹配
- 浏览器阻止了自动填充行为
- 目标网站使用了防自动填充技术

**解决方法：**
1. 检查插件是否正常安装并启用
2. 验证应用配置中的 HTML 元素 ID 或名称是否正确
3. 确认使用的是最新版本的插件
4. 使用浏览器开发者工具查看元素 ID 并更新配置
</Accordion>

<Accordion title="跳转后显示空白页面">
**可能原因：**
- 应用 URL 配置错误
- 网络连接问题
- 目标站点存在跨域限制

**解决方法：**
1. 检查应用配置中的登录地址是否正确
2. 测试网络连接，确保能够正常访问目标应用
3. 确认目标应用是否有 IP 访问限制
4. 检查目标站点的安全策略是否阻止了 iframe 加载
</Accordion>

<Accordion title="无法在移动设备上使用密码代填">
密码代填功能目前仅支持 Chrome 浏览器，不支持移动设备。如需在移动设备上访问应用，建议：
1. 使用支持标准单点登录协议的应用
2. 通过企业应用分发功能部署移动应用
3. 使用响应式 Web 应用替代原生移动应用
</Accordion>

<Accordion title="登录成功但应用内部操作失败">
**可能原因：**
- 账号权限不足
- 会话状态问题
- 应用内部逻辑限制

**解决方法：**
1. 确认账号在目标系统中拥有足够权限
2. 检查应用是否有额外的身份验证步骤
3. 联系应用管理员检查账号状态
</Accordion>
</Accordions>
