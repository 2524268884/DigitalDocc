## 概述

<ImageZoom
  src="/images/应用中心/单点登录配置/overview.png"

  width={1200}
  height={720}

单点登录（SSO）是专属集成平台的核心功能，允许用户通过一次身份验证访问多个应用系统，提升用户体验和安全性。

**主要特点：**
- **统一认证入口**：所有应用共享统一的登录界面
- **多协议支持**：支持 SAML 2.0、OIDC、OAuth2.0、CAS 等主流协议
- **统一会话管理**：集中管理用户会话状态和超时策略
- **跨域访问支持**：解决不同域名应用间的认证问题

### SSO 工作原理

**标准模式：**

```
用户 → 访问应用 → 应用重定向到认证中心 → 用户认证
     ← 认证中心返回令牌 ← 应用验证令牌 ← 授权访问
```

**代理模式（适用于无法改造的传统应用）：**

1. 用户访问应用网关
2. 网关重定向到认证中心完成身份验证
3. 网关代理用户向后端应用发起请求（支持表单登录或 HTTP 头传递身份信息）

## 前提条件

- 拥有专属集成平台管理员账号
- 已在应用中心完成应用的基础创建
- 了解目标应用支持的 SSO 协议类型
- 生产环境必须配置 HTTPS

<ImageZoom
  src="/images/应用中心/单点登录配置/step-02.png"
  alt="选择认证协议截图"
  width={1200}

根据应用类型选择合适的协议：

| 协议 | 适用场景 |
|------|---------|
| **SAML 2.0** | 传统企业应用，支持跨域 SSO |
| **OIDC** | 现代 Web 应用，提供丰富用户信息 |
| **OAuth 2.0** | API 授权场景 |
| **CAS** | 高校、科研机构及内部简单应用 |
| **表单代理** | 无法改造的传统表单登录应用 |
| **标准 HTTP 代理** | 无法改造的 HTTP 应用 |
</Step>

### 步骤 3：配置协议参数

<ImageZoom

<Tabs items={['SAML 2.0', 'OIDC', 'OAuth2', 'CAS']}>
<Tab value="SAML 2.0">
| 参数 | 说明 |
|------|------|
| 服务提供者 ID | 应用的唯一标识符（来自 SP 元数据） |
| ACS URL | 断言消费服务 URL（来自 SP 元数据） |
| 断言加密 | 是否加密 SAML 断言 |
| 签名算法 | 推荐选择 RSA-SHA256 |
| 摘要算法 | 推荐选择 SHA-256 |

配置完成后，下载 **IdP Metadata 文件**，导入到应用的 SAML 配置中。

📖 详细配置参见：[单点登录-SAML 协议](/docs/应用中心/应用单点登录/单点登录-SAML协议)
</Tab>

<Tab value="OIDC">
| 参数 | 说明 |
|------|------|
| 客户端 ID | 系统自动生成，不可编辑 |
| 客户端密钥 | 点击"生成密钥"获取 |
| 重定向 URI | 认证完成后的跳转地址（需与应用回调地址一致） |
| 授权类型 | 推荐选择 `authorization_code` |
| 访问令牌有效期 | 建议 1 小时 |
| 刷新令牌有效期 | 建议 7 天 |
| ID 令牌签名算法 | 推荐 RS256 |

📖 详细配置参见：[单点登录-OIDC](/docs/应用中心/应用单点登录/单点登录-OIDC)
</Tab>

<Tab value="OAuth2">
参考 OIDC 配置 Client ID 和密钥，并根据应用需求配置授权范围（Scope）。

📖 详细配置参见：[单点登录-OAuth2](/docs/应用中心/应用单点登录/单点登录-OAuth2)
</Tab>

<Tab value="CAS">
| 参数 | 说明 |
|------|------|
| Login URL | 应用的登录回调地址 |
| Logout URL | 应用的登出地址 |
| Ver2.2 Validate URL | CAS 2.2 版本的票据验证 URL |
| Ver3.0 Validate URL | CAS 3.0 版本的票据验证 URL（支持用户属性返回） |
| Service URL | 目标应用提供的服务 URL |

📖 详细配置参见：[单点登录-CAS 协议](/docs/应用中心/应用单点登录/单点登录-CAS协议)
</Tab>

## 高级配置

### 会话管理

| 参数 | 说明 | 建议值 |
|------|------|--------|
| 会话超时时间 | 用户无操作后自动退出 | 30 分钟 |
| 最大会话数 | 每个用户允许的并发会话数 | 5 |
| 会话续期 | 是否允许活跃用户自动续期 | 开启 |
| 强制重新认证 | 是否定期要求重新认证 | 按需 |

### 多因子认证集成

在**多因子认证**选项卡中可启用 MFA，支持以下触发策略：

- **始终要求**：每次登录都需要 MFA
- **风险感知**：根据登录风险动态决定
- **特定用户**：仅对高权限用户要求 MFA

## 验证

配置完成后，可通过以下方式验证：

1. 以普通用户身份访问目标应用
2. 确认页面正确跳转到专属集成平台登录页
3. 完成认证后，确认自动跳回目标应用并已登录
4. 在**审计日志 > 用户行为日志**中查看 SSO 登录记录

## 常见问题

<Accordions>
<Accordion title="用户无法通过 SSO 登录">
1. 检查用户是否有访问应用的权限（授权范围配置）
2. 验证 SSO 配置参数是否正确
3. 清除浏览器 Cookie 和缓存后重试
4. 查看**审计日志 > 用户行为日志**中的错误信息
</Accordion>

<Accordion title="SAML 断言验证失败">
- 检查 IdP 和 SP 的服务器时钟是否同步（误差需 < 5 分钟）
- 验证签名证书是否在有效期内
- 确认签名算法配置在双方一致
- 使用 SAML-tracer 浏览器插件抓包分析断言内容
</Accordion>

<Accordion title="OIDC 重定向 URI 错误">
- 检查应用中配置的重定向 URI 是否与实际跳转地址完全一致（包括末尾斜杠）
- 确认 URI 已添加到专属集成平台应用的**允许重定向列表**中
</Accordion>

<Accordion title="登录后出现「未授权」错误">
**可能原因：**
- 用户属性映射配置错误，SP 收到的 NameID 与系统中不一致
- 用户未被授权访问该应用
- 令牌验证失败（签名算法或公钥不匹配）
- 会话已过期

按以上原因逐一排查，重点检查属性映射配置和授权范围。
</Accordion>
</Accordions>
