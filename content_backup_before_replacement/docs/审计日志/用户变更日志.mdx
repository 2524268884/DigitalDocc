## 概述

<ImageZoom
  src="/images/审计日志/用户变更日志/overview.png"
  alt="用户变更日志概览截图"
  width={1200}
  height={720}
/>

用户变更日志模块用于记录和展示专属集成平台中每位用户的变更历史，包括新增、修改、删除操作。管理员可通过该模块快速定位用户信息变更的来源和详情，支持筛选、排序和导出功能，便于审计和问题排查。

<Callout type="info">
日志默认保存最近 **180 天**，具体保存时间可在系统设置中查看或修改。
</Callout>

## 界面说明

日志列表展示以下字段：

| 字段 | 说明 |
|-----|------|
| 用户 ID | 用户的唯一标识 |
| 用户名 | 用户登录账号 |
| 变更类型 | 新增 / 修改 / 删除 |
| 变更状态 | 成功 / 失败 |
| 操作者 | 执行变更的管理员用户名或「通讯录集成」 |
| 来源 | 后台操作、OPEN API 或集成名称 |
| 变更时间 | 变更操作执行的时间 |

## 操作步骤

### 步骤 1：进入用户变更日志页面

{/* DOC_IMAGE src="/images/审计日志/用户变更日志/step-01.png" alt="进入用户变更日志页面截图" width="1200" height="720" */}

在左侧导航栏选择 **审计日志** → **用户变更日志**。

### 步骤 2：查看日志列表

{/* DOC_IMAGE src="/images/审计日志/用户变更日志/step-02.png" alt="查看日志列表截图" width="1200" height="720" */}

默认显示最近一周的变更记录，点击行左侧展开按钮可查看详情：

- **新增成功**：显示全部非空用户属性（字段名-字段值，带标签）
- **修改成功**：以表格展示修改字段及前后值
- **删除成功**：显示删除类型（如直接删除）
- **失败**：显示失败原因

### 步骤 3：筛选日志

{/* DOC_IMAGE src="/images/审计日志/用户变更日志/step-03.png" alt="筛选日志截图" width="1200" height="720" */}

1. 在页面顶部使用日期范围筛选器选择变更时间范围
2. 点击「高级筛选」按钮，弹出配置弹窗
3. 选择筛选条件：
   - **更新来源**：全部、后台操作、通讯录集成名称（包括已删除的）
   - **操作状态**：全部、新增、修改、删除
   - **用户名**：输入用户名
   - **姓名**：输入姓名（关联当前用户名）
   - **手机号**：输入手机号
   - **邮箱**：输入邮箱
4. 点击「确定」应用筛选；点击「重置」清除条件

<Callout type="warn">
高级筛选按钮在设置条件后将高亮显示。
</Callout>

### 步骤 4：排序日志

{/* DOC_IMAGE src="/images/审计日志/用户变更日志/step-04.png" alt="排序日志截图" width="1200" height="720" */}

点击列表头部「最后更新时间」列，可进行升序/降序排序。

### 步骤 5：导出日志

{/* DOC_IMAGE src="/images/审计日志/用户变更日志/step-05.png" alt="导出日志截图" width="1200" height="720" */}

- 若无筛选条件，点击「导出全部数据」按钮
- 若有筛选条件，点击「导出筛选数据（总数）」按钮（限制 2 万行）

系统将生成 Excel 文件下载，文件格式为 XLSX。

## 常见问题

<Accordions>
<Accordion title="操作者和来源字段有什么区别？">

操作者包括管理员用户名或「通讯录集成」；来源包括后台操作、OPEN API 或集成名称。
</Accordion>
<Accordion title="导出的 Excel 文件包含哪些字段？">

导出文件包含用户 ID、变更类型、时间等字段。
</Accordion>
</Accordions>
