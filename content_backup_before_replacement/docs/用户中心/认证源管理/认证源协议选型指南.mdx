## 概述

专属集成平台支持多种认证源接入协议。本文档通过对比分析帮助您快速确定适合自身场景的认证源协议。

## 快速选型表

| 场景 | 推荐协议 | 理由 |
|------|---------|------|
| 已有企业 Windows AD | **AD** | 原生支持，配置最简单 |
| 已有 LDAP 目录服务 | **LDAP** | 通用协议，兼容性最强 |
| 已在用钉钉/飞书/企微 | **钉钉/飞书/企微认证** | 直接扫码免密登录 |
| 微软云（Azure AD/M365） | **Azure AD / OIDC** | OIDC 是 Microsoft 推荐方式 |
| Google Workspace | **Google Workspace** | OAuth2/OIDC 基础 |
| 对接现代 SaaS 应用 | **OIDC** | 行业现代标准，Token 机制更安全 |
| 对接传统企业应用 | **SAML** | 企业级 SSO 老标准，兼容性最强 |
| 对接旧系统/高校系统 | **CAS** | 老版 SSO 系统的事实标准 |
| 微信/公众号登录 | **微信认证** | 官方 OAuth2 接口 |

## 协议详细对比

### LDAP / AD

| 属性 | 说明 |
|------|------|
| **适用场景** | 企业内网目录服务（Windows AD 域、OpenLDAP）|
| **认证机制** | 使用目录服务账号密码验证 |
| **部署要求** | 集成平台需能访问 LDAP/AD 服务器（389/636端口）|
| **优势** | 企业内网成熟稳定，属性丰富，支持 JIT 自动建号 |
| **劣势** | 不适合外网用户，需代理网关打通内网 |
| **参考文档** | [认证源-AD](./认证源-AD)、[认证源-LDAP](./认证源-LDAP) |

### OIDC（OpenID Connect）

| 属性 | 说明 |
|------|------|
| **适用场景** | 对接现代身份提供商（Azure AD、Google、企业自建 IdP）|
| **认证机制** | OAuth2 授权码流 + ID Token（JWT）|
| **部署要求** | 需要 IdP 支持 OIDC 端点（`/.well-known/openid-configuration`）|
| **优势** | 行业现代标准，Token 轻量，移动端友好，支持刷新令牌 |
| **劣势** | 部分老系统不支持 |
| **参考文档** | [认证源-OIDC协议](./认证源-OIDC协议) |

### OAuth2

| 属性 | 说明 |
|------|------|
| **适用场景** | 社交登录（微信、第三方平台）、API 授权 |
| **认证机制** | 授权码模式 / 隐式模式 / 客户端凭证模式 |
| **部署要求** | 需要第三方平台开放 OAuth2 授权接口 |
| **优势** | 适合开放平台授权，第三方应用接入标准方式 |
| **劣势** | 标准 OAuth2 不含身份声明，需额外获取用户信息接口 |
| **参考文档** | [认证源-OAuth2协议](./认证源-OAuth2协议) |

### SAML 2.0

| 属性 | 说明 |
|------|------|
| **适用场景** | 对接企业级传统应用（SAP、Salesforce、老版 OA）|
| **认证机制** | XML 断言，SP 发起或 IdP 发起两种流 |
| **部署要求** | 需要 SP 提供 SAML Metadata，配置 ACS URL |
| **优势** | 企业级兼容性最强，大量遗留系统支持 |
| **劣势** | XML 格式复杂，调试门槛高，移动端支持弱 |
| **备注** | 专属集成平台作为 IdP 支持 SAML SSO；作为认证源（SP）敬请关注后续版本 |

### CAS

| 属性 | 说明 |
|------|------|
| **适用场景** | 高校系统、老版企业 SSO（Apereo CAS）|
| **认证机制** | 基于 Ticket（ST/TGT）的票据机制 |
| **部署要求** | 需要 CAS 服务器地址和版本信息 |
| **优势** | 高校和政府系统广泛使用 |
| **劣势** | 非主流现代标准，新项目不推荐 |
| **参考文档** | [认证源-CAS协议](./认证源-CAS协议) |

### 钉钉 / 飞书 / 企业微信

| 属性 | 说明 |
|------|------|
| **适用场景** | 企业已全面使用钉钉/飞书/企微作为协作工具 |
| **认证机制** | 移动端扫码 / SSO 免密 |
| **部署要求** | 需在对应开放平台创建应用并获取 AppKey/AppSecret |
| **优势** | 零输入密码，用户体验最佳，自带组织绑定 |
| **劣势** | 依赖移动端 App，PC 端有限制 |
| **参考文档** | [认证源-钉钉](./认证源-钉钉)、[认证源-飞书](./认证源-飞书)、[认证源-企业微信](./认证源-企业微信) |

## 决策流程图

```
您的认证来源是？
│
├─ 企业内网目录服务
│   ├─ Windows Server AD 域  → AD 认证源
│   └─ OpenLDAP 等           → LDAP 认证源
│
├─ 微软云
│   ├─ Azure Active Directory → Azure AD 认证源（OIDC）
│   └─ Microsoft 365          → Azure AD 认证源
│
├─ 谷歌云
│   └─ Google Workspace       → Google Workspace 认证源
│
├─ 中国主流协作工具
│   ├─ 钉钉                  → 钉钉认证源
│   ├─ 飞书                  → 飞书认证源
│   └─ 企业微信              → 企业微信认证源
│
├─ 微信生态
│   ├─ 微信（个人/企业）     → 微信认证源
│   └─ 微信公众号            → 微信公众号认证源
│
├─ 标准协议对接
│   ├─ 现代应用（建议新项目） → OIDC 认证源
│   ├─ 传统企业应用           → SAML（作为 SP，待支持）
│   ├─ 高校/旧版 SSO         → CAS 认证源
│   └─ 开放平台授权          → OAuth2 认证源
│
└─ 其他行业系统
    └─ 美云智数               → 美云智数认证源
```

## 多认证源优先级说明

专属集成平台支持同时配置多个认证源。登录时系统按以下逻辑处理：

1. 用户选择认证方式（登录页展示已启用的认证源入口）
2. 用户通过选定的认证源完成身份验证
3. 系统根据账号关联映射规则，匹配集成平台内的用户账号
4. 若账号不存在且启用了 JIT，则自动创建账号

> **注意**：建议至少保留一个「平台账密」登录方式，防止认证源异常时管理员无法登录。
