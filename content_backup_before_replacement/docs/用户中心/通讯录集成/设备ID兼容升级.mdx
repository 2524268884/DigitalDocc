---
title: 钉钉 Device ID 可信设备识别升级
description: 在钉钉客户端内使用钉钉原生 Device ID 替代浏览器指纹进行可信设备识别，提升设备识别准确性，避免跨设备误判导致 MFA 被错误豁免。
---

## 功能概述

专属集成平台在钉钉客户端环境下，现已将可信设备识别方式从**浏览器指纹**升级为**钉钉原生 Device ID**。Device ID 绑定于具体设备硬件，在钉钉 WebView 中保持稳定，不受浏览器缓存、隐私模式等因素影响，是更可靠的设备识别依据。

**升级解决的问题：**

| 问题 | 原因 | 升级后 |
|------|------|--------|
| 不同钉钉设备被识别为同一设备，MFA 被错误豁免 | 钉钉 WebView 与系统浏览器生成相同指纹 | 使用设备硬件级 Device ID，不同设备唯一区分 |
| 清除浏览器缓存后可信设备失效 | 指纹依赖本地存储 | Device ID 不依赖缓存，持久稳定 |

## 升级影响说明

<Callout type="info">
本次升级为**无感知自动迁移**，存量已绑定可信设备的用户无需重新绑定。系统在用户下次通过钉钉客户端登录时，自动完成 Device ID 与浏览器指纹的双标识兼容过渡。
</Callout>

**迁移过渡策略：**

| 场景 | 处理方式 |
|------|---------|
| 用户通过钉钉客户端登录，历史可信设备为浏览器指纹 | 系统自动将 Device ID 关联至该可信设备记录，用户无需重新验证 MFA |
| 用户通过普通浏览器登录 | 继续使用浏览器指纹识别，不受影响 |
| 新设备首次登录（钉钉客户端）| 直接使用 Device ID 注册为可信设备 |

## 管理员操作

本次升级无需管理员进行任何配置操作，系统自动完成识别方式切换。

如需查看用户的可信设备列表（包含 Device ID 和浏览器指纹两种标识类型），可在 **用户中心 > 用户管理 > 用户详情 > 可信设备** 中查看：

| 字段 | 说明 |
|------|------|
| 设备标识类型 | `device_id`（钉钉）/ `browser_fingerprint`（浏览器）|
| 设备标识值 | 设备的唯一标识符（已脱敏显示）|
| 绑定时间 | 该设备首次被识别为可信设备的时间 |
| 最近使用时间 | 该设备最近一次通过可信设备验证的时间 |

## 常见问题

<Accordions>
<Accordion title="钉钉 Device ID 升级后，用户在钉钉端需要重新做 MFA 验证吗？">
不需要。系统自动完成 Device ID 与历史可信设备的关联，存量用户在可信设备有效期内无需重新验证 MFA。
</Accordion>
<Accordion title="用户同时在钉钉客户端和普通浏览器登录，两端的可信设备是独立计算的吗？">
是的。钉钉端使用 Device ID 识别设备，浏览器端使用浏览器指纹识别设备，两者作为独立的可信设备记录管理。
</Accordion>
</Accordions>
