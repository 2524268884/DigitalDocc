---
title: 通讯录连接器告警推送配置
description: 为通讯录同步任务配置告警通知，当同步失败或异常时自动推送消息至钉钉、邮件或 Webhook，帮助管理员及时感知并处理问题。
---

## 功能概述

专属集成平台支持为通讯录连接器（LDAP、AD、SCIM、钉钉、飞书、企业微信等）配置**告警推送**，当同步任务因网络异常、目标服务限流、凭证失效等原因失败时，系统主动通知管理员，避免数据不一致问题被延迟发现。

**支持的告警通知方式：**
- 钉钉机器人消息推送
- 邮件通知
- Webhook 回调（自定义接收端）

## 告警触发条件

| 触发条件 | 说明 |
|---------|------|
| 同步任务执行失败 | 单次同步任务因任何原因未能完成 |
| 连续失败超过阈值 | 连续 N 次同步均失败（可配置阈值）|
| 凭证失效 | 访问目标系统的 Token / 密码等凭证已过期或无效 |
| 目标服务不可达 | 目标系统地址无法访问或连接超时 |

## 配置步骤

<Steps>
<Step>
### 步骤一：进入通讯录连接器设置

在专属集成平台管理控制台，进入 **用户中心 > 通讯录集成**，选择需要配置告警的连接器，进入「连接器设置」页面。
</Step>
<Step>
### 步骤二：进入告警配置

在连接器设置页面中，找到「告警推送」选项卡，点击进入配置。
</Step>
<Step>
### 步骤三：选择通知方式并填写配置

<Tabs items={["钉钉机器人", "邮件通知", "Webhook"]}>
<Tab value="钉钉机器人">

| 配置项 | 说明 |
|--------|------|
| Webhook 地址 | 钉钉自定义机器人的 Webhook URL |
| 加签密钥 | 钉钉机器人安全设置中的加签密钥（如已启用）|
| 通知范围 | 全部告警 / 仅连续失败 / 仅凭证失效 |

</Tab>
<Tab value="邮件通知">

| 配置项 | 说明 |
|--------|------|
| 收件人邮箱 | 告警邮件接收地址，多个地址用逗号分隔 |
| 通知范围 | 全部告警 / 仅连续失败 / 仅凭证失效 |

</Tab>
<Tab value="Webhook">

| 配置项 | 说明 |
|--------|------|
| Webhook URL | 接收告警事件的 HTTP 端点地址 |
| 请求头 | 自定义请求头（可选），如鉴权 Token |
| 通知范围 | 全部告警 / 仅连续失败 / 仅凭证失效 |

Webhook 请求体格式：
```json
{
  "connectorName": "钉钉通讯录连接器",
  "tenantId": "xxx",
  "alertType": "SYNC_FAILED",
  "failedCount": 3,
  "message": "同步任务连续失败 3 次，最近一次失败原因：网络连接超时",
  "occurredAt": "2026-04-03T11:00:00+08:00"
}
```

</Tab>
</Tabs>
</Step>
<Step>
### 步骤四：配置告警阈值

| 配置项 | 说明 | 默认值 |
|--------|------|--------|
| 连续失败告警阈值 | 连续失败多少次后触发告警 | 3 次 |
| 告警冷却时间（分钟）| 同一告警在冷却期内不重复发送 | 60 分钟 |
</Step>
<Step>
### 步骤五：测试并保存

点击「发送测试消息」验证配置是否正确，收到测试消息后点击「保存」完成配置。
</Step>
</Steps>

## 告警消息示例

**钉钉推送示例：**

```
⚠️ 通讯录同步告警

连接器：钉钉通讯录连接器
租户：xxx 企业
告警类型：同步任务连续失败
连续失败次数：3 次
最近一次失败原因：API 请求频率超限（HTTP 429）
发生时间：2026-04-03 11:00:00

请及时登录管理控制台查看详情并处理。
```

## 常见问题

<Accordions>
<Accordion title="同步偶尔失败一次，也会触发告警吗？">
取决于配置的连续失败阈值。若阈值设置为 3，则连续失败 3 次才触发。单次偶发失败不会触发告警，系统会自动重试。
</Accordion>
<Accordion title="告警通知是否支持按严重程度区分？">
目前所有告警使用统一格式推送，不区分严重程度。如需精细化处理，可选择 Webhook 方式，在自定义接收端根据 `alertType` 字段进行分级处理。
</Accordion>
</Accordions>
