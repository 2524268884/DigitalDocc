description: 在连接流执行前对输入数据进行格式、类型和业务规则校验，阻断不符合要求的数据流转，实现数据质量关口前置。
---

## 功能概述

**数据校验节点（Guard）** 在数据进入核心处理逻辑前进行前置校验，确保只有符合预设规则的数据才能继续流转。当数据不符合条件时，节点会生成结构化的错误报告，并可配置阻断流程或跳转到错误处理分支。

**核心能力：**
- 支持字段存在性、非空、类型、格式、值范围等多种校验规则
- 校验失败时输出结构化错误报告，便于问题定位
- 可配置校验失败后的行为：终止流程或跳转错误分支

## 适用场景

| 场景 | 说明 |
|------|------|
| Webhook 数据入口校验 | 外部系统推送数据时字段不稳定，在处理前先校验必填字段是否存在 |
| API 参数格式校验 | 验证传入参数的类型和格式，如手机号格式、日期格式等 |
| 业务规则前置拦截 | 在执行写操作前验证业务条件，如金额不得为负、状态值必须在枚举范围内 |

## 配置步骤

<Steps>
<Step>
### 步骤一：添加数据校验节点

在 AI 连接流画布中，将「数据校验」节点拖入需要校验的位置，通常放在触发器节点之后、核心处理节点之前。

<ImageZoom src="/images/ai-flow/guard-node-position.png" alt="数据校验节点位置示意" width={800} height={450} />
</Step>
<Step>
### 步骤二：配置校验规则

在节点配置面板中，添加一条或多条校验规则：

| 校验类型 | 说明 | 示例 |
|---------|------|------|
| 字段存在 | 检查指定字段是否存在于输入数据中 | `order_id` 必须存在 |
| 非空检查 | 字段存在但值不能为空字符串或 null | `customer_name` 不能为空 |
| 类型校验 | 字段值必须为指定类型（string/number/boolean/array/object）| `amount` 必须为 number |
| 格式校验 | 字段值必须匹配正则表达式 | 手机号格式 `^1[3-9]\d{9}$` |
| 枚举校验 | 字段值必须在指定列表中 | `status` 只能是 `paid`/`pending`/`cancelled` |
</Step>
<Step>
### 步骤三：配置校验失败行为

| 行为 | 说明 |
|------|------|
| 终止流程并报错 | 校验失败时流程中断，输出错误信息，在运行日志中可查看详情 |
| 跳转错误分支 | 配合条件分支节点，将不合规数据路由到专门的错误处理流程 |

<Callout type="warn">
选择「终止流程并报错」时，后续节点将不会执行。建议在关键写入操作前使用此模式，避免脏数据写入目标系统。
</Callout>
</Step>
</Steps>

## 输出参数

校验通过时，数据校验节点透传输入数据，不修改原始内容。

校验失败时（配置为跳转错误分支），输出以下结构：

```json
{
  "passed": false,
  "errors": [
    {
      "field": "amount",
      "rule": "type",
      "message": "字段 amount 期望类型 number，实际为 string"
    }
  ]
}
```

## 常见问题

<Accordions>
<Accordion title="多条校验规则的执行顺序是什么？">
所有规则从上到下依次执行，发现第一个不满足的规则后即停止，输出该规则的错误信息。如需收集全部错误，请在配置中开启「收集全部错误」选项。
</Accordion>
<Accordion title="校验节点能否校验嵌套对象内的字段？">
支持。字段路径使用点号表示法，例如 `order.customer.name` 表示校验嵌套对象中的 `name` 字段。
</Accordion>
</Accordions>
