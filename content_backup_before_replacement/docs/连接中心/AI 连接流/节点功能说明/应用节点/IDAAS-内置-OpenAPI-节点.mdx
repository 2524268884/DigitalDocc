---

## 功能概述

**专属集成平台内置 OpenAPI 节点**将专属集成平台自身的组织架构和用户管理 API 封装为可视化节点，实施顾问和业务人员可通过拖拽配置完成集成编排，无需编写代码即可实现用户入职、离职、调岗、权限分配等高频业务场景的自动化。

**支持的 API 模块：**
- **用户管理**：创建/更新/停用/删除用户，重置密码，查询用户信息
- **部门管理**：创建/更新/删除部门，查询部门及下级成员
- **角色管理**：为用户添加/移除角色，查询角色成员
- **组织架构**：查询组织树结构，获取上下级关系

## 适用场景

| 场景 | 说明 |
|------|------|
| 员工入职自动化 | HR 系统触发 Webhook 后，自动在专属集成平台创建账号并分配部门、角色 |
| 员工离职处理 | 接收离职事件后，自动停用账号并移除角色权限 |
| 组织架构变更同步 | 接收上游系统的部门变更事件，自动同步到专属集成平台组织树 |
| 权限动态调整 | 根据业务系统中的审批结果，自动为用户添加或移除指定角色 |

## 配置步骤

<Steps>
<Step>
### 步骤一：添加内置 OpenAPI 节点

在 AI 连接流画布中，在「应用节点」分类下找到「专属集成平台」节点，拖入画布中。
</Step>
<Step>
### 步骤二：选择 API 操作

在节点配置面板中，从下拉列表选择需要执行的 API 操作：

<Tabs items={["用户管理", "部门管理", "角色管理"]}>
<Tab value="用户管理">

| 操作 | 说明 |
|------|------|
| 创建用户 | 在指定部门下创建新用户账号 |
| 更新用户信息 | 修改用户的基础字段或扩展属性 |
| 停用用户 | 禁止用户登录，保留账号数据 |
| 删除用户 | 永久删除用户账号（不可逆）|
| 重置密码 | 为指定用户重置登录密码 |
| 查询用户详情 | 根据用户 ID 或账号查询完整信息 |

</Tab>
<Tab value="部门管理">

| 操作 | 说明 |
|------|------|
| 创建部门 | 在指定上级部门下创建子部门 |
| 更新部门信息 | 修改部门名称或上级部门 |
| 删除部门 | 删除指定部门（需确认部门下无用户）|
| 查询部门成员 | 获取指定部门下的所有用户列表 |

</Tab>
<Tab value="角色管理">

| 操作 | 说明 |
|------|------|
| 为用户添加角色 | 将指定用户加入某角色的成员列表 |
| 移除用户角色 | 将指定用户从某角色的成员列表中移除 |
| 查询角色成员 | 获取指定角色下的所有用户列表 |

</Tab>
</Tabs>
</Step>
<Step>
### 步骤三：配置请求参数

根据所选操作，填写必要的请求参数。所有参数均支持引用上游节点的输出变量。

<Callout type="warn">
**删除用户**操作不可逆，建议在调用前添加条件分支节点进行二次确认，或仅在有明确业务触发条件时使用。
</Callout>
</Step>
<Step>
### 步骤四：处理返回结果

API 调用成功后，返回结果可在后续节点中引用：

```text
{{$node['内置OpenAPI节点ID'].output.data.userId}}
```

常用返回字段：

| 字段 | 说明 |
|------|------|
| `data.userId` | 操作的目标用户 ID |
| `data.deptId` | 操作的目标部门 ID |
| `success` | 操作是否成功（boolean）|
| `message` | 操作结果描述 |
</Step>
</Steps>

## 常见问题

<Accordions>
<Accordion title="内置 OpenAPI 节点使用的是哪个账号的权限？">
节点使用在「凭证管理」中配置的 M2M 凭证（客户端模式）调用 API，凭证需具备对应模块的管理权限。
</Accordion>
<Accordion title="API 调用失败时，流程会怎么处理？">
默认情况下，API 调用失败会导致当前流程实例报错中断。可在节点配置中开启「失败时继续」选项，或在流程中添加错误处理分支。
</Accordion>
</Accordions>
