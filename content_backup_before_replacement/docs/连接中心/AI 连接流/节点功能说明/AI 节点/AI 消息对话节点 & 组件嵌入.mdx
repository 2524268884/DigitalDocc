## 功能概述

本功能用于将 AI 连接流的能力，以 对话形式嵌入到用户自有系统中。
通过 Chatbox 对话组件 接收用户输入，由 对话触发（Chat Trigger）节点 启动 AI 连接流执行，并通过 回复对话消息节点 将流程执行结果实时回传至对话窗口，实现从「对话输入 → 流程执行 → 对话回复」的完整闭环。
1. 适用场景
该能力适用于以下场景：
- 将 AI Agent 嵌入到官网、业务系统、内部平台中使用
- 用户通过自然语言对话触发后台业务流程
- 希望通过对话方式获取流程执行结果
- 面向外部用户提供 AI 服务入口
- 构建以对话为入口的业务自动化能力
1. 对话触发（Chat Trigger）节点说明
### 节点能力
对话触发节点具备以下能力：
- 作为 AI 连接流的 起始节点
- 在用户通过 Chatbox 组件发送消息时触发流程
- 支持基于消息内容设置触发条件
- 输出对话上下文信息（会话 ID、用户 ID、组件 ID 等）
- 与 Chatbox 组件一一配合使用
1. 如何插入 对话触发 节点
### 插入入口
## 在 AI 连接流画布中点击「插入节点」
- 选择 Trigger 节点
1. 选择 对话触发（Chat Trigger）
### 节点使用限制
- 仅支持作为流程的第一个节点
- 一个流程仅允许存在一个对话触发节点


- 为流程添加对话相关节点
### 对话触发（Trigger）
用于接收用户在 Chatbox 中输入的内容，并触发流程执行。
### 回复对话消息（Action）
用于将流程执行结果，回复到对应的聊天窗口中。
- 一个流程中可配置多个回复对话消息节点
- 实际回复内容以流程执行到的节点为准


## 参数配置说明（重点）
### 对话触发（Trigger）参数说明
参数
	
说明


触发条件
	
指定何种对话消息触发流程


关键词
	
仅在触发条件为「包含指定关键词」时生效
触发条件支持：
- 收到任意消息时（默认）
- 包含指定关键词
### 回复对话消息（Action）参数说明
参数
	
说明


聊天组件 id
	
需要回复的 Chatbox 组件 ID


聊天会话 id
	
当前对话的 session id


回复内容
	
返回给用户的消息内容
说明：
- 组件 ID、会话 ID 通常来自对话触发节点输出
- 回复内容为必填项
### AI 生成模式
- 本功能本身不提供 AI 生成能力配置
- AI 能力由连接流中其他节点提供
- 对话节点仅负责输入触发与结果回传
### 限制说明
- 对话必须通过 Chatbox 组件发起
- 未绑定流程的组件不可启用
- 流程触发节点变更后，组件需重新启用
- 会话存在过期机制，过期后将生成新会话
1. 节点输出说明
### 对话触发（Trigger）输出

```json
{
  "sessionId": "xxxxxx",
  "userid": "xxxxxx",
  "chatboxid": "xxxxx",
  "type": "text | file",
  "content": "用户输入内容",
  "file_url": [],
  "customparams": {}
}
```

### 回复对话消息（Action）异常输出示例

```json
{
  "sessionId": "xxxxxx",
  "userid": "xxxxxx",
  "chatboxid": "xxxxx",
  "status": "fail",
  "description": "当前会话id已过期"
}
```
### 默认状态提示
场景
	
默认提示


未关联流程
	
当前对话暂无关联流程


流程无输出
	
当前关联流程无内容输出


组件未启用
	
当前对话组件尚未启用


组件被删除
	
当前组件已被删除
## 组件集成（Chatbox）配置与生效逻辑
### 创建组件
路径：集成平台 → 连接流 → 组件集成
- 系统自动生成组件 ID（不可修改）
- 组件名称可编辑


### 组件配置项说明
## 配置项
	
说明


组件名称
	
用于展示在 Chatbox 顶部标题。


关联流程
	
1. 仅可选择「已上线」且触发节点为「对话触发」的流程2. 未关联流程的组件不允许启用


允许的输入类型
	
- 文本输入- 语音输入- 文件上传


输入框引导文案
	
展示在输入框中的占位提示。


欢迎语
	
在用户开启新会话时自动发送。


会话过期时间
	
指定会话在无操作情况下的过期时间。


允许开启新会话
	
是否允许用户手动创建新的 session。


机器人头像
	
用于 Chatbox 中的机器人头像展示。
### 组件启用规则（非常重要）
- 未关联流程 → 不允许启用
- 关联流程被删除 → 自动关闭
- 流程触发节点被修改 → 自动关闭
- 所有条件满足后需手动重新开启



## Chatbox 组件嵌入说明
### 支持的嵌入方式
- 悬浮球嵌入（JS）
- iframe 嵌入
- 独立页面嵌入
### 嵌入参数说明
参数
	
说明


chatbox-id
	
组件唯一标识（必填）


userid
	
外部系统用户标识


custom_params
	
自定义参数（JSON）
使用建议（产品经验）
- 一个 Chatbox 组件建议仅绑定一个流程
- 流程中应确保至少存在一个「回复对话消息」节点
- 对执行时间较长的流程，建议先回复中间提示
- 文件上传场景需在流程中显式处理 file_url
- 会话过期时间不宜设置过短，避免上下文频繁丢失
