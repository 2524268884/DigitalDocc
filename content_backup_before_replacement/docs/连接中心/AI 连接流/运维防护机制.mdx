---

## 功能概述

**流程运维防护机制**在流程维度提供执行边界控制，当流程触发死循环、处理超大数据集或执行时间过长时，系统自动中断流程并发出告警，保障平台整体稳定性。

**核心防护能力：**
- **执行超时**：流程总执行时间超过阈值时自动终止
- **最大循环次数**：单次循环节点迭代次数上限，防止无限循环
- **最大执行步数**：流程中节点执行总步数上限，防止环路导致的无限跳转
- **异常告警推送**：触发防护机制时，向管理员推送告警通知

## 适用场景

| 场景 | 防护机制 |
|------|---------|
| For Each 遍历超大数组 | 最大循环次数限制 |
| 循环节点条件配置错误导致无限循环 | 最大循环次数 + 执行超时 |
| 流程拓扑形成环路 | 最大执行步数限制 |
| 外部 API 响应极慢拖垮流程 | 执行超时 |

## 配置步骤

<Steps>
<Step>
### 步骤一：进入流程设置

打开目标连接流，点击画布右上角的「流程设置」图标，进入流程级别配置面板。
</Step>
<Step>
### 步骤二：配置防护参数

在「运维防护」选项卡中，配置以下参数：

| 参数 | 说明 | 推荐值 |
|------|------|--------|
| 执行超时（秒） | 单次流程实例从触发到结束的最大允许时间 | 根据业务场景，建议 300 秒以内 |
| 最大循环次数 | 单个循环节点在一次流程执行中的最大迭代次数 | 建议不超过 10,000 次 |
| 最大执行步数 | 一次流程执行中所有节点累计执行次数上限 | 建议不超过 50,000 步 |

<Callout type="warn">
防护参数触发后，当前流程实例将被**强制终止**，已执行的节点操作（如写入数据）不会自动回滚。建议在关键写入操作前添加数据校验节点，减少因防护中断导致的脏数据风险。
</Callout>
</Step>
<Step>
### 步骤三：配置告警通知

在「告警设置」中，选择防护机制触发时的通知方式：

- **钉钉推送**：向指定人员或群组发送告警消息
- **邮件通知**：发送告警邮件至指定邮箱
- **Webhook 回调**：向指定 URL 推送告警事件，供自定义处理
</Step>
</Steps>

## 告警信息说明

防护机制触发时，告警消息包含以下信息：

| 字段 | 说明 |
|------|------|
| 流程名称 | 触发防护的连接流名称 |
| 触发类型 | 执行超时 / 超过最大循环次数 / 超过最大执行步数 |
| 实例 ID | 本次执行的唯一标识，可用于查询运行日志 |
| 触发时间 | 防护机制触发的时间戳 |
| 当前步数/循环次数 | 触发时已执行的步数或循环次数 |

## 常见问题

<Accordions>
<Accordion title="防护机制触发后，流程会自动重试吗？">
不会。防护机制触发后流程实例直接终止，不会自动重试。需人工排查流程配置后手动重新触发。
</Accordion>
<Accordion title="如何区分是防护机制终止还是正常节点报错？">
在流程运行日志中，防护机制终止的实例状态标记为「防护中断」，节点报错的实例状态标记为「执行失败」，两者可明确区分。
</Accordion>
<Accordion title="防护参数是否对所有版本的流程生效？">
防护参数与流程配置一起版本化，修改后需重新上线才对新触发的实例生效。历史已执行的实例不受影响。
</Accordion>
</Accordions>
