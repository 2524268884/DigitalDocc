## 概述

<ImageZoom
  src="/images/参数配置/授权管理/overview.png"

  width={1200}
  height={720}

授权管理模块是专属集成平台与企业协同办公应用建立安全连接的核心配置中心，用于集中管理第三方平台的认证凭证和 API 访问授权信息。通过标准化的认证协议与密钥管理机制，确保平台在数据交换和身份集成过程中的安全合规性，同时降低系统集成的管理复杂度。

目前支持 **钉钉**、**企业微信**、**飞书** 三种企业协同办公平台。创建完成的授权信息可在通讯录集成、通讯录同步和企业认证源三个核心功能模块中直接引用。

## 功能说明

### 应用场景

| 场景 | 说明 |
|------|------|
| 多源身份集成 | 为通讯录集成提供授权基础，建立与钉钉、企业微信、飞书之间的安全连接，实现用户和组织数据导入 |
| 身份数据同步 | 为通讯录同步模块提供授权配置，支持将平台内用户和组织数据同步至目标系统 |
| 企业认证源配置 | 为企业认证源模块提供基础授权信息，支持通过第三方平台实现单点登录和身份验证 |

### 授权管理类型

- **钉钉**：阿里巴巴旗下的智能移动办公平台
- **企业微信**：腾讯推出的企业级通讯与办公协作平台
- **飞书**：字节跳动旗下的新一代企业协作与管理平台

### 钉钉平台授权配置

钉钉平台认证基于 OAuth 2.0 协议，需要配置企业应用的基本认证参数，用于建立专属集成平台与钉钉开放平台之间的安全通信通道。

#### 配置参数说明

| 参数名称 | 说明 | 必填 | 获取方式 |
|----------|------|------|----------|
| 企业名称 | 用于在平台中标识该授权的显示名称 | 是 | 自定义 |
| CorpId | 钉钉企业唯一标识符 | 是 | 钉钉管理后台 → 工作台 → 管理员工具 |
| 应用 Key（AppKey） | 钉钉应用的唯一标识符 | 是 | 钉钉开发者后台 → 应用开发 → 应用信息 |
| 应用 Secret | 钉钉应用的安全密钥 | 是 | 钉钉开发者后台 → 应用开发 → 应用信息 |
| 应用 AgentId | 钉钉应用实例标识符 | 是 | 钉钉开发者后台 → 应用开发 → 应用信息 |

<Callout type="warn">
应用 Secret 为高敏感信息，平台使用 AES-256 加密算法存储。请妥善保管，避免泄露。
</Callout>

#### 所需授权范围

在钉钉开发者后台配置应用时，需申请以下 API 权限：

- 通讯录管理权限（读取/写入企业通讯录）
- 身份验证权限（身份验证、OrgUser.read）
- 消息通知权限（工作通知消息）

### 飞书平台授权配置

飞书平台的授权基于 OAuth 2.0 和自定义 Token 认证机制，需要配置企业自建应用的基本认证参数。

#### 配置参数说明

| 参数名称 | 说明 | 必填 | 获取方式 |
|----------|------|------|----------|
| 企业名称 | 用于在平台中标识该授权的显示名称 | 是 | 自定义 |
| App ID | 飞书应用的唯一标识符 | 是 | 飞书开发者后台 → 应用详情 |
| App Secret | 飞书应用的安全密钥 | 是 | 飞书开发者后台 → 应用详情 |
| Encrypt Key | 消息加密密钥 | 否 | 飞书开发者后台 → 事件订阅 |
| Verification Token | 事件订阅验证令牌 | 否 | 飞书开发者后台 → 事件订阅 |

<Callout type="info">
建议通过飞书开发者平台创建「自建应用」而非「商店应用」，自建应用更适合企业内部身份和数据集成场景。
</Callout>

#### 所需授权范围

- 通讯录权限（读取/写入企业通讯录）
- 身份验证权限（获取用户身份）
- 消息权限（发送消息）

### 企业微信授权配置

企业微信的授权基于内建的 API 密钥管理机制，需要配置企业自建应用的基本认证参数。

#### 配置参数说明

| 参数名称 | 说明 | 必填 | 获取方式 |
|----------|------|------|----------|
| 企业名称 | 用于在平台中标识该授权的显示名称 | 是 | 自定义 |
| 企业 ID（CorpId） | 企业微信中企业的唯一标识 | 是 | 企业微信管理后台 → 我的企业 → 企业信息 |
| 应用 ID（AgentId） | 企业微信应用的实例标识 | 是 | 企业微信管理后台 → 应用管理 → 自建应用 |
| Secret | 企业微信应用的安全密钥 | 是 | 企业微信管理后台 → 应用管理 → 自建应用 |
| 应用名称 | 企业微信中应用的名称 | 是 | 企业微信管理后台 → 应用管理 → 自建应用 |

<Callout type="warn">
企业微信的 Secret 凭证具有较高权限，建议每 90 天轮换更新一次，并限制知悉人员范围。
</Callout>

#### 所需授权范围

- 通讯录管理权限（读取/修改通讯录）
- 身份验证权限（获取员工身份信息）
- 消息推送权限（发送应用消息）

## 最佳实践

### 安全管理建议

- **权限最小化**：仅为应用申请必要的 API 权限
- **定期凭证轮换**：建议每 90 天更新一次 Secret 等敏感凭证
- **访问控制审计**：定期审核授权应用列表，撤销不再使用的授权
- **IP 访问限制**：在支持的平台上，建议设置 API 调用的 IP 白名单

### 多环境授权管理

对于拥有多个环境（开发、测试、生产）的企业，推荐为每个环境创建独立的第三方平台应用和授权配置，避免跨环境的数据污染风险。

命名示例：
- `钉钉-生产环境`
- `钉钉-测试环境`
- `钉钉-开发环境`

## 常见问题

<Accordions>
<Accordion title="授权验证失败怎么处理？">
可能原因为凭证信息错误或过期。请检查并更新密钥信息，同时确认第三方平台中应用状态正常。
</Accordion>
<Accordion title="出现权限不足错误怎么办？">
应用未申请所需 API 权限所致。请在第三方平台开发者中心为应用追加所需权限，如通讯录管理、身份验证等权限。
</Accordion>
<Accordion title="连接超时如何排查？">
可能是网络问题或第三方服务异常。请检查网络连接，并确认第三方服务当前状态是否正常。
</Accordion>
<Accordion title="IP 被限制访问怎么解决？">
第三方平台启用了 IP 限制。请在第三方平台配置中，将专属集成平台服务器 IP 添加到 API 调用白名单中。
</Accordion>
<Accordion title="授权信息失效后系统会如何处理？">
系统会在日志中记录授权失效事件，暂停相关同步任务或集成流程，并向管理员发送告警通知，待管理员更新授权信息后自动恢复相关任务。
</Accordion>
</Accordions>
