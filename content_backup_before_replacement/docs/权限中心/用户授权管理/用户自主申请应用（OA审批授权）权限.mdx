## 概述

<ImageZoom
  src="/images/权限中心/用户授权管理/用户自主申请应用（OA审批授权）权限/overview.png"
  alt="用户自主申请应用（OA审批授权）权限概览截图"
  width={1200}
  height={720}
/>

用户（尤其是企业外部用户）临时要访问某个应用，申请权限，之前的方式可能是单独找管理员，沟通申请或者是填写申请单，比较麻烦，管理员的工作量也比较大。现在可以通过自助门户进行自助申请，申请审批还将关联钉钉的 OA 审批，方便高效还可以留档追溯。

<Callout type="info">
自助申请流程：进入工作台 → 申请应用权限 → 填写申请信息 → 管理员审批 → 权限生效
</Callout>

## 自助申请优势

| 优势 | 说明 |
|-----|------|
| 简化申请流程 | 用户可直接在门户提交申请，无需额外沟通 |
| 审批流程标准化 | 统一的审批流程，保证审批规范 |
| 自动记录存档 | 所有申请和审批记录自动保存，便于追溯 |
| 减轻管理员负担 | 管理员只需处理审批，无需手动授权 |
| 与 OA 系统集成 | 与企业现有 OA 审批流程无缝集成 |

## 申请参数说明

| 参数 | 说明 | 是否必填 |
|-----|------|---------|
| 申请原因 | 说明申请应用权限的业务需求 | 是 |
| 使用时间 | 预计需要使用该应用的时间段 | 是 |
| 权限范围 | 申请的具体权限范围（如果应用支持） | 否 |
| 紧急程度 | 权限申请的紧急程度 | 否 |

## 操作步骤

### 步骤 1：进入自助门户

<ImageZoom
  src="/images/权限中心/用户授权管理/用户自主申请应用（OA审批授权）权限/step-01.png"
  alt="进入自助门户截图"
  width={1200}
  height={720}
/>

在管理平台的右上角，点击「用户服务台」，即可进入用户自助门户界面。工作台可以进行应用权限自助申请操作。

### 步骤 2：申请应用权限

<ImageZoom
  src="/images/权限中心/用户授权管理/用户自主申请应用（OA审批授权）权限/step-02.png"
  alt="申请应用权限截图"
  width={1200}
  height={720}
/>

1. 在工作台界面里，可以看到应用中心
2. 用户没有权限访问的应用，右上角会有一个「申请」标志
3. 点击之后弹出申请界面
4. 填写申请原因和使用时间
5. 点击「提交」完成权限申请

### 步骤 3：管理员审批

{/* DOC_IMAGE src="/images/权限中心/用户授权管理/用户自主申请应用（OA审批授权）权限/step-03.png" alt="管理员审批截图" width="1200" height="720" */}

1. 申请完成后，钉钉工作台会收到通知
2. 管理员可以点击「同意」或「拒绝」
3. 审批通过后，权限自动生效

### 步骤 4：账号续期申请（如需要）

{/* DOC_IMAGE src="/images/权限中心/用户授权管理/用户自主申请应用（OA审批授权）权限/step-04.png" alt="账号续期申请（如需要）截图" width="1200" height="720" */}

1. 在工作台界面点击右上角，选择「个人中心」
2. 进入个人中心界面，可以进行密码修改、账号续期操作
3. 点击「账号续期申请」，弹出申请界面
4. 填写续期原因和续期时间
5. 提交后，审批结果呈现给管理员的钉钉账号进行审批

<Callout type="warn">
账号续期申请的审批可以和钉钉的审批打通，审批结果呈现给管理员的钉钉账号进行审批。
</Callout>

## 最佳实践

1. **明确填写申请理由**：详细说明业务场景和必要性，提高审批通过率
2. **合理设置权限期限**：根据实际需求设置合适的权限期限，避免长期闲置权限
3. **分级审批流程**：针对不同重要程度的应用，设置不同级别的审批流程
4. **定期权限复查**：管理员应定期检查已授予的临时权限，及时回收过期权限

## 常见问题

<Accordions>
<Accordion title="用户如何查看申请状态？">

用户可以在自助门户的「我的申请」中查看申请状态。
</Accordion>
<Accordion title="审批通过后权限立即生效吗？">

是的，审批通过后权限自动生效，用户可以立即访问对应的应用。
</Accordion>
<Accordion title="申请被拒绝后可以重新申请吗？">

可以。用户可以根据拒绝原因修改申请信息后重新提交。
</Accordion>
</Accordions>
