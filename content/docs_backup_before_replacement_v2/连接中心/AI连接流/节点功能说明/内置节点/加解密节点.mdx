数据加解密处理器，支持多种对称和非对称加密算法，包括国际标准算法和国密算法。通过配置不同的算法、模式和密钥，实现数据的加密保护和解密还原。

## 功能特性
- **多种加密算法**：支持AES、DES、RSA等多种标准算法
- **国密算法支持**：集成SM2、SM3、SM4国密算法
- **灵活编码**：支持Base64、Hex等多种编码格式
- **模式选择**：支持ECB、CBC等多种加密模式
- **密钥管理**：支持不同格式的密钥输入
- **安全可靠**：基于标准加密库实现

## 支持的加密算法

### 对称加密算法
- **AES**：高级加密标准，支持128/192/256位密钥
- **DES**：数据加密标准
- **DESede**：三重DES加密
- **SM4**：国密对称加密算法

### 非对称加密算法
- **RSA**：RSA公钥加密算法
- **SM2**：国密椭圆曲线公钥密码算法

### 哈希算法
- **SM3**：国密哈希算法

### 编码算法
- **BASE64**：Base64编码/解码

## 加密模式

### ECB (Electronic Codebook)
- **特点**：简单分组模式，相同明文产生相同密文
- **适用**：短数据加密，不适合流式数据

### CBC (Cipher Block Chaining)
- **特点**：链式分组模式，安全性更高
- **需要**：初始化向量(IV)
- **适用**：大多数加密场景

## 配置参数

### DecryptOrEncryptAction

#### 基础配置
- **type**：操作类型（ENCRYPT加密/DECRYPT解密）
- **alg**：加密算法
- **mode**：加密模式（ECB/CBC）
- **fillMode**：填充模式（PKCS5\_PADDING）

#### 内容配置
- **content**：待处理内容
- **contentEncode**：内容编码格式（PlainText/Base64/Hex）

#### 密钥配置
- **secret**：加密密钥
- **keyEncode**：密钥编码格式（PlainText/Base64/Hex）
- **secretLength**：密钥长度（AES需要）
- **secretType**：密钥类型（RSA需要，公钥/私钥）

#### 高级配置
- **randomVector**：初始化向量（CBC模式需要）
- **encode**：输出编码格式（Base64/Hex）

## 使用方法

### AES加密
```
{
  "type": "ENCRYPT",
  "alg": "AES",
  "mode": "CBC",
  "fillMode": "PKCS5_PADDING",
  "content": "Hello World",
  "contentEncode": "PlainText",
  "secret": "1234567890123456",
  "keyEncode": "PlainText",
  "secretLength": 128,
  "randomVector": "1234567890123456",
  "encode": "Base64"
}
```

### AES解密
```
{
  "type": "DECRYPT",
  "alg": "AES",
  "mode": "CBC",
  "fillMode": "PKCS5_PADDING",
  "content": "加密后的Base64字符串",
  "contentEncode": "Base64",
  "secret": "1234567890123456",
  "keyEncode": "PlainText",
  "secretLength": 128,
  "randomVector": "1234567890123456",
  "encode": "PlainText"
}
```

### 国密密钥
- **SM2**：椭圆曲线密钥对
- **SM4**：32字节对称密钥

## 常见问题

### Q: 如何选择合适的加密算法？

**A:** 根据数据类型和安全需求选择：
- 通用数据：AES-128
- 敏感数据：AES-256或国密算法
- 文件传输：SM4
- 数字签名：RSA或SM2

### Q: CBC模式需要初始化向量吗？

**A:** 是的，CBC模式必须提供16字节的初始化向量。ECB模式不需要IV，但安全性较低。

### Q: 密钥长度对安全性有什么影响？

**A:** 密钥越长，安全性越高，但计算开销也越大。AES-128在大多数场景下已足够安全。

### Q: 如何处理二进制数据？

**A:** 对于二进制数据，使用Base64或Hex编码作为输入。加密后的结果也会根据encode参数进行编码。

### Q: 支持流式加密吗？

**A:** 当前版本主要支持块加密。对于大文件，建议分块处理或使用专门的文件加密工具。

### Q: 国密算法和国际算法有什么区别？

**A:** 国密算法（SM2/SM3/SM4）是由中国国家密码管理局制定的标准算法，在国内有强制性要求。国际算法（AES/RSA）是全球标准，在国际场景使用更广泛。
