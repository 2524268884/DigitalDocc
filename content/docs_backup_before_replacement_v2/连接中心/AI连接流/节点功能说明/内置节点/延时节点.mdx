## 概述

SleepActionHandler 是AI FLOW中的延时执行处理器，用于在流程执行过程中暂停指定时间。该功能适用于需要等待外部系统响应、控制执行频率、模拟异步处理等场景。

## 功能特性
- **精确延时**：支持毫秒级延时控制
- **时间限制**：最大延时300秒（5分钟）
- **线程安全**：使用Thread.sleep实现
- **中断处理**：支持线程中断响应
- **状态管理**：延时完成后返回成功状态

## 配置参数

### SleepAction
- **millisecond**：延时时间（毫秒），必填
    - 类型：Long
    - 范围：1-300000（1毫秒到5分钟）
    - 示例：5000（5秒）

## 使用方法

### 1\. 基本延时
```
{
  "millisecond": 5000
}
```

延时5秒后继续执行

### 2\. 短时间延时
```
{
  "millisecond": 1000
}
```

延时1秒

### 3\. 长时间延时
```
{
  "millisecond": 300000
}
```

延时5分钟（最大值）

### 4\. 动态延时
```
{
  "millisecond": "${delayTime}"
}
```

使用变量控制延时时间

## 性能考虑

### 系统资源
- 延时期间占用线程资源
- 长延时可能影响并发性能
- 建议合理控制延时时间

### 替代方案
- 对于短延时（\<1秒），考虑直接执行
- 对于长延时，考虑使用定时任务

## 常见问题

### Q: 最大延时时间是多少？

**A:** 最大延时300000毫秒（5分钟）。超过此时间会抛出异常。

### Q: 延时期间会发生什么？

**A:** 线程进入休眠状态，不占用CPU资源，但会占用内存和线程资源。

### Q: 支持并发延时吗？

**A:** 是的，每个延时节点独立执行，不影响其他节点的并发执行。

### Q: 如何取消延时？

**A:** 可以通过线程中断机制取消延时。系统会响应中断并继续执行后续节点。

### Q: 延时精度如何？

**A:** 使用Thread.sleep实现，精度取决于操作系统调度器，通常在毫秒级。
