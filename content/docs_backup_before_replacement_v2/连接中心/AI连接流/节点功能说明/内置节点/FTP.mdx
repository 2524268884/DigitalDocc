## 概述

FTP文件传输处理器，支持从FTP服务器下载文件。通过配置FTP服务器连接参数，可以实现安全高效的文件传输功能。

## 功能特性
- **文件下载**：从FTP服务器下载文件到本地
- **多种传输模式**：支持ASCII和二进制传输模式
- **连接管理**：自动管理FTP连接和断开
- **错误处理**：完善的异常捕获和错误信息返回
- **安全认证**：支持用户名密码认证

## 支持的操作类型

### DOWNLOAD - 文件下载

从FTP服务器下载指定文件到本地临时目录。

## 配置参数

### FTPAction

#### 基础配置
- **opType**：操作类型（DOWNLOAD/UPLOAD，目前仅支持DOWNLOAD）
- **remoteFilePath**：FTP服务器上的文件路径（必需）
- **localFilePath**：本地保存路径（可选，默认为临时目录）

### FTP服务器配置（通过账号管理）
- **host**：FTP服务器地址
- **port**：FTP服务器端口
- **username**：用户名
- **password**：密码
- **transferMode**：传输模式（binary/ASCII）

## 使用方法

### 文件下载
```
{
  "opType": "DOWNLOAD",
  "remoteFilePath": "/data/reports/monthly_report.pdf"
}
```

下载FTP服务器上的monthly\_report.pdf文件到本地/tmp/access/\{tenantId\}/目录

### 完整配置示例
```
{
  "opType": "DOWNLOAD", 
  "remoteFilePath": "/ftp/upload/user_data.csv",
  "localFilePath": "/tmp/custom_path/user_data.csv"
}
```

## 传输模式说明

### Binary（二进制）模式
- 适用于：图片、视频、压缩文件、程序文件等
- 特点：逐字节传输，保持文件完整性

### ASCII（文本）模式
- 适用于：纯文本文件
- 特点：自动转换换行符，适用于不同系统的文本文件

## 常见问题

### Q: 如何处理大文件下载？

**A:** 系统支持大文件下载，但建议分块处理超大文件。对于超大文件，考虑使用其他传输方式或分多次下载。

### Q: 支持哪些FTP服务器？

**A:** 支持标准FTP协议的服务器，包括常见的FTP服务器软件如vsftpd、FileZilla Server等。不支持SFTP（SSH FTP）。

### Q: 文件下载到哪里？

**A:** 默认下载到/tmp/access/\{tenantId\}/目录。可以通过localFilePath参数自定义保存路径，但需要确保目录存在且有写权限。

### Q: 如何处理文件覆盖？

**A:** 如果本地文件已存在，会被覆盖。如需保留历史版本，建议在文件名中添加时间戳或其他标识。

### Q: 支持断点续传吗？

**A:** 目前不支持断点续传。如果下载过程中断，需要重新下载整个文件。
