## 1\. 概述

专属集成平台与LDAP as a Service的通讯录同步功能，支持将LDAP as a Service中统一维护的组织机构和用户数据同步至平台，实现企业内部身份数据的统一管理与一致性。通过该功能，您可以从LDAP中获取用户数据和组织架构，降低管理成本，提高数据准确性，并确保各系统间身份信息的实时同步。

### 同步优势
- **数据自动获取**：配置完成后，平台可按设定周期自动从LDAP获取数据，无需人工干预
- **身份统一管理**：通过平台集中管理各系统账号，实现"一处变更，处处生效"
- **减少管理开销**：降低人工维护多个系统账号的工作量，提高IT管理效率
- **数据一致性保障**：确保各系统间的身份数据保持同步，避免信息孤岛
- **安全机制保障**：内置删除阈值机制，防止因数据异常造成大规模用户误删

### 适用场景
- **多系统身份打通**：企业已有LDAP目录，需要与多个应用系统打通用户体系
- **基于组织架构的权限分配**：需要将企业组织架构同步至各应用系统，用于自动化权限分配
- **员工入离职流程自动化**：通过LDAP集中管理员工账号，确保员工入离职时身份信息在各系统同步变更
- **集中账号管理**：利用LDAP的优势，实现企业账号的集中管理和统一认证

### LDAP as a Service说明

LDAP as a Service是一种轻量级目录访问协议服务，专为企业用户身份和组织结构管理设计：

| 特性 | 说明 | 优势 |
|------|------|------|
| **集中目录服务** | 提供标准化的LDAP目录服务，用于存储和管理用户信息和组织结构 | 统一管理入口，简化管理流程 |
| **标准协议支持** | 基于开放标准的LDAP协议，兼容大多数应用系统 | 广泛的系统兼容性，降低集成难度 |
| **高性能查询** | 针对身份认证和授权查询进行了优化 | 快速响应用户认证请求，提升用户体验 |
| **企业级安全** | 支持加密传输和细粒度的访问控制 | 保障企业身份数据安全 |

## 2\. 前置准备

### 必要条件
- **LDAP服务部署**：已成功部署LDAP as a Service（参考[《LDAP as a Service部署说明》](https://alidocs.dingtalk.com/i/nodes/YMyQA2dXW7lx40bGt76mv9BaVzlwrZgb)）
- **LDAP管理权限**：获取LDAP服务的管理员账号和配置信息
- **平台权限**：拥有专属集成平台的管理员权限，用于配置同步连接器
- **网络要求**：确保专属集成平台服务器可以访问LDAP服务
- **数据准备**：LDAP中已配置完善的组织结构和用户数据

### LDAP配置信息获取

成功配置LDAP通讯录同步需要获取以下关键信息：
1. **BaseDN**：LDAP目录的基础识别名称，定义了搜索的根起点
2. **管理员账号**：具有足够权限读取目录信息的LDAP账号
3. **管理员密码**：对应管理员账号的密码
4. **服务器地址**：LDAP服务器的IP地址或域名
5. **服务器端口**：LDAP服务的端口号，标准端口为389(非加密)或636(加密)

### 获取LDAP配置信息的步骤
1. **联系LDAP管理员**：向您的LDAP系统管理员请求所需配置信息
2. **查看LDAP服务器配置**：
    - 如果您有LDAP服务器的管理权限，登录LDAP管理控制台
    - 查看服务器配置信息，记录BaseDN、服务器地址和端口
3. **确认管理员账号权限**：
    - 确保管理员账号具有以下权限：
        - 读取组织结构的权限
        - 读取用户信息的权限
        - 执行搜索操作的权限

## 3\. 专属集成平台配置

### 3.1 创建同步连接器
1. **导航至通讯录同步**：登录专属集成平台管理控制台，在左侧导航栏选择「用户中心 → 通讯录同步」
2. **添加连接器**：点击页面右上角的「添加连接器」按钮
3. **选择连接器类型**：在弹出的平台选择面板中，选择「LDAP as a Service」图标
4. **填写基本信息**：根据收集的LDAP配置信息，填写以下字段：
    - BaseDN：填写LDAP的基础DN，如"dc=example,dc=com"
    - 管理员账号：填写具有读取权限的LDAP管理员账号
    - 管理员密码：填写对应的密码
    - 服务器地址：填写LDAP服务器的IP地址或域名
    - 服务器端口：填写LDAP服务的端口号
    - 测试账号：填写一个存在于LDAP中的用户账号，用于验证配置是否正确

### 3.2 配置搜索条件

LDAP同步需要配置用户和组织的搜索条件，以确定从LDAP目录中获取哪些数据：

| 参数名称 | 界面显示名称 | 说明 | 建议配置 | 适用场景 |
|------------|------------------|------|------------|------------|
| 用户组搜索条件 | 用户组搜索条件 | 定义如何搜索组织结构 | 推荐使用默认值: (objectClass=groupOfNames) | 标准LDAP目录 |
| 用户搜索条件 | 用户搜索条件 | 定义如何搜索用户 | 推荐使用默认值: (objectClass=inetOrgPerson) | 标准LDAP目录 |

**搜索条件详细说明**：
- **用户组搜索条件**：用于识别LDAP目录中的组织结构或群组。默认值`(objectClass=groupOfNames)`适用于大多数标准LDAP配置。如果您的LDAP使用不同的对象类来表示组织，可能需要调整此值，例如`(objectClass=organizationalUnit)`或`(objectClass=group)`。
- **用户搜索条件**：用于识别LDAP目录中的用户条目。默认值`(objectClass=inetOrgPerson)`适用于大多数标准LDAP配置。如果您的LDAP使用不同的对象类来表示用户，可能需要调整此值，例如`(objectClass=person)`或`(objectClass=user)`。

**特殊场景搜索条件示例**：
```
(&(objectClass=inetOrgPerson)(departmentNumber=IT))

# 仅同步启用状态的用户
(&(objectClass=inetOrgPerson)(!(userAccountControl:1.2.840.113556.1.4.803:=2)))

# 仅同步特定角色的用户
(&(objectClass=inetOrgPerson)(employeeType=FullTime))
```

### 3.3 配置同步策略

完成基本信息和搜索条件填写后，配置以下同步策略参数：

| 参数名称 | 界面显示名称 | 说明 | 建议配置 | 适用场景 |
|------------|------------------|------|------------|------------|
| 同步方式 | 同步方式 | 指定同步的方式 | 选择自动增量、自动全量或手动全量 | 自动增量：日常变更自动全量：定期校验手动全量：大规模变更 |
| 用户同步周期 | 同步周期 | 执行同步的频率和时间 | 建议初始设置为"手动"稳定后设置为"1小时" | 测试阶段：手动生产环境：自动（1-6小时） |
| 删除用户阈值 | 删除用户阈值 | 触发删除保护的阈值 | 建议值：5% | 小型企业：10%中大型企业：5% |

**同步方式说明**：
- **自动增量同步**：
    - 只同步变更的数据，不显示删除用户阈值
    - 适用于日常运维，频繁小规模变更
    - 资源消耗较少，同步速度快
- **自动全量同步**：
    - 同步所有数据，显示同步周期和删除用户阈值
    - 适用于定期数据校验，确保数据一致性
    - 可纠正历史数据错误
- **手动全量同步**：
    - 需要手动触发同步，显示删除用户阈值
    - 适用于大规模组织变更、系统初始化
    - 不显示同步周期设置

**删除用户阈值详细说明**：

删除用户阈值是一项安全机制，用于防止因数据异常导致大量用户被误删。当同步过程中检测到需要删除的用户数量占总用户数的比例超过设定阈值时，系统会将这些用户放入"待删除用户"列表，而不是直接删除。

**计算公式**：
```
删除阈值(%) = (待删除用户数 ÷ 总用户数) × 100%
```

**不同规模企业的建议阈值设置**：
- **小型企业(\<500用户)**：建议设置为8-10%原因：小型企业用户基数小，少量用户变动可能导致较高比例变化
- **中型企业(500-2000用户)**：建议设置为5-8%原因：中型企业组织相对稳定，此阈值可以有效防止异常删除
- **大型企业(\>2000用户)**：建议设置为3-5%原因：大型企业用户数量多，较低阈值可以更好地保护数据安全

### 3.4 配置属性映射

属性映射是将LDAP目录中的用户属性与平台中的用户属性建立对应关系的过程。通过属性映射，系统可以准确地将用户数据从LDAP推送到平台，确保数据的完整性和一致性。
1. **导航至属性映射**：完成同步策略配置后，点击「下一步」或切换至「属性映射」选项卡
2. **配置用户属性映射**：根据业务需求配置用户属性映射关系
3. **配置部门属性映射**：配置部门属性的映射关系

**映射方式详细说明**：
- **不映射**：该属性不会被同步到平台
    - 适用场景：无需展示的内部属性
    - 影响：平台中不会创建或更新该属性
- **仅创建**：只在创建用户/部门时同步该属性，更新时不同步
    - 适用场景：初始化后不应被自动更新的属性，如员工编号
    - 影响：用户/部门创建后，该属性的变更不会自动同步到平台
- **创建和更新**：创建和更新用户/部门信息时均同步该属性
    - 适用场景：需要保持实时同步的核心属性，如姓名、手机号、部门名称
    - 影响：LDAP中该属性变更时，会自动同步到平台

**用户属性映射表**：

| LDAP属性 | 平台属性 | 数据类型 | 必填 | 建议映射方式 | 转换规则示例 |

| cn | name | String | 是 | 创建且更新 | 直接映射 |
| mobile | mobile | String | 是 | 创建且更新 | 直接映射 |
| mail | email | String | 否 | 创建且更新 | 直接映射 |

| departmentNumber | department\_ids | Array | 否 | 创建且更新 | 直接映射 |
| manager | manager\_id | String | 否 | 创建且更新 | 直接映射 |

## 4\. 测试与验证

### 连接测试步骤
1. **准备测试账号**：确认一个确定存在于LDAP中的有效用户账号
2. **填写测试账号**：在LDAP连接器配置页面的"测试账号"字段中填入该账号
3. **执行连接测试**：点击「测试连接」按钮
4. **查看测试结果**：系统将尝试使用配置的参数连接LDAP服务器并检索测试账号信息
5. **验证返回数据**：测试成功时，系统会显示从LDAP中检索到的测试账号详细信息
6. **检查错误信息**：如测试失败，仔细阅读错误信息，根据提示进行故障排查

**连接测试失败排查路径**：
- 错误提示"连接失败"：检查服务器地址和端口是否正确，网络连接是否畅通
- 错误提示"认证失败"：检查管理员账号和密码是否正确
- 错误提示"找不到用户"：检查BaseDN和用户搜索条件是否正确，以及测试账号是否存在
- 连接超时：检查网络连接和防火墙设置，确认LDAP服务器是否在线

### 数据同步验证方法
1. **首次同步触发**：
    - 在同步连接器配置页面，点击「保存并同步」按钮触发首次同步
    - 或在通讯录同步列表页，找到已创建的连接器，点击「立即同步」按钮
2. **同步状态监控**：
    - 在通讯录同步列表页，观察连接器的同步状态
    - 同步进行中会显示进度百分比
    - 同步完成后会显示同步结果（成功/失败）和同步时间
3. **成功完成判断标准**：
    - 同步状态显示为"同步成功"
    - 同步日志中无错误信息
    - 同步数据统计显示预期的创建、更新和删除数量
4. **数据验证**：
    - 在平台的用户管理页面，检查是否成功同步了用户数据
    - 验证关键属性（姓名、手机号等）是否同步正确
    - 验证组织架构是否正确同步

### 增量同步测试方法
1. **准备测试数据**：
    - 在LDAP中做以下测试性变更：
        - 新增一个测试用户
        - 修改一个已存在用户的信息（如职位或电话）
        - 调整用户的部门归属
        - 创建一个新的组织单位
2. **触发增量同步**：
    - 手动触发：点击连接器的"立即同步"按钮
    - 自动触发：等待下一个自动同步周期
3. **验证增量同步结果**：
    - 检查新增用户是否被同步到平台
    - 验证修改的用户信息是否在平台中更新
    - 确认用户部门变更是否生效
    - 验证新组织单位是否正确创建

### 同步日志查看和解析
1. **访问同步日志**：
    - 在通讯录同步列表页，点击目标连接器的「日志」按钮
    - 或在同步详情页面，切换至「同步日志」选项卡
2. **日志关键信息解读**：
    - 时间戳：记录操作发生的具体时间
    - 操作类型：FETCH(获取)、CREATE(创建)、UPDATE(更新)、DELETE(删除)
    - 目标对象：操作的对象类型(USER/DEPARTMENT)和ID
    - 状态码：操作结果代码，200表示成功
    - 错误信息：如操作失败，显示具体错误原因
3. **常见日志模式和含义**：
    - "开始同步数据"：同步操作启动
    - "从LDAP获取数据成功"：成功从LDAP服务器获取数据
    - "创建/更新/删除对象成功"：对应操作成功完成
    - "同步完成"：整个同步过程结束
    - "错误：xxx"：操作失败，后跟具体错误信息

### 5.2 待删除用户处理

**待删除用户功能机制和意义**：

待删除用户功能是专属集成平台的一项重要安全机制，旨在防止因LDAP数据异常导致大量用户被错误删除。当系统检测到需要删除的用户数量超过设定的阈值时，这些用户会被自动标记为"待删除"状态，而不会被立即从系统中删除。这些用户会被放入待删除用户列表，等待管理员进行人工审核和确认。

只有经过管理员确认后，这些用户才会在下次同步中被实际删除。这一机制为企业提供了额外的数据安全保障，防止因LDAP数据异常、网络问题或配置错误导致的大规模用户误删，保护企业核心业务系统的稳定运行。

**待删除用户管理界面访问路径**：
1. 登录专属集成平台管理控制台
2. 在左侧导航栏选择「用户中心 → 通讯录同步」
3. 找到目标LDAP同步连接器，点击「待删除用户」按钮
4. 系统将显示当前所有处于待删除状态的用户列表

**待删除用户信息解读**：
- **用户基本信息**：显示用户姓名、员工编号、部门、邮箱等基本信息
- **删除原因**：标明用户被标记为待删除的原因，常见原因包括：
    - "源数据不存在"：LDAP中该用户已被删除
    - "不在同步范围"：用户所在组织不在当前配置的搜索范围内
    - "搜索条件过滤"：用户不满足配置的用户搜索条件
- **待删除时间**：用户被标记为待删除状态的时间
- **来源信息**：标明用户来自哪个LDAP服务器

**处理操作的详细步骤**：
1. **单个用户确认删除**：
    - 在待删除用户列表中，找到目标用户
    - 点击操作列中的「确认删除」按钮
    - 在弹出的确认对话框中，再次确认操作
    - 系统将在下次同步时删除该用户
2. **批量确认删除**：
    - 在待删除用户列表中，选中多个需要删除的用户（可使用列表顶部的复选框全选）
    - 点击列表上方的「批量确认删除」按钮
    - 在弹出的确认对话框中，查看选中的用户数量并确认操作
    - 系统将在下次同步时批量删除所选用户
    - **注意事项**：批量删除前务必仔细核对用户列表，确保不会误删重要账号
3. **拒绝删除操作**：
    - 在待删除用户列表中，找到不应删除的用户
    - 点击操作列中的「拒绝删除」按钮
    - 在弹出的对话框中说明拒绝原因（可选）
    - 确认操作后，该用户将被从待删除列表中移除，保持在系统中的现有状态

## 6\. 常见问题与故障排除

### 同步失败的可能原因和解决方案

| 问题类别 | 问题描述 | 可能原因 | 解决方案 |
|------------|------------|------------|------------|
| 连接问题 | 无法连接到LDAP服务器 | 服务器地址错误或服务器不可用 | <ul><li>1. 检查服务器地址和端口是否正确2\. 确认LDAP服务器是否在线3\. 检查网络连接，确认防火墙配置</li></ul> |
| 认证问题 | LDAP绑定失败 | 管理员账号或密码错误 | <ul><li>1. 检查管理员账号和密码是否正确2\. 确认管理员账号是否有效3\. 尝试使用LDAP浏览工具验证账号</li></ul> |
| 搜索问题 | 无法获取用户或组织数据 | BaseDN或搜索条件配置错误 | <ul><li>1. 验证BaseDN是否正确2\. 检查用户搜索条件和组织搜索条件3\. 使用LDAP浏览工具测试搜索查询</li></ul> |
| 属性映射问题 | 同步成功但数据不完整 | 属性映射配置不正确 | <ul><li>1. 检查LDAP属性名称是否正确2\. 验证必填属性是否已映射3\. 确认属性数据类型是否匹配</li></ul> |
| 数据格式问题 | 同步失败，提示数据格式错误 | LDAP中的数据格式不符合要求 | <ul><li>1. 检查LDAP中特殊字符或格式问题2\. 验证日期、电话等特殊格式字段3\. 考虑使用转换规则处理特殊格式</li></ul> |
| 性能问题 | 同步超时或极慢 | LDAP数据量过大或查询效率低 | <ul><li>1. 优化搜索条件，减少数据范围2\. 增加LDAP服务器资源3\. 调整同步周期，错开高峰期</li></ul> |

### 数据不完整问题排查方法
1. **LDAP数据验证**：
    - 使用LDAP浏览工具（如Apache Directory Studio）直接连接LDAP服务器
    - 执行相同的搜索查询，验证数据是否存在且完整
    - 检查关键属性（如uid、cn、mail等）是否有值
2. **属性映射验证**：
    - 检查属性映射配置，确保必要字段已正确映射
    - 验证LDAP属性名称拼写是否正确（区分大小写）
    - 确认数据类型转换是否有效（如日期格式）
3. **搜索条件验证**：
    - 检查用户和组织搜索条件是否过于严格
    - 临时放宽搜索条件，测试是否能获取更多数据
    - 验证BaseDN是否涵盖所有需要同步的数据范围

### 组织架构不匹配问题解决
1. **结构层级问题**：
    - LDAP的组织结构与平台结构可能存在差异
    - 检查LDAP中的组织单位（OU）结构
    - 确认父子关系映射是否正确
2. **解决方法**：
    - 调整LDAP中的组织结构以更好地匹配目标系统
    - 或自定义属性映射逻辑，适应不同的结构模型
    - 考虑使用中间表或转换规则处理特殊结构
3. **特殊场景处理建议**：
    - 扁平结构转层级结构：使用分隔符或命名规则创建层级
    - 多父级问题：确定主要父级，其他关系通过辅助属性维护
    - 虚拟组织：考虑使用特殊标识或前缀标记虚拟组织单位

## 7\. 最佳实践

### 初始配置建议
1. **配置前的准备工作**：
    - 全面了解LDAP目录结构，绘制组织架构图
    - 确认LDAP中必要字段（如uid、cn、mail等）的数据完整性
    - 准备测试账号，确保包含各种典型用户情况（不同部门、职位等）
    - 制定同步策略、范围和计划，包括回滚方案
2. **分步骤实施建议**：
    - **第一阶段 - 连接验证**：
        - 配置基本连接参数（服务器地址、端口、管理员账号等）
        - 使用测试账号验证连接是否成功
        - 确认能正确检索LDAP数据
    - **第二阶段 - 小范围测试**：
        - 配置有限范围的搜索条件（限定特定部门）
        - 设置关键字段的属性映射
        - 执行手动同步并验证结果
    - **第三阶段 - 全面部署**：
        - 扩大搜索范围至完整组织
        - 完善所有属性映射
        - 设置适当的同步周期和删除阈值
    - **第四阶段 - 监控与优化**：
        - 建立同步监控机制
        - 定期检查同步日志和数据一致性
        - 根据实际情况调整配置参数
3. **避免常见配置错误的方法**：
    - BaseDN设置过高（层级过浅）可能导致数据不完整，应确保包含所有需要同步的数据
    - 搜索条件设置过于严格可能过滤掉有效用户，初期可适当放宽
    - 避免在生产环境直接测试配置变更，先在测试环境验证
    - 保留初始配置备份，方便出现问题时回滚

### 运行维护建议
1. **日常监控要点**：
    - 定期检查同步状态，确保按计划执行且成功完成
    - 监控LDAP服务器负载，避免同步操作影响LDAP服务性能
    - 定期抽样对比LDAP数据和同步后的数据，验证一致性
    - 关注异常同步日志，特别是批量用户变动情况
2. **周期性维护任务**：
    - 每日：检查同步日志，确认正常运行
    - 每周：处理待删除用户列表，确认是否有异常情况
    - 每月：执行一次全量同步，确保数据完整性和一致性
    - 每季度：评估和优化同步策略，调整参数配置
3. **性能优化策略**：
    - 合理设置搜索范围和搜索条件，减少不必要的数据检索
    - 优化LDAP索引，提高查询效率
    - 错开同步时间，避免在业务高峰期执行大量同步操作
    - 合理调整同步周期，根据数据变化频率设置适当的增量同步间隔

### LDAP同步安全建议
1. **LDAP连接安全**：
    - 优先使用LDAPS（加密连接），而非标准LDAP
    - 管理员账号应具有只读权限，避免使用完全管理权限
    - 定期更换管理员密码，遵循密码复杂度要求
2. **数据安全措施**：
    - 敏感字段（如密码哈希）不应被同步
    - 考虑对特定敏感信息（如手机号、身份证号）进行掩码处理
    - 实施数据访问审计，监控异常数据访问行为
3. **删除操作安全**：
    - 合理设置删除阈值，防止大量误删
    - 对于关键用户组（如管理员）考虑实施额外的删除保护机制
    - 维护已删除用户的备份记录，方便必要时恢复

## 8\. 附录

### LDAP基础概念说明

| 概念 | 说明 | 示例 |
|------|------|------|
| DN (Distinguished Name) | 目录中条目的唯一标识 | cn=John Smith,ou=IT,dc=example,dc=com |
| BaseDN | 搜索的起始点，表示目录树的根 | dc=example,dc=com |
| 对象类 (objectClass) | 定义条目可以包含的属性类型 | inetOrgPerson, organizationalUnit |
| 属性 (Attribute) | 存储在目录条目中的数据字段 | cn (Common Name), uid (User ID) |
| RDN (Relative DN) | DN相对于其父条目的部分 | cn=John Smith |
| LDIF | LDAP数据交换格式 | 用于导入/导出LDAP数据的文本格式 |

### 常见搜索条件参考

| 用途 | 搜索条件 | 说明 |
|------|------------|------|
| 标准用户搜索 | (objectClass=inetOrgPerson) | 匹配所有标准用户对象 |
| 标准组织搜索 | (objectClass=organizationalUnit) | 匹配所有组织单位 |

### 系统要求
1. **LDAP服务器要求**：
    - 支持标准LDAP v3协议
    - 推荐使用OpenLDAP、Microsoft Active Directory、ApacheDS等主流LDAP服务器
    - 建议配置适当的资源，以支持预期的用户量和查询频率
2. **网络要求**：
    - 专属集成平台必须能够访问LDAP服务器
    - LDAP连接端口(389/636)需开放访问
    - 网络延迟应低于300毫秒，以确保同步效率
    - 建议配置固定IP和防火墙规则，增强安全性
3. **性能考量**：
    - 同步10,000个用户的典型时间：初次全量同步约10-15分钟，增量同步约1-3分钟
    - LDAP服务器负载：全量同步期间CPU使用率可能增加20-30%
    - 网络带宽需求：全量同步峰值约5-10Mbps，增量同步约1-2Mbps
    - 存储需求：每千用户约需2-5MB存储空间（取决于属性数量和数据复杂度）

### 常见应用场景示例

**场景1：企业级统一身份管理**
```
配置：
- BaseDN: dc=company,dc=com
- 用户搜索条件: (objectClass=inetOrgPerson)
- 组织搜索条件: (objectClass=organizationalUnit)
- 同步方式: 自动增量
- 同步周期: 1小时
- 删除阈值: 5%

适用于大型企业环境，将LDAP作为企业身份数据的权威源，定期自动增量同步确保各系统的用户数据一致性，同时通过适当的删除阈值防止意外大规模删除。
```

**场景2：特定部门账号管理**
```
配置：
- BaseDN: ou=Finance,dc=company,dc=com
- 用户搜索条件: (&(objectClass=inetOrgPerson)(departmentNumber=FIN*))
- 组织搜索条件: (objectClass=organizationalUnit)
- 同步方式: 手动全量
- 删除阈值: 10%

适用于仅需管理特定部门(如财务部)用户的场景，通过精确设置BaseDN和搜索条件限定同步范围，手动控制同步时机，适合敏感部门的特殊管理需求。
```

**场景3：多域环境集成**
```
配置：
- 为每个域创建单独的LDAP连接器
- 域A配置:
  * BaseDN: dc=domainA,dc=com
  * 同步方式: 自动全量
  * 同步周期: 24小时
- 域B配置:
  * BaseDN: dc=domainB,dc=com
  * 同步方式: 自动全量
  * 同步周期: 24小时

适用于多域环境或企业合并场景，通过配置多个LDAP连接器分别同步不同域的用户数据，实现跨域身份集成管理。
```
