## 1\. 概述

本指南详细说明如何在专属集成平台中配置汇联易通讯录同步，实现专属集成平台系统中维护的组织架构和用户数据自动同步至汇联易。通过此同步连接器，企业可以保持内部各应用系统的数据一致性，简化身份管理流程，提高企业数据治理效率。

### 1.1 集成价值
- **数据一致性**：确保企业内部身份数据的一致性，消除信息孤岛
- **自动化同步**：减少手动维护工作，降低人为错误风险
- **提高效率**：简化用户管理流程，提高IT管理效率
- **身份统一管理**：集中管理企业内各系统的用户身份
- **权限精确控制**：基于组织架构实现精细化权限分配

### 1.2 适用场景
- **多系统身份打通**：需要将多个业务系统的用户数据通过专属集成平台同步至汇联易
- **员工生命周期管理**：员工入职、转岗、离职等状态变更需自动同步至相关系统
- **财务报销与身份管理集成**：将汇联易的组织和人员数据与企业其他业务系统集成

## 2\. 前提条件

在开始配置前，您需要确保满足以下条件：

### 2.1 账号与权限要求
- **汇联易系统要求**：
    - 获取汇联易系统的管理员账号权限
    - 获取汇联易API访问所需的企业ID和密钥
    - 具备查看和导出组织架构与用户数据的权限
- **专属集成平台要求**：
    - 拥有专属集成平台的管理员账号
    - 具备通讯录同步模块的操作权限

### 2.2 获取必要的配置信息

在配置前，需要从汇联易系统获取以下信息：

| 配置项 | 说明 | 获取方式 |
|---------|------|------------|
| 服务器地址 | 汇联易API服务器地址 | 联系汇联易技术支持获取 |
| 企业ID | 企业在汇联易系统的标识 | 汇联易管理后台-企业信息中查看 |
| 企业密钥 | 访问汇联易API的密钥 | 汇联易管理后台-企业信息中查看 |
| 公司编码 | 企业在汇联易的内部编码 | 汇联易管理后台-企业信息中查看 |
| 根组织ID | 需要同步的组织根节点ID | 汇联易管理后台-组织管理中查看 |

### 2.3 系统与网络环境
- 确保专属集成平台可以通过网络访问汇联易API服务器
- 如有防火墙，需要开放相应的网络端口和访问权限
- 建议使用稳定的网络环境，避免同步过程中断

## 3\. 配置步骤

### 3.1 创建汇联易通讯录连接器
1. 使用管理员账号登录专属集成平台
2. 在左侧导航栏选择「用户中心 → 通讯录同步」
3. 点击页面右上角的「添加连接器」按钮
4. 在连接器类型选择界面，找到并选择「汇联易」选项

### 3.2 配置基本连接信息

在连接器基本信息配置页面，填写以下参数：

| 参数名称 | 说明 | 填写指南 |
|------------|------|------------|
| 名称 | 连接器显示名称 | 自定义名称，建议使用有业务含义的名称，如"汇联易-总部" |
| 服务器地址 | 汇联易API服务器地址 | 填写完整的服务器地址，包含协议前缀(http/https) |
| 企业ID | 企业在汇联易的唯一标识 | 从汇联易管理后台获取，确保准确无误 |
| 企业密钥 | 企业API访问密钥 | 从汇联易管理后台获取，注意保密 |
| 公司编码 | 企业在汇联易的编码 | 从汇联易管理后台获取，通常为字母或数字组合 |

**填写建议**：
- **名称**：建议使用描述性名称，方便后续识别和管理
- **服务器地址**：确保填写正确的URL格式，包含http/https协议前缀
- **企业ID和密钥**：直接从汇联易复制粘贴，避免手动输入错误
- 填写完基本信息后，点击「下一步」进入高级配置

### 3.3 配置同步策略

在高级配置页面，设置以下同步参数：

| 参数名称 | 说明 | 建议配置 | 适用场景 |
|------------|------|------------|------------|
| 根组织ID | 需要同步的组织根节点ID | 填写部门ID，全部部门填写"0" | 全部同步填"0"；部分同步填特定部门ID |
| 全部同步 | 是否同步所有部门和用户 | 视需求选择"是"或"否" | 需要完整组织架构选"是"；特定范围同步选"否" |
| 同步方式 | 数据同步策略 | 初次配置建议"手动全量同步"验证后改为"自动增量同步" | 初次配置验证用"手动全量"日常运行用"自动增量" |
| 同步周期 | 自动同步的频率 | 根据数据变更频率选择通常建议6小时 | 变更频繁选较短周期(30min-1h)变更不频繁选较长周期(12h-24h) |
| 开始时间 | 自动同步起始时间 | 业务低峰期，如凌晨 | 避免业务高峰期执行同步 |
| 删除用户阈值 | 限制删除比例 | 小型企业(\<500)：20-30%中型企业(500-2000)：10-20%大型企业(\>2000)：5-10% | 详见下方"删除用户阈值说明" |

**关于删除用户阈值的详细说明**：

删除用户阈值是专属集成平台的安全机制，用于防止因配置错误或数据异常导致的大规模用户误删除。当单次同步中待删除用户的比例超过此阈值时，这些用户会被标记为"待删除"状态，需管理员确认后才执行实际删除操作。

**计算公式**：
```
删除用户比例 = (本次待删除用户数 / 已同步用户总数) × 100%
```

**阈值设置建议**：
- **小型企业(\<500用户)**：建议设置20-30%
    - 理由：用户基数小，正常情况下单次变动比例可能较大
    - 风险：过低的阈值可能导致频繁人工确认
- **中型企业(500-2000用户)**：建议设置10-20%
    - 理由：平衡自动化效率和安全性
    - 风险：单次大规模组织调整可能触发阈值
- **大型企业(\>2000用户)**：建议设置5-10%
    - 理由：大型企业通常组织相对稳定，单次变动比例不应过高
    - 风险：过高阈值可能无法及时发现异常数据同步问题

**特殊情况的阈值调整**：
- 组织大规模调整前（如部门合并、裁员）：可临时调高阈值
- 日常稳定运行期：可适当调低阈值增强安全性
- 初始数据导入期：可设置为较高值以适应数据初始化

完成同步策略配置后，点击「保存」按钮进入下一步。

### 3.4 配置属性映射

在属性映射配置页面，设置汇联易数据字段与专属集成平台字段的对应关系：

**用户属性映射建议**：

| 汇联易属性 | 专属集成平台属性 | 映射方式 | 说明 |
|---------------|------------------|------------|------|
| userId | sub | 创建且更新 | 用户唯一标识 |
| username | username | 创建且更新 | 登录名 |
| name | name | 创建且更新 | 用户姓名 |
| mobile | phone\_number | 创建且更新 | 手机号码 |
| email | email | 创建且更新 | 电子邮箱 |
| departmentId | primary*org*id | 创建且更新 | 主部门ID |
| status | active | 创建且更新 | 用户状态 |
| position | title | 创建且更新 | 职位名称 |

**部门属性映射建议**：

| 汇联易属性 | 专属集成平台属性 | 映射方式 | 说明 |
|---------------|------------------|------------|------|
| departmentId | connector*org*id | 创建且更新 | 部门唯一标识 |
| departmentName | name | 创建且更新 | 部门名称 |
| parentId | parentRefId | 创建且更新 | 父部门ID |
| orderNo | seq | 创建且更新 | 部门排序值 |
| code | code | 创建且更新 | 部门编码 |

**属性映射注意事项**：
- 确保必要字段正确映射，特别是用户和部门的唯一标识字段
- 对于重要字段，建议使用"创建且更新"映射方式
- 如汇联易中有自定义字段，可根据需要进行映射

完成属性映射配置后，点击「保存」按钮完成连接器创建。

## 4\. 验证与测试

### 4.1 连接测试
1. 在连接器配置完成后，点击「测试」按钮
2. 系统将尝试连接汇联易API并验证配置的有效性
3. 测试成功将显示"连接成功"的提示
4. 测试失败将显示错误信息，根据提示进行问题排查

**测试建议**：
- 确保填写的所有配置信息准确无误
- 验证网络连接是否正常
- 检查汇联易API服务是否可用

### 4.2 执行同步测试
1. 连接测试成功后，点击「保存」按钮创建连接器
2. 在连接器列表页，找到新创建的汇联易连接器
3. 点击「立即同步」按钮触发一次手动同步
4. 观察同步进度和状态，等待同步完成
5. 同步完成后，查看同步结果统计：
    - 成功同步的用户数量
    - 成功同步的部门数量
    - 失败记录数量及原因

**首次同步建议**：
- 首次同步建议选择非工作高峰期执行
- 对于大量数据，首次同步可能需要较长时间，请耐心等待
- 密切关注同步日志，及时发现并解决问题

### 4.3 数据验证

同步完成后，验证同步结果是否符合预期：
1. 导航至「用户管理 → 用户列表」页面
2. 使用筛选条件，选择汇联易连接器作为认证源
3. 验证同步的用户数据：
    - 抽查用户信息是否完整准确
    - 确认用户与部门的关联关系是否正确
    - 验证用户状态（启用/禁用）是否符合预期
4. 导航至「用户管理 → 组织管理」页面
5. 检查组织架构是否正确同步：
    - 验证部门层级结构是否正确
    - 确认部门名称和排序是否符合预期
    - 检查特殊组织结构的处理是否符合要求

**验证要点**：
- 用户总数是否与汇联易系统中的预期数量相符
- 部门结构是否与汇联易系统一致，特别是层级关系
- 关键用户的属性值是否正确映射
- 用户与部门的关联关系是否准确

## 5\. 常见问题与故障排除

### 5.1 连接问题

#### 5.1.1 无法连接汇联易API

**可能原因**：
- 服务器地址错误
- 企业ID或密钥填写错误
- 网络连接问题
- 汇联易API服务异常

**解决方案**：
1. 验证服务器地址是否正确，包括协议前缀(http/https)
2. 确认企业ID和密钥是否正确
3. 检查网络连通性，确认能够访问汇联易服务器
4. 联系汇联易技术支持，确认API服务状态

#### 5.1.2 认证失败

**可能原因**：
- 企业ID与密钥不匹配
- 企业账号权限不足
- 密钥已过期或被重置

**解决方案**：
1. 重新获取并验证企业ID和密钥
2. 联系汇联易管理员检查企业账号权限
3. 如密钥已重置，获取新密钥并更新配置

### 5.2 同步数据问题

#### 5.2.1 部分用户或部门未同步

**可能原因**：
- 根组织ID设置不正确
- 用户或部门状态异常
- API权限不足
- 属性映射配置不正确

**解决方案**：
1. 检查根组织ID配置，确保覆盖所需同步的范围
2. 在汇联易中确认用户和部门状态是否正常
3. 确认API访问权限是否完整
4. 检查并修正属性映射配置

#### 5.2.2 组织架构层级错误

**可能原因**：
- 部门ID和父部门ID映射不正确
- 根部门处理不当
- 特殊组织结构未正确处理

**解决方案**：
1. 检查部门属性映射，确保ID和父部门ID正确映射
2. 调整根部门的处理方式
3. 针对特殊组织结构制定专门的处理方案

#### 5.2.3 同步性能问题

**可能原因**：
- 数据量过大
- 网络连接质量不佳
- 系统资源不足

**解决方案**：
1. 考虑分批同步，先同步核心部门
2. 选择网络状况良好的时段执行同步
3. 优化服务器资源配置
4. 调整同步周期，避免频繁同步导致系统负载过高

## 6\. 最佳实践

### 6.1 初始配置建议
1. **分步实施**：
    - 从小范围测试开始，如先同步单个部门
    - 验证配置无误后，再扩大同步范围
    - 完成小范围测试后，再进行全量同步
2. **关键字段映射**：
    - 确保唯一标识字段(ID)正确映射
    - 用户名、姓名、手机号等核心字段使用"创建和更新"模式
    - 确保部门ID和父部门ID映射关系正确
3. **同步策略优化**：
    - 首次配置使用手动全量同步，验证数据准确性
    - 验证无误后，改为自动增量同步
    - 根据组织变更频率合理设置同步周期

### 6.2 运维管理建议
1. **定期监控**：
    - 定期检查同步日志，关注同步状态
    - 监控同步数据量变化，及时发现异常
    - 定期抽查同步数据的准确性
2. **变更管理**：
    - 汇联易系统组织架构发生重大变更前，提前评估影响
    - 大规模变更前调整删除用户阈值
    - 重要变更后执行全量同步，确保数据一致
3. **问题处理**：
    - 建立同步异常快速响应机制
    - 记录常见问题及解决方法
    - 与汇联易系统管理员保持沟通，及时了解系统变更

### 6.3 安全建议
1. **访问凭证保护**：
    - 安全保存企业ID和密钥，避免泄露
    - 定期更新密钥
    - 限制访问凭证的知情范围
2. **数据保护**：
    - 合理设置删除用户阈值，防止误操作
    - 实施数据变更审计
    - 定期备份关键配置和数据
3. **权限控制**：
    - 严格控制有权修改连接器配置的管理员范围
    - 建立配置变更审批流程
    - 定期审计权限设置

## 7\. 同步周期与阈值设置参考

### 7.1 同步周期参考值

| 企业规模与特点 | 推荐同步周期 | 说明 |
|---------------------|------------------|------|
| 小型企业(≤100人) | 6小时 | 数据量小，变更不频繁 |
| 中型企业(100-500人) | 3小时 | 平衡实时性和系统负载 |
| 大型企业(\>500人) | 6-12小时 | 数据量大，避免频繁同步 |
| 人员变动频繁 | 1-2小时 | 保证数据及时更新 |
| 人员变动不频繁 | 12-24小时 | 减少系统负载 |

### 7.2 删除阈值参考值

| 企业规模与特点 | 推荐阈值值 | 说明 |
|---------------------|---------------|------|
| 小型企业(≤100人) | 25-30% | 小规模企业人员变动比例可能较大 |
| 中型企业(100-500人) | 15-20% | 平衡安全性和便利性 |
| 大型企业(\>500人) | 5-10% | 大规模企业不应有高比例变动 |
| 组织架构稳定 | 5-10% | 严格控制，提高安全性 |
| 组织架构频繁调整 | 20-30% | 适应大规模变更 |
| 初始数据导入阶段 | 50-100% | 允许大规模数据变更 |
