## 概述

本文档介绍专属集成平台中通讯录数据的两种主要流转方式：**通讯录集成**和**通讯录同步**，帮助您理解两者的区别并选择适合您业务场景的方案。

## 核心概念对比

| 特性 | 通讯录集成 | 通讯录同步 |
|------|-----------|-----------|
| **数据方向** | 上游拉取 | 下游推送 |
| **典型场景** | 将第三方身份源的用户数据拉取到专属集成平台 | 将专属集成平台的用户数据推送到第三方系统 |
| **触发方式** | 事件驱动或定时拉取 | 事件驱动或定时推送 |
| **数据所有者** | 第三方系统（AD、钉钉、飞书等） | 专属集成平台 |

## 通讯录集成（上游拉取）

### 什么是通讯录集成

通讯录集成是指将第三方身份源系统（如AD、钉钉、飞书、企业微信等）的用户和组织数据拉取到专属集成平台的过程。数据的所有权和维护点在第三方系统，专属集成平台作为数据的消费者。

### 支持的身份源

| 身份源类型 | 说明 |
|-----------|------|
| **钉钉** | 拉取钉钉企业用户和组织架构 |
| **企业微信** | 拉取企业微信用户和组织架构 |
| **飞书** | 拉取飞书用户和组织架构 |
| **AD（Active Directory）** | 拉取Windows AD域用户和组织架构 |
| **LDAP** | 拉取LDAP目录用户和组织架构 |
| **Azure AD** | 拉取Azure AD用户和组织架构 |
| **SCIM** | 通过SCIM协议拉取用户数据 |
| **泛微** | 拉取泛微OA用户和组织架构 |
| **致远** | 拉取致远OA用户和组织架构 |
| **北森** | 拉取北森HR系统用户数据 |
| **睿人事** | 拉取睿人事HR系统用户数据 |
| **用友EHR** | 拉取用友EHR系统用户数据 |
| **数据库** | 从自定义数据库拉取用户数据 |

### 适用场景

- 企业已有完善的身份源系统（如AD），希望统一管理
- 需要将多个第三方系统的用户数据汇聚到统一平台
- 希望用户使用原有系统账号登录所有应用

### 配置要点

1. **配置身份源连接**：提供第三方系统的连接信息（地址、端口、凭证）
2. **配置字段映射**：将第三方系统字段映射到专属集成平台字段
3. **配置同步策略**：确定同步频率、冲突处理方式等
4. **启用JIT配置**：可选，配置是否支持Just-In-Time用户创建

## 通讯录同步（下游推送）

### 什么是通讯录同步

通讯录同步是指将专属集成平台中管理的用户和组织数据推送到第三方应用或系统的过程。数据的所有权和维护点在专属集成平台，第三方系统作为数据的接收者。

### 支持的目标系统

| 目标系统类型 | 说明 |
------------|------|
| **钉钉** | 同步用户和组织架构到钉钉 |
| **企业微信** | 同步用户和组织架构到企业微信 |
| **飞书** | 同步用户和组织架构到飞书 |
| **AD/LDAP** | 同步用户到Windows AD或LDAP目录 |
| **泛微OA-E9** | 同步用户到泛微OA-E9 |
| **网易邮箱** | 同步用户到网易邮箱系统 |
| **用友U8C** | 同步用户到用友U8C |
| **汇联易** | 同步用户到汇联易系统 |
| **企企通** | 同步用户到企企通系统 |
| **LDAP as a Service** | 通过LDAP协议同步到云目录服务 |

### 适用场景

- 企业希望在专属集成平台集中管理用户，然后分发到各个下游应用
- 需要保证多个系统间用户数据的一致性
- 希望简化下游系统的用户管理工作

### 配置要点

1. **创建同步任务**：在专属集成平台中创建新的同步任务
2. **配置目标连接**：提供目标系统的连接信息
3. **配置字段映射**：将专属集成平台字段映射到目标系统字段
4. **配置同步规则**：确定同步范围（全部用户/指定部门/指定角色）

## 如何选择

### 选择通讯录集成的情况

- 您已有主要的身份源系统（如AD、钉钉等）
- 第三方系统是用户数据的主要来源
- 您希望用户使用原有系统账号登录
- 您不希望在多个系统中重复创建用户

### 选择通讯录同步的情况

- 您希望在专属集成平台中集中管理所有用户
- 您需要将用户数据分发到多个下游系统
- 下游系统需要接收来自专属集成平台的用户更新
- 您需要控制哪些用户可以访问哪些下游系统

### 两者同时使用的情况

在复杂的企业环境中，您可能需要同时使用通讯录集成和通讯录同步：

1. **场景示例**：企业使用AD作为主要身份源，但需要将用户同步到钉钉和企业微信

2. **配置方案**：
   - 配置**通讯录集成**：将AD用户拉取到专属集成平台
   - 配置**通讯录同步**：将专属集成平台用户同步到钉钉和企业微信

3. **数据流向**：
   ```
   AD（身份源） → [通讯录集成] → 专属集成平台 → [通讯录同步] → 钉钉/企业微信
   ```

## 冲突处理策略

当同一用户从多个身份源汇聚时，可能发生数据冲突。专属集成平台提供以下冲突处理策略：

| 策略 | 说明 |
|------|------|
| **身份源优先** | 以先到达的身份源数据为准 |
| **最新优先** | 以最新更新的数据为准 |
| **手动合并** | 保留所有来源数据，由管理员手动合并 |
| **指定主身份源** | 指定一个主身份源，以其数据为准 |
