## 1\. 概述

本指南详细说明如何配置企业微信通讯录与专属集成平台的集成，实现组织架构和用户信息的自动同步。通过此集成，您可以将企业微信中的部门和人员数据无缝导入专属集成平台，简化身份管理流程，提高企业数据治理效率。

### 1.1 集成优势
- **数据自动同步**：减少手动维护人员信息的工作量，确保各系统人员数据实时一致
- **身份统一管理**：集中管理企业内各系统的用户身份，提升账号生命周期管理效率
- **权限精确控制**：基于组织架构实现精细化权限分配，降低安全风险
- **流程自动化**：支持员工入职、离职等场景的自动化流程，优化人力资源管理
- **系统互通互联**：打通企业微信与其他业务系统，实现无缝数据流转

### 1.2 适用场景
- **多系统身份打通**：实现企业微信与其他业务系统的身份信息同步，避免重复账号管理
- **基于组织架构的权限分配**：利用部门层级关系自动分配应用访问权限
- **员工生命周期管理**：员工入职、转岗、离职等状态变更自动同步至各系统
- **审批流程优化**：基于组织架构实现审批流程自动化，提高工作效率

## 2\. 使用限制说明

⚠️ **重要提示：SaaS平台使用限制**

由于企业微信通讯录集成规则限制了集成的IP地址及域名与企业微信组织必须保持一致，请勿使用SaaS平台集成多个客户的通讯录。此功能主要适用于私有化部署版本。如确有必要在SaaS平台进行集成配置，请提交工单获取专业支持。

**详细限制说明**：
1. 企业微信要求调用API的IP地址必须在企业微信管理后台的IP白名单中
2. 网页授权回调域名必须事先在企业微信后台进行登记
3. 一个企业微信应用只能对应一个组织的通讯录同步
4. 私有化部署版本可以灵活配置服务器IP和域名，因此更适合此类集成

## 4\. 专属集成平台配置

### 4.1 新增企业微信授权
1. 登录专属集成平台管理控制台
2. 导航至「参数配置 → 授权管理」
3. 点击页面右上角的「新增授权」按钮
4. 在弹出的授权类型选择界面中选择「企业微信」
5. 填写以下信息：
    - **CorpID**：填写从企业微信管理后台获取的企业CorpID
    - **AgentID**：填写从企业微信管理后台获取的应用AgentID
    - **AppSecret**：填写从企业微信管理后台获取的应用Secret
    - **名称**：输入便于识别的授权名称（如"总部企业微信"、"销售部企业微信"等）
    - **校验文件**：上传从企业微信管理后台下载的校验文件（保留文件后缀名）
6. 点击「测试连接」验证配置的有效性
7. 连接成功后，点击「确定」保存配置

**常见问题排查**：
1. **连接测试失败**：
    - 检查CorpID、AgentID、AppSecret是否正确输入，避免复制时包含多余空格
    - 确认企业微信应用是否已配置正确的API权限
    - 验证IP白名单是否已添加专属集成平台服务器IP
    - 确认校验文件是否正确上传，包含原始文件后缀
2. **校验文件错误**：
    - 确保上传的是未修改的原始校验文件
    - 校验文件名必须与企业微信后台生成的完全一致，包括文件后缀
    - 如文件无法上传，可联系技术支持协助处理
3. **权限不足错误**：
    - 返回企业微信管理后台，确认已开启所有必要的通讯录权限
    - 权限修改后可能需要等待几分钟生效

### 4.2 创建企业微信连接器
1. 导航至「用户中心 → 通讯录集成」
2. 点击页面右上角的「新建连接器」按钮
3. 在弹出的连接器列表中，找到并选择「企业微信集成」选项
4. 在基本信息配置页面，填写以下内容：
    - **企业选择**：从下拉菜单中选择在4.1步骤中添加的企业微信授权
    - **名称**：输入自定义连接器名称，建议使用具有业务含义的名称（例：总部人员同步、研发部门同步等）
    - **描述**：简要描述连接器的用途，便于后续管理（可选）
5. 点击「下一步」进入同步策略配置

**连接器命名最佳实践**：
- 使用"\[业务单位\]-\[数据类型\]-\[用途\]"的命名模式，如"总部-企微-用户同步"
- 避免使用特殊字符，以防在日志搜索时出现问题
- 若有多个同类连接器，可在名称中包含环境标识（如"生产"、"测试"）

### 4.3 配置同步策略

在同步策略配置页面，您需要设置数据同步的范围、方式和条件。以下是各配置项的详细说明：

| 参数名称 | 界面显示名称 | 说明 | 建议配置 | 适用场景 |
|------------|------------------|------|------------|------------|
| 上游根组织ID | 上游根组织ID | 指定企业微信中的起始同步节点ID，为空则同步整个组织 | 大型组织建议按部门设置，避免首次同步量过大 | 希望仅同步特定部门时使用 |
| 同步范围 | 同步范围 | 指定同步的数据范围，可选"用户和部门"或"仅用户" | 「用户和部门」 | 需完整复制组织架构时选"用户和部门"；仅需用户数据时选"仅用户" |
| 本地挂载节点 | 在本地挂载节点 | 指定数据同步到平台后的认证方式 | 「SAAS认证」 | 标准集成场景建议使用SAAS认证 |
| 挂载方式 | 挂载方式 | 数据同步到平台的处理方式 | 「在选择节点下新建」 | 希望保持独立组织树时使用 |
| 同步方式 | 同步方式 | 决定同步的触发方式和范围 | 初次配置选择「手动全量同步」，验证无误后改为「自动增量同步」 | 初次配置验证时使用手动全量；日常运行使用自动增量 |
| 同步周期 | 同步周期 | 自动同步的时间间隔 | 根据企业人员变动频率选择，一般建议设置为「6小时」 | 人员变动频繁时可设为较短周期，如1小时 |
| 开始时间 | 开始时间 | 指定同步执行的起始时间点 | 建议选择业务低峰期，如午夜或凌晨 | 避开业务高峰期，减少系统负载 |
| 删除用户阈值 | 删除用户阈值 | 限制单次同步可删除的用户占比，超过阈值将进入待删除状态 | 小型企业(\<500): 30%\<br\>中型企业(500-2000): 20%大型企业(\>2000): 10% | 员工变动大时可适当调高；稳定期可降低以增加安全性 |
| 账号关联 | (映射规则中设置) | 设置企业微信身份源与专属集成平台的用户映射关系 | 常用"企业微信用户名与专属集成平台用户名"映射 | 需要与现有账号关联时使用 |
| 筛选规则 | 账号关联 | 基于JavaScript表达式的过滤条件 | 初次配置可不设置，后续根据需要添加 | 需要排除特定用户群体时使用 |
| 连接测试 | 连接测试 | 验证配置的有效性 | 建议使用确认存在于企业微信中的账号进行测试 | 验证连接配置正确性 |

**关于删除用户阈值的详细说明**：

删除用户阈值是专属集成平台的安全机制，用于防止因配置错误或数据异常导致的大规模用户误删除。当单次同步中待删除用户的比例超过此阈值时，这些用户会被标记为"待删除"状态，需管理员确认后才执行实际删除操作。

**计算公式**：
```
删除用户比例 = (本次待删除用户数 / 已同步用户总数) × 100%
```

**建议阈值设置**：
- **小型企业(\<500用户)**：建议设置为30%
    - 理由：用户基数小，正常情况下单次变动比例可能较大
    - 风险：过低的阈值可能导致频繁人工确认
- **中型企业(500-2000用户)**：建议设置为20%
    - 理由：平衡自动化效率和安全性
    - 风险：单次大规模组织变更可能触发阈值
- **大型企业(\>2000用户)**：建议设置为10%
    - 理由：大型企业通常组织相对稳定，单次变动比例不应过高
    - 风险：过高阈值可能无法及时发现异常数据同步问题

**阈值触发后的系统行为**：
1. 系统将待删除用户标记为"待删除"状态
2. 在连接器详情页的"待删除用户"选项卡中显示这些用户
3. 发送通知给管理员（如已配置通知）
4. 这些用户仍然保留在系统中，直到管理员确认删除

**特殊情况的阈值调整**：
- 组织大规模调整前（如部门合并、裁员）：可临时调高阈值
- 日常稳定运行期：可适当调低阈值增强安全性
- 初始数据导入期：可设置为较高值（如50%）以适应数据初始化

### 4.4 配置属性映射

属性映射决定了企业微信中的用户和组织信息如何对应到专属集成平台中的字段。合理的属性映射是确保数据完整性和准确性的关键。

\[图片：属性映射配置界面截图位置\]

**属性映射方式说明**：
- **不映射**：不同步该属性至平台，属性值将保持为空或默认值。适用于不需要在平台中使用的属性或敏感数据。
- **仅创建**：仅在创建用户时同步该属性，后续更新用户信息时不会变更该属性值。适用于固定不变或仅初始化时需要的属性，如初始密码。
- **创建和更新**：创建和更新用户信息时均同步该属性，确保数据始终与源系统保持一致。适用于需要实时更新的属性，如姓名、手机号等。

**映射颜色标识含义**：
- **紫色**：表示"创建和更新"映射方式，数据会持续保持同步
- **绿色**：表示"仅创建"映射方式，仅在首次创建时同步
- **灰色**：表示"不映射"，该字段不会从企业微信同步到平台

#### 4.4.1 用户属性映射

在"属性映射"配置页面的"用户"标签页中，可配置以下映射关系。

**注意**：企业微信API直接提供的人员属性有限，某些属性（如手机号等）需要在企业微信中设置特定的可见范围或通过扩展属性获取。

| 企业微信属性 | 专属集成平台属性 | 数据类型 | 必填 | 建议映射方式 | 转换规则示例 |
|------------------|------------------|------------|------|------------------|------------------|
| appuser.userid | sub | String | 是 | 创建且更新 | 直接映射 |
| appuser.name | name | String | 是 | 创建且更新 | 直接映射 |
| appuser.mobile | phone\_number | String | 否 | 创建且更新 | 直接映射 |
| appuser.email | email | String | 否 | 创建且更新 | 直接映射 |
| appuser.department\[0\] | primary*org*id | String | 否 | 创建且更新 | 主部门ID |
| appuser.department | connector*org*ids | Array | 否 | 创建且更新 | 所有部门ID |
| appuser.status | active | Boolean | 是 | 创建且更新 | appuser.status==1 ? false : true |
| appuser.gender | gender | String | 否 | 创建 | appuser.gender1 ? "男" : (appuser.gender2 ? "女" : "未知") |
| appuser.position | title | String | 否 | 创建且更新 | 直接映射 |
| appuser.alias | nickname | String | 否 | 创建且更新 | 直接映射 |

**企业微信可用人员属性**：
- userid：成员UserID
- name：成员名称
- mobile：手机号码
- department：部门ID列表
- position：职务信息
- email：邮箱
- avatar：头像URL

- telephone：座机
- alias：别名
- status：激活状态
- gender：性别

#### 4.4.3 角色/标签属性映射（如适用）

企业微信的标签功能可映射为专属集成平台的角色，实现基于标签的权限管理：

| 企业微信属性 | 专属集成平台属性 | 数据类型 | 必填 | 建议映射方式 | 转换规则示例 |
|------------------|------------------|------------|------|------------------|------------------|
| approle.tagid | connector*role*id | String | 是 | 创建且更新 | 直接映射 |
| approle.tagname | name | String | 是 | 创建且更新 | 直接映射 |
| approle.userlist | user\_ids | Array | 否 | 创建且更新 | 直接映射 |

**常见映射问题及解决方法**：
1. **无法获取手机号等敏感信息**：
    - 解决方案：确保企业微信应用有足够权限，并在企业微信后台设置允许该应用获取敏感信息
    - 操作步骤：在企业微信管理后台 - 应用管理 - 选择应用 - 企业微信授权 - 勾选"通讯录联系方式"等敏感权限
2. **部门层级结构不正确**：
    - 解决方案：检查parentid映射是否正确，特别是根部门（通常parentid为1或0）的处理
    - 映射示例：`parentid != 1 ? parentid : null`（假设1为根部门ID）
3. **用户部门关系处理**：
    - 解决方案：对于多部门用户，设置主部门逻辑
    - 映射示例：`appuser.department.length > 0 ? appuser.department[0] : ""`
4. **特殊字符处理**：
    - 解决方案：使用替换函数移除特殊字符
    - 映射示例：`appuser.name.replace(/[^\w\s]/gi, "")`

## 5\. 测试与验证

### 5.1 连接测试
1. 在同步策略配置页面底部，找到「连接测试」区域
2. 在测试账号输入框中，输入一个确认存在于企业微信中的账号ID或名称
3. 点击「测试」按钮
4. 系统将尝试连接企业微信API并验证授权有效性
5. 测试成功将显示绿色的"连接成功"提示和账号基本信息
6. 如测试失败，系统会显示错误信息，请参考错误代码进行问题排查

### 5.2 创建连接器并同步数据
1. 完成所有配置后，点击「保存并下一步」或「确定」按钮创建连接器
2. 在连接器列表中找到新创建的连接器，点击其名称进入详情页
3. 在连接器详情页面顶部，点击「立即同步」按钮触发一次手动同步
4. 系统会开始同步过程，状态栏会显示当前同步进度
5. 等待同步过程完成，可能需要几分钟到几十分钟不等（取决于数据量大小）
6. 同步完成后，系统会显示同步结果，包括成功/失败数量和详细日志

### 5.3 验证同步结果
1. 导航至「用户管理 → 用户列表」页面
2. 使用筛选条件，选择新建连接器作为认证源
3. 验证是否成功同步用户数据的方法：
    - 检查用户总数是否与企业微信中的预期数量相符
    - 抽查几个关键用户，确认其属性是否正确映射（尤其是姓名、邮箱等）
    - 验证组织架构是否与企业微信中的结构一致
    - 检查部门层级和用户所属部门关系是否正确
4. 针对不同类型数据的验证重点：
    - **用户数据**：身份信息完整性、核心属性映射正确性
    - **部门数据**：部门层级关系、部门名称、排序顺序
    - **标签/角色数据**（如适用）：标签分配、标签名称

### 5.4 增量同步测试

    - 添加或移动测试用户
2. 在专属集成平台连接器详情页中：
    - 手动方式：点击「立即同步」按钮触发一次手动同步
    - 自动方式：等待下一个自动同步周期（根据配置的同步周期）
3. 同步完成后，验证变更是否正确同步到平台：
    - 检查修改的用户信息是否更新
    - 验证结构变更是否正确反映
    - 确认新增用户是否被添加
4. 监控同步日志，确认只有变更的数据被处理，而非全量数据

### 6.2 待删除用户处理

"待删除用户"是专属集成平台的重要安全机制，当同步过程中发现大量用户需要删除且比例超过配置的删除阈值时，这些用户会被标记为待删除状态而非直接删除，需要管理员审核确认后才会执行真正的删除操作。

**待删除用户功能机制和意义**：
- 当单次同步中待删除用户比例超过阈值时，系统自动将这些用户标记为"待删除"
- 这是一个重要的安全措施，防止因企业微信数据异常或API错误导致的大量用户误删
- 只有经过管理员明确确认后，这些用户才会从系统中实际删除
- 此机制特别适用于大型企业或关键业务系统，为管理员提供额外的安全检查点

**待删除用户管理操作流程**：
1. 导航到连接器详情页，切换到「待删除用户」选项卡
2. 系统显示所有待删除用户列表，包含以下信息：
    - **姓名**：用户显示姓名
    - **用户名**：登录账号
    - **用户ID**：系统唯一标识符
    - **部门**：所属部门信息
    - **来源**：标记用户来自哪个同步操作
    - **删除原因**：标记删除触发的原因（如"源系统不存在"）
3. 处理待删除用户的操作选项：
    - **确认删除**（单个）：点击单个用户行末的"确认"按钮
    - **批量确认删除**：选择多个用户，点击列表上方的"批量确认"按钮
    - **拒绝删除**：选择用户，点击"拒绝删除"，这些用户将保留在系统中
    - **导出列表**：点击"导出"按钮，将待删除用户列表导出为Excel文件，便于离线审核
4. 确认删除的操作后果：
    - 确认删除后，用户将从专属集成平台中彻底删除
    - 用户的所有访问权限将被撤销
    - 相关的授权记录会被删除
    - 操作不可逆，需谨慎执行

<img src="/images/通讯录集成-企业微信_images/8d04f8a8-7183-47dc-badd-362fa5785172.png" alt="image.png" />

**安全建议和最佳实践**：
- 始终先导出待删除用户列表进行备份，再执行批量删除操作
- 对于大批量删除操作（超过100人），建议分批次处理
- 执行删除前，与相关业务部门确认这些用户是否确实已离职或调岗
- 设置操作权限，限制有权处理待删除用户的管理员范围
- 保持删除阈值在合理范围，避免过高（无安全保障）或过低（频繁人工干预）

### 7.2 数据不完整问题排查方法

**源数据完整性检查**：
1. 登录企业微信管理后台，确认用户和部门信息是否完整
2. 特别关注必要字段（如姓名、邮箱、手机号等）是否已填写
3. 检查特殊字符或格式（如国际电话号码、非标准邮箱等）

**权限范围验证**：
1. 确认企业微信应用已获得所有必要的API权限
2. 检查应用的可见范围设置是否包含所有需要的部门
3. 验证敏感信息权限（如手机号、邮箱等）是否已授权

**属性映射验证**：
1. 检查关键字段的映射方式（不映射/仅创建/创建和更新）
2. 确认自定义表达式语法正确（如数据处理函数）
3. 测试特殊情况处理（如null值、空字符串处理）

**敏感信息获取问题**：
1. 检查企业微信应用是否被授予获取敏感信息的权限
2. 确认企业微信后台的隐私保护设置
3. 联系管理员调整信息可见性设置
4. 考虑采用其他属性或扩展属性作为替代方案

### 7.3 组织架构不匹配问题解决

**结构层级问题**：
- **原因**：部门父子关系映射不正确或部门ID引用错误
- **解决方法**：
    1. 检查部门属性中parentid的映射配置
    2. 确认企业微信部门ID与父部门ID的对应关系
    3. 特别注意根部门的处理（通常parentid为1）
    4. 执行全量同步，刷新整个组织结构

**部门属性问题**：
- **原因**：部门名称、排序等属性映射不当或格式不兼容
- **解决方法**：
    1. 检查部门名称、排序字段的映射规则
    2. 确认特殊字符处理（如括号、换行符等）
    3. 验证数字类型字段的格式转换
    4. 为空值或无效值设置合理的默认值

**特殊组织结构处理方法**：
- **矩阵式组织**：
    1. 企业微信支持用户属于多个部门，确保正确映射多部门关系
    2. 设置主部门的判断逻辑（通常使用department数组的第一个元素）
    3. 考虑使用标签/角色补充组织关系
- **跨公司组织**：
    4. 使用命名规则区分不同公司的部门
    5. 考虑设置多个连接器分别同步不同公司
    6. 使用前缀或后缀明确标识组织来源

### 7.4 获取敏感信息的技巧

企业微信默认对某些敏感信息（如手机号、邮箱）有访问限制，以下是获取这些信息的方法：
1. **权限设置**：
    - 在企业微信管理后台 → 应用管理 → 选择应用 → 企业微信授权中
    - 勾选"通讯录联系方式"等敏感信息权限
    - 注意需要企业管理员审批此类权限
2. **可见范围调整**：
    - 确保应用的可见范围包含需要同步的所有部门
    - 权限仅对可见范围内的成员有效
3. **自定义字段**：
    - 如果无法直接获取某些信息，可考虑使用企业微信的自定义字段功能
    - 在管理后台 → 通讯录设置 → 字段与标签中添加自定义字段
4. **API参数优化**：
    - 使用企业微信API的用户详情接口时，确保包含fetch\_child参数
    - 部分信息可能需要单独API调用获取，如外部联系人信息

### 8.3 删除阈值设置建议

**小型企业(\<500用户)**：
- **建议值**：25-30%
- **依据**：小型组织结构变动较频繁，单次变更比例可能较大
- **调整策略**：
    1. 组织相对稳定时可降至20%
    2. 大规模调整期间可临时提高至40-50%
    3. 业务关键系统可适当降低至15-20%

**中型企业(500-2000用户)**：
- **建议值**：15-20%
- **依据**：平衡自动化效率和安全性，适应正常组织变动
- **调整策略**：
    1. 季度组织调整前可临时提高至25%
    2. 敏感系统可降低至10-15%
    3. 多业务线可按部门设置不同阈值

**大型企业(\>2000用户)**：
- **建议值**：5-10%
- **依据**：大型组织通常结构相对稳定，大比例变更应引起警觉
- **调整策略**：
    1. 分支机构调整可临时提高特定连接器阈值
    2. 核心业务系统建议维持在5%以下
    3. 实施分级阈值管理，不同部门设置不同标准

- **概要日志**：显示每次同步的基本信息（时间、状态、数据量等）
- **详细日志**：点击具体同步记录后可查看的API调用和处理细节
- **错误日志**：记录同步过程中的错误和异常情况
- **操作日志**：记录管理员执行的手动操作（如确认删除）

**常用日志分析方法**：
1. **模式识别**：分析多次失败是否存在共同特征
2. **时间相关性**：检查失败是否与特定时间段相关
3. **数据量分析**：观察大数据量同步时的性能和稳定性
4. **错误代码分类**：按错误类型分组，找出高频问题
5. **解决方案效果验证**：比较问题修复前后的日志差异
