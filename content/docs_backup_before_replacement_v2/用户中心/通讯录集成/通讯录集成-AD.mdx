## 1\. 概述

本指南详细说明如何在专属集成平台中配置Active Directory(AD)连接器，实现将AD系统中统一维护的组织架构和用户数据同步至平台，并可进一步同步至钉钉等应用，确保企业内各应用系统的身份数据一致性。

通过AD集成，您可以获得以下好处：
- 从现有AD基础设施集中管理用户，减少多系统维护工作
- 自动化用户配置和取消配置流程，降低管理复杂度
- 利用单点登录(SSO)功能简化用户访问体验\*: 显著减少管理开销和人工操作
- 通过一致的访问控制策略提高整体安全性
- 确保各系统身份数据同步一致，避免信息孤岛

## 2\. 前提条件

在开始配置前，请确保满足以下条件：

### 2.1 网络连接要求
- **专属集成平台和AD服务器之间的网络连接**
    - 使用 代理网关：可实现公网到内网的连接（平台在公网，AD在内网）
    - 不使用网关：两个系统必须在同一网络或能够直接通信
    - 确保相关端口开放（默认389用于LDAP，636用于LDAPS）

### 2.2 AD服务账户准备
- **创建专用服务账户**
    - 建议使用专门的服务账户进行集成，避免使用个人账户或域管理员
    - 设置"密码永不过期"属性，但仍应定期更换密码（90-180天）
    - 基本权限：对目标OU和用户对象的读取访问权限
    - 如需密码回写功能，还需额外权限（详见第4节）

### 2.3 所需AD信息
- **基本连接信息**
    - AD服务器地址(FQDN或IP地址)
    - LDAP端口(通常为389，或LDAPS的636)
    - 服务账户用户名和密码
    - 目标组织的BaseDN（基准专有名称）
    - 如使用TLS/SSL，还需相关证书信息

### 2.4 平台权限
- **专属集成平台管理员权限**
    - 确保您拥有创建和管理连接器的权限
    - 熟悉平台的用户和组织管理功能

## 3\. 理解关键AD概念

### 3.1 专有名称(Distinguished Names, DN)

专有名称在AD中唯一标识一个对象，是一个分层结构的字符串。例如：
```
CN=张三,OU=销售部,OU=用户,DC=公司,DC=com
```

各部分含义：
- **CN (Common Name)**: 对象的通用名称
- **OU (Organizational Unit)**: 组织单位
- **DC (Domain Component)**: 域组件

### 3.2 BaseDN查找方法

BaseDN确定将同步AD目录的哪一部分，是配置的关键参数：

**获取BaseDN方法**：
1. 打开"Active Directory用户和计算机"
2. 右键点击要同步的OU（如SSO部门）
3. 选择"属性"
4. 转到"属性编辑器"（如不可见，请从菜单选择"查看→高级功能"）
5. 查找"distinguishedName"属性
6. 复制此值作为BaseDN

BaseDN示例：
- 整个域：`DC=company,DC=com`
- 特定部门：`OU=IT部门,DC=company,DC=com`

### 3.3 常用AD属性参考

| 属性名 | 说明 | 用途 |
|---------|------|------|
| sAMAccountName | Windows登录用户名 | 用户标识，登录名 |
| userPrincipalName | 用户主体名称(user@domain格式) | SSO标识 |
| mail | 电子邮件地址 | 联系方式 |
| displayName | 显示全名 | 界面显示 |
| objectGUID | AD对象唯一标识符 | 对象标识 |
| memberOf | 用户所属的组 | 组成员关系 |
| telephoneNumber | 电话号码 | 联系方式 |
| mobile | 手机号 | 移动联系方式 |
| title | 职位 | 角色信息 |
| department | 部门 | 组织归属 |

## 4\. AD服务账户权限配置

### 4.1 只读同步所需权限

如果只需同步用户数据而不需要回写（如密码修改），服务账户需要以下最小权限：
1. **基本读取权限配置**
    - 打开"Active Directory用户和计算机"
    - 右键点击目标OU（要同步的组织单位）
    - 选择"属性" → "安全" → "添加"
    - 添加服务账户
    - 勾选权限：
        - "读取所有属性"
        - "列出内容"
        - "读取权限"
2. **权限配置示例（PowerShell）**
```
$ou = "OU=员工,DC=公司,DC=com" # 替换为您的OU路径
$serviceAccount = "svc- -ad" # 替换为您的服务账户名
$acl = Get-Acl "AD:$ou"
$sid = (Get-ADUser $serviceAccount).SID

# 创建只读访问控制条目
$objectGuid = New-Object Guid "4c164200-20c0-11d0-a768-00aa006e0529" # 用户对象GUID
$inheritanceType = [System.DirectoryServices.ActiveDirectorySecurityInheritance]::Descendents
$adRights = [System.DirectoryServices.ActiveDirectoryRights]::ReadProperty -bor [System.DirectoryServices.ActiveDirectoryRights]::ReadControl
$accessControlType = [System.Security.AccessControl.AccessControlType]::Allow
$ace = New-Object System.DirectoryServices.ActiveDirectoryAccessRule($sid, $adRights, $accessControlType, $objectGuid, $inheritanceType)

# 添加访问控制条目并应用
$acl.AddAccessRule($ace)
Set-Acl -AclObject $acl -Path "AD:$ou"
```

### 4.2 密码回写功能配置

专属集成平台支持密码回写功能，允许用户在平台中修改密码并同步回AD。此功能需要额外的服务账户权限：
1. **密码重置权限配置**
    - 打开"Active Directory用户和计算机"
    - 右键点击目标OU
    - 选择"委派控制"
    - 点击"下一步" → "添加" → 添加服务账户
    - 点击"下一步" → 选择"创建自定义任务"
    - 点击"下一步" → 选择"仅以下对象" → 勾选"用户对象"
    - 点击"下一步" → 勾选：
        - "重置密码"
        - "读取和写入所有属性"
        - "写入pwdLastSet属性"
    - 完成委派设置
2. **密码回写安全要求**
    - 必须启用TLS加密连接保护密码传输
    - 配置连接器时选择"是"启用TLS选项
    - 确保AD服务器有有效的SSL证书
    - 定期审计密码操作日志
3. **特殊AD字段的LDAP过滤器**
    - 识别已启用用户的过滤器示例：```(&(objectCategory=person)(objectClass=user)(!(userAccountControl:1.2.840.113556.1.4.803:=2)))```
    - 识别密码永不过期用户的过滤器：```(userAccountControl:1.2.840.113556.1.4.803:=65536)```
    - 识别锁定用户的过滤器：```(userAccountControl:1.2.840.113556.1.4.803:=16)```

### 5.3 配置同步策略

选择适合的同步策略和相关参数：

| 参数说明 | 说明 | 建议配置 |
|------------|------|------------|
| **同步方式** | 决定数据同步模式 |  |
|  | **自动增量**：只同步变更数据，显示同步周期，删除用户阈值不展示 | 生产环境推荐 |
|  | **自动全量**：同步所有数据，显示同步周期，删除用户阈值展示 | 适用于定期数据校验 |
|  | **手动全量**：手动触发同步，不显示同步周期，删除用户阈值展示 | 测试阶段推荐 |
| **用户同步周期** | 自动同步执行频率 |  |
|  | 支持手动、30min、1小时、6小时、12小时、24小时 | 根据数据实时性需求选择 |
|  | 可指定具体执行时间 | 建议选择非业务高峰期 |
| **同步用户组** | 是否将上游组织部门集成到专属集成平台 | 建议启用 |
|  | 不勾选则用户存放到指定部门或默认组中 |  |
| **是否创建默认组** | 组织结构导入方式 | 根据需求选择 |
|  | 勾选：在指定部门下创建根节点 | 保持独立层级结构 |
|  | 不勾选：与指定部门合并，指定部门作为根 | 整合到现有结构 |
| **删除用户阈值** | 设置当源系统变动的阈值保护比例 | 建议10% |
|  | 阈值 =（平台已回收数据和本次回收数据的差异值 / 已回收数据）\* 100% |  |
|  | 当超过设定阈值，平台不执行禁用/删除操作 | 防止意外大规模删除 |
| **账号关联** | 源系统与目标系统账号映射关系 | "AD用户名与专属集成平台用户名" |
| **测试账号** | 填写AD中存在的用户名进行测试 | 选择确定存在的标准用户 |

**同步策略建议**：
- 初始测试阶段使用"手动全量"，便于控制和验证
- 生产环境使用"自动增量"，减少系统和网络负载
- 大型环境(\>1万用户)建议安排在非工作时段执行同步
- 删除阈值建议按企业规模设置：小型(\<500用户)可设15-20%，大型(\>2000用户)建议5-10%

### 5.4 配置属性映射

完成基本配置后，需要设置AD属性与平台属性的映射关系：
1. 点击「保存并下一步」进入属性映射页面
2. 或导航至「用户中心 \> 属性管理」找到已创建的AD连接器，点击「映射」

**属性映射支持**：
- **用户属性映射**：AD用户属性映射到平台用户属性
- **组织属性映射**：AD组织单位属性映射到平台组织属性

**推荐映射关系**：

| AD属性 | 平台属性 | 建议映射方式 | 转换规则 |
|--------|------------|------------------|------------|
| objectGUID | sub | 仅创建 | 直接映射 |
| sAMAccountName | username | 创建且更新 | =TOLOWERCASE(sAMAccountName) |
| displayName | name | 创建且更新 | 直接映射 |
| mail | email | 创建且更新 | =TOLOWERCASE(mail) |
| telephoneNumber | telephone\_number | 创建且更新 | 直接映射 |
| mobile | phone\_number | 创建且更新 | =TRIM(mobile) |
| title | position | 创建且更新 | 直接映射 |
| department | department | 创建且更新 | 直接映射 |

**映射方式说明**：
- **不映射**：该属性不会同步到平台
- **仅创建**：仅在首次创建用户时同步该属性，后续更新不同步
- **创建且更新**：创建和更新用户时都同步该属性

**数据转换函数**：
- TOLOWERCASE()：转为小写，用于规范化用户名和邮箱
- TRIM()：去除前后空格
- CONCAT()：连接字符串，如：=CONCAT(firstName, " ", lastName)
- JSON.PARSE()：将文本转换为JSON对象，用于复杂结构

## 6\. 常见问题与故障排除

### 6.1 连接问题
1. **连接超时**
    - **原因**：网络连接问题、防火墙阻止LDAP端口、AD服务器地址错误
    - **解决方法**：
        - 使用telnet测试连接（如：`telnet ad.company.com 389`）
        - 检查防火墙规则，确保LDAP端口开放
        - 验证服务器地址正确性
2. **SSL/TLS证书错误**
    - **原因**：自签名证书未被信任、证书过期、证书主体与服务器地址不匹配
    - **解决方法**：
        - 更新或重新生成有效证书
        - 导入证书到信任存储
        - 确保使用证书中包含的服务器名称
3. **认证失败**
    - **原因**：服务账户密码错误、账户被锁定、用户名格式不正确
    - **解决方法**：
        - 重置服务账户密码
        - 检查账户状态，解锁被锁定的账户
        - 尝试不同的用户名格式（domain\\\\user或user@domain）

### 6.2 同步问题
1. **无用户返回**
    - **原因**：BaseDN配置错误、搜索过滤条件过严、权限不足
    - **解决方法**：
        - 验证BaseDN正确性
        - 临时放宽搜索条件
        - 检查服务账户权限
2. **用户属性缺失**
    - **原因**：属性映射配置错误、AD中属性不存在、权限问题
    - **解决方法**：
        - 调整属性映射配置
        - 确认AD中属性存在并有值
        - 检查服务账户对属性的读取权限
3. **中文乱码问题**
    - **原因**：字符编码不匹配
    - **解决方法**：
        - 确保使用UTF-8编码
        - 配置适当的字符集处理
4. **删除阈值触发**
    - **原因**：大规模组织变更、AD结构调整
    - **解决方法**：
        - 确认变更是否预期
        - 适当调整删除阈值
        - 分批执行大规模变更

### 6.3 常见问题解答 (FAQ)
1. **Q: 专属集成平台如何与Active Directory集成？****A:** 专属集成平台通过LDAP协议与AD进行集成，支持用户和组的同步、认证和授权。
2. **Q: 集成AD需要哪些准备工作？****A:** 需要准备AD服务器信息（地址、端口）、具有读取权限的账号、以及要同步的OU（组织单位）信息。
3. **Q: 专属集成平台支持哪些AD认证方式？****A:** 支持简单绑定（Simple Bind）和TLS认证方式。
4. **Q: 专属集成平台和AD必须在同一个网络内吗？****A:** 非必须。若不使用 网关，则需要网络互通。若使用 网关，可以采用公网穿透内网方式（ 服务在公网，AD在内网，通过 网关打通网络）。
5. **Q: 专属集成平台如何处理AD用户的密码？****A:** 平台通常不会同步或存储AD用户的密码，认证时会将凭证传递给AD进行验证。
6. **Q: 支持AD密码反写吗？****A:** 是的，支持AD密码反写，打开AD认证源 \> 勾选"是否允许同步修改密码"。
7. **Q: 如何解决AD同步过程中的错误？****A:** 查看平台的同步日志，检查网络连接、AD账号权限等常见问题。
8. **Q: 专属集成平台支持多域AD环境吗？****A:** 支持。可以配置多个AD连接，分别同步不同域的用户和组信息。
9. **Q: 如何确保AD集成的安全性？****A:** 使用SSL/TLS加密LDAP连接，采用最小权限原则配置AD账号，定期审计访问日志。
10. **Q: 可以自定义AD用户属性的映射吗？****A:** 可以。专属集成平台提供灵活的属性映射配置，支持自定义字段映射。

## 7\. 最佳实践

### 7.1 初始配置建议
- 首次配置时建议使用手动全量同步，验证数据正确性后再启用自动同步
- 从小范围OU开始测试，验证成功后再扩展到更大组织单位
- 对于属性映射，先配置基本关键属性，确认正确后再添加更多属性

### 7.2 生产环境配置
- 使用专用服务账户进行集成，定期更换密码
- 对于大型AD目录，建议使用增量同步降低资源消耗
- 安排同步任务在非业务高峰期执行

### 7.3 安全建议
- 确保AD管理员账号具有足够权限但遵循最小权限原则
- 在生产环境中启用TLS加密保护数据传输安全
- 定期审计同步日志和访问记录
- 密码回写功能必须使用TLS加密保护

### 7.4 维护建议
- 定期检查同步日志，及时发现并解决潜在问题
- 合理设置删除用户阈值，避免意外大规模删除用户
- 对于多域环境，建议为每个域创建单独的连接器
- 在大规模组织变更前，提前调整同步策略和删除阈值

### 7.5 问题处理
- 建立问题处理流程，明确责任人
- 保留同步配置和日志的完整记录
- 对于关键同步问题，采取分步骤排查和解决策略
- 定期备份关键配置和用户数据

## 8\. 高级配置与场景

### 8.1 多域AD环境配置

在复杂企业环境中，可能存在多个AD域需要集成。以下是多域配置建议：
1. **多域架构类型识别**
    - **单一林多域**：同一个AD林中的多个域，存在信任关系
    - **多林多域**：多个AD林，每个林有自己的域，可能无信任关系
    - **资源域与账户域分离**：用户账户和资源分别在不同域管理
2. **多域集成策略**
    - 为每个域创建独立的AD连接器
    - 配置不同的同步时间，避免同时执行多个同步任务
    - 在连接器命名中明确标识域名，如"销售域-AD"、"研发域-AD"
3. **用户去重策略**
    - 使用唯一标识属性(如邮箱或员工ID)作为匹配键
    - 配置冲突解决规则，如保留先同步的用户或特定域的用户
    - 考虑使用域前缀或后缀标识用户来源

### 8.2 高级属性映射

专属集成平台支持多种高级属性映射场景：
1. **数据转换函数使用**
    - TOLOWERCASE()：转为小写，用于规范化用户名和邮箱
    - TRIM()：去除前后空格
    - JSON.PARSE()：将文本转换为JSON对象
    - 自定义函数：支持JavaScript表达式进行自定义转换
2. **条件映射设置**
    - 根据条件设置不同映射规则，如：```=IF(ISEMPTY(displayName), sAMAccountName, displayName)```
3. **组合映射应用**
    - 多源属性组合映射，如名字和姓氏组合：```=CONCAT(firstName, " ", lastName)```

### 8.3 高可用与灾备配置

对于企业关键系统，确保AD集成的高可用性至关重要：
1. **冗余AD服务器配置**
    - 配置多个AD服务器地址，确保主服务器不可用时可使用备用服务器
    - 使用负载均衡器分配连接请求
2. **同步策略优化**
    - 配置自动重试机制，处理临时网络问题
    - 设置合理的超时阈值，避免长时间挂起
    - 实施错误告警机制
3. **灾备方案**
    - 定期备份连接器配置
    - 制定在AD服务不可用情况下的应急预案
    - 考虑配置备用身份源作为灾备

### 9.2 userAccountControl属性值参考

userAccountControl属性是一个位掩码，包含多个用户账户设置：

| 十进制值 | 十六进制值 | 常量名 | 说明 |
|------------|---------------|---------|------|
| 2 | 0x002 | ADS*UF*ACCOUNTDISABLE | 账户已禁用 |

| 65536 | 0x10000 | ADS*UF*DONT*EXPIRE*PASSWD | 密码永不过期 |

### 9.3 常用LDAP过滤器示例
1. **查找所有启用的用户账户**```(&(objectCategory=person)(objectClass=user)(!(userAccountControl:1.2.840.113556.1.4.803:=2)))```
2. **查找特定部门的用户**```(&(objectCategory=person)(objectClass=user)(department=IT))```
3. **查找最近30天修改过的用户**```(&(objectCategory=person)(objectClass=user)(whenChanged\>=20230101000000.0Z))```
4. **查找所有组织单位**```(objectClass=organizationalUnit)```
5. **查找邮箱地址为空的用户**```(&(objectCategory=person)(objectClass=user)(!(mail=\*)))```

### 9.5 基本排障流程

遇到同步问题时，建议按照以下流程进行排查：
1. **检查连接**
    - 验证网络连通性
    - 确认服务器地址和端口正确
    - 测试服务账户凭据
2. **检查权限**
    - 确认服务账户具有必要权限
    - 验证BaseDN和搜索范围
    - 测试特定对象的访问权限
3. **检查配置**
    - 验证同步策略设置
    - 检查属性映射配置
    - 确认过滤条件正确
4. **检查数据**
    - 验证源数据质量和完整性
    - 检查特殊字符和编码问题
    - 确认数据转换规则正确
5. **检查日志**
    - 分析同步日志，找出错误模式
    - 查看详细错误信息和代码
    - 跟踪问题发生的时间和条件
