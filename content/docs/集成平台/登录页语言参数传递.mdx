## 功能概述

数犀集成平台的登录页现已支持**语言参数传递**功能。当管理员或用户通过浏览器直接访问登录页并指定语言（如英文）完成认证后，SSO 跳转至三方应用时可同步传递语言参数，使三方应用能够感知并同步展示对应语言，解决语言体验割裂问题。

**支持的参数：**

| 参数名 | 说明 | 示例值 |
|--------|------|--------|
| `locale` | 语言标识 | `zh-CN` / `en-US` / `ja-JP` |
| `redirect_uri` | SSO 目标跳转地址 | —（原有参数）|

**使用方式：**

```
# 指定语言并跳转
https://{your-domain}/login?locale=en-US&redirect_uri=https://target-app.com/callback
```

登录页识别 `locale` 参数后，页面自动切换为对应语言，用户完成登录后跳转目标应用时，语言参数会自动附加在跳转 URL 中。

## SSO 跳转时的语言参数传递

当用户通过浏览器访问带语言参数的登录页并完成认证后，目标应用接收到的回调 URL 中会携带语言参数：

```
https://target-app.com/callback?code=xxx&state=xxx&locale=en-US
```

三方应用可通过解析 `locale` 参数获取用户在登录页使用的语言，从而在自身系统中同步展示对应语言。

<Callout type="info">
语言参数传递功能需要目标应用（SP）支持解析 URL 参数。若目标应用不支持接收额外参数，可通过自定义 URL 映射或后端处理的方式获取语言信息。
</Callout>

## 管理员配置

此功能为平台内置能力，管理员无需额外配置。默认支持的语言包括：

| 语言代码 | 显示名称 |
|---------|---------|
| `zh-CN` | 简体中文 |
| `en-US` | English |
| `ja-JP` | 日本語 |

如需启用其他语言或调整默认语言，可在 **设置 > 基本设置 > 国际化** 中进行配置。

## 常见问题

<Accordions>
<Accordion title="如果目标应用不支持 locale 参数，会不会导致跳转失败？》
不会。语言参数作为 URL 查询参数附加在目标地址后，不影响原有的 code 和 state 参数。若应用不需要此参数，可直接忽略，不会导致认证失败。
</Accordion>
<Accordion title="用户在钉钉容器内访问登录页，语言参数是否生效？》
钉钉内嵌浏览器环境下，登录页会优先使用钉钉应用的语言设置，URL 中的 locale 参数在这种情况下可能不生效。建议在非钉钉环境的浏览器中使用此功能。
</Accordion>
</Accordions>
