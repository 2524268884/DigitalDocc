## 概述

应用网关是专属集成平台提供的安全代理服务，实现"外网访问内网"的安全通道。通过应用网关，企业可以让员工安全地从外部网络访问内部应用系统，无需复杂的 VPN 配置。

**主要特点：**
- **安全代理**：通过加密通道安全地将外部请求转发到内网应用
- **身份认证**：集成企业认证体系，确保只有授权用户可以访问内网应用
- **访问控制**：精细的权限管理，可按组织、角色设置访问权限
- **请求转换**：支持请求头、URL 路径和响应内容的自定义转换规则
- **匿名访问**：支持配置不需要认证的公开访问路径

## 前提条件

- 已在平台中部署并配置好代理网关服务
- 代理网关服务器能够访问目标内网应用（位于同一网络或有网络互通策略）
- 目标内网应用提供 Web 服务接口

## 操作步骤

<Steps>
<Step>
### 步骤 1：准备应用

在左侧导航栏选择**应用中心 > 应用管理**，可以新建应用或使用现有应用进行配置。

<Callout type="info">
应用网关功能支持任意类型和协议的应用。虽然不必专门新建应用，但对于初次使用者，建议创建专用应用以避免影响现有配置。
</Callout>
</Step>

<Step>
### 步骤 2：配置网关

在应用详情页面，选择**网关**选项卡，点击**新建**按钮。选择已配置好的代理网关服务（Proxy），如果尚未配置代理网关，需要先完成代理网关的配置。
</Step>

<Step>
### 步骤 3：配置域名

| 参数 | 说明 |
|------|------|
| 内网域名 | 输入需要访问的内网应用完整 URL（包含协议、域名和端口） |
| 外网统一访问域名 | 设置外部用户访问的域名。如不填写，系统会自动生成一个随机值 |
</Step>

<Step>
### 步骤 4：配置高级选项（可选）

| 功能 | 说明 | 示例 |
|------|------|------|
| **匿名访问路径** | 设置不需要认证即可访问的 URL 路径，使用正则表达式 | `/login`、`/static/.*` |
| **请求头重写** | 修改转发到内网应用的 HTTP 请求头 | 将 Host 头设置为内网应用期望的值 |
| **请求路径重写** | 将请求路径映射到内网应用的不同路径 | `/api/v1` → `/internal/api` |
| **响应重写** | 对内网应用返回的响应内容进行修改 | - |

```nginx
# 请求头重写示例
Host: internal-app.example.com
```

```nginx
# 匿名访问路径示例（允许 /login 和 /static/ 下所有路径匿名访问）
/login
/static/.*
```
</Step>

<Step>
### 步骤 5：配置授权范围

1. 切换到**授权范围**选项卡
2. 可基于组织架构、角色或用户属性设置访问权限
3. 只有授权范围内的用户才能通过应用网关访问内网应用

<Callout type="info">
应用网关配置是对应用的扩展配置，两者之间的关系：
- 应用负责身份认证和授权
- 网关负责网络代理和请求转发
- 网关配置不会影响应用的基本配置
</Callout>
</Step>
</Steps>

## 验证

配置完成后，可以通过以下方式验证：

1. 使用授权用户从外部网络访问外网访问域名
2. 确认页面正确跳转到专属集成平台登录页
3. 完成认证后，验证是否成功访问到内网应用
4. 检查内网应用日志，确认请求已正确转发

## 常见问题

<Accordions>
<Accordion title="应用网关配置后无法访问内网应用">
**可能原因：**
- 代理网关服务与内网应用之间存在网络隔离
- 内网应用对请求的 Host 等头信息有特殊要求
- 内网应用的 URL 路径配置不正确

**解决方法：**
1. 检查代理网关服务器是否能直接访问目标内网应用
2. 尝试配置请求头重写，特别是 Host 头，设置为内网应用期望的值
3. 检查并修正内网域名配置，确保包含正确的协议、域名和端口
</Accordion>

<Accordion title="如何实现部分路径匿名访问？">
内网应用的某些路径（如登录页、静态资源）需要允许匿名访问，而其他部分需要认证。

**解决方法：**
使用正则表达式在匿名访问路径中精确定义允许匿名访问的 URL 模式：

```nginx
# 示例：允许 /login 和 /static/ 下的所有路径匿名访问
/login
/static/.*
```
</Accordion>

<Accordion title="如何关闭或删除应用网关配置？">
当不再需要应用网关功能时，可以通过以下步骤删除配置：
1. 进入应用详情页面，选择**网关**选项卡
2. 找到要删除的网关配置，点击**删除**按钮
3. 确认删除操作
</Accordion>
</Accordions>

## 最佳实践

**安全性建议**
- **最小化匿名访问**：仅将必要的路径配置为匿名访问，减少安全风险
- **定期审核授权**：定期检查应用网关的授权配置，移除不再需要访问权限的用户
- **启用访问日志**：配置并监控访问日志，便于及时发现异常访问
- **使用 HTTPS**：优先使用 HTTPS 协议，确保数据传输的安全性

**性能优化**
- **选择合适的代理网关节点**：代理网关服务器应尽量靠近内网应用，减少网络延迟
- **资源缓存**：对于静态资源，考虑配置缓存策略，减轻代理负担
- **连接复用**：确保代理服务配置了适当的连接池设置，提高并发处理能力
