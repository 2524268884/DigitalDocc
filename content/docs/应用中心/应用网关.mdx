## 概述

应用网关提供「外网安全访问内网应用」的反向代理能力。员工在外部网络通过统一域名访问企业内网系统，网关在转发请求的同时完成身份验证和权限控制，无需搭建 VPN。

**主要能力：**
- **安全代理**：加密通道转发请求，内网应用地址对外不可见
- **身份认证**：仅允许已登录且有权限的用户访问
- **访问控制**：按部门/角色精细控制访问权限
- **请求转换**：支持 Header、URL 路径、响应内容的自定义改写
- **匿名路径**：支持配置无需认证的公开访问路径

## 前提条件

| # | 条件 | 说明 |
|---|------|------|
| 1 | 已部署代理网关服务 | 见 [代理网关配置](/docs/参数配置/代理网关) |
| 2 | 代理网关可访问目标内网应用 | 两者需在同一网络或有互通策略 |
| 3 | 目标内网应用提供 Web 服务 | 支持 HTTP/HTTPS |

## 配置步骤

<Steps>
<Step>
### 选择或创建应用

在**应用中心 > 应用管理**中选择已有应用，或新建一个专用应用（建议初次使用时新建，避免影响现有配置）。
</Step>
<Step>
### 新建网关配置

在应用详情页面选择「网关」选项卡，点击「新建」，选择已配置好的代理网关服务（Proxy）。
</Step>
<Step>
### 配置域名

| 配置项 | 说明 |
|--------|------|
| 内网域名 | 内网应用的完整 URL，含协议+域名+端口，如 `http://192.168.1.100:8080` |
| 外网访问域名 | 员工通过外网访问的域名，不填则系统自动生成 |

</Step>
<Step>
### 高级配置（可选）

**匿名访问路径**：配置不需要登录即可访问的 URL 路径（使用正则表达式）：

```
# 示例：允许登录页和静态资源匿名访问
/login
/static/.*
```

**请求头重写**：修改转发到内网的 HTTP 请求头，解决 Host 头等特殊要求：

```
Host: internal-app.example.com
```

**请求路径重写**：将外网访问路径映射到内网应用的不同路径。

**响应重写**：修改内网应用返回的响应内容。
</Step>
<Step>
### 配置访问权限

切换到「授权范围」选项卡，配置哪些用户/部门/角色可以通过应用网关访问内网应用。
</Step>
</Steps>

## 常见问题

<Accordions>
<Accordion title="配置完成后无法访问内网应用？">
逐项排查：
1. 确认代理网关服务器**能直接访问**目标内网 IP 和端口（可在网关服务器上用 curl 测试）
2. 检查「内网域名」配置是否包含正确的协议（`http://` 或 `https://`）
3. 如目标应用对 Host 头有特殊要求，配置「请求头重写」将 Host 改为内网应用期望的值
4. 检查内网应用的防火墙是否允许来自代理网关服务器 IP 的访问
</Accordion>
<Accordion title="如何只让部分路径需要认证？">
在「高级配置 > 匿名访问路径」中使用正则表达式定义公开路径，未配置的路径默认需要认证：

```
/login          # 登录页匿名访问
/static/.*      # 静态资源匿名访问
/api/public/.*  # 公开 API 匿名访问
```
</Accordion>
<Accordion title="如何删除网关配置？">
进入应用详情 → 「网关」选项卡 → 找到目标网关配置 → 点击「删除」→ 确认即可。删除后外网访问域名立即失效。
</Accordion>
</Accordions>
