## 概述

单点登出（SLO，Single Logout）允许用户在专属集成平台退出后，自动退出所有已对接 SSO 的应用（SP）。同样，用户在 SP 应用发起退出时，也会触发专属集成平台及其他 SP 的联动登出。

**主要特点：**
- **统一登出入口**：在任意平台退出，即可完成所有已对接应用的登出
- **多通道支持**：支持前端通道（Front-Channel）和后端通道（Back-Channel）两种登出方式
- **统一会话管理**：确保登出后所有会话同步失效

<Callout type="info">
当前专属集成平台仅支持 **OIDC 协议**的单点登出，SAML 协议单点登出即将支持。
</Callout>

## 单点登出原理

### 平台发起的登出流程

1. 用户在专属集成平台点击"退出登录"
2. 平台向所有已登录的 SP 应用发送登出请求
3. 各 SP 应用接收请求并清除本地会话
4. 平台清除平台侧的会话记录
5. 用户被重定向到登录页面

### SP 发起的单点登出流程

1. 用户在 SP 应用中点击"退出"
2. SP 应用向专属集成平台发送 SLO 请求
3. 专属集成平台清除平台会话
4. 专属集成平台向其他已登录的 SP 应用发送登出请求
5. 所有 SP 应用清除本地会话

### OIDC 单点登出机制

| 方式 | 说明 |
|------|------|
| **前端通道登出** | 用户退出时，平台引导浏览器访问各 SP 的登出端点，SP 接收后清除本地会话 |
| **后端通道登出** | 平台向 SP 的登出回调地址发送 POST 请求，携带用户 Token 信息，SP 收到后清除对应会话 |

## 前提条件

- 已完成 OIDC 应用的单点登录（SSO）配置
- SP 应用支持 OIDC 协议的登出回调（登出端点可被专属集成平台访问）

## 操作步骤

<Steps>
<Step>
### 步骤 1：启用 OIDC 协议

1. 进入**应用中心 > 应用管理**，选择目标应用，点击**编辑**
2. 切换到**单点配置**选项卡，启用**单点协议**
3. 选择 **OIDC** 认证协议类型
</Step>

<Step>
### 步骤 2：配置登出回调地址

1. 在应用配置中找到**登出回调地址**字段
2. 填写 SP 应用提供的登出端点 URL（需为平台可访问的地址）
3. 点击**保存**
</Step>

<Step>
### 步骤 3：验证登出功能

1. 以用户身份登录专属集成平台，并访问已配置 SSO 的 OIDC 应用
2. 在专属集成平台点击"退出登录"
3. 验证 SP 应用的会话是否也已同步清除（尝试直接访问 SP 应用，应跳转到登录页）
</Step>
</Steps>

<Callout type="info">
配置登出回调地址后，用户在专属集成平台退出时，系统将携带用户登录 Token 自动回调该地址，完成单点登出。
</Callout>

## 密码修改后强制登出

如需在用户密码修改后强制退出所有会话，可进行以下配置：

进入**参数配置 > 平台设置 > 登录策略**，开启**密码修改后强制登出**选项。

开启后，用户密码被修改将导致其所有现有会话立即失效，需要重新登录。

## 常见问题

<Accordions>
<Accordion title="配置了登出回调地址，但登出后应用没有响应">
请检查以下事项：
1. 确认登出回调地址可被专属集成平台从网络侧访问（公网可访问或内网互通）
2. 确认 SP 应用的登出端点能正确处理 POST 请求
3. 检查 SP 应用日志，确认是否收到登出请求及处理结果
</Accordion>

<Accordion title="OIDC 应用支持 SAML 协议的单点登出吗？">
目前专属集成平台仅支持 OIDC 协议的单点登出，SAML 协议单点登出功能正在开发中。
</Accordion>

<Accordion title="用户未点击退出直接关闭浏览器，会话如何处理？">
用户直接关闭浏览器而未点击"退出登录"时，系统会创建新会话（如下次登录），原会话在超时后自动失效。如需确保安全，建议要求用户通过正式退出流程登出，或启用较短的会话超时时间。
</Accordion>

<Accordion title="部分 SP 登出失败，会影响用户在平台的登录状态吗？">
不会影响。用户在平台的退出操作会清除平台会话，即使部分 SP 登出回调失败，用户在平台侧已登出，无法再通过 SSO 访问这些应用。建议检查相关 SP 的回调地址是否可访问。
</Accordion>
</Accordions>
