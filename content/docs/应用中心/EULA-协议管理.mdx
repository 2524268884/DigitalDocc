## 功能概述

**EULA（最终用户许可协议）管理**功能允许企业管理员在专属集成平台中配置用户登录前必须签署的协议条款。用户首次登录或协议更新后，系统自动弹出协议签署页面，用户确认同意后方可进入系统。

**核心能力：**
- 支持配置多份协议（如用户服务协议、隐私政策）
- 协议内容更新后，已登录用户下次访问时自动要求重新签署
- 签署记录持久化存储，支持按用户或协议版本查询
- 支持设置协议的生效范围（全员 / 指定组织 / 指定角色）

## 适用场景

| 场景 | 说明 |
|------|------|
| 新员工入职合规 | 新员工首次登录时签署用户协议和隐私政策 |
| 合规更新 | 法规变化导致协议更新，要求全员重新确认 |
| 差异化协议 | 不同部门或角色签署不同版本的协议条款 |

## 配置步骤

<Steps>
<Step>
### 步骤一：进入 EULA 管理页面

在专属集成平台管理控制台中，进入 **设置 > EULA 协议管理** 页面。
</Step>
<Step>
### 步骤二：新建协议

点击「新建协议」，填写以下信息：

| 配置项 | 说明 |
|--------|------|
| 协议名称 | 展示给用户的协议标题，如「用户服务协议」 |
| 协议内容 | 支持富文本编辑，可包含标题、段落、列表等格式 |
| 协议版本 | 版本号，每次修改协议内容时建议更新版本号 |
| 生效范围 | 全员 / 指定部门 / 指定角色 |
| 是否强制同意 | 开启后，用户必须勾选同意才能继续登录 |
</Step>
<Step>
### 步骤三：发布协议

协议内容填写完成后，点击「发布」使协议生效。已发布的协议将在用户下次登录时展示。

<Callout type="warn">
发布后已登录用户的当前会话不受影响，下次登录或会话过期后重新登录时才会看到新协议。若需立即强制所有用户签署，请在发布后同时执行「强制下线」操作。
</Callout>
</Step>
</Steps>

## 更新协议内容

<Steps>
<Step>
### 步骤一：编辑现有协议

在 EULA 管理列表中，点击目标协议的「编辑」按钮。
</Step>
<Step>
### 步骤二：修改内容并更新版本号

修改协议内容后，更新版本号（如从 `v1.0` 改为 `v1.1`）。

<Callout type="info">
版本号更新后，系统会将所有用户的该协议签署状态重置，要求用户重新签署新版协议。若仅修改文字错误且不希望用户重新签署，可保持版本号不变。
</Callout>
</Step>
<Step>
### 步骤三：发布新版本

点击「发布」，新版协议生效，已签署过旧版本的用户将在下次登录时看到新版协议。
</Step>
</Steps>

## 查询签署记录

在 EULA 管理页面，点击协议名称进入详情页，切换到「签署记录」选项卡，可按以下维度查询：

| 查询维度 | 说明 |
|---------|------|
| 用户 | 查询指定用户签署了哪些协议及签署时间 |
| 协议版本 | 查询某版本协议的签署用户列表及签署率 |
| 时间范围 | 查询指定时间段内的签署记录 |

## 常见问题

<Accordions>
<Accordion title="用户拒绝签署协议会怎样？">
若协议设置为「强制同意」，用户点击拒绝后将无法进入系统，页面提示联系管理员。若未设置强制，用户可跳过签署正常登录，但签署状态记录为「已跳过」。
</Accordion>
<Accordion title="协议内容支持外部链接吗？">
支持在富文本编辑器中插入超链接，用户点击可跳转至外部页面查看完整条款（如已备案的法律文件页面）。
</Accordion>
</Accordions>
