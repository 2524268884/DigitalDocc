单点登录（SSO）是集成平台的核心功能，允许用户通过一次身份验证访问多个应用系统，提高用户体验和安全性。

## 功能特点

- **统一认证入口**：为所有应用提供统一的认证登录界面
- **多协议支持**：支持 SAML、OIDC、OAuth2.0 等主流 SSO 协议
- **统一会话管理**：集中管理用户会话状态和超时策略
- **跨域访问支持**：解决不同域名应用之间的认证问题
- **安全防护**：防止会话劫持、CSRF 等安全威胁

## 单点登录原理

### 标准模式

```
用户 → 访问应用 → 应用重定向到认证中心 → 用户认证
     ← 认证中心返回令牌 ← 应用验证令牌 ← 授权访问
```

### 代理模式

对于无法改造的传统应用，平台提供代理模式：

1. 用户访问应用网关
2. 网关重定向到认证中心完成身份验证
3. 网关代理用户向后端应用发起请求（支持表单登录或 HTTP 头传递身份信息）

## 基础配置步骤

<Steps>
<Step>
### 启用单点登录
1. 进入**应用中心 > 应用管理**
2. 选择需要配置 SSO 的应用，点击"编辑"
3. 在**认证设置**选项卡中，启用**单点登录**开关
4. 选择认证协议类型，点击"保存"
</Step>

<Step>
### 选择认证协议
根据应用类型选择合适的协议：
| 协议 | 适用场景 |
|------|---------|
| SAML 2.0 | 传统企业应用，支持跨域 SSO |
| OIDC | 现代 Web 应用，提供丰富用户信息 |
| OAuth 2.0 | API 授权场景 |
| 表单代理 | 无法改造的传统表单登录应用 |
| 标准 HTTP 代理 | 无法改造的 HTTP 应用 |
</Step>

<Step>
### 配置协议参数
<Tabs items={['SAML 2.0', 'OIDC', 'OAuth2']}>
<Tab value="SAML 2.0">
选择 SAML 2.0 协议后，配置以下参数：
| 参数 | 说明 |
|------|------|
| 服务提供者 ID | 应用的唯一标识符 |
| ACS URL | 断言消费服务 URL |
| 断言加密 | 是否加密 SAML 断言 |
| 签名算法 | 推荐选择 RSA-SHA256 |
| 摘要算法 | 推荐选择 SHA-256 |
配置完成后，访问 Metadata 地址下载 IdP 元数据文件，导入到应用的 SAML 配置中。
📖 参考：[单点登录-SAML协议](/docs/应用中心/应用单点登录/单点登录-SAML协议)
</Tab>
<Tab value="OIDC">
选择 OIDC 协议后，配置以下参数：
| 参数 | 说明 |
|------|------|
| 客户端 ID | 系统自动生成的客户端标识 |
| 客户端密钥 | 点击"生成密钥"生成 |
| 重定向 URI | 认证完成后的跳转地址 |
| 授权类型 | 支持的授权流程类型 |
| 访问令牌有效期 | 建议 1 小时 |
| 刷新令牌有效期 | 建议 7 天 |
| ID 令牌签名算法 | 推荐 RS256 |
📖 参考：[单点登录-OIDC](/docs/应用中心/应用单点登录/单点登录-OIDC)
</Tab>
<Tab value="OAuth2">
选择 OAuth2 协议后，参照 OIDC 配置客户端 ID 和密钥，并根据应用需求配置授权范围（Scope）。
📖 参考：[单点登录-OAuth2](/docs/应用中心/应用单点登录/单点登录-OAuth2)
</Tab>
</Tabs>
</Step>

<Step>
### 配置属性映射
在**属性映射**区域，添加需要传递给应用的用户属性：
- `NameID`：用户的唯一标识（如邮箱、用户名）
- `姓名`：用户姓名
- `邮箱`：用户邮箱
- `部门`：用户所属部门
- `角色`：用户角色信息
</Step>

<Step>
### 配置授权范围
在应用的**授权范围**选项卡中，配置哪些用户可以访问此应用。
</Step>
</Steps>

## 高级配置

### 会话管理

在**会话管理**选项卡中可配置：

| 参数 | 说明 | 建议值 |
|------|------|--------|
| 会话超时时间 | 用户无操作后自动退出 | 30 分钟 |
| 最大会话数 | 每个用户允许的并发会话数 | 5 |
| 会话续期 | 是否允许活跃用户自动续期 | 开启 |
| 强制重新认证 | 是否定期要求重新认证 | 按需 |

### 多因子认证集成

在**多因子认证**选项卡中启用 MFA 集成，支持以下触发策略：
- **始终要求**：每次登录都需要 MFA
- **风险感知**：根据登录风险动态决定
- **特定用户**：仅对高权限用户要求 MFA

## 常见问题排查

<Accordions>
<Accordion title="用户无法通过 SSO 登录">
1. 检查用户是否有访问应用的权限
2. 验证 SSO 配置是否正确
3. 清除浏览器 Cookie 和缓存后重试
4. 查看**审计日志 > 用户行为日志**中的错误信息
</Accordion>
<Accordion title="SAML 断言验证失败">
- 检查 IdP 和 SP 的服务器时钟是否同步（误差需 < 5 分钟）
- 验证签名证书是否在有效期内
- 确认签名算法配置在双方一致
- 使用 SAML-tracer 浏览器插件抓包分析断言内容
</Accordion>
<Accordion title="OIDC 重定向 URI 错误">
- 检查应用中配置的重定向 URI 是否与实际跳转地址完全一致（包括末尾斜杠）
- 确认 URI 已添加到集成平台应用的**允许重定向列表**中
</Accordion>
<Accordion title="登录后出现「未授权」错误">
可能原因：
- 用户属性映射配置错误，SP 收到的 NameID 与系统中不一致
- 用户未被授权访问该应用
- 令牌验证失败（签名算法/公钥不匹配）
- 会话已过期
</Accordion>
</Accordions>
