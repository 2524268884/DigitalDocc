## 概述

应用授权管理包含两个层面：
- **用户访问授权**：控制哪些用户可以在门户中看到并访问该应用
- **M2M 接口授权**：控制该应用可以调用哪些平台 API（机器间调用）

---

## 用户访问授权

### 授权方式对比

| 授权方式 | 说明 | 适用场景 |
|---------|------|---------|
| **全员授权** | 所有平台用户均可访问 | 企业通用系统（OA、邮箱）|
| **组织架构授权** | 按部门/用户组批量授权，自动同步人员变动 | 部门专属应用 |
| **角色授权（RBAC）** | 按静态或动态角色授权 | 基于职能的访问控制 |
| **用户授权** | 指定具体用户 | 特定人员专用应用 |

<Callout type="info">
多种授权方式可同时使用，取**并集**生效：满足任意一种授权条件的用户均可访问应用。授权变更**实时生效**，无需用户重新登录。
</Callout>

### 配置步骤

<Steps>
<Step>
### 进入授权配置

在**应用中心 > 应用管理**中找到目标应用，点击「编辑」，切换到「授权范围」选项卡。
</Step>
<Step>
### 选择授权方式

**全员授权**：直接开启「允许所有人使用」开关。

**组织架构授权**：点击「选择部门」，在组织树中勾选目标部门（勾选父部门自动包含子部门）。

**角色授权**：点击「选择角色」，勾选静态角色或动态角色。

**用户授权**：点击「添加用户」，通过用户名/姓名/手机号/邮箱搜索并添加。
</Step>
<Step>
### 保存配置

点击「保存」，授权立即生效。
</Step>
</Steps>

## M2M 接口授权

M2M（Machine-to-Machine）授权用于第三方应用或企业自建系统调用数犀集成平台 Open API 时的权限控制。

### 可授权的 API 操作

**用户管理**：查询用户 / 创建用户 / 编辑用户 / 调整部门 / 删除用户

**部门管理**：查询部门 / 创建部门 / 编辑部门 / 删除部门

**角色管理**：查询角色 / 创建角色 / 编辑角色 / 删除角色 / 角色成员管理

<Callout type="warn">
创建、编辑、删除类操作存在数据风险，系统会显示安全警示。建议遵循**最小权限原则**，只授予实际需要的操作。
</Callout>

### 配置 M2M 授权

<Steps>
<Step>
### 开启 M2M 授权

进入应用编辑页面，切换到「权限范围」选项卡，找到「M2M 授权」区域，开启「M2M 应用授权设置」开关。
</Step>
<Step>
### 选择 API 权限

勾选该应用需要调用的 API 操作权限类别和具体接口，点击「保存」。
</Step>
</Steps>

### API 调用认证

配置 M2M 授权后，下游系统通过以下方式获取访问令牌并调用 API：

**获取 Token：**
```bash
POST /iam/token
Content-Type: application/x-www-form-urlencoded
Authorization: Basic base64({client_id}:{client_secret})

grant_type=client_credentials
```

**使用 Token 调用 API：**
```bash
GET /iam/api/users
Authorization: Bearer {access_token}
```

详细接口说明见 [API 开放文档](/docs/开发指引/API%20开放文档)。

## 常见问题

<Accordions>
<Accordion title="用户反馈无法访问已授权应用？">
逐项检查：
1. 用户是否属于已授权的部门/角色，或在用户授权列表中
2. 用户账号是否处于活跃状态（未被停用）
3. 应用本身是否处于启用状态
4. 是否存在多种授权方式冲突（注意取并集，有任一满足即可）
</Accordion>
<Accordion title="M2M API 调用返回 403？">
1. 检查 Token 是否在有效期内（重新获取 Token）
2. 确认该应用是否已开启对应接口的 M2M 授权
3. 确认 API 请求格式是否正确（请求头是否携带 Bearer Token）
4. 查看 [接口调用日志](/docs/审计日志/接口调用日志) 获取详细错误信息
</Accordion>
<Accordion title="授权变更后用户仍无法访问怎么办？">
授权变更实时生效，但用户的浏览器可能缓存了旧的权限状态。建议：
- 让用户退出后重新登录
- 清除浏览器缓存后刷新页面
</Accordion>
</Accordions>
