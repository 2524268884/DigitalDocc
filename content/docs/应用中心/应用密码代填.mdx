---
title: "应用密码代填"
description: "为不支持 SSO 协议的应用配置密码自动填充，用户通过门户一键登录，无需手动输入账号密码。"
---

## 概述

密码代填适用于无法改造为支持 SSO 协议的传统应用或第三方系统。通过浏览器插件与平台配合，用户点击应用图标后自动跳转登录页并填充账密，实现「零输入」登录。

## 适用场景

- 传统旧系统：无法改造支持 OIDC/SAML 等标准 SSO 协议
- 第三方供应商应用：不提供单点登录接口
- 临时使用的系统：不值得做复杂 SSO 配置

## 前提条件

| # | 条件 | 说明 |
|---|------|------|
| 1 | Chrome 浏览器 | 密码代填插件**仅支持 Chrome** |
| 2 | 已安装密码代填插件 | 离线安装「密码代填插件 Beta」 |
| 3 | 管理员已配置应用 | 后台已添加密码代填应用并配置账密 |

### 安装 Chrome 插件

<Steps>
<Step>
### 打开扩展程序管理

在 Chrome 浏览器中，进入「管理扩展程序」（地址栏输入 `chrome://extensions/`）。
</Step>
<Step>
### 加载插件

开启「开发者模式」，点击「加载已解压的扩展程序」，选择插件目录完成安装。
</Step>
</Steps>

## 管理员配置步骤

<Steps>
<Step>
### 创建密码代填应用

进入**应用中心 > 应用管理**，点击「添加应用 > 自定义应用」，选择「密码代填」类型。

<Callout type="warn">
**不要**为密码代填应用同时配置单点登录协议，否则会触发 SSO 认证流程而非密码代填。
</Callout>
</Step>
<Step>
### 填写应用信息

| 配置项 | 说明 |
|--------|------|
| 应用名称 | 清晰易识别的名称 |
| 应用图标 | 建议上传 128×128 像素图标 |
| 登录地址 | 目标应用的登录页面 URL |
| 账号字段 | 登录页面用户名输入框的元素 ID 或 name 属性 |
| 密码字段 | 登录页面密码输入框的元素 ID 或 name 属性 |
| 提交按钮 | 登录按钮的元素 ID 或 name 属性 |

</Step>
<Step>
### 配置账号映射

选择账号映射方式：
- **固定账号**：所有用户使用同一账号（适合共享账号场景）
- **属性映射**：使用用户的平台属性（如用户名/工号）作为登录账号
</Step>
</Steps>

## 用户使用流程

1. 登录数犀集成平台门户
2. 在应用列表中找到密码代填应用，点击图标
3. 浏览器自动跳转到目标应用登录页面
4. 插件自动识别登录表单并填充账号密码
5. 插件触发提交，用户直接进入应用系统

<Callout type="info">
在飞书客户端内使用时，需先点击「独立窗口打开」再通过 Chrome 访问，才能使插件正常工作。
</Callout>

## 常见问题

<Accordions>
<Accordion title="插件无法自动填充密码？">
1. 确认插件已正确安装且已启用（Chrome 扩展管理页面检查）
2. 使用浏览器开发者工具（F12）检查登录表单的实际元素 ID 或 name，确认与应用配置一致
3. 检查目标网站是否使用了防自动填充技术（`autocomplete="off"`），如有需要特殊处理
4. 确认插件版本是最新版本
</Accordion>
<Accordion title="跳转后显示空白页面？">
- 检查「登录地址」URL 是否正确
- 测试是否能在普通浏览器中正常打开该 URL
- 检查目标应用是否对来源 IP 有限制
</Accordion>
<Accordion title="移动设备上可以使用密码代填吗？">
密码代填插件目前**仅支持 PC 端 Chrome 浏览器**，不支持移动端。如需移动端访问，建议将该应用改造为支持标准 SSO 协议（OIDC/SAML），或通过企业移动应用分发解决。
</Accordion>
</Accordions>
