## 概述

单点登录（SSO）允许用户只需登录一次，即可访问所有已集成到数犀集成平台中的应用系统。

```
未使用 SSO：
用户 → 应用 A（需登录）→ 应用 B（需登录）→ 应用 C（需登录）

使用 SSO：
用户 → 数犀集成平台（登录一次）→ 应用 A → 应用 B → 应用 C
```

## 前置准备

| 准备项 | 说明 |
|--------|------|
| 数犀集成平台管理员账号 | 需要应用中心管理员权限 |
| 目标应用 | 需要支持 SAML、OIDC、OAuth2 或 CAS 协议 |
| 了解应用支持的协议 | 查阅目标应用的对接文档 |
| HTTPS 环境 | 生产环境必须使用 HTTPS |

## 选择合适的协议

| 协议 | 适用场景 | 复杂度 | 代表应用 |
|------|----------|--------|----------|
| **OIDC** | 现代 Web 应用、SPA、移动 APP | 低 | 自定义 Web 应用、钉钉 |
| **SAML 2.0** | 企业级应用、传统软件、硬件设备 | 中 | 泛微 OA、致远 OA、阿里云 |
| **OAuth 2.0** | API 授权、第三方登录 | 中 | 阿里云、纷享销客 |
| **CAS** | 高校、科研机构 | 低 | 校级系统、宜搭 |

<Callout type="info">
如果应用同时支持多种协议，推荐优先使用 **OIDC**，配置更简单。
</Callout>

## 方式一：OIDC 接入（以自定义 Web 应用为例）

### 整体流程

```
用户 → 应用 → 授权码请求 → 数犀集成平台 → 用户登录 → 授权码 → 应用 → 换取 Token → 完成
```

### 操作步骤

<Steps>
<Step>
### 步骤 1：创建应用

进入**应用中心 > 应用管理**，点击**添加应用 > 自定义应用**，填写应用基本信息。
</Step>

<Step>
### 步骤 2：配置 OIDC 参数

1. 进入应用详情，点击**单点登录**选项卡
2. 选择协议类型：**OIDC**
3. 配置以下参数：

| 参数 | 说明 | 示例值 |
|------|------|--------|
| 客户端 ID | 自动生成 | `oidc_app_12345` |
| 客户端密钥 | 点击"生成" | `****` |
| 重定向 URI | 授权后跳转地址 | `https://app.example.com/callback` |
| 授权类型 | 推荐 Authorization Code | ✓ |
| 作用域 | openid profile email | ✓ |
| 访问令牌有效期 | 建议 1 小时 | 3600 |
| 刷新令牌有效期 | 建议 7 天 | 604800 |
| ID 令牌签名算法 | 推荐 RS256 | RS256 |

4. 点击**保存**
</Step>

<Step>
### 步骤 3：在应用端集成

使用 OIDC 客户端库集成（以 Node.js 为例）：

```javascript
const oidc = require('openid-client');

const client = new oidc.Client({
  client_id: '{your-client-id}',
  client_secret: '{your-client-secret}',
  redirect_uris: ['https://app.example.com/callback'],
  issuer: 'https://{your-tenant}.dingtalk.com'
});

// 1. 生成授权 URL
const authUrl = client.authorizationUrl({
  scope: 'openid profile email'
});

// 2. 交换 Token
const tokenSet = await client.callback(
  'https://app.example.com/callback',
  code
);

// 3. 获取用户信息
const userinfo = await client.userinfo();
```
</Step>

<Step>
### 步骤 4：测试验证

1. 以普通用户身份访问目标应用
2. 确认页面正确跳转到数犀集成平台登录页
3. 完成认证后，自动跳回目标应用并已登录
</Step>
</Steps>

📖 详细配置参见：[单点登录-OIDC](/docs/应用中心/应用单点登录/单点登录-OIDC)

## 方式二：SAML 2.0 接入（以阿里云 RAM 为例）

### 整体流程

```
数犀集成平台 (IdP)          阿里云 RAM (SP)
      │                           │
      │←─── 用户访问应用 ──────────│
      │                           │
      │──── SAML Request ────────→│
      │                           │
      │←─── 用户登录 ─────────────│
      │                           │
      │──── SAML Response ───────→│
      │                           │
      │←─── 登录成功 ─────────────│
```

### 操作步骤

<Steps>
<Step>
### 步骤 1：获取 SP 的 SAML 配置信息

1. 访问[阿里云 RAM 控制台](https://ram.console.aliyun.com/providers)
2. 选择**SSO 管理 > 用户 SSO**
3. 查看 **SAML 服务提供方元数据 URL**，记录：
   - `entityID`：阿里云的 SAML 实体 ID
   - `SSO URL`：单点登录服务地址
</Step>

<Step>
### 步骤 2：在数犀集成平台创建 SAML 应用

1. 登录数犀集成平台，导航至**应用管理 > 应用市场**
2. 搜索并添加"阿里云控制台"应用
3. 进入应用配置页，切换到**单点配置**选项卡
4. 配置以下参数：

| 参数 | 说明 |
|------|------|
| SSO URL | 来自 SP 元数据 |
| 接受者 URL | 通常与 SSO URL 相同 |
| SP 实例 ID | 服务提供商的 entityID |
| 用户 ID 格式 | 建议选择"邮箱" |
| 应用用户名 | 选择与 SP 用户标识匹配的字段 |

5. 点击**保存**
</Step>

<Step>
### 步骤 3：获取并配置 IdP 元数据

1. 在应用配置完成后，访问 **Metadata URL**
2. 将元数据内容保存为 XML 文件
3. 返回阿里云 RAM 控制台 **SSO 管理 > 用户 SSO**
4. 点击**编辑**，开启 SSO 功能
5. 上传数犀集成平台的 SAML 元数据 XML 文件
6. 点击**确定**保存
</Step>

<Step>
### 步骤 4：配置用户映射

1. 确保阿里云 RAM 用户的登录名称与数犀集成平台中对应用户的邮箱地址一致
2. 为用户分配适当的权限策略
</Step>

<Step>
### 步骤 5：测试验证

1. 访问阿里云 RAM 用户登录页面
2. 选择**企业账号登录**
3. 系统自动跳转到数犀集成平台登录页
4. 完成认证后，自动跳回阿里云控制台
</Step>
</Steps>

📖 详细配置参见：[单点登录-SAML 协议](/docs/应用中心/应用单点登录/单点登录-SAML协议)

## 方式三：OAuth 2.0 接入

### 操作步骤

<Steps>
<Step>
### 步骤 1：创建 OAuth2 应用

在数犀集成平台中添加自定义应用，选择**OAuth2** 协议。
</Step>

<Step>
### 步骤 2：配置参数

| 参数 | 说明 |
|------|------|
| RedirectURIs | 平台的回调地址 |
| 授权类型 | 推荐 `authorization_code` |
| 授权范围 | 如 `profile`、`email` |
| Access Token 有效期 | 建议 1 小时 |
| Refresh Token 有效期 | 建议 7 天 |
</Step>

<Step>
### 步骤 3：在目标平台配置认证源

在目标平台添加 OAuth2 企业认证源，配置以下端点：

| 端点 | 说明 |
|------|------|
| 授权端点 | `https://[域名]/oauth2/v1/authorize` |
| 令牌端点 | `https://[域名]/oauth2/v1/token` |
| 用户信息端点 | `https://[域名]/oauth2/v1/userinfo` |
</Step>

<Step>
### 步骤 4：测试验证

按照 OAuth2 授权码流程完成登录测试。
</Step>
</Steps>

📖 详细配置参见：[单点登录-OAuth2](/docs/应用中心/应用单点登录/单点登录-OAuth2)

## 方式四：CAS 接入（以宜搭为例）

### 操作步骤

<Steps>
<Step>
### 步骤 1：创建 CAS 应用

在数犀集成平台中添加自定义应用，选择 **CAS** 协议。
</Step>

<Step>
### 步骤 2：配置 CAS 参数

| 参数 | 说明 |
|------|------|
| Login URL | 应用的登录回调地址 |
| Logout URL | 应用的登出地址 |
| Ver3.0 Validate URL | CAS 3.0 验证 URL |
| Service URL | 目标应用提供的服务 URL |
</Step>

<Step>
### 步骤 3：在宜搭配置 CAS 客户端

1. 确认宜搭支持的 CAS 版本（2.2 或 3.0）
2. 填写 CAS 服务器地址、Client ID、Client Secret
3. 配置回调 URL
</Step>

<Step>
### 步骤 4：测试验证

访问宜搭应用，验证是否能通过数犀集成平台完成登录。
</Step>
</Steps>

📖 详细配置参见：[单点登录-CAS 协议](/docs/应用中心/应用单点登录/单点登录-CAS协议)

## 调试与排错

### 常用调试工具

| 工具 | 用途 |
|------|------|
| SAML Tracer（浏览器插件） | 查看 SAML 请求/响应 XML |
| JWT.io | 解码和验证 JWT Token |
| Postman | 测试 OIDC 端点 |

### 常见错误

| 错误现象 | 可能原因 | 解决方案 |
|----------|----------|----------|
| 登录跳转循环 | ACS URL 配置错误 | 检查回调地址是否正确 |
| 断言验证失败 | 证书过期 | 更新 IdP 证书 |
| 属性为空 | 属性映射未配置 | 检查映射规则 |
| CORS 错误 | 重定向 URI 不匹配 | 确认 URI 白名单 |

### 排查步骤

1. **检查浏览器开发者工具**
   - Network：查看请求/响应详情
   - Console：查看错误日志

2. **查看数犀集成平台日志**
   - 进入**审计日志 > 操作日志**
   - 筛选 SSO 相关操作

3. **验证配置一致性**
   - Entity ID 是否完全一致
   - ACS URL 是否可访问
   - 证书是否有效

## 最佳实践

**安全建议**
- 所有 SSO 相关地址必须使用 **HTTPS**
- 合理设置令牌有效期（访问令牌建议 1-2 小时）
- 启用最小权限许可，仅请求必要的权限
- 定期轮换密钥（建议每年一次）

**性能建议**
- 缓存 IdP 元数据，减少网络请求
- 使用会话 Cookie，避免每次操作都验证 Token
- 合理设置作用域，只请求必要的用户信息
