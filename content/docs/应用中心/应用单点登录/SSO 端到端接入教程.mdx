本文档提供从零开始配置单点登录（SSO）的完整指南。涵盖两种主流协议：**SAML 2.0** 和 **OIDC**，并提供常见应用的对接示例。

---

## 什么是单点登录

单点登录（Single Sign-On，SSO）允许用户只需登录一次，即可访问所有已集成的中犀集成平台中的应用系统。

```
未使用 SSO：
用户 → 应用 A（需登录）→ 应用 B（需登录）→ 应用 C（需登录）

使用 SSO：
用户 → 数犀集成平台（登录一次）→ 应用 A → 应用 B → 应用 C
```

---

## 前置准备

| 准备项 | 说明 |
|--------|------|
| 数犀集成平台管理员账号 | 需要应用中心管理员权限 |
| 目标应用 | 需要支持 SAML 或 OIDC 协议 |
| 了解应用支持的协议 | 查阅目标应用的对接文档 |
| HTTPS 环境 | 生产环境必须使用 HTTPS |

---

- 选择合适的协议

| 协议 | 适用场景 | 复杂度 | 代表应用 |
|------|----------|--------|----------|
| **SAML 2.0** | 企业级应用、传统软件、硬件设备 | 中 | 泛微 OA、致远 OA、SAP |
| **OIDC** | 现代 Web 应用、SPA、移动 APP | 低 | 自定义 Web 应用 |
| **OAuth 2.0** | API 授权、第三方登录 | 中 | 阿里云、纷享销客 |
| **CAS** | 高校、科研机构 | 低 | 校级系统 |

> **建议**：如果应用同时支持多种协议，推荐优先使用 **OIDC**，配置更简单。

- **方式一**：SAML 2.0 接入（以泛微 OA 为例）

### 整体流程

```
数犀集成平台 (IdP)          泛微 OA (SP)
      │                           │
      │←─── 用户访问应用 ──────────│
      │                           │
      │──── SAML Request ────────→│
      │                           │
      │←─── 用户登录 ─────────────│
      │                           │
      │──── SAML Response ───────→│
      │                           │
      │←─── 登录成功 ─────────────│
```

### 步骤 1：在数犀集成平台创建应用

- 进入「应用中心 > 应用创建与配置」
1. 点击「新建应用」
- 填写应用信息：
   - 应用名称：泛微 OA
   - 应用类型：企业应用
   - 描述：内部 OA 系统
1. 点击「创建」

### 步骤 2：配置 SAML 认证

- 进入应用详情，点击「单点登录」tab
- **选择协议类型**：**SAML 2.0**
- 配置 IdP 元数据：

| 参数 | 说明 | 示例值 |
|------|------|--------|
| SP ID / Entity ID | 应用的唯一标识 | `https://oa.example.com/saml/metadata` |
| ACS URL | 断言消费服务地址 | `https://oa.example.com/saml/acs` |
| 签名算法 | 推荐 RSA-SHA256 | RSA-SHA256 |

- 点击「保存」
1. 下载 **IdP 元数据文件**（XML 格式）

### 步骤 3：在泛微 OA 配置 SAML

- 登录泛微 OA 管理后台
1. 进入「系统管理 > 认证管理 > SSO 配置」
- 选择 SAML 协议
1. 上传数犀集成平台的 IdP 元数据文件
- 配置属性映射：

| 数犀集成平台属性 | 泛微 OA 字段 |
|-----------------|-------------|
| username | 泛微用户账号 |
| email | 邮箱 |
| department | 部门 |

- 保存配置

### 步骤 4：测试验证

- 在数犀集成平台点击「测试登录」
1. 系统跳转至泛微 OA
2. 如果已登录，自动返回
3. 如果未登录，显示数犀集成平台登录页
4. 登录后验证是否能正常访问

详细 SAML 配置：[单点登录-SAML协议](./单点登录-SAML协议.mdx)

- **方式二**：OIDC 接入（以自定义 Web 应用为例）

### 整体流程

```
用户 → 应用 → 授权码请求 → 数犀集成平台 → 用户登录 → 授权码 → 应用 → 换取 Token → 完成
```

### 步骤 1：在数犀集成平台创建应用

- 进入「应用中心 > 应用创建与配置」
1. 点击「新建应用」
- 选择应用类型：自定义应用 / OIDC 应用
1. 填写应用信息

### 步骤 2：配置 OIDC 参数

- 进入应用详情，点击「单点登录」tab
- **选择协议类型**：**OIDC**
- 配置参数：

| 参数 | 说明 | 示例值 |
|------|------|--------|
| 客户端 ID | 自动生成 | `oidc_app_12345` |
| 客户端密钥 | 点击生成 | `****` |
| 重定向 URI | 授权后跳转地址 | `https://app.example.com/callback` |
| 授权类型 | Authorization Code | ✓ |
| 作用域 | openid profile email | ✓ |

- 配置令牌参数：

| 参数 | 说明 | 推荐值 |
|------|------|--------|
| 访问令牌有效期 | | 1 小时 |
| 刷新令牌有效期 | | 7 天 |
| ID 令牌签名算法 | | RS256 |

- 点击「保存」

### 步骤 3：获取 IdP 配置信息

在应用详情页面获取：

```json
{
  "issuer": "https://{your-tenant}.idaas.example.com",
  "authorization_endpoint": "https://{your-tenant}.idaas.example.com/oauth2/authorize",
  "token_endpoint": "https://{your-tenant}.idaas.example.com/oauth2/token",
  "userinfo_endpoint": "https://{your-tenant}.idaas.example.com/oauth2/userinfo",
  "jwks_uri": "https://{your-tenant}.idaas.example.com/.well-known/jwks.json"
}
```

### 步骤 4：在应用端集成

使用 OIDC 客户端库集成（以 Node.js 为例）：

```javascript
const oidc = require('openid-client');

const client = new oidc.Client({
  client_id: '{your-client-id}',
  client_secret: '{your-client-secret}',
  redirect_uris: ['https://app.example.com/callback'],
  issuer: 'https://{your-tenant}.idaas.example.com'
});

// 1. 生成授权 URL
const authUrl = client.authorizationUrl({
  scope: 'openid profile email'
});

// 2. 交换 Token
const tokenSet = await client.callback(
  'https://app.example.com/callback',
  code
);

// 3. 获取用户信息
const userinfo = await client.userinfo();
```

详细 OIDC 配置：[单点登录-OIDC](./单点登录-OIDC.mdx)

## 调试与排错

### 常用调试工具

| 工具 | 用途 |
|------|------|
| SAML Tracer (浏览器插件) | 查看 SAML 请求/响应 XML |
| JWT.io | 解码和验证 JWT Token |
| Postman | 测试 OIDC 端点 |

### 常见错误

| 错误现象 | 可能原因 | 解决方案 |
|----------|----------|----------|
| 登录跳转循环 | ACS URL 配置错误 | 检查回调地址是否正确 |
| 断言验证失败 | 证书过期 | 更新 IdP 证书 |
| 属性为空 | 属性映射未配置 | 检查映射规则 |
| CORS 错误 | 重定向 URI 不匹配 | 确认 URI 白名单 |

### 排查步骤

1. **检查浏览器开发者工具**
   - Network：查看请求/响应详情
   - Console：查看错误日志

2. **查看数犀集成平台日志**
   - 进入「审计日志 > 操作日志」
   - 筛选 SSO 相关操作

3. **验证配置一致性**
   - Entity ID 是否完全一致
   - ACS URL 是否可访问
   - 证书是否有效

## 最佳实践

### 安全建议

1. **使用 HTTPS**：所有 SSO 相关地址必须使用 HTTPS
2. **合理设置令牌有效期**：访问令牌建议 1-2 小时
3. **启用增量许可**：仅请求必要的权限
4. **定期轮换密钥**：建议每年更换一次

### 性能建议

1. **缓存 IdP 元数据**：减少网络请求
2. **使用会话 Cookie**：避免每次操作都验证 Token
3. **合理设置作用域**：只请求必要的用户信息
