---
title: 应用级 Passkey（FIDO2）MFA 配置
description: 在应用的单点登录配置中单独启用 Passkey 认证方式，实现应用级别的差异化 MFA 策略，支持继承平台设置或单独配置。
---

## 功能概述

数犀集成平台支持在单个应用的登录配置中独立启用 **Passkey（FIDO2）** 认证方式，管理员可为不同安全级别的应用配置不同的 MFA 策略，无需全局开启 Passkey 即可针对特定应用启用。

**配置模式：**

| 模式 | 说明 |
|------|------|
| 继承平台设置 | 使用全局 MFA 配置中的 Passkey 开关状态 |
| 应用单独配置 | 忽略全局设置，为该应用单独启用或关闭 Passkey |

## 适用场景

| 场景 | 说明 |
|------|------|
| 高安全级别应用 | 财务系统、研发平台等核心应用单独强制要求 Passkey 验证 |
| 差异化 MFA 策略 | 不同应用启用不同 MFA 组合，如财务应用启用 Passkey + TOTP，普通应用仅启用短信 |

## 配置步骤

<Steps>
<Step>
### 步骤一：进入应用单点登录配置

在数犀集成平台管理控制台中，进入 **应用中心 > 应用管理**，找到目标应用，点击「设置」进入应用配置页面，切换到「单点登录」选项卡。
</Step>
<Step>
### 步骤二：找到 MFA 认证方式配置区域

在单点登录配置页面中，找到「MFA 认证方式」配置区域，可以看到短信验证、邮箱、TOTP 等已有的 MFA 选项。
</Step>
<Step>
### 步骤三：启用 Passkey 选项

在 MFA 认证方式列表中，找到「Passkey（FIDO2）」选项并开启。

<Callout type="warn">
开启 Passkey 后，系统将显示以下提示：

**「Passkey 验证依赖浏览器 WebAuthn 支持，部分客户端内嵌浏览器（如钉钉、企业微信内嵌浏览器）可能不支持 Passkey，建议同时开启其他 MFA 方式作为备选。」**

建议同时保持短信或 TOTP 等备选方式处于开启状态。
</Callout>
</Step>
<Step>
### 步骤四：保存配置

点击「保存」，配置立即生效。用户下次登录该应用时，MFA 验证阶段将显示 Passkey 选项。
</Step>
</Steps>

## 浏览器兼容性说明

| 浏览器环境 | Passkey 支持情况 | 处理方式 |
|-----------|----------------|---------|
| Chrome / Safari / Firefox（最新版）| ✅ 完整支持 | 正常显示 Passkey 选项 |
| 钉钉内嵌浏览器 | ⚠️ 可能不支持 | 隐藏 Passkey 选项，仅显示其他 MFA |
| 企业微信内嵌浏览器 | ⚠️ 可能不支持 | 同上 |
| IE / 旧版浏览器 | ❌ 不支持 | 隐藏 Passkey 选项 |

<Callout type="info">
当浏览器不支持 Passkey 时，系统会自动隐藏 Passkey 验证选项，用户只能使用其他已开启的 MFA 方式完成验证。如果应用强制要求 Passkey 但环境不支持，系统将提示用户使用系统浏览器打开或联系管理员。
</Callout>

## 常见问题

<Accordions>
<Accordion title="全局 MFA 未开启 Passkey，应用级别可以单独开启吗？">
可以。选择「应用单独配置」模式后，应用级别的 Passkey 设置独立于全局配置，可单独开启。
</Accordion>
<Accordion title="用户需要提前注册 Passkey 才能使用吗？">
是的。用户需要在个人设置中预先绑定 Passkey（如系统 Touch ID、Face ID 或安全密钥），之后才能在 MFA 验证阶段使用 Passkey 完成验证。
</Accordion>
</Accordions>
