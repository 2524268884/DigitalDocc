## 概述
本文档详细介绍如何配置集成平台与金蝶云星空系统之间的单点登录功能。通过此配置，企业用户可以使用集成平台的身份认证直接登录金蝶云星空系统，无需重复输入账号密码，显著提升用户体验和工作效率。
金蝶云星空是一款企业级云ERP系统，为企业提供财务管理、供应链管理、生产制造、人力资源等全面的业务管理解决方案。通过单点登录集成，可实现企业统一身份认证体系与金蝶云星空的无缝连接。

### 中文编码问题解决方案
在使用中文语言ID时，可能会遇到中文名无法正确传入系统的问题，只能到达登录界面而无法登录成功。这是由于字符编码问题导致的，解决方法如下：
方法一：修改金蝶系统配置文件
需要金蝶客户方协助在金蝶系统的common.config文件中添加以下配置：

```xml
&lt;!-- 默认为操作系统默认编码，例如简体服务器为GB2312编码器，如果需要指定，可以用下面参数指定utf-8 --&gt;
&lt;add key="StartAppPluginUDConfig" value="{encoding:'utf-8'}" /&gt;
```

同时，在生成单点登录参数时，使用UTF-8编码并进行Base64转换：

```csharp
string argJsonBase64 = System.Text.UTF8Encoding.UTF8.GetBytes(argJosn).ToBase64(); //base64编码
```
方法二：使用URL参数指定编码（适用于2021年3月后的版本）
在构建URL时添加参数udencoding=utf-8：
## 将生成好的ud参数整体进行基于UTF-8的Base64编码
- 在URL地址中增加参数&udencoding=utf-8
示例URL格式：

```
https://xxx/k3cloud/html5/index.aspx?udencoding=utf-8&ud=###
```
参考资料：更多详细信息可参考金蝶云星空社区的集成文档
### 授权范围配置
配置可访问金蝶云星空应用的用户范围。集成平台支持以下三种授权方式：
## 在集成平台中，配置可访问金蝶应用的用户范围：
- 在应用详情页，切换到"授权范围"选项卡
## 可通过以下方式进行授权：
  - 基于组织架构和用户授权：授权特定部门或用户
基于组织架构和用户授权

  - 基于角色授权(RBAC)：通过静态标签(角色)授权
基于角色授权（RBAC）

  - 基于用户属性授权(ABAC)：通过动态标签(用户属性)授权
基于用户属性授权（ABAC）

## 根据企业需求选择合适的授权方式并配置授权范围
## 保存配置
根据企业实际需求，选择合适的授权方式，并配置相应的授权范围，确保只有授权用户才能访问金蝶云星空应用。
## 用户映射配置
为确保单点登录成功，需要正确配置集成平台用户与金蝶云星空系统用户之间的映射关系：
## 确定映射字段：
  - 通常使用用户名、工号或邮箱作为映射字段
  - 确保两个系统中对应字段的值保持一致
## 用户数据准备：
  - 确认金蝶云星空系统中已创建相应用户
  - 确保集成平台中的用户属性值与金蝶系统匹配
## 特殊字符处理：
  - 注意用户名中的特殊字符可能需要进行编码处理
  - 对于中文用户名，参考前述编码解决方案
