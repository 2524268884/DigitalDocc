## 概述
本文档详细介绍如何配置纷享销客与集成平台之间的待办消息免登功能。此功能允许用户在收到纷享销客待办消息后，点击待办消息可以无需重复登录即可直接进入相应的待办页面，极大地提升用户体验和工作效率。
### 功能价值
- 简化用户操作流程：用户点击待办消息后无需额外登录，直接进入待办处理页面
- 提高工作效率：减少重复登录步骤，加快待办事项处理速度
- 增强用户体验：流畅的操作体验，降低用户操作门槛
- 支持多渠道应用：可应用于钉钉等多种消息推送渠道
### 技术实现原理
该功能通过纷享销客的"自定义认证服务商"功能实现，利用Java代码编写自定义认证逻辑，结合集成平台的身份认证服务，实现用户身份自动识别和无缝跳转至指定待办页面。
1. 前提条件
在开始配置前，请确保满足以下条件：
- 账号准备：
  - 拥有纷享销客管理员账号，具备认证服务商配置权限
  - 拥有集成平台管理员账号
- 用户数据准备：
  - 集成平台中已创建相应用户
  - 纷享销客中已创建对应用户
  - 关键：确保纷享销客用户的手机号与集成平台中对应用户的手机号保持一致
- 技术要求：
  - 对Java编程有基本了解，能够理解和修改示例代码
  - 了解OAuth 2.0协议基本原理
  - 对API调用有基本认识

关键代码部分需要修改：
- **应用登录地址**：修改为集成平台的应用登录地址

```javascript
return "https://您的域名/iam/authorize?response_type=code&" +
      "response_mode=query&" +
      "scope=openid%20profile%20address%20phone%20email%20offline_access&" +

```




- **待办跳转逻辑**：根据待办消息传递的参数，处理跳转逻辑

```java
@Override
String redirect(FunctionContext context, Map&lt;String, List&lt;String&gt;&gt; params) {
    if(params["state"] != null && params["state"].size() > 0 && params["state"].get(0)!=""){
        return params["state"].get(0);
    }else{
        // 默认跳转逻辑
        String s = params["continue"].get(0);
        String substring = params["continue"].get(0).substring(74, s.length());
        return "https://www.fxiaoke.com/XV/UI/Home#objdetail/AccountObj/" + substring;
    }
}
```



示例：https://\{your-tenant\}.my.fxiaoke.com/oauth/sp/sso/todo?continue= 代办地址两次编码
continue 参数可以修改，但是定义好之后就不能变
不相同continue 的话 可以用task todo等，脚本中也会跟着变
- **自动登出URL**：配置登出时跳转的地址

```java
@Override
String logoffUrl(FunctionContext context) {
    return "https://&lt;您的域名&gt;/iam/logout"
}
```



- **用户信息处理**：处理集成平台与纷享销客之间的用户匹配
注意：纷享销客登录的用户账号是不带"+86"前缀的手机号，而集成平台集成钉钉的用户手机号带有"+86"前缀，需要特殊处理以确保用户匹配

```java
String user = (String)o.content["phone_number"];
if(user.size() == 15){
    user = user.substring(4, 15); // 移除+86前缀
}
```



第三方用户信息获取接口：专属集成平台客户环境获取用户信息接口：https://&lt;您的域名&gt;/iam/userinfo
- **用户信息获取接口**：修改为您的集成平台环境地址

HttpResult o = (HttpResult) http.get("https://您的域名/iam/userinfo", ["Authorization": "Bearer "+accessToken]).getData();

### 待办消息推送配置
向消息推送开发人员提供待办消息跳转URL，格式为：

```plaintext
https://您的域名.my.fxiaoke.com/oauth/sp/sso/todo?continue=待办页面URL两次编码
```

说明：continue参数是待办详情页的URL，需要进行两次URL编码。参数名称（如continue）可以根据需要修改为task、todo等，但修改后脚本中的参数名也需要同步修改。

## 常见问题(FAQ)
### 配置问题
Q1: 待办消息点击后无法正常跳转到集成平台
A: 可能原因及解决方案：
- 检查纷享销客中配置的应用登录地址是否正确
- 验证URL中的client_id是否与集成平台应用一致
- 确认redirect_uri是否正确编码
Q2: 认证成功后未跳转到正确的待办页面
A: 可能原因：
- 检查redirect函数中的跳转逻辑是否正确
- 确认continue参数是否正确传递并解析
- 验证continue参数的URL编码是否正确（需两次编码）
Q3: 用户认证失败，提示"找不到对应的用户"
A: 这通常是由于用户匹配问题导致：
- 确认纷享销客中用户的手机号与集成平台用户手机号一致
- 检查代码中对手机号前缀（+86）的处理逻辑是否正确
- 验证查询PersonnelObj时使用的字段名是否正确
### 开发问题
Q4: 如何调试自定义认证服务商代码？
A: 可以通过以下方式进行调试：
- 在代码中使用log.info()输出关键变量和状态
- 通过纷享销客后台的APL日志查看执行情况
- 根据日志信息定位和解决问题
Q5: 如何处理不同类型的待办消息跳转？
A: 可以通过continue参数传递不同的目标URL，并在redirect函数中进行判断：

```java
if(continueUrl.contains("AccountObj")) {
    // 客户相关待办处理逻辑
} else if(continueUrl.contains("OpportunityObj")) {
    // 商机相关待办处理逻辑
} else {
    // 默认处理逻辑
}
```

Q6: 代码中的clientid和clientsecret如何获取？
A: 这些参数来自集成平台创建的应用：
- client_id对应应用的Client ID
- client_secret对应应用的Client Secret
- 这些信息可在集成平台应用详情页查看
