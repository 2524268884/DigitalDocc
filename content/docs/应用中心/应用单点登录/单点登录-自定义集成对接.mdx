## 概述
自定义单点登录配置是 专属集成平台提供的高级 SSO 集成能力，专为复杂的单点登录场景设计。通过 AI FLOW 的可视化流程编排，用户可以灵活配置多步骤 API 调用，最终生成符合第三方平台标准的单点登录 URL，实现高度定制化的身份认证集成。
功能特性
- ✅ 多 API 编排：支持调用多个 API 接口获取登录参数
- ✅ 灵活数据处理：自定义参数拼接和数据转换逻辑
- ✅ 可视化配置：基于 AI FLOW 的拖拽式流程设计
- ✅ 标准返回值：固定返回格式 `\{sso_url: "xxxxx"\}`
- ✅ 流程保护：防止误删除正在使用的 SSO 流程
- ✅ 实时测试：支持使用实际用户数据进行流程测试
1. 适用场景
1. 复杂 API 调用链：需要调用多个 API 接口才能完成登录准备
- **动态参数生成**：登录 URL 需要根据用户属性动态生成
- **自定义认证流程**：标准 SSO 协议无法满足的特殊认证需求
- **第三方系统集成**：如网易邮箱、企业内部系统等复杂 SSO 场景
核心价值
- 解决复杂集成：突破标准 SSO 协议的限制，支持任意复杂度的认证流程
- 降低开发成本：无需编写复杂代码，通过可视化配置实现
- 提高集成效率：从传统开发模式大幅缩短集成周期
- 增强可维护性：可视化流程便于理解和后续维护
功能架构
技术实现原理
- **协议扩展**：在现有 SSO 协议基础上新增"自定义集成对接"类型
- **流程引擎**：利用 AI FLOW 的强大编排能力处理复杂业务逻辑
3. Webhook 触发：通过 HTTP Webhook 接收用户访问请求和参数
- **数据桥接**：在 AI FLOW 和 SSO 系统间建立数据桥梁
5. URL 重定向：前端接收处理结果后自动跳转完成登录
数据流转过程

```
用户访问 SSO URL → 前端发起认证请求 → 触发 AI FLOW Webhook
    ↓
AI FLOW 接收参数 → 执行编排的 API 调用 → 数据处理和转换
    ↓
生成最终 SSO URL → 返回标准格式结果 → 前端重定向完成登录
```
1. 管理员配置指南
2. 创建自定义单点登录连接
步骤一：进入应用管理
1. 登录管理后台
  - 使用管理员账号访问 专属集成平台管理控制台
1. 选择目标应用
  - 导航至"应用管理"页面
  - 找到需要配置 SSO 的应用
  - 点击"编辑"或"SSO 配置"


步骤二：选择协议类型
1.
- 查看协议选项
  - 选择"自定义集成对接"
2.
- 查看功能描述
  - 系统显示功能说明：> 通过自定义集成对接功能，可以灵活编排单点登录（SSO）URL 的获取方式，并根据平台标准规范设置流程返回值，从而实现高度可控、可扩展的单点登录配置能力。
步骤三：进入流程编排
1. 点击编排按钮
  - 在配置页面找到"进入流程编排"按钮
  - 点击按钮跳转至 AI FLOW 配置界面
2.
- 查看默认流程
  - 系统自动创建基础流程结构：
■
触发节点：Webhook 节点（默认配置）
■
执行节点：返回变量节点（默认返回 `\{sso_url: "xxxxx"\}`）
1. 了解测试数据
  - Webhook 节点的测试数据包含当前租户的全部用户字段
  - 支持实时预览和编辑测试数据
步骤五：配置认证流程
在流程中完成对应的 API 调用，并将 API 最终返回的认证 url结果作为返回值完成配置
1. 修改返回变量节点
  - 系统默认创建的返回节点
  - 确保返回格式为：`\{sso_url: "xxxxx"\}`
