## 概述

企业应用分发帮助企业将自研或采购的移动应用（iOS/Android）集中分发给内部员工，员工通过统一门户下载安装，无需上架公开应用市场。

## 两种分发模式

| 模式 | 说明 | 适用场景 |
|------|------|---------|
| **启用平台分发** | 应用包存储在 OSS，员工直接从平台下载 | SaaS 环境直接可用；私有化部署需提前配置 OSS |
| **不启用平台分发** | 填写外部下载链接，平台做跳转 | 已有自建分发渠道，不想占用平台流量 |

<Callout type="info">
私有化部署环境使用平台分发需要额外配置 OSS（对象存储），请联系商务采买配置。SaaS 环境开箱即用。
</Callout>

## 上架 iOS 应用

<Steps>
<Step>
### 上传应用包

进入**应用中心 > 企业应用分发**，点击「上传」，选择本地 `.ipa` 文件。系统自动解析并提取应用图标和基本信息。
</Step>
<Step>
### 填写应用信息

| 参数 | 说明 | 是否必填 |
|------|------|---------|
| 下载地址 | 应用下载地址，由管理员根据分发模式填写 | 是 |
| 分发模式 | `in-house`（内部应用）或 `unlisted`（未上市应用）| 是 |
| 系统要求 | 最低 iOS 版本要求 | 否 |
| 兼容设备 | 支持的设备类型 | 否 |
| 开发商 | 开发商名称 | 否 |
| 应用简介 | 应用功能描述 | 否 |

</Step>
<Step>
### 上架应用

信息填写完成后，点击「上架应用」。上架后的应用出现在用户下载页面，员工可以扫码或点击链接下载安装。

<Callout type="warn">
上架后的应用**不能直接编辑**。如需修改信息，必须先「下架」，修改后重新上架。删除操作同样需要先下架。
</Callout>
</Step>
</Steps>

## 上架 Android 应用

<Steps>
<Step>
### 上传 APK

进入**应用中心 > 企业应用分发**，切换到「Android 应用」标签，点击「上传」，选择 `.apk` 文件。系统自动解析并提取图标。
</Step>
<Step>
### 填写应用信息

| 参数 | 说明 |
|------|------|
| 下载地址 | 应用下载地址 |
| 系统要求 | 最低 Android 版本要求 |
| 开发商 | 开发商名称 |
| 应用简介 | 应用功能描述 |

</Step>
<Step>
### 上架应用

点击「上架」，应用进入分发页面供员工下载。
</Step>
</Steps>

## 应用版本管理

上架应用支持多版本管理：

- **新增版本**：上传新的应用包并配置版本信息，与旧版本共存
- **强制更新**：可设置最低版本要求，低于该版本的用户将被强制更新
- **下架旧版**：不再需要的版本可单独下架，不影响其他版本

## 常见问题

<Accordions>
<Accordion title="员工下载后安装失败（iOS）？">
iOS 企业应用需要信任开发者证书。安装时若提示「不受信任的企业级开发者」，员工需要：
1. 进入 iPhone「设置」→「通用」→「VPN 与设备管理」
2. 找到对应的开发者证书，点击「信任」
</Accordion>
<Accordion title="私有化部署环境如何配置 OSS？">
请联系数犀集成平台商务团队，说明您的私有化部署环境，获取 OSS 配置指引和采买方案。
</Accordion>
</Accordions>
