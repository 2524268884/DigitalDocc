## 概述

企业应用分发用于向企业内部人员分配应用程序，满足工作需求。平台支持多种终端平台的应用分发：

- **iOS 应用**：IPA 包上传分发
- **Android 应用**：APK 包上传分发
- **HarmonyOS Next 应用**：通过信息配置创建
- **Windows 应用**：支持 32 位和 64 位版本
- **Mac 应用**：支持 M 芯片和 Intel 芯片版本

### 分发方式

| 方式 | 说明 |
|------|------|
| **启用平台分发** | 应用存储于 OSS 中，用户下载时消耗平台流量。私有化部署需配置 OSS，SaaS 环境无需额外配置 |
| **不启用平台分发** | 用户下载时跳转至填写的下载链接 |

## 前提条件

- 拥有数犀集成平台管理员账号
- 准备待分发的应用安装包（IPA/APK 等格式）
- 了解应用的系统要求、兼容性等信息

## 操作步骤

<Tabs items={['iOS 应用', 'Android 应用', 'HarmonyOS Next', 'Windows 应用', 'Mac 应用']}>
<Tab value="iOS 应用">
<Steps>
<Step>
### 步骤 1：上传应用包

进入**应用中心 > 企业应用分发**，点击**上传**，选择本地的 iOS 应用包（.ipa）上传。系统会自动解析应用包并提取应用图标和基本信息。
</Step>

<Step>
### 步骤 2：填写应用信息

| 参数 | 说明 |
|------|------|
| 下载地址 | 应用的下载地址（平台分发或外部链接） |
| 分发模式 | in-house（内部应用）/ unlisted（未上市应用） |
| 系统要求 | 应用要求的最低 iOS 版本 |
| 兼容设备 | iPhone / iPad |
| 开发商 | 开发商名称 |
| 应用简介 | 应用介绍文字 |
</Step>

<Step>
### 步骤 3：上架应用

点击**上架**，应用将显示在用户的下载页面中。

<Callout type="warn">
上架后的应用不能直接编辑，如需修改需先下架，下架后才能删除。
</Callout>
</Step>
</Steps>

## 应用上下架管理

| 操作 | 说明 |
|------|------|
| **上架** | 点击"上架"按钮，应用状态变为"已上架"，用户可在下载页面看到并下载 |
| **下架** | 已上架的应用如需修改或删除，必须先下架。点击"下架"按钮确认后状态变为"未上架" |
| **编辑** | 应用下架后，点击编辑图标可修改应用信息，修改完成后重新上架 |
| **删除** | 应用下架后，点击删除图标可永久删除（不可恢复） |

<Callout type="warn">
上架的应用不能直接编辑或删除，必须先下架后才能执行这些操作，确保用户访问的应用信息保持一致性。
</Callout>

## 分发下载页面配置

该配置应用于所有类型的应用（企业应用分发和 ABM 应用分发共用同一个分发下载页面）。

| 参数 | 说明 |
|------|------|
| 页面 Logo | 自定义应用图标 |
| 页面背景图 | 自定义背景图片 |
| 按钮颜色 | 管理员自定义颜色 |
| 标题文案 | 页面显示的标题，中英文用 `\` 分隔 |
| 底部提示文案 | 页面底部提示，中英文用 `\` 分隔 |
| 下载说明 | 页面显示的应用说明，中英文用 `\` 分隔 |
| 验证方式 | 不验证 / 提取码 / 短信 |

<Callout type="info">
下载页面支持中英文版本。移动端扫码后，根据设备和系统自动显示对应的系统应用选项。
</Callout>

## 最佳实践

**应用包管理建议**
- 版本命名使用统一的语义化版本号（Semantic Versioning）
- 大型应用考虑提供增量更新包，减少下载流量
- 确保所有应用包经过正确签名，以便用户设备验证
- 完整填写应用信息，提供清晰的应用描述和使用指南

**分发策略建议**
- 根据部门或角色进行分组分发，确保用户只能下载与其工作相关的应用
- 重要应用更新时先向小部分用户推送，验证稳定性后再全量发布
- 对包含重要安全更新的版本可配置强制更新策略
- 保留历史版本，新版本出现问题时可快速回滚

**安全建议**
- 建立应用上架前的安全审核机制
- 上传应用前进行病毒和恶意代码扫描
- 审查应用请求的权限，确保符合最小权限原则
- 设置 IP 限制或组织网络限制，防止应用包外泄

## 常见问题

<Accordions>
<Accordion title="iOS 应用无法安装">
**可能原因：**
- 企业证书过期
- 设备不信任企业证书
- 应用包签名错误

**解决方法：**
1. 更新企业证书
2. 在设备中信任企业开发者
3. 重新签名应用包
</Accordion>

<Accordion title="Android 应用安装失败">
**可能原因：**
- APK 签名验证失败
- 系统版本不兼容
- 设备未开启"未知来源"安装权限

**解决方法：**
1. 检查 APK 签名
2. 确认应用与设备系统兼容性
3. 指导用户开启"未知来源"安装权限
</Accordion>

<Accordion title="下载速度慢">
**可能原因：**
- 应用包体积过大
- 网络带宽限制
- 下载服务器负载高

**解决方法：**
1. 优化应用包大小
2. 使用 CDN 加速下载
3. 增加下载服务器资源
</Accordion>

<Accordion title="应用上架后无法编辑">
**解决方法：**
1. 先将应用下架
2. 下架后可以进行编辑操作
3. 编辑完成后重新上架
</Accordion>
</Accordions>
