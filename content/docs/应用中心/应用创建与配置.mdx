## 概述

应用创建与配置是数犀集成平台的基础功能，通过该功能可以将各类企业应用纳入统一管理，实现单点登录和访问控制。平台支持以下核心能力：

- **多种添加方式**：从应用市场添加、创建自定义应用、导入第三方工作台应用
- **应用基础信息配置**：设置应用名称、图标、分类、描述等
- **单点登录配置**：支持 SAML、OIDC、OAuth2、CAS 等多种协议
- **授权管理**：控制哪些用户可以访问应用
- **生命周期管理**：启用、禁用、删除应用

## 前提条件

- 拥有数犀集成平台管理员账号
- 已规划好应用分类体系
- 对于第三方工作台导入，需要具备对应平台（钉钉、企业微信等）的管理员权限

## 操作步骤

### 方式一：创建自定义应用

<Steps>
<Step>
### 步骤 1：进入应用管理

登录管理控制台，进入**应用中心 > 应用管理**，点击**添加应用 > 自定义应用**。
</Step>

<Step>
### 步骤 2：设置应用基本信息

填写以下基本信息后点击**下一步**：

| 参数 | 说明 |
|------|------|
| 应用名称 | 应用在系统内的显示名称，建议简洁易识别 |
| 打开方式 | 当前页面 / 新窗口 / 用户自定义（用户可在服务台自选） |
| 应用图标 | 上传应用 Logo，建议尺寸 128×128 像素（PNG/JPG） |
| 应用分类 | 影响应用在门户中的展示位置 |
| 应用简介 | 简要描述应用的功能和用途 |
| 首页 URL | 应用访问的入口地址 |
| 是否允许自助申请 | 用户是否可自行申请应用权限 |
</Step>

<Step>
### 步骤 3：配置单点登录参数（可选）

根据应用支持的协议选择对应配置：

- **SAML 协议**：配置 SP 实体 ID、ACS URL、断言属性映射
- **OIDC 协议**：配置 RedirectURIs、授权类型、Token 有效期
- **OAuth2 协议**：配置客户端凭证、授权范围
- **CAS 协议**：配置服务 URL、票据校验方式

<Callout type="info">
如暂不配置单点登录，可跳过此步骤，后续在应用详情页随时补充配置。
</Callout>
</Step>

<Step>
### 步骤 4：设置授权范围

切换到**授权范围**选项卡，选择可访问此应用的用户范围（组织架构、角色或指定用户）。
</Step>

<Step>
### 步骤 5：保存并完成

点击**保存**完成应用创建。应用将出现在应用列表中，状态为"已启用"。
</Step>
</Steps>

---

### 方式二：从应用市场添加

<Steps>
<Step>
### 步骤 1：进入应用市场

在**应用中心 > 应用管理**页面，点击**应用市场**按钮。
</Step>

<Step>
### 步骤 2：找到并选择应用

浏览或搜索目标应用，点击应用卡片查看详情，确认后点击**添加应用**。
</Step>

<Step>
### 步骤 3：完成配置

按需配置应用基本信息和单点登录参数，点击**保存**完成添加。
</Step>
</Steps>

---

### 方式三：导入第三方工作台应用

数犀集成平台支持从钉钉、企业微信等第三方工作台导入已有应用。

<Steps>
<Step>
### 步骤 1：选择第三方应用来源

在**应用中心 > 应用管理**，点击**添加应用 > 第三方应用**，选择应用来源（如"钉钉应用"）。
</Step>

<Step>
### 步骤 2：选择要导入的应用

选择企业账号，在列表中勾选需要导入的应用。
</Step>

<Step>
### 步骤 3：完成导入

确认后点击**导入**，应用将显示在应用列表中。
</Step>
</Steps>

<Callout type="info">
对于已导入的钉钉应用，支持以下操作：
- **禁用**：禁用后该应用不可在专属钉钉上使用
- **删除**：当应用在钉钉开放平台有信息变动时，可删除后重新导入
- **更新**：再次导入时系统会更新基础信息，同时保留原有配置
</Callout>

## 应用管理

### 应用列表操作

| 操作 | 说明 |
|------|------|
| 搜索 | 通过应用名称、类型等条件快速查找 |
| 筛选 | 按状态、类型、分类等条件筛选 |
| 编辑 | 修改基本信息、SSO 配置和授权范围 |
| 启用 / 禁用 | 控制应用是否可被访问 |
| 删除 | 从系统中永久移除应用 |
| 查看统计 | 查看访问量、用户数等统计信息 |

### 应用分类管理

<Steps>
<Step>
### 步骤 1：新建分类

进入**应用中心 > 应用分类管理**，点击**新建分类**，输入分类名称和描述，按需设置分类图标。
</Step>

<Step>
### 步骤 2：调整分类顺序

通过拖拽调整分类顺序，点击**保存排序**使更改生效。
</Step>
</Steps>

## 最佳实践

**应用命名规范**
- 使用简洁明了的名称，同类应用使用统一命名前缀（如 `HR-xxx`、`IT-xxx`）
- 测试环境应用名称前加 `测试-` 前缀，与生产环境区分

**应用图标规范**
- 推荐使用 128×128 像素的正方形图标，格式为 PNG 或 JPG
- 保持图标风格统一，避免过于复杂的设计

**应用分类建议**
- 基于业务域划分，如"HR 系统"、"财务系统"
- 控制分类数量，避免过多分类导致导航复杂
- 为常用应用创建"常用应用"分类

## 常见问题

<Accordions>
<Accordion title="应用创建后无法访问">
**可能原因：**
- 应用 URL 配置错误
- 单点登录配置参数不正确
- 网络或防火墙限制

**解决方法：**
1. 检查应用首页 URL 是否正确（包含协议头，如 `https://`）
2. 验证单点登录配置参数，与 SP 侧保持一致
3. 测试应用服务器是否可从平台所在网络访问
4. 检查网络及防火墙设置，确保相关端口已开放
</Accordion>

<Accordion title="导入第三方应用失败">
**可能原因：**
- 第三方平台授权已过期
- 账号权限不足
- 应用信息不完整

**解决方法：**
1. 重新授权第三方平台账号
2. 确认当前账号具有应用管理权限
3. 完善应用必要信息后重新导入
</Accordion>

<Accordion title="应用图标显示异常">
**可能原因：**
- 图标格式不支持（仅支持 PNG/JPG）
- 图标文件过大
- 浏览器缓存问题

**解决方法：**
1. 确保图标为 PNG 或 JPG 格式
2. 调整图标尺寸至 128×128 像素，压缩文件大小
3. 清除浏览器缓存后重试
</Accordion>
</Accordions>
