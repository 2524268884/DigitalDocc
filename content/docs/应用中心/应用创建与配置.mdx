## 概述

应用创建与配置是数犀集成平台的基础功能，通过统一管理所有企业应用，实现单点登录、访问控制和应用门户集成。

## 三种添加方式

| 方式 | 适用场景 |
|------|---------|
| **自定义应用** | 企业自研系统、需要手动配置 SSO 的系统 |
| **应用市场** | 市场中已有模板的主流 SaaS 应用（钉钉、阿里云等）|
| **第三方工作台导入** | 已在钉钉/企微工作台上架的应用，直接导入 |

---

## 方式一：创建自定义应用

<Steps>
<Step>
### 进入应用管理

在左侧导航选择**应用中心 > 应用管理**，点击「添加应用 > 自定义应用」。
</Step>
<Step>
### 填写基本信息

| 配置项 | 说明 |
|--------|------|
| 应用名称 | 在平台和门户中的显示名称 |
| 应用分类 | 影响应用在门户中的展示分类 |
| 打开方式 | 当前页面 / 新窗口 / 用户自选 |
| 应用图标 | 建议上传 128×128 像素的 PNG/JPG |
| 应用简介 | 简要描述应用功能 |
| 首页 URL | 应用访问入口地址 |
| 是否允许自助申请 | 开启后用户可自行申请访问权限 |

</Step>
<Step>
### 配置单点登录（可选）

根据应用支持的协议选择配置方式：

| 协议 | 主要配置项 |
|------|-----------|
| OIDC | 授权模式、Redirect URI、Token 有效期 |
| SAML | SP 实体 ID、ACS URL、断言属性映射 |
| OAuth2 | 客户端凭证、授权范围、Token 设置 |
| CAS | 服务 URL、票据校验方式 |

详细配置见 [SSO 端到端接入教程](/docs/应用中心/应用单点登录/SSO%20端到端接入教程)。
</Step>
<Step>
### 设置访问权限

在「授权范围」选项卡中，配置哪些用户可以访问该应用（部门/角色/用户/全员）。
</Step>
<Step>
### 保存应用

点击「保存」完成创建，应用出现在应用列表中。
</Step>
</Steps>

---

## 方式二：从应用市场添加

<Steps>
<Step>
### 打开应用市场

进入**应用中心 > 应用管理**，点击「应用市场」，浏览或搜索目标应用。
</Step>
<Step>
### 添加并配置

点击应用卡片查看详情，点击「添加应用」，按照应用模板填写必要的配置参数，点击「保存」完成。
</Step>
</Steps>

---

## 方式三：导入第三方工作台应用

<Steps>
<Step>
### 选择导入来源

进入**应用中心 > 应用管理**，点击「添加应用 > 第三方应用」，选择来源（钉钉应用 / 企业微信应用）。
</Step>
<Step>
### 选择要导入的应用

选择企业账号，勾选需要导入的工作台应用，点击确认完成导入。
</Step>
</Steps>

**导入后可执行的操作：**

| 操作 | 说明 |
|------|------|
| 禁用 | 禁用后该应用在专属工作台不可使用 |
| 删除 | 应用在第三方平台信息变动后，可删除再重新导入 |
| 更新 | 重新导入时更新基础信息，保留原有配置 |

---

## 应用管理操作

| 操作 | 说明 |
|------|------|
| 编辑 | 修改基本信息、SSO 配置、访问权限 |
| 启用/禁用 | 控制应用是否可被用户访问 |
| 删除 | 从系统中永久移除应用 |
| 查看统计 | 查看访问量、活跃用户数等数据 |

## 常见问题

<Accordions>
<Accordion title="应用创建后用户无法访问？">
请检查以下几点：
1. **首页 URL** 是否配置正确，且可被用户浏览器访问
2. **SSO 配置参数** 是否与应用系统一致（Redirect URI、SP 实体 ID 等）
3. **授权范围** 是否已包含该用户
4. 应用服务器是否正常运行，防火墙是否允许访问
</Accordion>
<Accordion title="从应用市场导入失败怎么办？">
- 检查第三方平台（钉钉/企微）授权是否已过期，需重新授权
- 确认当前账号具有应用管理权限
- 确认应用必要信息已完整填写
</Accordion>
<Accordion title="应用图标显示异常？">
- 支持格式：PNG 或 JPG
- 推荐尺寸：128×128 像素
- 如已上传正确图标但仍显示异常，尝试清除浏览器缓存后刷新
</Accordion>
</Accordions>
