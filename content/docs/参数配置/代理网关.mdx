## 概述

代理网关功能用于实现公网 SaaS 环境与企业内网环境之间的安全连接。在通讯录集成等应用场景中，当需要访问企业内网资源（如 AD 域控制器、内部数据库、内部 API 等）时，代理网关提供了一种安全、高效的内网穿透方案，无需将内网资源直接暴露在公网上。

### 工作原理

代理网关采用**客户端-服务端**架构：

- **网关服务端**：已在专属集成平台 SaaS 环境中部署完毕，无需用户配置
- **网关客户端（connector）**：需要部署在企业内网环境中，作为内网穿透的桥梁
- **工作流程**：网关客户端主动连接服务端，建立加密长连接隧道 → 集成平台请求通过服务端转发 → 客户端转发至内网目标资源 → 响应原路返回

### 主要优势

- 内网穿透，无需将内网资源暴露在公网
- 数据全程加密传输，确保安全
- 无需复杂的 VPN 或防火墙配置
- 支持集中管理多个内网环境连接

### 典型应用场景

- **AD/LDAP 集成**：连接内网 AD 域控制器，实现用户和组织结构同步
- **数据库集成**：连接内网数据库服务器，获取用户和组织数据
- **内部 API 集成**：访问企业内网的 REST API 或 Web 服务
- **内部 ERP/OA 系统集成**：连接企业内部业务系统实现数据集成

## 前提条件

### 环境要求

| 项目 | 要求 |
|------|------|
| 操作系统 | Linux（推荐 CentOS 7+、Ubuntu 18.04+）或 Windows Server 2012 R2+ |
| Java 运行环境 | JRE/JDK 1.8+（必须） |
| 最低硬件配置 | 1 核 CPU、1 GB 内存、10 GB 磁盘 |
| 网络要求 | 内网服务器需能访问公网（出站连接），无需开放入站防火墙端口 |

### 权限要求

- 拥有专属集成平台的管理员权限
- 内网服务器具有安装和运行软件的权限
- 内网服务器能访问目标内网资源（如 AD 服务器、数据库等）

<Callout type="info">
SaaS 版本网关服务端已部署完毕，用户只需在内网服务器上安装 connector 客户端即可使用。私有化部署版本需参照「应用网关部署」文档部署完整的网关组件。
</Callout>

## 操作步骤

### 创建代理网关组

<Steps>
<Step>
### 步骤 1：进入代理网关配置页面

登录专属集成平台管理控制台，从左侧导航栏选择 **参数配置 → 代理网关**。
</Step>
<Step>
### 步骤 2：创建网关组

点击「创建代理网关组」按钮，系统会自动默认创建一个网关组。根据需要编辑网关组名称，建议按照业务分类或区域设置不同网关组（如「总部 AD 网关组」、「ERP 网关组」）。
</Step>
</Steps>

<Callout type="info">
一个网关组可以包含多个代理服务，实现负载均衡和高可用。对于不同业务系统或区域网络，建议创建独立的网关组进行管理。
</Callout>

### 创建代理服务并安装客户端

<Steps>
<Step>
### 步骤 1：新建代理服务

在已创建的网关组下，点击「新建代理服务」按钮，配置以下参数：

- **代理网关名称**：使用有意义的名称，如 `AD-Proxy`
- **操作系统类型**：选择 Linux 或 Windows
- **关联代理组**：选择所属的网关组

点击「下一步」进入下载安装页面。
</Step>
<Step>
### 步骤 2：下载代理客户端

**Linux 环境：**

```bash
# 创建安装目录
mkdir -p /opt/example/gateway
cd /opt/example/gateway

# 下载客户端（使用页面提供的 wget 命令）
wget https://[提供的下载链接]/connector.jar

# 设置文件权限
chmod 755 connector.jar
```

**Windows 环境：**

```powershell
# 创建安装目录后，使用 PowerShell 下载
Invoke-WebRequest -Uri "https://[提供的下载链接]/connector.jar" -OutFile "connector.jar"
```

<Callout type="warn">
下载前请确保服务器已安装 Java 1.8+。可通过 `java -version` 命令验证。未安装时：CentOS 执行 `yum install java-1.8.0-openjdk`，Ubuntu 执行 `apt install openjdk-8-jre`。
</Callout>
</Step>
<Step>
### 步骤 3：启动代理客户端

**Linux 环境（推荐配置为 systemd 服务）：**

```bash
# 创建 systemd 服务文件
cat > /etc/systemd/system/example-gateway.service << EOF
[Unit]
Description=example Gateway Service
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/opt/example/gateway
ExecStart=/usr/bin/java -jar connector.jar [生成的连接参数]
Restart=always
RestartSec=10

[Install]
WantedBy=multi-user.target
EOF

# 启用并启动服务
systemctl daemon-reload
systemctl enable example-gateway
systemctl start example-gateway
```

**Windows 环境（推荐使用 NSSM 创建服务）：**

```batch
# 使用 NSSM 创建 Windows 服务
nssm install exampleGateway "java" "-jar connector.jar [生成的连接参数]"
nssm set exampleGateway AppDirectory C:\example\Gateway
nssm start exampleGateway
```
</Step>
<Step>
### 步骤 4：测试网关连接

返回平台配置页面，点击「测试连接」按钮，系统将测试网关服务端与客户端的连接状态。

- 测试成功：显示「连接成功」提示，代理服务在列表中显示为「在线」状态
- 测试失败：检查网络连通性、Java 版本和连接参数是否正确

点击「完成」按钮完成代理服务配置。
</Step>
</Steps>

## 进阶配置

### 高可用配置

- 在同一网关组下部署多个网关客户端，分布在不同的内网服务器上
- 系统会自动进行负载均衡和故障转移
- 配置 systemd 或 Windows 服务确保客户端异常退出时自动重启

### 性能优化

**JVM 调优**（高并发场景）：

```bash
java -Xms512m -Xmx1024m -jar connector.jar [连接参数]
```

推荐硬件配置（高并发场景）：2 核 CPU、4 GB 内存或更高。

### 安全加固

- 使用非 root/管理员账户运行代理客户端
- 仅允许必要的出站连接
- 定期更新客户端以修复安全漏洞
- 确保启用 TLS 加密传输

## 常用命令参考

**Linux 环境：**

```bash
# 检查网关进程
ps -ef | grep connector

# 检查 Java 版本
java -version

# 后台运行网关客户端
nohup java -jar connector.jar [连接参数] > gateway.log 2>&1 &

# 服务管理
systemctl status example-gateway
systemctl start example-gateway
systemctl stop example-gateway
systemctl restart example-gateway

# 查看服务日志
journalctl -u example-gateway
tail -f /opt/example/gateway/logs/connector.log
```

**Windows 环境：**

```batch
# 检查网关进程
tasklist | findstr java

# 服务管理
sc query example-Gateway
sc start example-Gateway
sc stop example-Gateway
```

## 常见问题

<Accordions>
<Accordion title="网关客户端无法连接服务端怎么排查？">
请依次检查：
1. 内网服务器是否能访问公网（尝试 ping 外网地址）
2. 连接参数是否完整正确（从页面重新复制）
3. Java 版本是否为 1.8 或以上（执行 `java -version` 验证）
4. 防火墙是否阻止了出站连接
</Accordion>
<Accordion title="连接不稳定或频繁断开如何处理？">
建议检查网络稳定性，并将客户端配置为系统服务（systemd 或 Windows Service）以确保自动重启。同时检查内网服务器的 CPU 和内存使用情况，避免资源耗尽导致进程异常退出。
</Accordion>
<Accordion title="通过代理的集成运行缓慢怎么优化？">
可以从以下方面优化：检查内网服务器和网络负载；使用 JVM 参数增加内存分配（`-Xms512m -Xmx1024m`）；考虑分批同步以减轻单次负载。
</Accordion>
<Accordion title="常见错误码如何处理？">
| 错误码 | 描述 | 解决方案 |
|--------|------|----------|
| E001 | 连接拒绝 | 检查网络和防火墙设置 |
| E002 | 认证失败 | 验证连接参数正确性 |
| E003 | 内存不足 | 增加 JVM 内存分配或优化资源使用 |
| E004 | 连接超时 | 检查网络质量，调整超时设置 |
| E005 | 服务未注册 | 重新创建代理服务获取新连接参数 |
</Accordion>
</Accordions>
