## 概述

专属集成平台支持对登录流程进行顺序配置编排，管理员可针对密码修改、身份验证、多因子认证、账号绑定等步骤灵活调整执行顺序，以适应企业的实际业务场景。

## 功能说明

### 登录流程步骤说明

| 步骤 | 触发条件 | 说明 |
|------|----------|------|
| **密码修改** | 首次登录需修改密码 / 密码到期 / 管理员重置密码后 | 引导用户设置新密码 |
| **身份验证** | 平台设置中开启了身份验证属性 | 要求用户验证与平台已集成身份属性一致的关键信息 |
| **多因子认证** | 平台设置或应用设置中开启了 MFA 因子 | 需要验证 MFA 认证因子，或首次登录时补充认证因子信息 |
| **账号绑定** | 开启了微信认证源 | 要求用户绑定微信账号 |

### 默认执行顺序

```
平台登录 → 修改密码 → 身份验证 → 多因子认证 → 账号绑定
```

<Callout type="info">
每个登录步骤满足启用条件后，状态为「已启用」的步骤才会在登录过程中生效，未启用的步骤将被自动跳过。
</Callout>

### 步骤启用条件说明

- **身份验证**：需在「平台设置」中开启并配置身份验证属性后，该步骤才会启用
- **多因子认证**：需在「平台设置」或「应用设置」中开启认证因子后，该步骤才会启用

---

## 操作步骤

### 调整登录流程顺序

进入 **参数配置 → 平台设置 → 登录流程配置**，通过各步骤卡片左侧的拖拽区域，将步骤拖动到期望的位置，调整完成后点击「保存」，平台登录将按照新设置的顺序执行。

<Callout type="warn">
保存流程配置后，新的顺序将立即对所有用户生效，请在非业务高峰期进行配置变更。
</Callout>

## 常见问题

<Accordions>
<Accordion title="调整流程顺序后对正在登录的用户有影响吗？">
保存后立即生效，对当前正在登录的会话无影响，新会话将按照新顺序执行。
</Accordion>
<Accordion title="未启用的步骤是否可以调整顺序？">
可以，可以提前调整好顺序，待启用条件满足后，系统将按照设置的顺序执行。
</Accordion>
<Accordion title="如何关闭某个登录步骤？">
关闭登录步骤需在对应功能的配置入口进行操作，例如关闭多因子认证需在「平台设置」的 MFA 设置中关闭认证因子。
</Accordion>
</Accordions>
