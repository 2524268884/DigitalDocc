## 概述
集成平台提供自定义连接流功能，支持企业将第三方短信网关服务快速集成到平台中，实现短信发送功能的灵活定制与扩展。通过连接流配置，可以无缝对接主流运营商或第三方短信服务提供商的API接口，满足企业在登录验证、通知提醒等场景中的短信发送需求。
适用对象：系统管理员、平台运维人员
前置条件：已获取第三方短信网关的账号、密码、接口地址等接入凭证
技术复杂度：中等

响应结果处理
平台会解析短信网关返回的结果，常见响应格式如下：

```json
{
  "code": "0",
  "message": "发送成功",
  "transactionId": "202309151023453721",
  "failPhones": []
}
```

响应状态处理逻辑：
- 成功状态：系统记录发送成功日志
- 失败状态：记录错误码和失败原因，可配置重试机制
- 异常状态：捕获网络异常并记录日志，提供问题排查入口
常见问题与解决方案
1. 未购买连接流模块的配置方法
问题：客户未采购连接流模块，如何配置短信服务？
解决方案：可联系技术支持团队，提供相关商务合同信息，由技术人员协助配置webhook地址，确保短信服务正常使用。
1. 含URL短信发送失败问题
问题：发送的短信内容中包含URL链接，使用「中国联通Http短信」模板发送失败。
解决方案：在HTTP节点中对短信内容进行URL编码处理：

```javascript
var content = {{N1901925213942263809.value.content}};

var mobile = {{N1901925213942263809.value.mobile}};

const encodedContent = encodeURIComponent("【中国合格评定国家认可委员会】" + content);

return `http://xx.xx.xx.xx:xxxxx/xxxxx.aspx?userid=xxxx&action=send&timestamp=${time}&sign=${sign}&mobile=${mobile}&content=${encodedContent}&sendTime=&extno=`;
```

1. 短信发送频率限制问题
问题：系统提示"发送频率超过限制"错误。
解决方案：
- 检查运营商对单个手机号的发送频率限制（通常为1分钟1条，1小时5条，1天10条）
- 在连接流中添加频率控制逻辑
- 必要时联系运营商调整频率限制参数

短信内容规范
1. 签名规范：
  - 确保短信签名已在运营商平台完成备案
  - 签名格式为【企业名称】，需放在短信正文开头
  - 签名内容应与备案信息完全一致
1. 内容规范：
  - 避免使用敏感词汇
  - 控制短信长度，建议不超过70个字符
  - 验证码类短信明确说明有效期
发送策略优化
- **错峰发送**：批量短信建议错峰发送，避免瞬时并发过高
- **重试机制**：针对网络波动导致的发送失败，建立合理的重试策略
- **黑名单机制**：建立短信黑名单机制，避免骚扰投诉
监控与告警
建议设置以下监控指标：
- 短信发送成功率告警（低于95%触发告警）
- 短信API接口可用性监控
- 短信发送量异常波动监控
