本文档定义数犀集成平台文档的标准模板，确保文档结构一致、内容完整。

---

- **模板 A**：系统对接指南

适用于：通讯录集成 / 认证源 / SSO 等配置类文档

### 结构

```markdown
# {系统名} + {功能}配置指南

## 概述
- 功能说明（1-2 句话）
- 适用场景
- 架构示意图（可选）

## 前提条件
| # | 条件 | 说明 | 验证方式 |
|---|------|------|----------|

## 第三方侧配置
（对方系统需要做的配置）

## 数犀集成平台侧配置
（本平台配置步骤）

## 测试与验证
| 步骤 | 操作 | 预期结果 |
|------|------|----------|

## 常见问题
### Q1: `{问题}`
**A**: `{答案}`

### Q2: `{问题}`
**A**: `{答案}`

## 相关文档
- [文档名称](./path/to/doc.mdx)
```

### 示例结构

```markdown
# 钉钉 + 通讯录集成配置指南

## 概述
介绍钉钉通讯录集成功能，适用场景说明。

## 前提条件
| # | 条件 | 说明 | 验证方式 |
|---|------|------|----------|
| 1 | 钉钉企业账号 | 已创建钉钉企业 | 登录钉钉管理后台 |
| 2 | 管理员权限 | 需要通讯录读写权限 | 确认权限 |

## 第三方侧配置
（钉钉开放平台配置步骤）

## 数犀集成平台侧配置
（本平台配置步骤）

## 测试与验证
（测试步骤和预期结果）

## 常见问题
（FAQ）

## 相关文档
（交叉引用链接）
```

- **模板 B**：节点功能说明

适用于：连接流 / AI 连接流 节点文档

### 结构

```markdown
# {节点名}

## 功能概述
（1-2 句话说明节点作用）

## 适用场景
- 场景 1
- 场景 2

## 配置参数
| 参数名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|

## 使用示例

### 示例 1：{场景描述}
**输入**：
```json
{
  // 输入 JSON
}
```

**配置**：
1. 步骤 1
2. 步骤 2

**输出**：
```json
{
  // 输出 JSON
}
```

## 输出说明
| 输出字段 | 类型 | 说明 |
|----------|------|------|
| field1 | string | 字段说明 |

## 错误处理
| 错误码 | 说明 | 解决方案 |
|--------|------|----------|
| E001 | 错误描述 | 解决步骤 |

## 最佳实践
- 实践 1
- 实践 2

## 常见问题
### Q: `{问题}`
**A**: `{答案}`
```

### 示例结构

```markdown
# HTTP 请求

## 功能概述
HTTP 请求节点用于向外部 API 发起 HTTP 请求，获取或提交数据。

## 适用场景
- 调用第三方 REST API
- 获取外部数据
- 提交表单数据

## 配置参数
| 参数名 | 类型 | 必填 | 说明 | 示例 |
|--------|------|------|------|------|
| url | string | 是 | 请求地址 | https://api.example.com |
| method | string | 是 | 请求方法 | GET/POST |

## 使用示例

### 示例 1：GET 请求
**输入**：
```json
{
  "url": "https://api.example.com/users"
}
```

**输出**：
```json
{
  "status": 200,
  "body": { ... }
}
```

## 输出说明
（输出字段说明）

## 错误处理
（错误码和解决方案）

## 最佳实践
（最佳实践）

## 常见问题
（FAQ）
```

- **模板 C**：排障指南

适用于：故障排除 / FAQ 类文档

### 结构

```markdown
# {模块}故障排除

## 快速诊断流程
（诊断流程图或步骤列表）

## 常见错误码表
| 错误码 | 错误描述 | 可能原因 |
|--------|----------|----------|
| E001 | 错误描述 | 原因1、原因2 |

## 按症状分类排查

### 症状 1：{表现}
**可能原因**：
- 原因 1
- 原因 2

**排查步骤**：
1. 步骤 1
2. 步骤 2

**解决方案**：
- 方案 1
- 方案 2

### 症状 2：{表现}
（同样结构）

## 日志分析指引
（如何查看和分析日志）

## 升级支持路径
（如何获取更多帮助）
```

### 示例结构

```markdown
# 单点登录故障排除

## 快速诊断流程
1. 检查用户是否有应用访问权限
2. 检查 SSO 配置是否正确
3. 检查浏览器 Cookie 和缓存
4. 查看认证日志
5. 检查网络连接

## 常见错误码表
| 错误码 | 错误描述 | 可能原因 |
|--------|----------|----------|
| SSO-001 | 无效的 SAML 断言 | 证书过期 |
| SSO-002 | ACS URL 不匹配 | 配置错误 |

## 按症状分类排查

### 症状：用户无法登录
**可能原因**：
- 用户未被授权
- SSO 配置错误
- 会话已过期

**排查步骤**：
1. 确认用户有应用访问权限
2. 检查 SSO 配置
3. 清除浏览器缓存后重试

**解决方案**：
- 为用户分配应用权限
- 重新配置 SSO
- 联系管理员

## 日志分析指引
进入「审计日志 > 操作日志」，筛选 `sso` 相关操作。

## 升级支持路径
1. 查看常见问题
2. 联系技术支持
```

## 通用规范

### Front Matter

所有文档必须包含以下 front matter：

```yaml
---
title: "文档标题"
description: "文档简短描述"
source_portal: "原始文档链接"
docKey: "文档唯一标识"
exported_at: "导出日期"
path:
  - 目录1
  - 目录2
  - 文档名
content_hash: "内容hash"
---
```

### 标题层级

```
# 一级标题（页面标题）
## 二级标题（主要章节）
### 三级标题（子章节）
#### 四级标题（细节内容）
```

### 表格格式

```
| 列1 | 列2 | 列3 |
|------|------|------|
| 内容 | 内容 | 内容 |
```

### 代码块

````markdown
```语言
代码内容
```
````

### 交叉引用

```markdown
[文档名称](./path/to/doc.mdx)
```

### 注意事项标记

| 标记 | 含义 |
|------|------|
| ⚠️ 注意 | 需要特别关注的警告 |
| 💡 提示 | 有用的技巧或建议 |
| 🔧 操作 | 需要执行的具体步骤 |
