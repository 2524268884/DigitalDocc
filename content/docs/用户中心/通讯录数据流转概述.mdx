## 概述

数犀集成平台通讯录功能分为两个方向，**通讯录集成**（上游拉取）和**通讯录同步**（下游推送），初次接触容易混淆。本文档帮助您快速理解两者的定义、区别与选型建议。

## 核心概念区分

### 通讯录集成（上游 → 集成平台）

**定义**：将外部系统（钉钉、飞书、AD、LDAP 等）的用户和组织数据**拉取**到数犀集成平台，使集成平台成为企业统一身份数据的汇聚中心。

**数据流向**：

```
外部身份源（钉钉/飞书/AD/LDAP...）
        ↓  拉取
  数犀集成平台（统一身份中心）
```

**典型场景**：
- 以钉钉作为主数据源，将钉钉组织架构同步到集成平台
- 以 AD 作为权威目录，将企业用户账号导入集成平台
- 多个身份源合并，统一管理用户身份

---

### 通讯录同步（集成平台 → 下游）

**定义**：将数犀集成平台中的用户和组织数据**推送**到下游业务系统（另一个钉钉、泛微OA、用友等），实现跨系统账号自动化管理。

**数据流向**：

```
  数犀集成平台（统一身份中心）
        ↓  推送
下游业务系统（泛微OA/用友/钉钉/AD...）
```

**典型场景**：
- 集成平台的用户变更自动同步至 OA 系统
- 新员工入职后账号自动开通到多个业务系统
- 员工离职后批量停用下游系统账号

---

## 两者对比一览

| 对比维度 | 通讯录集成 | 通讯录同步 |
|---------|-----------|-----------|
| **数据流向** | 外部 → 集成平台（拉取） | 集成平台 → 下游（推送） |
| **平台角色** | 数据消费方 | 数据提供方 |
| **目的** | 将外部身份源引入集成平台 | 将集成平台身份数据分发至下游 |
| **触发方式** | 定时拉取 / Webhook 事件 | 定时推送 / 用户变更触发 |
| **典型上游** | 钉钉、飞书、企微、AD、LDAP、SCIM、北森、泛微... | — |
| **典型下游** | — | 钉钉、企微、AD、泛微OA、用友、网易邮箱... |
| **所在菜单** | 用户中心 > 通讯录集成 | 用户中心 > 通讯录同步 |

## 选型决策树

```
我要处理的是哪个方向的数据？
│
├─ 外部系统 → 集成平台（我想把外部用户导进来）
│     └─ 选择：通讯录集成
│           ├─ 钉钉/飞书/企微  → 通讯录集成-钉钉/飞书/企微
│           ├─ AD / Azure AD  → 通讯录集成-AD / Azure AD
│           ├─ LDAP            → 通讯录集成-LDAP
│           ├─ SCIM 协议       → 通讯录集成-SCIM
│           ├─ HR 系统（北森/睿人事/Moka）→ 对应系统文档
│           ├─ OA 系统（泛微/致远）→ 对应系统文档
│           └─ 数据库           → 通讯录集成-数据库
│
└─ 集成平台 → 下游系统（我想把用户推出去）
      └─ 选择：通讯录同步
            ├─ 推送到钉钉      → 通讯录同步-钉钉
            ├─ 推送到企微      → 通讯录同步-企业微信
            ├─ 推送到 AD       → 通讯录同步-AD
            ├─ 推送到泛微 OA   → 通讯录同步-泛微OA-E9
            ├─ 推送到用友U8C   → 通讯录同步-用友U8C
            ├─ 推送到邮箱系统  → 通讯录同步-网易邮箱
            └─ REST API 推送   → 身份自动化同步-REST API
```

## 常见组合方案

### 方案一：单向拉取（最常见）

**场景**：企业以钉钉/飞书为主数据源，集成平台作为 SSO 和权限管理中心。

```
钉钉/飞书（主数据源）
     ↓ 通讯录集成
数犀集成平台
     ↓ SSO 授权
各业务应用（无需单独管用户）
```

### 方案二：双向同步

**场景**：以 AD 为权威数据源，集成平台作为中间层，再推送到各云应用。

```
Windows AD（权威目录）
     ↓ 通讯录集成
数犀集成平台
     ↓ 通讯录同步
钉钉 / 飞书 / 泛微 OA...
```

### 方案三：多源合并 + 统一推送

**场景**：多个上游（HR 系统 + AD），统一到集成平台后推送下游。

```
北森 HR + Windows AD
     ↓ 通讯录集成（多源）
数犀集成平台（统一身份中心）
     ↓ 通讯录同步
钉钉 / 企微 / 用友 U8C...
```

---

## 注意事项

1. **优先选一个主数据源**：多个上游集成时，需在集成平台配置用户匹配规则，避免重复账号
2. **集成与同步可并行**：同一套配置中，可以同时运行上游集成和下游同步
3. **删除策略需谨慎**：建议先设置删除阈值为 0 观察，再逐步放开（详见[删除用户方式配置](./通讯录集成/通讯录集成-同步策略-删除用户方式)）
4. **代理网关**：如果上下游系统在内网，需先配置代理网关（详见[通讯录集成-代理网关](./通讯录集成/通讯录集成-代理网关)）
