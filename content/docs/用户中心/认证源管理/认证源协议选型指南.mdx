---

## 协议概览

| 协议 | 类型 | 适用场景 | 复杂度 | 主流支持 |
|------|------|----------|--------|----------|
| **SAML 2.0** | 企业级 SSO | 传统企业应用、硬件设备 | 中 | 高 |
| **OIDC** | 现代 SSO | Web 应用、SPA、移动 APP | 低 | 高 |
| **OAuth 2.0** | 授权框架 | API 授权、第三方登录 | 中 | 高 |
| **CAS** | SSO 协议 | 高校、科研机构 | 低 | 中 |
| **LDAP** | 目录协议 | 企业内网、Active Directory | 低 | 高 |
| **AD** | 目录协议 | Windows 企业环境 | 低 | 高 |

## SAML 2.0

### 什么是 SAML

SAML（Security Assertion Markup Language）是一种基于 XML 的开放标准，用于在身份提供商（IdP）和服务提供商（SP）之间交换认证和授权数据。

### 适用场景

| 场景 | 推荐度 | 说明 |
|------|--------|------|
| 企业内部应用 | ⭐⭐⭐⭐⭐ | 泛微 OA、致远 OA、金蝶等 |
| 传统软件（SaaS） | ⭐⭐⭐⭐⭐ | 不支持 OIDC 的应用 |
| 硬件设备登录 | ⭐⭐⭐⭐ | 打印机、网络设备 |
| 现代 Web 应用 | ⭐⭐⭐ | 可用但不推荐 |
| 移动 APP | ⭐⭐ | 实现复杂，不推荐 |

### 优点

- 广泛支持，几乎所有企业应用都支持
- XML 格式，安全性经过验证
- 支持断言加密和签名
- 适合复杂的企业环境

### 缺点

- XML 配置相对复杂
- 移动端支持不如 OIDC
- 不适合纯 API 场景

### 配置复杂度

```
低 ◀ ─────────────────────────────── ▶ 高
     │        │        │        │
   LDAP     CAS     OIDC    SAML
```

## OIDC

### 什么是 OIDC

OIDC（OpenID Connect）是基于 OAuth 2.0 的身份层协议，在授权的基础上提供了用户身份信息。

### 适用场景

| 场景 | 推荐度 | 说明 |
|------|--------|------|
| 现代 Web 应用（SPA） | ⭐⭐⭐⭐⭐ | React、Vue、Angular 应用 |
| 移动 APP | ⭐⭐⭐⭐⭐ | iOS、Android 应用 |
| 自定义开发应用 | ⭐⭐⭐⭐⭐ | 自主研发系统 |
| 需要用户信息 | ⭐⭐⭐⭐⭐ | 获取用户名、邮箱等 |
| 传统企业应用 | ⭐⭐⭐ | 可用但不如 SAML 成熟 |

### 优点

- JSON 格式，现代且易于理解
- 与 OAuth 2.0 兼容，学习成本低
- 移动端支持好
- 可获取丰富的用户信息

### 缺点

- 部分传统应用不支持
- 需要 HTTPS 环境
- 比 CAS 复杂度略高

### 配置复杂度

```
低 ◀ ─────────────────────────────── ▶ 高
     │        │        │        │
   LDAP     CAS     SAML    OIDC
```

## OAuth 2.0

### 什么是 OAuth 2.0

OAuth 2.0 是一个授权框架，用于授予第三方应用访问用户资源的权限，但不提供身份认证。

### 适用场景

| 场景 | 推荐度 | 说明 |
|------|--------|------|
| 第三方登录 | ⭐⭐⭐⭐⭐ | 微信、微博登录 |
| API 授权 | ⭐⭐⭐⭐⭐ | 开放平台授权 |
| 不想暴露密码 | ⭐⭐⭐⭐⭐ | 安全地授权第三方 |
| 已有 OIDC 需求 | ⭐⭐ | 建议直接用 OIDC |

### 重要说明

> ⚠️ **OAuth 2.0 不是认证协议**：如果您需要用户身份认证，请使用 OIDC。OAuth 2.0 只负责授权，不提供用户身份信息。

## CAS

### 什么是 CAS

CAS（Central Authentication Service）是一种专为 Web 应用设计的单点登录协议，简洁易用。

### 适用场景

| 场景 | 推荐度 | 说明 |
|------|--------|------|
| 高校系统 | ⭐⭐⭐⭐⭐ | 学校教务系统、图书馆系统 |
| 科研机构 | ⭐⭐⭐⭐⭐ | 实验室管理系统 |
| 简单 SSO 需求 | ⭐⭐⭐⭐ | 不需要复杂的企业功能 |
| 跨域应用 | ⭐⭐⭐⭐ | 支持良好 |

### 优点

- 协议简单，容易理解
- 实现和学习成本低
- 支持代理认证
- 适合单一应用场景

### 缺点

- 不适合复杂企业场景
- 移动端支持有限
- 社区活跃度下降

## LDAP / AD

### 什么是 LDAP

LDAP（Lightweight Directory Access Protocol）是一种访问目录服务的协议，常用于企业用户和组织架构管理。

### 什么是 AD

AD（Active Directory）是微软的目录服务，基于 LDAP 协议，是 Windows 企业环境的标配。

### 适用场景

| 场景 | 推荐度 | 说明 |
|------|--------|------|
| Windows 企业环境 | ⭐⭐⭐⭐⭐ | 使用 Active Directory |
| 统一身份管理 | ⭐⭐⭐⭐⭐ | 员工账号管理 |
| 同步组织架构 | ⭐⭐⭐⭐⭐ | 部门、用户组同步 |
| 内部系统认证 | ⭐⭐⭐⭐ | 企业内网应用 |
| 跨平台目录 | ⭐⭐⭐ | 需要 LDAP 兼容 |

### 优点

- 企业标配，普及率高
- 支持复杂的组织架构
- 密码策略完善
- 与 Windows 集成良好

### 缺点

- 配置相对复杂
- 需要网络可达
- 不适合纯云环境

## 选型决策树

```
开始
  │
  ▼
需要用户身份认证？
  │
  ├─ 否 ──▶ 需要 API 授权？ ──▶ 是 ──▶ OAuth 2.0
  │                │
  │                └─ 否 ──▶ 不需要 SSO？请使用应用内置认证
  │
  └─ 是 ──▶ 是什么类型的应用？
              │
              ├─ 传统企业软件（泛微/致远/SAP）──▶ SAML 2.0
              │
              ├─ 现代 Web 应用（SPA/移动 APP）──▶ OIDC
              │
              ├─ 高校/科研系统 ─────────────────▶ CAS
              │
              └─ 需要接入目录服务？
                  │
                  ├─ Windows 环境 ───────────────▶ AD
                  │
                  └─ 其他目录 ─────────────────▶ LDAP
```

## 快速参考

### 按场景选择

| 您的场景 | 推荐协议 |
|----------|----------|
| 泛微 OA、致远 OA | SAML 2.0 |
| 自建 Web 系统 | OIDC |
| 企业微信、钉钉、飞书 | OIDC / 平台自带认证 |

| 应用类型 | 通常支持 |
|----------|----------|
| 企业级 SaaS | SAML 2.0, OIDC |
| 自研应用 | OIDC |
| 国产化应用 | SAML 2.0 |
| 移动应用 | OIDC |
| 硬件设备 | SAML 2.0 |
