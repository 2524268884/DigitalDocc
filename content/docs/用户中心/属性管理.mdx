## 概述

属性管理是数犀集成平台的核心功能，用于定义和管理系统中各实体（用户、部门、角色）的属性字段，支持基础属性和扩展属性的灵活配置，并实现与外部系统的属性映射，确保多系统间数据的准确同步。

## 功能说明

### 属性管理范围

| 类型 | 说明 |
|------|------|
| **平台属性** | 用户、部门、角色的基础属性和自定义扩展属性 |
| **通讯录集成源属性** | 创建通讯录集成时自动生成，支持与平台属性映射 |
| **通讯录同步源属性** | 创建通讯录同步应用时自动生成，支持与平台属性映射 |
| **认证源属性** | 创建认证源应用时自动生成，支持 JIT 用户创建的属性映射 |

### 属性类型

- **基础属性**：系统内置的核心字段（如用户名、手机号、邮箱等），不可删除，仅部分特性可修改
- **扩展属性**：管理员自定义添加的字段，支持完整的增删改操作

### 属性特性说明

| 特性 | 说明 |
|------|------|
| **必填** | 创建或编辑实体时必须提供该属性的值 |
| **全局唯一** | 属性值在所有实体中必须唯一（如用户名、手机号、邮箱默认开启） |
| **个人中心展示** | 属性是否在用户服务台的个人信息页面显示 |
| **允许用户修改** | 用户是否可在个人中心自行修改该属性值 |

<Callout type="info">
「允许用户修改」的属性必须同时启用「个人中心展示」。
</Callout>

## 操作步骤

### 新增扩展属性

<Steps>
<Step>
### 步骤 1：选择实体类型

导航至 **用户中心 → 属性管理**，选择实体类型标签页（用户、部门或角色）。
</Step>
<Step>
### 步骤 2：新增属性

点击「新增属性」按钮，填写以下信息：

- **属性名称**：系统内部标识，建议使用英文（如 `employee_id`）
- **显示名称**：界面上显示的名称（如「员工编号」）
- **数据类型**：选择文本、时间或数字
</Step>
<Step>
### 步骤 3：配置属性特性

根据业务需求配置：
- 是否必填
- 是否全局唯一
- 是否在个人中心展示
- 是否允许用户修改
</Step>
<Step>
### 步骤 4：保存

填写字段描述（可选），点击「确定」完成创建。
</Step>
</Steps>

### 编辑属性

<Steps>
<Step>
### 步骤 1：找到目标属性

在属性列表中找到需要修改的属性，点击「编辑」按钮。
</Step>
<Step>
### 步骤 2：修改配置

- **基础属性**：仅支持修改特性设置（必填、唯一性等），不可修改名称和数据类型
- **扩展属性**：支持修改所有信息
</Step>
<Step>
### 步骤 3：保存更改

点击「确定」保存更改。
</Step>
</Steps>

### 删除扩展属性

<Callout type="warn">
基础属性不支持删除。删除扩展属性后，相关数据将无法恢复，请谨慎操作。
</Callout>

在属性列表中找到目标扩展属性，点击「删除」按钮，确认删除操作即可。

## 属性映射配置

属性映射用于在不同系统间建立属性关联关系，实现数据同步和转换。

### 映射类型

| 类型 | 方向 | 典型场景 |
|------|------|----------|
| 上游映射 | 外部系统 → 集成平台 | 通讯录集成、认证源 JIT 创建用户 |
| 下游映射 | 集成平台 → 外部系统 | 通讯录同步到钉钉/企业微信/AD 等 |

### 映射策略

| 策略 | 说明 |
|------|------|
| **仅创建** | 仅在创建实体时同步该属性 |
| **创建且更新** | 创建和更新实体时都同步该属性 |
| **不映射** | 不同步该属性 |

### 配置映射关系

<Steps>
<Step>
### 步骤 1：进入属性映射配置

在属性管理中选择相应的外部源标签页（通讯录集成源、通讯录同步源或认证源）。
</Step>
<Step>
### 步骤 2：配置映射关系

配置源属性和目标属性的对应关系，设置映射策略，必要时使用表达式处理属性值。
</Step>
<Step>
### 步骤 3：保存映射配置

点击保存，配置将在下次数据同步时生效。
</Step>
</Steps>

## 表达式支持

属性映射支持使用 JavaScript 表达式对属性值进行处理和转换（基于 GraalVM，支持 ECMAScript 2022）。

### 常用表达式示例

**格式化与转换：**

```javascript
// 去除电话号码前缀
user.mobile.replace("+86", "+86-")

// 提取邮箱用户名
user.email.split('@')[0]

// 字段拼接
user.first_name + " " + user.last_name
```

**状态码转文本：**

```javascript
// 在职状态
user.status === "1" ? '在职' : '离职'

// 性别代码
user.gender === "M" ? '男' : user.gender === "F" ? '女' : '未知'
```

**处理组织关系：**

```javascript
// 用户与部门关系
user.departments && user.departments.map(dept => ({id: dept.id, is_primary: dept.is_primary}))

// 下游根部门映射（用本地部门 ID 映射钉钉根部门）
org.id == '1858377061035466753' ? 1 : org.connector_org_id
```

**日期处理：**

```javascript
// 计算服务年限
user.hire_date && Math.floor((new Date() - new Date(user.hire_date)) / (365*24*60*60*1000))

// 格式化日期
user.birth_date && new Date(user.birth_date).toLocaleDateString('zh-CN')
```

**默认值处理：**

```javascript
// 多级备用值
user.display_name || user.full_name || (user.first_name + " " + user.last_name) || user.email

// 默认时区
user.timezone || "Asia/Shanghai"
```

## 常见问题

<Accordions>
<Accordion title="为什么基础属性不能删除或修改数据类型？">
基础属性是系统核心功能所依赖的字段，修改其数据类型或删除可能导致系统功能异常，因此平台限制这类操作。
</Accordion>
<Accordion title="新增的扩展属性可以在哪些地方使用？">
扩展属性可用于：用户/部门管理界面的创建和编辑表单；角色管理中的动态角色匹配规则；通讯录集成和同步的属性映射；用户服务台的个人信息页面（需配置展示权限）。
</Accordion>
<Accordion title="外部系统的属性如何映射到平台扩展属性？">
1. 先在平台创建相应的扩展属性
2. 在通讯录集成或认证源的属性映射中，选择源系统属性和对应的平台扩展属性
3. 设置合适的映射策略
4. 保存后，系统将根据映射关系在下次同步时同步数据
</Accordion>
<Accordion title="如何批量更新用户的扩展属性？">
可通过以下方式批量更新：
- 使用 Excel 批量导入功能，在模板中包含扩展属性列
- 通过通讯录集成同步，配置相应的属性映射关系
- 使用连接中心的批量数据处理能力
</Accordion>
</Accordions>
