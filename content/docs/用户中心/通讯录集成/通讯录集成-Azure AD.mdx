## 概述
本指南详细说明如何通过SCIM（跨域身份管理系统）协议将Microsoft Azure Active Directory (Azure AD)中的用户组和用户数据同步至集成平台。此集成能够实现企业内身份数据的自动化同步，保障各应用系统的用户数据和组织架构一致性，简化身份管理流程，提高企业数据治理水平。
### 集成价值
- 集中身份管理：利用Azure AD作为身份源，实现企业内部统一的身份管理
- 自动化同步：用户和组织信息变更自动同步，无需手动维护
- 提高安全性：集中控制用户访问权限，降低安全风险
- 降低管理成本：减少多系统账号维护工作，提高IT管理效率
- 提升用户体验：用户只需管理一个账号，即可访问多个应用系统
### 适用场景
- Azure AD作为主身份源：企业使用Azure AD/Microsoft 365作为主要身份管理系统
- 混合身份环境：需要连接本地Active Directory与云应用的企业
- 多云身份管理：跨多个云平台进行统一身份管理的企业
- 大规模企业：需要集中管理大量用户和复杂组织架构的企业
### 集成架构
本集成基于SCIM协议实现Azure AD与集成平台之间的身份数据同步：
1. Azure AD作为身份数据的来源（身份提供者）
1. 集成平台作为身份数据的消费者（服务提供者）
2. 通过SCIM协议实现两者之间的双向通信
4. Azure AD中的变更（如用户添加、修改、删除）会自动同步至集成平台
3. 前提条件
在开始配置前，请确保满足以下条件：
### Azure AD要求
- Azure AD账号：拥有Azure AD的全局管理员或云应用管理员权限
- Azure AD版本：需要Azure AD Premium P1或P2版本以支持SCIM配置
- 企业应用管理权限：能够创建和配置企业应用程序
- 授权范围：确定需要同步的用户和组范围
### 集成平台要求
- 管理员账号：拥有集成平台的管理员权限
- 应用创建：已在集成平台创建相关应用
- 系统版本：集成平台 v3.2.0及以上版本
### 网络环境要求
- 连接性：确保Azure AD能够通过互联网访问集成平台的SCIM端点
- 防火墙配置：如有防火墙，需确保允许从Azure AD到集成平台的SCIM通信
- SSL/TLS：集成平台SCIM端点必须支持HTTPS协议（TLS 1.2+）
1. 集成流程概览
整个集成过程分为以下五个主要步骤：
1. 配置SCIM连接器：在集成平台创建SCIM连接器，获取token
- **启动SCIM同步服务**：确保集成平台的SCIM服务正常运行
3. 创建Azure AD应用：在Azure AD中创建用于SCIM集成的企业应用程序
4. 分配用户与组：在Azure AD中将需要同步的用户/组分配给应用程序
5. 配置SCIM同步：在Azure AD中配置SCIM预配设置，启动同步
1. 详细操作步骤
### 配置SCIM连接器，获取token
#### 获取应用凭证
1. 使用管理员账号登录集成平台
2. 导航至「应用中心 → 应用管理 → 应用详情」
- 找到并打开需要与Azure AD集成的应用
4. 记录该应用的Client ID和Client Secret值，后续步骤将使用这些凭证
#### 创建SCIM连接器
## 在集成平台左侧导航栏选择「用户中心 → 通讯录集成」
- 点击页面右上角的「新建连接器」按钮
1. 在连接器类型选择界面，找到并选择「SCIM」身份源类型



#### 配置SCIM连接器基本信息
## 在基本信息配置页面，填写以下内容：
  - 名称：输入自定义的身份源名称，建议包含"Azure AD"便于识别
  - Client ID：填写步骤4.1.1中获取的Client ID
  - Client Secret：填写步骤4.1.1中获取的Client Secret
  - 根据界面提示填写其他必要参数
1. 点击「下一步」保存基本信息

#### 获取token
1. 连接器创建完成后，返回到SCIM连接器的基础信息页
2. 点击「获取token」按钮
3. 系统生成token信息，请妥善保存此token，后续在Azure AD中配置SCIM同步时需要使用
4. 同时记录"SCIM服务端地址"，这将作为Azure AD中的租户URL
注意事项：
- Token具有敏感性，请妥善保管，避免泄露
- 如需更新Token，可以在此页面重新获取
- SCIM服务端地址通常格式为：https://[域名]/scim/[标识符]
### 启动SCIM同步服务
## 在SCIM连接器详情页，确认连接器状态为"正常"
- 确认SCIM服务已启动并可正常访问
1. 可以通过点击"测试连接"按钮验证SCIM服务是否正常运行
### 在Azure AD中创建应用程序
1. 使用管理员账号登录Azure门户
2. 在主页左上角，点击导航菜单图标
3. 在左侧导航栏，依次选择「Azure Active Directory → 企业应用程序 → 所有应用程序」
- 点击「新建应用程序」按钮
1. 在「浏览Azure AD库」页面，点击「创建你自己的应用程序」
- 在「创建你自己的应用程序」对话框中：
  - 输入描述性应用程序名称（例如：集成平台-SCIM）
  - 选择「集成未在库中找到的任何其他应用程序（非库）」选项
  - 点击「创建」按钮


最佳实践：
- 应用程序名称应当清晰表明其用途，便于后续管理
- 建议在名称中包含"SCIM"字样，明确表明同步方式
- 如有多个环境（如测试、生产），应在名称中注明
### 在Azure AD中分配用户或用户组到应用程序
1. 在Azure门户左侧导航栏，依次选择「Azure Active Directory → 企业应用程序 → 所有应用程序」
- 在应用程序列表中，找到并点击刚创建的应用程序
1. 在左侧导航栏，选择「用户和组」
- 点击左上角的「添加用户/组」按钮
1. 在弹出的「添加分配」对话框中：
  - 点击「用户和组」字段，在弹出的用户/组选择面板中
  - 搜索并选择需要同步的用户或用户组
  - 可以选择多个用户或组
  - 点击「选择」按钮确认选择
1. 返回「添加分配」对话框，点击「分配」按钮完成用户/组分配


用户分配策略建议：
- 初始测试：先选择少量测试用户（2-3个），验证同步正常后再扩大范围
- 按组分配：优先使用用户组分配而非单个用户，便于管理
- 分批实施：对于大型组织，建议分批分配用户，避免一次同步大量数据
- 关键用户先行：优先同步关键业务部门和管理层用户
### 在Azure AD中配置SCIM同步
#### 基本配置
## 在应用程序详情页面，点击左侧导航栏的「预配」

1. 设置「预配模式」为「自动」
- 在「管理员凭据」区域，配置以下信息：

  - 密钥标记：输入步骤4.1.5中获取的SCIM token
  - 点击「测试连接」按钮，确保连接成功

### 连接测试失败
可能原因：
- SCIM服务端地址错误或无法访问
- Token无效或已过期
- SSL/TLS证书问题
- 网络连接或防火墙限制
解决方案：
1. 验证SCIM服务端地址：
  - 确认URL格式正确，包含正确的协议前缀(https://)
  - 尝试在浏览器中访问SCIM端点（会返回401错误，但证明端点可访问）
  - 确认域名解析正确
1. 检查Token：
  - 确认Token复制无误，没有多余空格
  - 在集成平台重新生成Token并更新Azure AD配置
  - 确认Token未过期（如适用）
1. 网络连接问题：
  - 确认Azure AD能够访问互联网
  - 检查防火墙设置，确保允许出站连接到SCIM端点
  - 如有代理服务器，确认配置正确
### 同步失败或数据不完整

- 属性映射配置错误
- 用户/组未正确分配给应用程序

- 数据格式不兼容
解决方案：
1. 检查属性映射：
  - 确认必要的属性映射已正确配置

  - 查看是否有嵌套组结构导致的问题

8. Azure AD SCIM集成常见问题解答
### 功能与限制
问题：Azure AD SCIM集成支持哪些用户属性同步？
答：标准支持的属性包括：
- 基本属性：用户名、姓名、邮箱、电话等
- 组织属性：部门、职位、管理者关系
- 状态属性：启用/禁用状态
- 自定义属性：可通过自定义属性映射支持
问题：是否支持复杂的组织结构同步？
答：是，Azure AD SCIM集成支持多级组织结构同步，包括：
- 层级部门结构
- 用户与多个组的关系
- 嵌套组结构
- 组织单位之间的层次关系
### 故障处理
问题：如何处理同步过程中的冲突？
答：同步冲突处理建议：
- 一般情况下，Azure AD为主要数据源，其数据优先级更高
- 可在属性映射中设置特定属性的优先级
- 对于双向同步环境，需明确定义数据权威源
- 冲突日志会记录在预配日志中，需定期检查
问题：如何快速恢复同步失败？
答：同步失败恢复步骤：
1. 在Azure AD预配页面查看具体错误信息
2. 根据错误类型采取针对性措施（参考故障排除章节）
3. 修复问题后，可点击"重启预配"按钮重新开始同步
4. 对于持续失败的情况，考虑暂时关闭自动同步，修复后再启用
