## 概念定义

数犀集成平台提供两种不同的通讯录数据管理能力：**通讯录集成**和**通讯录同步**。理解这两者的区别对于选择正确的解决方案至关重要。

### 通讯录集成（上游拉取）

**定义**：从外部身份源（如钉钉、企业微信、AD/LDAP 等）拉取用户和组织架构数据到数犀集成平台。

**数据流向**：
```
外部身份源（钉钉/AD/飞书等）  →  数犀集成平台
         ↑                              ↑
      用户和组织数据              平台存储和管理
```

**核心特点**：
- 数据存储在数犀集成平台
- 外部系统作为数据源
- 用户使用数犀集成平台账号登录各应用
- 支持 JIT（Just-In-Time）用户创建

### 通讯录同步（下游推送）

**定义**：将数犀集成平台中的用户和组织架构数据推送/同步到外部应用系统。

**数据流向**：
```
数犀集成平台  →  目标应用系统（泛微/致远/用友等）
     ↑                    ↑
  用户和组织数据       应用存储数据
```

**核心特点**：
- 数据源在数犀集成平台
- 外部系统作为目标接收方
- 支持实时或定时同步
- 可配置字段映射和转换

## 选型决策树

### 何时使用通讯录集成？

| 场景 | 说明 | 示例 |
|------|------|------|
| 统一身份认证 | 需要使用外部系统（钉钉/AD）账号登录多个应用 | 企业使用钉钉作为主身份源，员工使用钉钉账号SSO访问各业务系统 |
| 集中用户管理 | 希望在外部系统管理用户，在数犀集成平台集中管控 | HR系统在AD，IT希望统一在平台管理权限 |
| 简化账号管理 | 减少在多个系统中重复创建用户的工作 | 新员工入职只在钉钉添加，各系统自动同步 |

### 何时使用通讯录同步？

| 场景 | 说明 | 示例 |
|------|------|------|
| 下游系统需要用户数据 | 外部应用需要数犀集成平台中的用户/组织数据 | OA系统需要同步员工花名册 |
| 数据需要双向一致性 | 数犀集成平台作为主数据源，向下游系统分发 | 人事系统维护组织架构，同步到各业务系统 |
| 下游应用无API | 不能拉取，只能等待推送数据 | 某些传统系统只支持被动接收数据 |

### 何时同时使用两者？

| 场景 | 说明 |
|------|------|
| 混合身份环境 | 部分应用支持拉取（SSO），部分需要推送（下游系统） |
| 数据中转 | 从A系统拉取，同步到B系统（如AD → 平台 → 泛微） |

## 关键区别对比

| 对比维度 | 通讯录集成 | 通讯录同步 |
|----------|-----------|-----------|
| **数据方向** | 上游 → 平台 | 平台 → 下游 |
| **数据存储位置** | 数犀集成平台 | 目标应用系统 |
| **身份源** | 外部系统（如钉钉/AD） | 数犀集成平台 |
| **用户登录方式** | 使用外部系统账号登录 | 使用目标系统账号登录 |
| **同步触发** | 用户首次登录时触发（JIT） | 实时或定时任务触发 |
| **适用场景** | SSO、身份统一管理 | 下游系统需要用户数据 |

- 导航路径

| 功能 | 导航路径 |
|------|----------|
| 通讯录集成 | 用户中心 → 通讯录集成 |
| 通讯录同步 | 用户中心 → 通讯录同步 |
