1.
1. 概述
随着企业信息化建设的深入，通讯录数据的跨系统同步已成为常见的需求。本文档将指导用户如何将通讯录数据从泛微系统集成到集成平台，确保数据的准确性和一致性。通过本文档，您可以掌握从泛微系统导出通讯录数据，并将其成功同步到集成平台的操作流程。

## 前提准备
在进行通讯录数据集成前，请确保以下条件已满足：
- 版本支持：泛微OA ecology.E9版本
- 账号权限：
- 您需要拥有泛微系统的管理员权限，以便导出通讯录数据。
- 您需要拥有集成平台的管理员权限，以便配置数据同步功能。
- 网络配置：
- 确保泛微系统和集成平台在同一网络环境下，或网络连接稳定，能够正常访问。
- 确保两个系统之间的数据传输通道畅通。
- 数据准备：
- 确保泛微系统中的通讯录数据已更新至最新版本，且格式符合集成平台的导入要求。
- 确认集成平台中已具备相应的通讯录模板或数据结构，以便顺利导入。
- 认证方式配置：
- Token认证：确保集成平台已配置好泛微系统的API访问Token，并验证Token的有效性。
- IP认证：确保集成平台已配置好允许访问的泛微系统IP地址，并验证IP地址的准确性。

3.
1. 操作步骤

- 在集成平台中，导航至用户中心_通讯录集成模块。






- 认证方式包含2种：token认证和IP认证
1. token认证：具体token+appid生成方式 Token认证使用步骤V3
发放许可证(appid)，在ecology系统数据库执行以下脚本示例（请不要直接使用示例中的数据）：

```sql
INSERT INTO ECOLOGY_BIZ_EC(ID,APPID,NAME) VALUES
('123456','EEAA5436-7577-4BE0-8C6C-89E9D88805EA','上海泛微网络科技股份有限公司');
COMMIT;
```




字段描述：
- ID：数据库主键。保证与其它系统发放的许可证在数据库中的主键标识不冲突即可（对应示例：123456）
- APPID：许可证号码。最终发放给异构系统的许可证号码,多个许可证号码保证唯一（对应示例：EEAA5436-7577-4BE0-8C6C-89E9D88805EA）
- NAME：许可证名称。用于快速辨识许可证发放系统

2. IP认证：在/WEB-INF/prop/HrmWebserviceIP.properties配置客户端IP；多个值的以逗号分隔。调用的客户端ip正确，则无需提供任何额外的参数。直接调用业务接口。
在ecology系统代码目录中找到以下配置文件：ecology/WEB-INF/prop/weaver_rest_token.properties

```properties
# 限制Rest API Token Invoke注册端IP地址,多个IP地址之间用逗号分隔
# 例如：allowIp=127.0.0.1,192.168.0.1,172.10.0.1,10.10.10.01
allowIp=
```




在allowIp=后面添加客户调用方服务器实际的ip地址(不设置将代表不限制任何ip，生产环境建议设置)


- 确认所有配置无误后，点击“同步”按钮，启动通讯录数据同步。
- 等待同步完成，系统会提示同步成功或失败。
1. 注意事项
- 在同步过程中，建议先备泛微系统和集成平台的通讯录数据，以防数据丢失或损坏。
- 确保通讯录数据格式与集成平台的模板完全一致，避免导入失败。
- 如遇到数据导入失败的情况，请检查文件路径、格式、Token或IP地址是否正确。
- 定期同步通讯录数据，以保持两个系统之间的数据一致性。
