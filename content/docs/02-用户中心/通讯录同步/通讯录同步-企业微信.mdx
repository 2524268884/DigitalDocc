## 概述

专属集成平台与企业微信的通讯录同步功能，支持将企业微信中的组织架构、用户信息高效同步至平台，实现企业内部身份数据的统一管理与一致性。通过该功能，您可以从企业微信获取用户数据和组织架构，降低管理成本，提高数据准确性，并确保各系统间身份信息的实时同步。

⚠️**重要提示**：由于企业微信通讯录集成规则限制了集成的IP地址及域名与企业微信组织是否一致，所以只能在私有化版本中使用，不建议在SaaS平台配置多个企业微信组织的通讯录。如果需要在SaaS平台进行集成配置，请提交工单获取专业支持。

### 同步优势
- **数据自动获取**：配置完成后，平台可按设定周期自动从企业微信获取数据，无需人工干预
- **身份统一管理**：通过平台集中管理各系统账号，实现"一处变更，处处生效"
- **减少管理开销**：降低人工维护多个系统账号的工作量，提高IT管理效率
- **数据一致性保障**：确保各系统间的身份数据保持同步，避免信息孤岛
- **安全机制保障**：内置删除阈值机制，防止因数据异常造成大规模用户误删

### 适用场景
- **多系统身份打通**：企业使用企业微信作为组织管理工具，需要将用户数据同步至其他业务系统
- **基于组织架构的权限分配**：需要将企业微信组织架构同步至各应用系统，用于自动化权限分配
- **员工入离职流程自动化**：通过企业微信集中管理员工账号，确保员工入离职时身份信息在各系统同步变更
- **移动办公协同**：将企业微信作为移动端入口，与PC端各业务系统实现无缝对接

## 前置准备

### 必要条件
- **企业微信管理员权限**：拥有企业微信的管理员权限，用于创建应用和配置API权限
- **平台管理员权限**：拥有专属集成平台的管理员权限，用于配置同步连接器
- **网络要求**：确保专属集成平台服务器可以访问企业微信API接口
- **企业认证**：企业微信组织已完成企业认证（未认证组织无法配置自定义域名）
- **可信IP配置**：确保专属集成平台的服务器IP已添加至企业微信可信IP列表

### 企业微信应用配置步骤
1. **登录企业微信管理后台**：
    - 访问 https://work.weixin.qq.com/wework\_admin/frame
    - 使用管理员账号登录
2. **创建自建应用**：
    - 导航至「应用管理 → 应用 → 自建」
    - 点击「创建应用」按钮
    - 填写应用名称（如"专属集成平台"）、应用Logo
    - 选择应用可见范围（建议选择「全部成员」以获取完整组织架构）
    - 完成应用创建
3. **获取应用配置信息**：
    - 创建完成后，记录以下关键信息：
        - CorpID：企业微信的企业ID
        - AgentID：自建应用的AgentID
        - Secret：自建应用的Secret
4. **配置应用权限**：
    - 在应用详情页，导航至「权限管理」
    - 勾选以下权限：
        - 通讯录管理权限
        - 成员敏感信息权限（如需同步手机号等敏感信息）
        - 部门管理权限
5. **配置可信域名和IP**：
    - 在应用详情页，导航至「开发者接口」选项卡
    - 设置「企业可信IP」：添加专属集成平台服务器的公网出口IP
    - 设置「网页授权域名」：添加专属集成平台的域名
    - 下载校验文件，用于后续配置（文件名需包含后缀）

<Callout type="warn">
设置可信域名必须要配置企业微信的认证源信息，否则未认证的企业微信组织无法配置第三方域名。认证源配置请参考文档：
</Callout>

[企业微信开放平台创建应用](/docs/01-快速开始/企业微信开放平台创建应用)

## 专属集成平台配置

### 添加企业微信授权
1. **导航至授权管理**：登录专属集成平台管理控制台，在左侧导航栏选择「参数配置 → 授权管理」
2. **添加新授权**：点击页面右上角的「添加授权」按钮
3. **选择平台类型**：在弹出的对话框中，选择「企业微信」平台
4. **填写授权信息**：按照表单要求填写以下信息：
    - 授权名称：输入具有识别意义的名称，如"总部企业微信授权"
    - CorpID：填写企业微信的企业ID
    - AgentID：填写自建应用的AgentID
    - AppSecret：填写自建应用的Secret
    - 校验文件：上传从企业微信管理后台获取的网页授权校验文件
5. **测试连接**：点击「测试连接」按钮，验证授权配置是否正确
6. **保存授权**：测试成功后，点击「保存」按钮完成授权配置

### 创建同步连接器
1. **导航至通讯录同步**：在左侧导航栏选择「用户中心 → 通讯录同步」
2. **添加连接器**：点击页面右上角的「添加连接器」按钮
3. **选择连接器类型**：在弹出的平台选择面板中，选择「企业微信」图标
4. **连接器基本信息**：
    - 名称：建议使用具有业务含义的名称，如"总部企业微信同步"
    - 企业选择：从下拉菜单中选择之前配置的企业微信授权

### 配置同步策略

完成基本信息填写后，配置以下同步策略参数：

| 参数名称 | 界面显示名称 | 说明 | 建议配置 | 适用场景 |
|------------|------------------|------|------------|------------|
| 同步范围 | 同步范围 | 指定同步的内容类型 | 用户和部门：同步完整组织架构仅同步用户：只同步用户数据 | 完整管理：用户和部门仅用户管理：仅同步用户 |
| 下游根组织ID | 下游根组织ID | 组织架构同步起始节点 | 默认为"1"(同步完整结构)指定部门ID可限定范围 | 全量同步：填"1"部分同步：填指定部门ID |
| 强制创建 | 强制创建 | 是否强制在下游创建用户/部门 | 建议开启，确保数据一致性 | 新系统初始化：开启增量维护：可选关闭 |
| 组织架构同步范围 | 组织架构同步范围 | 同步组织的范围 | 全量：同步全部组织部分：仅同步选定组织 | 全部数据同步：全量特定部门同步：部分 |
| 请选择范围 | 请选择范围 | 指定部分同步的具体范围 | 根据实际需求选择特定部门 | 仅当"组织架构同步范围"选择"部分"时显示 |
| 同步方式 | 同步方式 | 指定同步的方式 | 自动增量：只同步变更数据自动全量：定期同步全部数据手动全量：手动触发全量同步 | 日常变更：自动增量定期校验：自动全量大规模变更：手动全量 |
| 同步周期 | 同步周期 | 执行同步的频率 | 初期建议"每小时"稳定后可改为"每6小时" | 频繁变更：每30分钟或每小时一般场景：每6小时或每天 |
| 开始时间 | 开始时间 | 同步执行的起始时间 | 建议选择业务低峰期 | 避开业务高峰期，减少系统负担 |
| 过滤器 | 过滤器 | 对同步数据进行筛选的条件 | 默认关闭，按需配置 | 需要特定条件筛选数据时开启 |

**同步方式详细说明**：
- **自动增量同步**：
    - 只同步企业微信中变更的数据
    - 适用于日常运维，频繁小规模变更
    - 资源消耗较少，同步速度快
- **自动全量同步**：
    - 按设定周期同步企业微信的所有数据
    - 适用于定期数据校验，确保数据一致性
    - 可纠正历史数据错误
- **手动全量同步**：
    - 需要管理员手动触发同步
    - 适用于大规模组织变更、系统初始化
    - 提供更高的控制度

**组织架构同步范围说明**：
- **全量**：同步企业微信中的完整组织架构和用户
- **部分**：仅同步指定部门的组织架构和用户，需要在"请选择范围"中指定具体部门

### 配置属性映射

属性映射是将企业微信中的用户/部门属性与平台中的属性建立对应关系的过程。通过属性映射，系统可以准确地将数据从企业微信同步到平台，确保数据的完整性和一致性。
1. **导航至属性映射**：完成同步策略配置后，点击「下一步」或切换至「属性映射」选项卡
2. **配置用户属性映射**：根据业务需求配置用户属性映射关系
3. **配置部门属性映射**：配置部门属性的映射关系

**映射方式详细说明**：
- **不映射**：该属性不会被同步到平台
    - 适用场景：无需在平台存储的企业微信特有属性
    - 影响：平台中不会创建或更新该属性
- **仅创建**：只在创建用户/部门时同步该属性，更新时不同步
    - 适用场景：初始化后不应被自动更新的属性，如员工编号
    - 影响：用户/部门创建后，该属性的变更不会自动同步
- **创建和更新**：创建和更新用户/部门信息时均同步该属性
    - 适用场景：需要保持实时同步的核心属性，如姓名、职位、部门名称
    - 影响：企业微信中该属性变更时，会自动同步到平台

**用户属性映射表**：
<Callout type="warn">
企业微信的用户属性有限，部分属性（如手机号）可能需要创建扩展属性获取。
</Callout>

| 企业微信属性 | 平台属性 | 数据类型 | 必填 | 建议映射方式 | 说明 |
|------------------|------------|------------|------|------------------|------|
| userid | sub | String | 是 | 创建且更新 | 用户唯一标识 |
| name | name | String | 是 | 创建且更新 | 用户姓名 |
| department | department\_ids | Array | 是 | 创建且更新 | 用户所属部门 |
| order | dept\_order | Array | 否 | 创建且更新 | 用户在部门中的排序 |
| position | position | String | 否 | 创建且更新 | 用户职位信息 |
| email | email | String | 否 | 创建且更新 | 用户邮箱地址 |
| avatar | avatar | String | 否 | 创建且更新 | 用户头像URL |
| thumb\_avatar | thumb\_avatar | String | 否 | 创建且更新 | 用户头像缩略图URL |
| address | address | String | 否 | 创建且更新 | 用户地址信息 |

**部门属性映射表**：

| 企业微信属性 | 平台属性 | 数据类型 | 必填 | 建议映射方式 | 说明 |
|------------------|------------|------------|------|------------------|------|
| id | id | String | 是 | 创建且更新 | 部门ID |
| name | name | String | 是 | 创建且更新 | 部门名称 |
| parentid | parent\_id | String | 是 | 创建且更新 | 父部门ID |
| order | order | Integer | 否 | 创建且更新 | 部门排序值 |

## 测试与验证

<Steps>
<Step>
### 步骤 1：连接测试步骤

1. **点击测试按钮**：在同步连接器配置页面，点击「测试」按钮
2. **查看测试结果**：系统将调用企业微信API进行测试，并显示测试结果
3. **验证API权限**：测试成功表明配置的授权信息正确，且应用具有必要的API权限
4. **检查错误信息**：如测试失败，仔细阅读错误信息，根据提示进行故障排查

**连接测试失败排查路径**：
- 错误提示"连接失败"：检查网络连接，确认服务器IP已添加至企业微信可信IP列表
- 错误提示"认证失败"：检查CorpID、AgentID和Secret是否正确填写
- 错误提示"权限不足"：检查应用是否具有通讯录读取权限
- 错误提示"域名未授权"：检查域名是否已正确配置为可信域名，校验文件是否已上传
</Step>
<Step>
### 步骤 2：数据同步验证方法

1. **首次同步触发**：
    - 在同步连接器配置页面，点击「保存并同步」按钮触发首次同步
    - 或在通讯录同步列表页，找到已创建的连接器，点击「立即同步」按钮
2. **同步状态监控**：
    - 在通讯录同步列表页，观察连接器的同步状态
    - 同步进行中会显示进度百分比
    - 同步完成后会显示同步结果（成功/失败）和同步时间
3. **成功完成判断标准**：
    - 同步状态显示为"同步成功"
    - 同步日志中无错误信息
    - 同步数据统计显示预期的创建、更新和删除数量
4. **数据验证**：
    - 导航至「用户中心 → 用户管理」，检查是否成功同步了用户数据
    - 验证关键属性（姓名、部门等）是否同步正确
    - 验证组织架构是否正确同步
</Step>
<Step>
### 步骤 3：增量同步测试方法

1. **准备测试数据**：
    - 在企业微信管理后台做以下测试性变更：
        - 新增一个测试用户
        - 修改一个已存在用户的信息（如职位）
        - 调整用户的部门归属
        - 创建一个新的部门
2. **触发增量同步**：
    - 手动触发：点击连接器的"立即同步"按钮
    - 自动触发：等待下一个自动同步周期
3. **验证增量同步结果**：
    - 检查新增用户是否被同步到平台
    - 验证修改的用户信息是否在平台中更新
    - 确认用户部门变更是否生效
    - 验证新部门是否正确创建
</Step>
<Step>
### 步骤 4：同步日志查看和解析

1. **访问同步日志**：
    - 在通讯录同步列表页，点击目标连接器的「日志」按钮
    - 或在同步详情页面，切换至「同步日志」选项卡
2. **日志关键信息解读**：
    - 时间戳：记录操作发生的具体时间
    - 操作类型：CREATE(创建)、UPDATE(更新)、DELETE(删除)
    - 目标对象：操作的对象类型(USER/DEPARTMENT)和ID
    - 状态码：操作结果代码，200表示成功
    - 错误信息：如操作失败，显示具体错误原因
3. **常见日志模式和含义**：
    - "开始同步数据"：同步操作启动
    - "从企业微信获取数据成功"：成功从企业微信获取数据
    - "创建/更新/删除对象成功"：对应操作成功完成
    - "同步完成"：整个同步过程结束
    - "错误：xxx"：操作失败，后跟具体错误信息
</Step>
</Steps>

## 常见问题与故障排除

### 同步失败的可能原因和解决方案

| 问题类别 | 问题描述 | 可能原因 | 解决方案 |
|------------|------------|------------|------------|
| 连接问题 | 无法连接到企业微信API | 网络连接问题或服务器IP未授权 | <ul><li>1. 检查网络连接是否正常2\. 确认服务器IP已添加至企业微信可信IP列表3\. 检查企业微信API服务是否可用</li></ul> |
| 认证问题 | API调用返回认证错误 | 授权信息错误或已过期 | <ul><li>1. 检查CorpID、AgentID和Secret是否正确2\. 确认自建应用状态正常3\. 尝试重新生成Secret</li></ul> |
| 权限问题 | 无法获取通讯录数据 | 应用权限不足 | <ul><li>1. 检查应用是否具有通讯录读取权限2\. 确认应用可见范围是否包含需同步的部门3\. 更新应用权限配置并等待生效</li></ul> |
| 域名问题 | 接口调用被拒绝 | 域名未授权或校验失败 | <ul><li>1. 确认平台域名已添加至企业微信可信域名2\. 检查校验文件是否正确上传3\. 重新配置域名授权</li></ul> |
| 数据问题 | 同步部分数据失败 | 数据格式不符合要求 | <ul><li>1. 检查用户属性是否符合要求2\. 确认必填字段值完整3\. 处理特殊字符或格式问题</li></ul> |
| API限制 | 同步被限流 | 短时间内API调用次数过多 | <ul><li>1. 调整同步周期，错开时间2\. 减少同步频率3\. 使用增量同步替代全量同步</li></ul> |

### 特定错误代码解析

| 错误代码 | 错误描述 | 解决方案 |
|------------|------------|------------|
| 40001 | 不合法的secret参数 | 检查Secret是否正确，必要时重新生成 |
| 40014 | 不合法的access\_token | access\_token已过期，系统会自动重新获取 |
| 40096 | 不合法的外部联系人userid | 检查用户ID格式是否符合要求 |
| 41001 | 缺少access\_token参数 | 检查授权配置是否完整 |
| 60011 | 用户不在权限范围内 | 检查应用可见范围设置，确保包含需同步的用户 |
| 60020 | 访问IP不在白名单中 | 将服务器IP添加至企业微信可信IP列表 |

### 数据不完整问题排查方法
1. **企业微信数据验证**：
    - 登录企业微信管理后台，验证用户数据是否完整
    - 检查关键字段（如姓名、部门等）是否有值
    - 确认用户是否在应用可见范围内
2. **属性映射验证**：
    - 检查属性映射配置，确保必要字段已正确映射
    - 验证映射方式是否符合预期（如"创建且更新"vs"仅创建"）
    - 对照日志检查实际同步的数据内容
3. **同步范围验证**：
    - 检查同步范围配置是否正确
    - 确认下游根组织ID设置是否符合预期
    - 验证部分同步时，选择的范围是否包含所需用户/部门

### 组织架构不匹配问题解决
1. **结构层级问题**：
    - 企业微信与平台的组织结构层级可能存在差异
    - 确认根部门ID配置正确（通常为"1"）
    - 检查特殊部门结构的映射逻辑
2. **部门顺序问题**：
    - 验证部门排序字段(order)是否正确映射
    - 调整排序值解决部门显示顺序问题
    - 对特殊排序需求使用自定义转换规则

## 最佳实践

### 初始配置建议
1. **配置前的准备工作**：
    - 梳理企业微信组织架构，确保结构清晰
    - 检查用户数据完整性，补充缺失信息
    - 确认企业微信管理员权限和应用权限
    - 准备服务器IP和域名信息，用于配置可信访问
2. **分步骤实施建议**：
    - **第一阶段 - 应用准备**：
        - 创建企业微信自建应用并配置权限
        - 配置可信IP和域名
        - 获取必要的配置参数（CorpID, AgentID, Secret）
    - **第二阶段 - 平台配置**：
        - 添加企业微信授权
        - 创建同步连接器，配置基本信息
        - 测试连接是否成功
    - **第三阶段 - 小范围测试**：
        - 先选择小部门进行同步测试
        - 验证数据同步是否准确
        - 调整配置参数，优化同步效果
    - **第四阶段 - 全面部署**：
        - 扩大同步范围至全组织
        - 配置自动同步策略
        - 建立监控机制
3. **避免常见配置错误的方法**：
    - 确保服务器IP和域名已正确配置为企业微信可信
    - 应用权限设置需包含通讯录管理权限
    - 同步范围要与应用可见范围一致
    - 属性映射配置要考虑数据变更频率和重要性

### 运行维护建议
1. **日常监控要点**：
    - 定期检查同步状态，确保按计划执行且成功完成
    - 监控同步日志，及时发现并解决异常情况
    - 抽样验证同步数据一致性，特别是新增和变更的用户
    - 关注企业微信组织变动，评估对同步的影响
2. **周期性维护任务**：
    - 每周：检查同步日志，确认无持续错误
    - 每月：执行一次全量同步，确保数据完整性
    - 每季度：评估同步策略和参数，进行必要调整
    - 每年：检查企业微信应用配置，确保权限和设置仍然有效
3. **安全建议**：
    - 定期更换企业微信应用Secret
    - 严格控制可信IP列表，删除不再使用的IP
    - 监控异常的同步行为，防止数据泄露
    - 对重要数据变更建立审批机制

### 企业微信特有注意事项
1. **应用可见范围影响**：
    - 企业微信应用可见范围直接影响可同步的用户范围
    - 如需同步全员数据，应将应用可见范围设为「全部成员」
    - 如只需同步部分部门，可相应限制应用可见范围
2. **敏感数据获取**：
    - 获取手机号等敏感信息需要特殊权限
    - 对于无法直接获取的字段，考虑使用扩展属性
    - 某些字段可能需要企业微信认证后才能获取
3. **同步频率控制**：
    - 企业微信API有调用频率限制
    - 大型组织应控制同步频率，避免触发限流
    - 优先使用增量同步，减少API调用次数
4. **企业认证影响**：
    - 未认证的企业微信组织功能受限
    - 域名配置等高级功能需要企业认证
    - 建议完成企业认证以获得完整功能

### 场景示例配置

**场景1：全公司人员同步**
```text
配置：
- 下游根组织ID: 1（顶级组织）
- 同步范围: 用户和部门
- 组织架构同步范围: 全量
- 同步方式: 自动增量
- 同步周期: 每小时
- 开始时间: 凌晨3点（非工作时间）

此配置适合将企业微信中的完整组织架构和全体人员信息同步到平台，每小时进行增量同步，确保数据及时更新，而初始同步安排在非工作时间，减少对业务的影响。
```

**场景2：特定部门同步**
```text
配置：
- 下游根组织ID: 特定部门ID（如：2001）
- 同步范围: 用户和部门
- 组织架构同步范围: 部分
- 请选择范围: 选择特定部门（如：市场部）
- 同步方式: 自动全量
- 同步周期: 每6小时
- 开始时间: 上午8点

此配置适合只需同步特定业务部门的情况，如将企业微信中的市场部人员和组织结构同步至平台，每6小时进行一次全量同步，确保数据一致性。
```

**场景3：仅用户数据同步**
```text
配置：
- 下游根组织ID: 1
- 同步范围: 仅同步用户
- 组织架构同步范围: 全量
- 同步方式: 手动全量
- 强制创建: 开启

此配置适合已有组织架构，只需同步用户数据的场景，通过手动控制同步时机，避免自动同步带来的潜在冲突，强制创建确保所有用户数据都能被正确同步。
```

### 系统要求
1. **服务器要求**：
    - 可访问互联网，能够连接企业微信API
    - 具有固定公网IP地址，用于配置企业微信可信IP
    - 如使用内网环境，需配置网络代理或NAT穿透
2. **网络要求**：
    - 服务器到企业微信API的网络延迟应低于500毫秒
    - 防火墙需允许对企业微信API的访问（https连接）
    - 建议配置专用网络通道，确保连接稳定性
3. **性能考量**：
    - 100人以下的小型组织：内存≥4GB，CPU≥2核
    - 100-1000人的中型组织：内存≥8GB，CPU≥4核
    - 1000人以上的大型组织：内存≥16GB，CPU≥8核
    - 存储空间需求：根据用户数量和同步历史记录保留策略确定，建议至少预留10GB
4. **企业微信组织要求**：
    - 建议完成企业微信组织认证，获得更完整的API功能
    - 确保组织架构清晰合理，避免过深的层级结构（建议不超过5级）
    - 用户数据尽量完整，特别是必填字段（如姓名、部门归属等）
