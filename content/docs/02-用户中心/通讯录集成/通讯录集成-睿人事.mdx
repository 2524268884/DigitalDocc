## 概述

本指南详细说明如何在专属集成平台中配置睿人事连接器，实现将睿人事系统中统一维护的组织架构和用户数据同步至专属集成平台。通过此集成，企业可以确保内部各应用系统的身份数据一致性，简化身份管理流程，提高企业数据治理效率。

### 集成价值
- **数据统一管理**：以睿人事作为人力资源主数据源，确保企业内部身份数据一致性
- **自动化同步**：减少手动维护工作，降低人为错误风险
- **提高效率**：简化用户管理流程，提高IT管理和HR协作效率
- **身份统一管理**：集中管理企业内各系统的用户身份
- **优化员工生命周期管理**：员工入职、异动、离职等状态变更自动同步至各相关系统

### 适用场景
- **人事与IT协同**：企业使用睿人事作为人力资源管理系统，需要与IT系统紧密集成
- **多系统身份打通**：将睿人事的组织和人员数据同步至多个业务系统
- **员工生命周期管理**：实现员工入职、转岗、离职等状态的自动化同步
- **组织变更高效应对**：组织架构调整后，自动同步至各集成系统，确保数据一致性

## 前提条件

在开始配置前，请确保满足以下条件：

### 睿人事系统要求
- **管理员权限**：拥有睿人事系统的管理员账号及密码
- **API访问**：睿人事系统已开启API接口访问权限
- **组织结构信息**：了解睿人事系统中的组织结构和部门编码体系
- **根组织ID**：获取睿人事系统中的根组织ID或需要同步的部门ID

### 专属集成平台要求
- **管理员账号**：拥有专属集成平台的管理员权限
- **通讯录集成权限**：具备创建和管理通讯录集成连接器的权限
- **系统版本**：专属集成平台 v3.2.0及以上版本

### 信息收集清单

在配置前，需要准备以下信息：

| 所需信息 | 说明 | 获取方式 |
|------------|------|------------|
| 服务器地址 | 睿人事系统的API访问地址 | 从睿人事系统管理员处获取 |
| CorpID | 企业在睿人事的标识 | 睿人事管理后台或联系系统管理员 |
| 根组织ID | 组织架构的根节点ID | 睿人事组织管理模块或咨询管理员 |
| 测试账号 | 用于验证集成的有效账号 | 选择睿人事中确实存在的账号 |
| 账号关联信息 | 用户标识字段(如工号、手机号) | 根据企业实际情况确定 |

**关于删除用户阈值的详细说明**：

删除用户阈值是专属集成平台的安全机制，用于防止因配置错误或数据异常导致的大规模用户误删除。当单次同步中待删除用户的比例超过此阈值时，这些用户会被标记为"待删除"状态，需管理员确认后才执行实际删除操作。

**计算公式**：
```text
删除用户比例 = (本次待删除用户数 / 已同步用户总数) × 100%
```

**阈值设置值及其含义**：
- **1**：当删除1个用户时，不执行删除操作
- **2**：当删除1个用户时执行删除；当删除2个及以上用户时，不执行删除
- **3**：当删除2个用户时执行删除；当删除3个及以上用户时，不执行删除
- **0**：同步时不删除任何用户，适用于仅需添加和更新用户的场景
- **-1**：不设置删除上限，无论删除多少用户都会同步执行删除（谨慎使用）

**关于账号关联的说明**：

账号关联用于建立睿人事用户与专属集成平台用户之间的映射关系。配置步骤：
1. 在左侧下拉菜单中选择睿人事识别字段（如 appuser.empid、appuser.mobile 等）
2. 在右侧选择专属集成平台对应字段（如 用户名、手机号等）
3. 可添加多个关联规则，规则之间为"OR"逻辑关系（满足任一规则即视为同一用户）

**建议配置**：
- 优先使用唯一标识字段，如工号(empid)
- 如工号不可用，可使用手机号作为备选
- 对于复杂环境，可设置多个规则增加匹配成功率

## 数据管理

### 查看同步用户
1. 导航至「用户中心 → 用户管理」页面
2. 使用筛选条件，选择睿人事连接器作为认证源
3. 查看已同步的用户列表
4. 点击用户名可查看详细信息

### 待删除用户处理

当同步过程中发现睿人事系统删除的用户数量超过设定的删除阈值时，这些用户会被标记为"待删除"状态：
1. 导航至连接器详情页，切换到「待删除用户」选项卡
2. 系统显示所有待删除用户列表，包含以下信息：
    - 用户姓名
    - 用户名
    - 部门
    - 标记为待删除的原因
    - 标记时间
3. 处理待删除用户的操作选项：
    - **确认删除**：点击用户行右侧的"确认"按钮，用户将被从系统中删除
    - **批量确认**：选择多个用户，点击"批量确认"按钮
    - **拒绝删除**：如确认某用户不应删除，可点击"拒绝"按钮，用户将保留在系统中

**删除用户操作建议**：
- 在确认删除前，先核实这些用户在睿人事系统中的状态
- 对于批量删除操作，建议先导出列表进行审核
- 建议定期处理待删除用户，避免积压过多

## 故障排除

### 连接问题

### 连接测试失败

**可能原因**：
- 服务器地址错误
- CorpID不正确
- 网络连接问题
- 睿人事API服务异常

**解决方案**：
1. 验证服务器地址是否正确，包括协议前缀(http/https)
2. 确认CorpID是否正确填写
3. 检查网络连通性，确认能够访问睿人事服务器
4. 联系睿人事技术支持，确认API服务状态

### 认证失败

**可能原因**：
- 账号权限不足
- API访问限制
- 系统限制IP访问

**解决方案**：
1. 确认使用的是管理员级别账号
2. 检查睿人事系统的API访问权限设置
3. 如有IP限制，确认专属集成平台服务器IP已被授权

### 同步数据问题

### 数据同步不完整

**可能原因**：
- 同步范围设置不正确
- 根组织ID配置错误
- 属性映射配置有误
- 睿人事系统中数据不完整

**解决方案**：
1. 检查并调整同步范围设置
2. 验证根组织ID是否正确
3. 修正属性映射配置
4. 在睿人事系统中确认数据完整性

### 组织架构错误

**可能原因**：
- 部门ID和父部门ID映射不正确
- 组织层级处理不当
- 特殊组织结构未正确处理

**解决方案**：
1. 检查部门属性映射，特别是ID和父部门ID
2. 调整组织层级处理方式
3. 针对特殊组织结构制定专门的处理方案

### 同步性能问题

**可能原因**：
- 数据量过大
- 网络连接质量不佳
- 系统资源不足

**解决方案**：
1. 考虑分批同步，先同步核心部门
2. 选择网络状况良好的时段执行同步
3. 优化服务器资源配置
4. 调整同步周期，避免频繁同步导致系统负载过高

## 最佳实践

### 初始配置建议
1. **分步实施**：
    - 从小范围测试开始，如先同步单个部门
    - 验证配置无误后，再扩大同步范围
    - 完成小范围测试后，再进行全量同步
2. **关键字段映射**：
    - 确保唯一标识字段(ID)正确映射
    - 用户名、姓名、手机号等核心字段使用"创建和更新"模式
    - 确保部门ID和父部门ID映射关系正确
3. **同步策略优化**：
    - 首次配置使用手动全量同步，验证数据准确性
    - 验证无误后，改为自动增量同步
    - 根据组织变更频率合理设置同步周期

### 运维管理建议
1. **定期监控**：
    - 定期检查同步日志，关注同步状态
    - 监控同步数据量变化，及时发现异常
    - 定期抽查同步数据的准确性
2. **变更管理**：
    - 组织架构发生重大变更前，提前评估影响
    - 大规模变更前调整删除用户阈值
    - 重要变更后执行全量同步，确保数据一致
3. **问题处理**：
    - 建立同步异常快速响应机制
    - 记录常见问题及解决方法
    - 与睿人事系统管理员保持沟通，及时了解系统变更

### 安全建议
1. **删除阈值设置**：
    - 根据企业规模和人员变动情况合理设置删除阈值
    - 避免设置为-1（无限制删除），除非确有必要
    - 定期审核待删除用户，及时处理
2. **权限控制**：
    - 严格控制有权修改连接器配置的管理员范围
    - 建立配置变更审批流程
    - 定期审计权限设置
3. **数据保护**：
    - 避免同步不必要的敏感信息
    - 定期备份关键配置和数据
    - 实施最小必要原则，只同步必需的数据

## 睿人事集成特别说明

### 人力资源数据集成

睿人事作为人力资源管理系统，通讯录集成后可实现以下功能：
1. **员工信息同步**：
    - 新员工信息自动同步至专属集成平台和下游系统
    - 员工信息变更自动更新
    - 离职员工自动禁用或删除
2. **组织架构同步**：
    - 部门设置和调整自动反映在专属集成平台
    - 员工部门归属变更自动更新
    - 支持复杂的组织结构和层级关系
3. **岗位和职级同步**：
    - 员工岗位变更自动更新
    - 职级体系可与权限管理集成
    - 支持基于岗位的权限分配

### 常见业务场景
1. **新员工入职**：
    - HR在睿人事系统创建新员工账号
    - 专属集成平台自动同步新员工信息
    - 下游系统自动创建相应账号
2. **员工信息变更**：
    - HR在睿人事系统更新员工信息
    - 变更信息自动同步至专属集成平台
    - 确保所有系统使用最新的员工信息
3. **员工离职**：
    - HR在睿人事系统标记员工离职
    - 系统自动将用户标记为待删除或禁用
    - 管理员确认后，完成账号禁用或删除
4. **组织调整**：
    - HR在睿人事调整组织架构
    - 新的组织关系自动同步至专属集成平台
    - 确保基于组织架构的权限分配保持更新
