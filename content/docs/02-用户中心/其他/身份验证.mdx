## 概述

身份验证是专属集成平台的高级安全功能，通过二次验证用户档案中的关键属性确保登录者身份真实性，有效防止账号被盗用的风险。当用户通过密码或其他方式完成初步验证后，系统会要求验证预先配置的关键属性（如身份证号、工号等），只有全部验证通过才能完成登录。

## 应用场景

身份验证适用于以下场景：
1. **高风险账号保护**：对管理员账号、财务账号等权限较高用户进行额外保护
2. **敏感信息访问**：访问包含敏感数据的系统或应用时进行身份再确认
3. **账号共享风险防范**：防止账号共享使用，确保登录者是账号真实所有者
4. **合规要求**：满足特定行业（如金融、医疗）的身份验证合规要求
5. **账号安全升级**：为现有账号体系增加额外安全层，无需更改现有验证流程

## 前提条件

- 用户已导入或集成到专属集成平台
- 管理员已在参数配置中启用身份验证功能
- 要验证的用户属性已在系统中录入（如员工工号、身份证号等）

## 配置步骤

### 开启身份验证
1. 以管理员身份登录专属集成平台
2. 导航到**参数配置** \> **平台设置**
3. 在"登录控制"区域找到"身份验证属性"选项
4. 点击开启身份验证
5. 从下拉列表中选择需要验证的属性（最多可选5个）：
    - 基础属性选项：姓名、手机号、身份证号、工号、邮箱
    - 也可选择已定义的扩展属性（系统会自动过滤不支持验证的属性类型，如下拉列表、时间类型）
6. 设置验证失败的最大尝试次数（建议3-5次）
7. 保存配置

## 用户身份验证流程

### 账号密码登录流程
1. 用户输入用户名和密码进行常规登录
2. 密码验证通过后，如用户有待验证的不可信属性，系统跳转到身份验证页面
3. 用户必须正确输入所有待验证属性的值
4. 全部属性验证通过后，完成登录
5. 如管理员还配置了MFA验证，则在身份验证通过后，继续进行MFA验证

### 密码重置流程
1. 用户在重置密码页面输入手机号或邮箱，点击获取验证码
2. 系统检查该用户是否有待验证的不可信属性
3. 如有，验证码验证通过后，用户需要先完成身份验证
4. 身份验证通过后，用户才能继续重置密码流程

## 特殊情况处理

### 验证失败处理
1. **属性验证失败**：
    - 当用户输入错误的属性值时，系统会提示错误并允许重新输入
    - 如连续失败次数达到设定的阈值（默认5次），验证项会被临时锁定
    - 用户账号也会被置为临时锁定状态
    - 锁定状态下，用户需要联系管理员解锁账号

1. **空属性处理**：
    - 如用户的某待验证属性值为空，系统会自动跳过该属性的验证
    - 但当该属性值后续通过其他方式（如通讯录集成）填充后，用户再次登录时仍需要验证
2. **新增属性处理**：
    - 当管理员新增需要验证的属性后，即使用户之前已通过身份验证
    - 下次登录时仍需要完成新增属性的验证

## 最佳实践

1. **选择合适的验证属性**：选择用户不易忘记但又不是公开信息的属性进行验证，如员工编号
2. **组合多种安全措施**：将身份验证与MFA验证结合使用，形成多层次防护
3. **优先保障可用性**：确保所有待验证属性在系统中有值，避免因属性缺失导致用户无法登录
4. **用户培训**：向用户清晰说明身份验证的目的和流程，减少使用障碍

## 与其他安全功能的关系

身份验证可与以下安全功能配合使用：
- **多因子验证(MFA)**：身份验证后再进行MFA验证，形成双重保护
- **登录策略**：与异常登录检测、IP限制等结合，增强安全性
- **验证源管理**：外部验证源登录时也可触发身份验证

## 常见问题

**Q: 用户忘记了需要验证的属性值怎么办？** A: 需要联系管理员，管理员可以更新用户的属性值或暂时禁用该用户的身份验证要求。

**Q: 身份验证与多因子验证(MFA)有什么区别？** A: 身份验证验证用户"是谁"（基于用户已知的个人信息），而MFA验证用户"拥有什么"（如手机）或"是什么"（如指纹）。两者结合可以提供更全面的安全保障。

**Q: 如何处理批量导入用户的身份验证问题？** A: 确保导入数据中包含所有需要验证的属性值。对于暂时无法提供完整信息的用户，可考虑暂不启用身份验证或设置临时的验证项。
