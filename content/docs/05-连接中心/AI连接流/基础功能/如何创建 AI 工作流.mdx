## 概述

AI 工作流（AI Flow）是专属集成平台提供的智能流程编排能力，通过可视化的拖拽操作，将 AI 节点与传统节点组合，实现跨系统的业务流程自动化与智能化处理。本文详细介绍如何从零开始创建一个完整的 AI 工作流。

## 前置条件

在开始创建 AI 工作流之前，请确保满足以下条件：

- 拥有专属集成平台的管理员权限或工作流创建权限
- 已配置必要的连接凭证（如需要连接外部系统）
- 了解需要实现的业务流程和目标

## 创建步骤

### 步骤一：创建工作流

1. 登录专属集成平台，进入 **连接中心 > AI连接流** 模块
2. 点击页面右上角的 **「创建连接流」** 按钮
3. 在弹出的对话框中，选择创建方式：
   - **手动创建**：从空白画布开始，自定义配置所有节点
   - **从模板创建**：选择一个预置模板，快速搭建常见场景的工作流

4. 填写工作流基本信息：
   - **名称**：为工作流设置一个描述性的名称
   - **描述**（可选）：说明工作流的用途
   - **分类**（可选）：将工作流归类到特定项目或业务线

### 步骤二：选择触发器

触发器决定了工作流的启动方式。点击画布上的 **「+」** 按钮，选择一个触发器节点：

| 触发器类型 | 说明 |
|-----------|------|
| **Webhook 触发** | 当收到外部 HTTP 请求时触发 |
| **定时触发** | 按照指定的时间计划周期执行 |
| **事件触发** | 当特定事件发生时触发（如用户登录、数据变更等） |

### 配置 Webhook 触发器

1. 将「Webhook 触发」节点拖拽到画布上
2. 点击节点进行配置：
   - **请求方法**：选择 GET、POST 等
   - **认证方式**：可选配置 API Key 或 Bearer Token 认证
3. 系统会自动生成 Webhook URL，复制该 URL 用于外部调用

### 配置定时触发器

1. 将「定时触发」节点拖拽到画布上
2. 点击节点进行配置：
   - **Cron 表达式**：设置执行时间规则
   - **时区**：选择适用的时区
   - **起始时间**：设置首次执行的时间

### 步骤三：添加并配置节点

根据业务需求，从左侧节点面板拖拽所需节点到画布上。

### 常用节点类型

| 节点类别 | 节点名称 | 用途 |
|---------|---------|------|
| **AI 节点** | AI 浏览器操作节点 | 使用 AI 控制浏览器执行操作 |
| **AI 节点** | AI 消息对话节点 | 实现与用户的自然语言对话 |
| **AI 节点** | AI Agent | 调用大语言模型进行智能决策 |
| **内置节点** | HTTP 请求 | 发送 HTTP 请求调用外部 API |
| **内置节点** | 条件分支 | 根据条件选择不同的执行路径 |
| **内置节点** | 循环执行 | 重复执行一组操作 |
| **内置节点** | 变量节点 | 创建和使用变量存储临时数据 |
| **数据处理节点** | 数组处理 | 对数组数据进行筛选、映射等操作 |
| **数据处理节点** | 文本处理节点 | 对文本进行提取、替换、格式化等操作 |

### 节点配置示例：配置 HTTP 请求节点

1. 将「HTTP 请求」节点拖拽到画布上
2. 连接触发器与 HTTP 请求节点
3. 点击 HTTP 请求节点进行配置：
   - **请求方法**：选择 GET 或 POST
   - **URL**：输入目标 API 地址
   - **请求头**：配置必要的请求头（如 Authorization）
   - **请求体**：配置 POST 请求的 Body 内容
4. 配置完成后，点击 **「测试」** 按钮验证请求

### 节点配置示例：配置 AI 对话节点

1. 将「AI 消息对话」节点拖拽到画布上
2. 连接前置节点与 AI 对话节点
3. 点击节点进行配置：
   - **模型选择**：选择使用的大模型（DeepSeek、GPT-4 等）
   - **系统提示词**：设置 AI 的角色和行为规则
   - **对话历史**：配置是否保留对话上下文
4. 在节点中定义输入变量，用于接收前置节点的数据

### 步骤四：连接节点

在画布上，通过拖拽操作将节点连接起来，形成完整的数据流转路径：

1. 鼠标悬停到节点的输出端口（右侧）
2. 拖拽到下一个节点的输入端口（左侧）
3. 如果节点有多个输出分支（如条件分支），可以为每个分支创建不同的连接

### 步骤五：配置错误处理

为工作流配置错误处理机制，确保运行异常时能够正确响应：

1. 点击画布右上角的 **「错误处理」** 按钮
2. 配置错误处理策略：
   - **重试次数**：节点执行失败时的自动重试次数
   - **重试间隔**：每次重试之间的等待时间
   - **兜底策略**：所有重试都失败时的备选方案
   - **告警通知**：执行失败时是否发送告警通知

### 步骤六：调试工作流

在正式上线前，使用调试功能验证工作流是否按预期运行：

1. 点击画布右上角的 **「调试」** 按钮进入调试模式
2. 根据触发器类型，输入测试数据：
   - Webhook 触发：发送测试 HTTP 请求
   - 定时触发：手动触发一次执行
3. 观察每个节点的执行状态和输出结果
4. 如有错误，根据错误提示调整节点配置
5. 调试模式下不会向外部系统发送真实请求，方便排查问题

### 步骤七：上线运行

调试通过后，将工作流正式上线：

1. 点击画布右上角的 **「保存」** 按钮保存工作流
2. 点击 **「上线」** 按钮将工作流状态切换为运行中
3. 工作流上线后：
   - 定时触发：按配置的时间计划自动执行
   - Webhook 触发：外部系统可以通过 Webhook URL 调用
   - 事件触发：当相应事件发生时自动触发

## 最佳实践

### 模块化设计

将复杂的工作流拆分为多个子工作流，通过「返回变量」和「触发子工作流」节点进行调用，提高复用性和可维护性。

### 善用变量

使用变量节点存储中间结果，避免重复计算和数据冗余。变量命名应具有描述性，便于理解和维护。

### 完善的错误处理

- 为每个可能失败的节点配置重试策略
- 设置合理的超时时间
- 配置告警通知，及时发现和处理问题

### 监控和日志

- 定期查看工作流运行日志
- 关注执行时长和资源消耗
- 根据监控数据持续优化工作流性能

## 常见问题

<Accordions>
<Accordion title="Q1：工作流触发后没有响应">
**可能原因**：
- 工作流未上线
- 触发器配置不正确
- 外部调用方式与配置不匹配

**解决方案**：
- 确认工作流状态为「运行中」
- 检查触发器配置是否正确
- 查看运行日志定位具体失败环节
</Accordion>
<Accordion title="Q2：节点执行超时">
**可能原因**：
- 外部 API 响应慢
- 网络连接不稳定
- 节点配置的超时时间过短

**解决方案**：
- 适当增加节点的超时时间配置
- 检查网络环境和外部服务状态
- 考虑添加重试机制
</Accordion>
<Accordion title="Q3：如何查看工作流的执行历史">
**解决方案**：
- 进入 **连接中心 > AI连接流**
- 点击目标工作流的 **「运行日志」** 按钮
- 可以查看每次执行的详细信息、输入输出数据和问题排查
</Accordion>
</Accordions>
