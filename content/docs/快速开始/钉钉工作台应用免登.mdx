title: "钉钉工作台应用免登"
description: "如何配置钉钉工作台免登功能，让员工在钉钉工作台中一键访问企业应用（如阿里云、OA 系统），无需重复登录。"
---

## 概述

钉钉工作台免登基于专属集成平台的单点登录（SSO）能力，实现用户从钉钉工作台点击应用图标后，自动完成身份验证并直接进入目标系统，无需再次输入账号密码。

## 前提条件

| # | 条件 | 说明 |
|---|------|------|
| 1 | 通讻录已同步 | 钉钉用户数据已同步到集成平台，或集成平台用户已同步到钉钉 |
| 2 | 目标应用已配置 SSO | 已在集成平台创建并配置了目标应用的单点登录 |
| 3 | 钉钉认证源已创建 | 已在集成平台添加钉钉企业认证源 |
| 4 | 用户关联关系正确 | SSO 应用中的用户与集成平台用户有正确的映射关系 |

<Callout type="info">
以阿里云为例演示完整配置流程，其他应用（如 OA、ERP）配置方式完全相同，只需替换对应的 SSO 配置文档。
</Callout>

---

## 配置步骤

<Steps>
<Step>
### 同步钉钉组织架构

确保钉钉的用户和组织数据已同步到集成平台。如尚未配置，请先完成通讻录集成。

参考文档：[通讻录集成-钉钉](/docs/用户中心/通讻录集成/通讻录集成-钉钉)

**关键检查点：**
- 进入**用户中心 > 用户管理**，确认可以看到钉钉同步过来的用户
- 确认组织架构层级与钉钉一致
</Step>
<Step>
### 配置目标应用的单点登录

以阿里云为例，在集成平台配置阿里云 SAML SSO 应用。

参考文档：[单点登录-阿里云控制台(RAM)](/docs/应用中心/应用单点登录/单点登录-阿里云控制台(RAM))

**关键信息：** 配置完成后，集成平台会生成一个 **SSO 登录链接**，格式类似：
```
https://your-tenant.idaas-domain.com/app/sso/aliyun?appId=xxx
```

记录此链接，后续配置钉钉应用首页地址时需要用到。
</Step>
<Step>
### 添加钉钉认证源

确保集成平台已配置钉钉企业认证，用于识别从钉钉跳转过来的用户身份。

1. 进入**用户中心 > 认证源管理**
2. 确认已有钉钉认证源，若无请点击「添加认证源」→ 选择「钉钉」

参考文档：[认证源-钉钉](/docs/用户中心/认证源管理/认证源-钉钉)
</Step>
<Step>
### 在钉钉开放平台创建工作台应用

1. 前往 [钉钉开放平台](https://open-dev.dingtalk.com/)
2. 选择**应用开发 > 企业内部开发**，创建新应用
3. 在应用配置中填写：

| 字段 | 说明 |
|------|------|
| 应用名称 | 建议与 SSO 应用名称保持一致，如「阿里云」 |
| 应用图标 | 上传应用图标 |
| 应用首页地址 | **填写第二步获取的 SSO 登录链接** |

<Callout type="warn">
「应用首页地址」必须填写集成平台生成的 SSO 登录链接，而不是业务系统的直接访问地址。否则用户点击后会直接跳转业务系统登录页，无法实现免登。
</Callout>

4. 完成权限设置并发布应用
</Step>
</Steps>

## 验证免登效果

<Steps>
<Step>
### 员工侧验证

1. 在钉钉移动端或 PC 端进入**工作台**
2. 找到已配置的应用图标（如「阿里云」）
3. 点击应用图标
4. 系统自动完成身份验证，直接进入目标系统（无需输入账号密码）
</Step>
<Step>
### 管理员侧检查

如员工反馈免登失败，可在集成平台查看登录日志：

进入**审计日志 > 用户行为日志**，筛选对应用户和时间段，查看登录记录和错误详情。
</Step>
</Steps>

## 常见问题

<Accordions>
<Accordion title="点击工作台应用后跳转到登录页，没有实现免登？">
逐项检查以下原因：

1. **SSO 登录链接是否正确**：确认钉钉应用首页地址填写的是集成平台的 SSO 链接，而非业务系统地址
2. **用户关联关系**：确认该用户在集成平台中存在，且钉钉账号与集成平台账号有正确映射
3. **认证源配置**：确认集成平台已添加钉钉认证源，且认证源已启用
4. **应用授权**：确认该用户已被授权访问此应用
</Accordion>
<Accordion title="提示「用户不存在」或「账号未关联」？">
需要确认以下两点：
- 通讻录集成是否已完成，该用户是否已同步到集成平台
- 认证源账号关联规则是否正确配置（通常按手机号或邮箱匹配）

进入**用户中心 > 用户管理**搜索该用户，查看其绑定的钉钉账号信息。
</Accordion>
<Accordion title="部分员工可以免登，部分不能？">
检查以下差异：
- 不能免登的员工是否已同步到集成平台（查用户管理）
- 不能免登的员工是否已被授权访问目标应用（查权限中心）
- 钉钉账号与集成平台账号的关联字段（手机号/邮箱）是否一致
</Accordion>
</Accordions>
