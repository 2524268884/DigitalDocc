---

## 功能概述

**子流程调用**能力允许将常用逻辑封装为独立的子流程，主流程通过「子流程触发」节点进行调用，子流程执行完成后将结果返回给主流程继续执行。

**核心能力：**
- 主流程支持同步调用子流程，等待子流程执行完成后继续
- 子流程支持接收主流程传入的参数
- 子流程执行结果可作为变量在主流程中引用
- 子流程可被多个主流程复用，无需重复配置相同逻辑

## 适用场景

| 场景 | 说明 |
|------|------|
| 公共逻辑复用 | 多个流程都需要调用同一套认证或数据处理逻辑，封装为子流程统一维护 |
| 复杂流程拆解 | 单一流程节点过多导致维护困难，按业务模块拆分为多个子流程 |
| 项目交付沉淀 | 将客户定制逻辑封装为子流程，在不同项目中快速复用 |

## 创建子流程

<Steps>
<Step>
### 步骤一：新建子流程

在 AI 连接流列表页，新建一个流程，将触发器设置为**「子流程触发」**（而非 Webhook 或定时触发）。

<ImageZoom src="/images/ai-flow/sub-flow-trigger.png" alt="子流程触发器配置" width={800} height={450} />
</Step>
<Step>
### 步骤二：定义入参

在「子流程触发」节点的配置面板中，声明该子流程接受的入参字段及类型。主流程调用时将按此结构传入数据。

| 配置项 | 说明 |
|--------|------|
| 参数名 | 主流程传入时使用的参数键名 |
| 类型 | string / number / boolean / object / array |
| 是否必填 | 主流程调用时若缺少必填参数，子流程将拒绝执行 |
</Step>
<Step>
### 步骤三：构建子流程逻辑

在子流程中添加业务处理节点，与普通连接流配置方式相同。在需要向主流程返回数据的位置，添加**「返回变量」**节点，指定返回值。
</Step>
<Step>
### 步骤四：上线子流程

子流程配置完成后，点击「上线」使其处于可调用状态。未上线的子流程无法被主流程调用。
</Step>
</Steps>

## 在主流程中调用子流程

<Steps>
<Step>
### 步骤一：添加「调用子流程」节点

在主流程画布中，在需要调用子流程的位置添加「调用子流程」节点。
</Step>
<Step>
### 步骤二：选择目标子流程

在节点配置面板中，从下拉列表选择已上线的目标子流程。
</Step>
<Step>
### 步骤三：配置传入参数

按照子流程定义的入参结构，填写传入的参数值。支持引用主流程中其他节点的输出变量。
</Step>
<Step>
### 步骤四：引用子流程返回值

子流程执行完成后，在后续节点中通过以下方式引用返回值：

```text
{{$node['调用子流程节点ID'].output.返回字段名}}
```
</Step>
</Steps>

<Callout type="warn">
子流程调用为**同步执行**，主流程会等待子流程执行完成后才继续。请注意子流程的执行超时设置，避免主流程因子流程超时而中断。
</Callout>

## 常见问题

<Accordions>
<Accordion title="子流程是否支持嵌套调用（子流程调用子流程）？">
支持，但建议嵌套层级不超过 3 层，过深的嵌套会增加调试难度并影响执行性能。
</Accordion>
<Accordion title="主流程和子流程运行日志如何查看？">
主流程运行日志中会包含「调用子流程」节点的执行记录，点击该节点可展开查看子流程的详细执行日志。
</Accordion>
<Accordion title="修改子流程后，已在使用中的主流程会受影响吗？">
子流程修改并重新上线后，所有调用该子流程的主流程在下次执行时会使用最新版本的子流程逻辑。
</Accordion>
</Accordions>
