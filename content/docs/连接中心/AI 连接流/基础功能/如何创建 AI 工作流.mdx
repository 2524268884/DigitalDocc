## 概述

本文档详细介绍如何在数犀集成平台创建和配置 AI 工作流。通过 AI 工作流，您可以将 AI 能力与传统业务流程结合，实现智能化的流程自动化。

## 前提条件

- 已拥有数犀集成平台的管理员权限
- 已配置必要的授权凭证（如大模型 API 密钥）
- 了解需要自动化的业务流程

- 创建流程

### 步骤一：进入 AI 连接流模块

1. 登录数犀集成平台管理控制台
2. 在左侧导航栏选择**连接中心 > AI 连接流**
2. 点击页面右上角的**「创建连接流」**按钮
3. 选择**「手动创建」**方式

### 步骤二：配置基础信息

在创建页面填写工作流的基础信息：

| 参数 | 说明 | 必填 |
|------|------|------|
| 名称 | 工作流的显示名称，建议使用有意义的命名 | 是 |
| 描述 | 工作流的功能说明，便于后续维护 | 否 |
| 触发方式 | 选择工作流的触发类型 | 是 |

### 步骤三：选择触发器

AI 工作流支持多种触发方式：

#### 1. 定时触发

- 按照预设的时间周期自动执行
- 支持 Cron 表达式配置复杂时间规则
- 适用于周期性数据处理、报表生成等场景

#### 2. Webhook 触发

- 通过 HTTP 请求从外部系统触发
- 支持 GET/POST 方法
- 适用于与第三方系统集成

#### 3. 事件触发

- 当特定事件发生时自动执行
- 适用于响应业务事件

### 步骤四：添加并配置节点

在工作流画布上添加所需的节点：

1. **添加节点**：从左侧节点面板拖拽节点到画布
2. **连接节点**：将节点按业务逻辑顺序连接
3. **配置节点参数**：点击节点配置其具体参数

#### 常用节点类型

- **AI 节点**：包含 AI Agent、AI 消息对话、AI 浏览器操作等
- **HTTP 请求**：调用外部 API
- **条件分支**：根据条件选择执行路径
- **循环执行**：重复执行某段逻辑
- **变量节点**：处理和存储数据
- **数据处理节点**：数组处理、文本处理等

### 步骤五：调试工作流

配置完成后，进行调试验证：

1. 点击画布右上角的**「调试」**按钮
2. 在调试面板填写测试参数
2. 点击**「执行」**运行工作流
3. 查看各节点的执行结果和输出
2. 如有错误，根据日志排查并修正

### 步骤六：上线工作流

调试成功后，正式上线工作流：

1. 确认所有节点配置正确
2. 点击**「保存」**按钮
2. 如需自动执行，开启**「启用调度」**
3. 监控工作流运行状态

## 最佳实践

### 设计原则

- **单一职责**：每个节点只完成一个明确的任务
- **错误处理**：为关键节点配置错误处理和重试策略
- **日志记录**：使用日志节点记录关键执行信息

### 性能优化

- 避免不必要的循环嵌套
- 合理设置超时时间
- 及时清理不需要的中间数据

## 常见问题

**Q1：工作流执行失败如何排查？**

A1：可以通过以下步骤排查：
1. 查看工作流运行日志，确认失败节点
- 检查节点的输入参数是否正确
1. 验证外部服务（API、数据库等）的连接状态
- 检查授权凭证是否有效

**Q2：如何选择合适的触发方式？**

A2：根据业务场景选择：
- 定时执行 → 选择定时触发
- 外部系统调用 → 选择 Webhook 触发
- 事件驱动 → 选择事件触发

**Q3：AI 节点返回的结果不符合预期？**

A3：检查以下配置：
1. 确认大模型 API 密钥正确配置
- 检查提示词（Prompt）是否清晰准确
1. 验证输入数据的格式和内容
