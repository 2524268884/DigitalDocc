## 概述
变量节点是AI FLOW中的变量操作处理器，用于在流程执行过程中设置和更新变量值。通过指定变量路径和新的值，可以动态修改流程上下文中的变量，支持复杂的数据结构和嵌套路径操作。
功能特性
- 变量设置：设置流程上下文中的变量值
- 路径支持：支持点分隔的嵌套路径访问
- 类型灵活：支持字符串、数字、对象等多种数据类型
- 动态更新：运行时动态修改变量值
- 错误处理：完善的异常捕获和错误信息返回
1. 配置参数
UpdateVar
- varPath：变量路径，支持点分隔的嵌套路径
- value：要设置的新值，支持各种数据类型
变量路径格式
- 简单变量：varName
- 嵌套对象：user.name 或 user.profile.email
- 数组元素：`list[0]` 或 `users[1].name`
- 动态路径：支持使用其他变量作为路径
使用方法
## 设置简单变量

```json
{
  "varPath": "userName",
  "value": "张三"
}
```

将变量 userName 设置为字符串 "张三"
## 设置嵌套对象属性

```json
{
  "varPath": "user.profile.age",
  "value": 25
}
```

设置用户配置文件的年龄为25
## 更新数组元素

```json
{
  "varPath": "userList[0].status",
  "value": "active"
}
```

更新用户列表第一个用户的状态
## 设置复杂对象

```json
{
  "varPath": "result",
  "value": {
    "code": 200,
    "message": "success",
    "data": {
      "id": 123,
      "name": "测试数据"
    }
  }
}
```

设置result变量为完整的对象
## 使用变量引用

```json
{
  "varPath": "processedData",
  "value": "${input.data}"
}
```

将输入数据赋值给processedData变量

常见错误情况
- **路径不存在**：尝试更新不存在的嵌套路径
- **类型不匹配**：值类型与预期不符
- **权限不足**：尝试修改只读变量
- **内存不足**：大型对象导致内存溢出

高级功能
路径表达式
支持复杂的路径表达式：

```json
{
  "varPath": "data[${index}].result",
  "value": "computed_value"
}
```

批量更新
通过循环调用实现批量变量更新：

```json
{
  "varPath": "batch[${loop.index}].processed",
  "value": true
}
```

条件更新
结合条件节点实现条件变量更新：

Condition → VarAction (满足条件时更新)

         → VarAction (不满足时更新其他变量)

1. 常见问题
Q: 如何更新整个对象？
A: 设置varPath为对象根路径，value为完整的对象。系统会替换整个对象。
Q: 支持数组操作吗？
A: 支持数组元素更新，如array[0]。但不支持数组整体操作如push/pop。
Q: 变量作用域是怎样的？
A: 变量存储在流程上下文(ContextHolder)中，整个流程生命周期内有效。子流程有独立的上下文。
Q: 如何删除变量？
A: 设置value为null可以清空变量值。但要完全删除，需要使用专门的删除操作。
Q: 支持动态路径吗？
A: 是的，可以使用其他变量作为路径的一部分，如`data[$\{index\}]`。但需要确保变量已定义。
