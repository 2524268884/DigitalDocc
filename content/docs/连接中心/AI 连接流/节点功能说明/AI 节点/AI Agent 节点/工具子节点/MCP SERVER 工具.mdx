## 功能概述
MCP（Model Context Protocol）用于将平台已有应用能力以「工具」的方式提供给 AI Agent 使用。
通过在画布中配置 MCP SERVER 节点，可以让大模型在合适的时机自动调用对应的业务动作（如查询、创建、更新等），实现 AI + 业务系统的联动执行  。

选择工具分类（本期仅支持 MCP 分类）

## 适用场景
- AI Agent 在对话过程中需要调用真实业务系统
- 希望由大模型自主判断「什么时候用哪个工具」
- 一个 Agent 需要组合多个业务动作（查询 → 判断 → 执行）
## MCP SERVER 节点说明
### 节点能力
- 每一个 MCP 节点对应 一个后端 MCP SERVER
- 一个 MCP 节点下 可以配置多个动作
- MCP 节点只作为 工具集合，不直接参与流程判断  。

## 参数配置说明（重点）
### 参数输入方式
根据所选动作，系统会自动生成对应的参数输入组件。
### AI 生成模式
- 每个参数输入框右侧提供 「AI 生成」按钮
- 点击后：
  - 参数值由 AI 从用户对话中自动提取
  - 输入框进入 AI 生成状态
  - 不允许手动输入或变量绑定
### 取消 AI 生成
- AI 生成状态下，右侧显示「取消」按钮
- 点击后恢复为普通输入模式
### 限制说明
- JSON 类型参数暂不支持 AI 生成模式

参数由 AI 自动生成

## 保存与生效逻辑（非常重要）
### MCP 节点保存规则
- 每一个 MCP 节点 = 一个 MCP SERVER
- 点击保存时，系统会：
a.
根据当前配置自动生成 MCP SERVER
b.
将该 MCP SERVER 与当前 AI Agent 绑定
### 动作变更规则
- 若在 MCP 节点下新增 / 修改动作
- 在动作配置保存时：
  - 后端会对 对应 MCP SERVER 进行更新
  - 不会新建 Server，而是增量更新  。

## 使用建议（产品经验）
- 动作描述一定要写「业务语义」，而不是技术字段
- 参数能用 AI 生成的，优先使用 AI 生成
- 一个 MCP 节点建议控制在 同一业务域
- 不建议把完全无关的动作堆在同一个 MCP 下
