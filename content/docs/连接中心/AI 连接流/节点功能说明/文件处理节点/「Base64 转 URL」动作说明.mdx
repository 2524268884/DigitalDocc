## 功能概述
流程节点往往会产出 Base64 编码的文件内容（如 Excel、PDF、图片等），但下游系统或前端页面通常需要一个 可直接访问的文件 URL，用于：
- 下载文件
- 页面预览
- 作为第三方接口参数传递
- 在对话或流程中返回给用户
1. 需求范围
- 在 文件处理应用 下
- 新增一个独立 Action 节点
- 节点能力：
  - 输入 Base64 编码的文件内容
  - 输出一个 带有效期的文件访问 URL
1. 方案说明

### 各配置项详细说明
1️⃣ Base64 编码内容
- 组件类型：long-text
- 说明：
输入需要转换的 Base64 编码字符串
- 典型来源：
  - AI 节点生成的文件内容
  - OCR / 解析节点输出
  - 上游接口返回的 Base64 数据
2️⃣ 文件名称（非必填）
- 组件类型：text
- 说明：
用于指定最终生成文件的名称
- 说明补充：
  - 不填写时，系统可使用默认名称
  - 仅影响文件名，不影响文件内容
3️⃣ 文件类型
- 组件类型：text
- 是否必填：必填
- 说明：
指定生成文件的格式，例如：
```plaintext
xlsx
pdf
png
docx
```
文件类型将直接影响文件的解析与下载行为
4️⃣ 文件过期时间
- 组件类型：number
- 单位：秒
- 是否必填：必填
- 说明：
设置生成 URL 的最长有效时间
超过有效期后，该 URL 将无法继续访问，用于保障文件安全性
1. 节点输出说明
输出结果
- 文件访问 URL
  - 一个可通过 HTTP 直接访问的临时 URL
  - 在有效期内可用于：
■
浏览器下载
■
前端 iframe / link 打开
■
作为参数传递给下游接口或消息节点
1. 典型使用场景
- AI 生成 Excel / PDF → 转 URL → 返回给用户下载
- OCR / 解析文件 → 转换为标准文件 → 提供外链
- 流程中间态文件 → 作为参数传递给第三方系统
- Agent 对话中直接返回「文件下载链接」
