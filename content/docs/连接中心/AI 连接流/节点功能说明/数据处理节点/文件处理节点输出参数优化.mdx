## 功能概述

文件处理节点的输出参数现已优化，支持输出原始文件名（filename），不再强制转换为拼音或唯一标识。这使得在循环同步/异步场景下，用户能够更直观地识别处理文件的来源。

**优化内容：**

| 优化项 | 优化前 | 优化后 |
|-------|-------|-------|
| 输出文件名 | 自动转换为拼音或唯一 ID | 支持保留原始文件名 |
| 循环同步场景 | 文件名丢失，难以对应来源 | 循环内可追溯每个文件的原始名称 |
| 异步处理场景 | 无法通过文件名关联处理结果 | 异步返回结果中包含原始文件名 |

## 适用场景

| 场景 | 说明 |
|------|------|
| 循环处理多个文件 | 循环对数组中的文件进行处理，完成后可通过原始文件名关联每条处理结果 |
| 异步文件处理 | 提交异步处理任务后，返回的处理结果中包含原始文件名，便于后续业务逻辑关联 |
| 日志追溯 | 在同步日志中可根据文件名快速定位问题文件 |

## 技术说明

文件处理节点支持以下输出参数：

| 输出参数 | 说明 |
|---------|------|
| `originalName` | 原始文件名（本次新增） |
| `processedName` | 处理后的文件名（拼音/唯一 ID） |
| `filePath` | 文件存储路径 |
| `fileSize` | 文件大小（字节） |

在流程中引用原始文件名：

```text
{{$node['文件处理节点ID'].output.originalName}}
```

<Callout type="info">
原始文件名仅在目标存储支持中文文件名时可用。若目标存储强制要求拼音或唯一 ID，系统仍会使用转换后的文件名，此时 `originalName` 与 `processedName` 值相同。
</Callout>
