## 功能概述

**全局变量**是在数犀集成平台级别统一定义的键值对，所有连接流均可引用。当多条连接流需要使用相同的配置参数（如第三方系统的 API 地址、认证 Token、业务常量等）时，通过全局变量统一管理，避免在每条流程中重复填写，并在参数变更时只需修改一处即可全量生效。

**核心能力：**
- 平台级统一管理，一处修改、全局生效
- 支持字符串、数字、布尔等类型
- 支持加密存储敏感值（如 Token、密钥）
- 所有连接流均可通过统一语法引用

## 适用场景

| 场景 | 说明 |
|------|------|
| 跨流程共享 API 地址 | 第三方系统的 API Base URL 被多条流程使用，统一配置在全局变量中 |
| Token 集中管理 | 认证 Token 或 API Key 通过全局变量统一存储，避免散落在各流程中 |
| 环境切换 | 通过修改全局变量的值，快速切换测试环境和生产环境地址 |

## 配置全局变量

<Steps>
<Step>
### 步骤一：进入全局变量管理页

在数犀集成平台管理控制台中，进入 **连接中心 > 全局变量** 页面。
</Step>
<Step>
### 步骤二：新增变量

点击「新增变量」，填写以下配置：

| 配置项 | 类型 | 必填 | 说明 |
|--------|------|------|------|
| 变量名 | string | ✅ | 全局唯一标识，建议使用英文下划线命名，如 `crm_api_base_url` |
| 变量值 | string / number / boolean | ✅ | 变量的实际值 |
| 是否加密 | boolean | — | 开启后变量值将加密存储，页面不可见，适用于 Token、密钥等敏感信息 |
| 描述 | string | — | 说明该变量的用途，便于团队协作 |

<Callout type="warn">
加密变量一旦保存，原始值将不可查看，仅可覆盖更新。请在保存前确认值的正确性。
</Callout>
</Step>
<Step>
### 步骤三：保存变量

点击「保存」，变量立即生效，所有连接流均可引用。
</Step>
</Steps>

## 在连接流中引用全局变量

在任意连接流节点的配置字段中，使用以下语法引用全局变量：

```text
{{$global.变量名}}
```

**示例：**

```text
{{$global.crm_api_base_url}}/api/v1/users
```

<Callout type="info">
加密变量在运行时会自动解密后传入节点，但在配置预览和调试日志中不会显示原始值。
</Callout>

## 常见问题

<Accordions>
<Accordion title="全局变量修改后，正在执行的流程会立即使用新值吗？">
已在执行中的流程实例不受影响，将继续使用触发时的变量值。修改后触发的新实例才会使用新值。
</Accordion>
<Accordion title="不同租户之间的全局变量是否隔离？">
是的，全局变量按租户隔离，不同租户间完全独立，不会互相访问。
</Accordion>
</Accordions>
