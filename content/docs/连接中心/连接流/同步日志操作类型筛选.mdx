---

## 功能概述

专属集成平台的**通讻录同步详情日志**页面新增「操作类型」筛选条件，管理员可以按同步操作类型（创建/更新/删除等）快速过滤日志记录，提升问题排查效率。

**支持的操作类型：**

| 操作类型 | 说明 |
|---------|------|
| 创建 | 在目标系统中新建用户或部门 |
| 更新 | 更新目标系统中已有用户或部门的信息 |
| 删除 | 从目标系统中删除用户或部门 |
| 跳过 | 数据无变化，本次同步跳过未操作 |
| 失败 | 本次操作执行失败 |

## 使用步骤

<Steps>
<Step>
### 步骤一：进入同步详情日志

在专属集成平台管理控制台中，进入 **用户中心 > 通讻录同步**，选择目标同步连接器，点击「同步记录」查看历史同步任务列表。
</Step>
<Step>
### 步骤二：点击某次同步任务查看详情

在同步任务列表中，点击某次同步任务的「查看详情」，进入该次同步的详情日志页面。
</Step>
<Step>
### 步骤三：使用操作类型筛选

在详情日志页面的筛选栏中，点击「操作类型」下拉框，选择需要查看的操作类型。

<ImageZoom src="/images/sync/sync-log-operation-filter.png" alt="操作类型筛选界面" width={800} height={450} />

支持多选，同时筛选多种操作类型。
</Step>
<Step>
### 步骤四：查看过滤结果

筛选生效后，日志列表只展示符合所选操作类型的同步记录，可快速定位特定类型的操作结果。
</Step>
</Steps>

## 常见问题

<Accordions>
<Accordion title="筛选条件支持与时间范围筛选同时使用吗？">
支持。操作类型筛选与时间范围、状态等其他筛选条件可以叠加使用，多个条件取交集过滤。
</Accordion>
<Accordion title="「跳过」类型的记录说明什么？">
「跳过」表示该用户或部门的数据与上次同步时完全一致，系统判定无需更新，本次同步跳过了该条数据，这是正常现象。
</Accordion>
</Accordions>
