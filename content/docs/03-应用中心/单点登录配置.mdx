## 概述

单点登录(SSO)是专属集成平台的核心功能，允许用户通过一次身份验证访问多个应用系统，提高用户体验和安全性。本文档详细介绍如何配置和使用单点登录功能。

## 功能概述

平台单点登录具有以下特点：
- **统一认证入口**：为所有应用提供统一的认证登录界面
- **多协议支持**：支持SAML、OIDC、OAuth2.0等主流SSO协议
- **统一会话管理**：集中管理用户会话状态和超时策略
- **跨域访问支持**：解决不同域名应用之间的认证问题
- **安全防护**：防止会话劫持、CSRF等安全威胁

## 单点登录原理

### 标准模式

平台的标准单点登录流程如下：
1. 用户访问受保护的应用
2. 应用重定向到认证中心
3. 用户在认证中心完成身份验证
4. 认证中心生成令牌，并重定向回应用
5. 应用验证令牌有效性，授权用户访问

### 代理模式

对于无法改造的传统应用，平台提供代理模式：
1. 用户访问应用网关
2. 网关重定向到认证中心
3. 用户完成身份验证
4. 网关代理用户向后端应用发起请求
5. 网关可模拟表单登录或HTTP头传递身份信息

## 单点登录配置

### 基础配置

### 启用单点登录
1. 进入应用中心 \> 应用管理
2. 选择需要配置SSO的应用，点击"编辑"
3. 在"认证设置"选项卡中
4. 启用"单点登录"开关
5. 选择认证协议类型
6. 点击"保存"

### 认证协议选择

根据应用类型选择合适的认证协议：
- **SAML 2.0**：适用于传统企业应用，支持跨域SSO
- **OIDC**：适用于现代Web应用，提供更丰富的用户信息
- **OAuth 2.0**：适用于API授权场景
- **表单代理**：适用于无法改造的传统表单登录应用
- **标准HTTP代理**：适用于无法改造的HTTP应用

### 示例：SAML 2.0配置

参考文档：[单点登录-SAML协议](/docs/03-应用中心/应用单点登录/单点登录-SAML协议)

### 提供者配置
1. 选择SAML 2.0协议后，设置以下参数：
    - 服务提供者ID：应用的唯一标识符
    - ACS URL：断言消费服务URL
    - 断言加密：是否加密SAML断言
    - 签名算法：选择签名算法（RSA-SHA256推荐）
    - 摘要算法：选择摘要算法（SHA-256推荐）
2. 访问metadata地址"下载元数据"获取IdP元数据文件
3. 将元数据文件导入到应用的SAML配置中

### 属性映射

配置SAML断言中包含的用户属性：
1. 在"属性映射"区域
2. 添加需要的属性映射：
    - NameID：用户的唯一标识（如邮箱、用户名）
    - 姓名：用户姓名
    - 邮箱：用户邮箱
    - 部门：用户所属部门
    - 角色：用户角色信息
3. 点击"保存"完成配置

### 示例：OIDC配置

参考文档：[单点登录-OIDC](/docs/03-应用中心/应用单点登录/单点登录-OIDC)

### 客户端配置
1. 选择OIDC协议后，设置以下参数：
    - 客户端ID：系统自动生成的客户端标识
    - 客户端密钥：系统自动生成的密钥
    - 重定向URI：认证完成后的跳转地址
    - 授权类型：支持的授权流程类型
    - 作用域：请求的用户信息范围
2. 点击"生成密钥"重新生成客户端密钥
3. 复制配置信息到应用的OIDC设置中

### 令牌设置

配置OIDC令牌参数：
1. 在"令牌设置"区域
2. 设置以下参数：
    - 访问令牌有效期：如1小时
    - 刷新令牌有效期：如7天
    - ID令牌签名算法：如RS256
3. 点击"保存"完成配置

## 高级配置

配置用户会话参数：
1. 在"会话管理"选项卡
2. 设置以下参数：
    - 会话超时时间：如30分钟
    - 最大会话数：每个用户允许的并发会话数
    - 会话续期：是否允许自动续期会话
    - 强制重新认证：是否定期要求重新认证
3. 点击"保存"完成配置

### 多因子认证集成

与MFA集成增强安全性：
1. 在"多因子认证"选项卡
2. 启用"MFA集成"
3. 选择MFA触发策略：
    - 始终要求：每次登录都需要MFA
    - 风险感知：根据登录风险决定是否要求MFA
    - 特定用户：仅对特定用户要求MFA
4. 配置MFA流程：OTP、短信、移动认证等
5. 点击"保存"完成配置

## 应用场景

### 企业门户集成

将SSO集成到企业门户：
1. 配置企业门户应用的SSO设置
2. 设置门户为默认应用
3. 配置应用导航和权限
4. 用户登录门户后可直接访问其他已授权应用

### 移动应用SSO

为移动应用启用SSO：
1. 配置移动应用的OIDC设置
2. 在移动应用中集成SDK
3. 用户首次登录后获得令牌
4. 应用保存令牌用于后续访问
5. 令牌过期后自动刷新

### 跨域SSO

解决不同域名应用之间的SSO：
1. 为不同域名应用配置相同的SSO设置
2. 启用跨域Cookie支持或使用令牌传递
3. 用户在一个应用登录后，可无缝访问其他域名应用

## 最佳实践

### 安全性最佳实践

提高SSO安全性的建议：
- 使用TLS/SSL加密所有SSO通信
- 启用断言加密和签名验证
- 实施适当的会话超时策略
- 集成多因子认证
- 定期轮换密钥和证书
- 启用登录异常检测和阻止

### 性能最佳实践

优化SSO性能的建议：
- 使用分布式会话管理
- 优化令牌大小和有效期
- 实施合理的缓存策略
- 监控认证服务器负载
- 考虑部署地理分布的认证节点

### 用户体验最佳实践

改善SSO用户体验的建议：
- 提供直观的登录失败提示
- 实现"记住我"功能
- 优化移动设备上的登录体验
- 提供自助密码重置功能
- 减少登录页面加载时间

## 常见问题

<Accordions>
<Accordion title="常见登录问题">
**问题：用户无法通过SSO登录应用**

排查步骤：
1. 检查用户是否有访问应用的权限
2. 验证SSO配置是否正确
3. 检查浏览器Cookie和缓存
4. 查看认证日志中的错误信息
5. 验证网络连接是否正常

**问题：登录后出现"未授权"错误**

可能原因：
1. 用户属性映射配置错误
2. 应用授权设置问题
3. 令牌验证失败
4. 会话已过期
</Accordion>
<Accordion title="SAML相关问题">
**问题：SAML断言验证失败**

排查步骤：
1. 检查IdP和SP的时钟同步
2. 验证证书是否有效
3. 确认签名算法配置正确
4. 检查断言格式和内容

**问题：SAML响应包含无效的目标URL**

解决方法：
1. 检查ACS URL配置
2. 确认SP元数据中的ACS URL是否正确
3. 验证重定向URL是否在允许列表中
</Accordion>
<Accordion title="OIDC相关问题">
**问题：无效的重定向URI错误**

排查步骤：
1. 检查应用中配置的重定向URI
2. 确认URI格式正确且包含协议前缀
3. 验证重定向URI是否在允许列表中

**问题：令牌验证失败**

解决方法：
1. 检查令牌签名算法配置
2. 验证应用使用的公钥是否正确
3. 确认令牌未过期
4. 检查令牌的audience值是否正确
</Accordion>
