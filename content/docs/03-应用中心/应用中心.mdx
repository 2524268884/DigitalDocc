## 概述

应用中心是专属集成平台的核心模块之一，为企业提供统一的应用管理、发布和访问控制能力。通过应用中心，您可以集中管理企业内的各类应用，实现单点登录、应用分发和安全访问控制，为员工提供一站式应用门户。
> **功能概览**：应用中心整合了应用管理、应用市场、ABM应用分发和企业应用分发等功能，支持CAS、OIDC、OAuth2和SAML四种标准协议以及定制化单点登录，满足企业多样化的应用集成需求。

## 快速入门

新用户可以按照以下步骤快速上手应用中心功能：
1. **添加应用**：从应用市场选择预配置应用，或创建自定义应用
2. **配置单点登录**：根据应用类型选择合适的单点登录协议
3. **设置授权范围**：在应用授权管理中配置可访问用户
4. **发布应用**：将配置好的应用发布到用户门户，供员工使用

## 应用接入流程

### 应用市场添加流程
1. 进入应用中心 \> 应用市场
2. 搜索并选择需要的应用
3. 点击"添加"按钮
4. 配置应用参数
5. 设置授权范围
6. 发布应用

### 自定义应用创建流程
1. 进入应用中心 \> 应用管理
2. 点击"新建应用"
3. 填写应用基本信息
4. 配置单点登录参数
5. 设置授权范围
6. 测试并发布应用

## 常见应用场景

### 企业应用门户

通过应用中心构建统一的企业应用门户，员工可以一次登录，访问所有授权应用。

**配置步骤**：
1. 在应用市场中添加常用企业应用
2. 配置各应用的单点登录参数
3. 设置应用访问权限
4. 配置应用门户样式和分类

### 内网应用外部访问

借助[应用网关](/docs/03-应用中心/应用网关)
功能，安全地将内网应用发布到外网，支持远程办公场景。

**配置步骤**：
1. 创建应用并配置应用网关
2. 设置内网域名和外网访问域名
3. 配置访问授权范围
4. 启用多因素认证提升安全性

### 移动应用管理

结合[企业应用分发](/docs/03-应用中心/企业应用分发)
和[ABM应用分发](/docs/03-应用中心/ABM应用分发)
功能，集中管理和分发企业移动应用。

**配置步骤**：
1. 上传企业移动应用安装包
2. 配置应用版本和描述信息
3. 设置应用分发范围
4. 向员工推送应用安装

## 常见问题

<Accordions>
<Accordion title="单点登录配置问题">
**Q: 配置单点登录后，跳转到应用时报错怎么办？**A: 常见原因包括重定向URI配置错误、协议参数不匹配或用户未授权。请检查单点登录配置参数，特别是回调地址和客户端ID/密钥是否正确。

**Q: 如何选择合适的单点登录协议？**A: 建议根据应用类型选择：现代Web应用优先选择OIDC，传统企业应用选择SAML，API授权场景选择OAuth2，Java应用可考虑CAS。
</Accordion>
<Accordion title="应用授权问题">
**Q: 用户无法看到已授权的应用怎么办？**A: 检查用户是否在应用的授权范围内，确认应用已发布且状态为启用。如授权基于角色，也需检查用户是否拥有相应角色。

**Q: 如何实现基于部门的应用授权？**A: 在[应用授权管理](/docs/03-应用中心/应用授权管理)
中，选择"组织架构授权"方式，然后选择需要授权的部门即可。
</Accordion>
<Accordion title="应用网关问题">
**Q: 配置应用网关后无法访问内网应用怎么办？**A: 检查代理网关是否能正常连接内网应用，可能需要配置请求头重写（如Host头）或检查网络连通性。详见[应用网关](/docs/03-应用中心/应用网关)
文档。

**Q: 如何实现部分路径匿名访问？**A: 在应用网关的高级配置中，通过正则表达式设置匿名访问路径，如`/login`或`/static/.*`。
</Accordion>
</Accordions>

## 最佳实践

### 安全性建议
1. 为关键应用启用多因子认证
2. 定期审计应用访问权限
3. 使用HTTPS协议保护数据传输
4. 设置合理的会话超时时间
5. 实施最小权限原则，精确控制应用访问范围

### 性能优化建议
1. 合理设置令牌缓存和有效期
2. 使用统一的会话管理减少重复认证
3. 将常用应用添加到快捷访问区
4. 根据访问频率优化应用分类和展示顺序
