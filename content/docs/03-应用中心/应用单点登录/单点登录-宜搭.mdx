## 概述

本文档详细介绍如何配置企业微信与宜搭平台之间的单点登录功能，通过专属集成平台实现用户身份的统一认证和无缝访问体验。通过此配置，用户可以从企业微信直接访问宜搭应用，无需重复登录，大幅提升用户体验和工作效率。

## 配置流程

配置企业微信单点登录宜搭主要分为以下几个步骤：
1. 专属集成平台应用配置
2. 宜搭平台单点登录设置
3. 企业微信自建应用配置
4. 用户关联设置

**参数说明**：

| 参数 | 说明 |
|------|------|
| Client ID | 填写专属集成平台生成的Client ID |
| Client Secret | 填写专属集成平台生成的Client Secret |
| Authorize URL | 专属集成平台租户的域名:端口\+/iam/authorize（注意：使用实际租户域名，不是代理域名） |
| Token URL | 专属集成平台租户的域名:端口\+/iam/token（注意：使用实际租户域名，不是代理域名） |
| Redirect URL | 填写在专属集成平台中配置的宜搭工作台地址 |
| Public Key | 从OIDC服务发现端点获取的公钥信息 |

## 企业微信单点登录宜搭

**5.1 企业微信自建应用配置**

参考文档：

[企业微信开放平台创建应用](/docs/01-快速开始/企业微信开放平台创建应用)

### 应用首页设置

在企业微信自建应用中设置应用首页地址，使用如下格式：
```text
https://<您的域名>/iam/authorize?response_type=code&tcode=【***】&client_id=【***】&scope=openid%20profile%20email%20address%20phone&redirect_uri=【****】
```

**参数说明**：
- client\_id：专属集成平台生成的Client ID
- redirect\_uri：专属集成平台中配置的宜搭应用地址
- tcode：专属集成平台租户的名称，例如tcode=bit
- 您的域名：需要 Nginx 代理域名

## 常见问题与排障

### 单点登录失败的常见原因
1. **sub值不匹配**：
    - 检查专属集成平台中的属性值和声明名称配置
    - 确认宜搭中配置的用户标识字段与专属集成平台返回的sub值一致
2. **Redirect URI不匹配**：
    - 确保专属集成平台中配置的Redirect URI与宜搭中设置的完全一致
    - 注意检查URL中的大小写和特殊字符
3. **用户未关联**：
    - 验证企业微信用户是否已与专属集成平台用户关联
    - 检查关联的账号状态是否正常

### 配置注意事项
1. **域名使用**：
    - Authorize URL和Token URL必须使用实际的租户域名，不是代理域名
    - 确保网络环境允许访问这些域名和端口
2. **Public Key更新**：
    - 如果遇到Token验证失败问题，尝试重新获取并更新Public Key
    - 注意移除jwks返回结果中的所有空格
3. **参数编码**：
    - 确保URL中的特殊字符被正确编码
    - 特别注意redirect\_uri参数的编码方式

## 附录

### 参数对照表

| 专属集成平台 | 宜搭平台 | 说明 |
|------------|------------|------|
| Client ID | Client ID | 应用唯一标识 |
| Client Secret | Client Secret | 应用密钥 |
| /iam/authorize | Authorize URL | 授权端点 |
| /iam/token | Token URL | 令牌端点 |
| RedirectURIs | Redirect URL | 回调地址 |

### 术语解释
- **OIDC**：OpenID Connect，基于OAuth 2.0的身份认证协议
- **jwks\_uri**：JSON Web Key Set的URI，用于获取验证Token签名的公钥
- **sub**：Subject的缩写，表示用户的唯一标识
- **tcode**：租户代码，用于识别特定的租户实例
