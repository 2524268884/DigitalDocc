## 这组接口适合做什么

组织架构同步、部门初始化、按部门维度查询成员范围。

<Callout>
本组共收录 **7** 个接口操作，推荐先阅读下面的调用顺序，再进入具体接口页。
</Callout>

## 推荐调用顺序

<Steps>
  <Step>

先用“获取部门列表”确认当前组织树结构。

  </Step>
  <Step>

需要新增或调整结构时，再调用创建、修改或删除部门接口。

  </Step>
  <Step>

如果只需要按条件定位部门，优先使用过滤部门接口。

  </Step>
</Steps>

## 全部接口

| 接口 | 方法 | 路径 | 说明 |
| --- | --- | --- | --- |
| [获取部门列表](/docs/08-API使用手册/API参考/组织与部门/获取部门列表) | `GET` | `/iam/api/orgs` | 通常作为组织架构联调的第一条验证接口。 |
| [创建部门](/docs/08-API使用手册/API参考/组织与部门/创建部门) | `POST` | `/iam/api/orgs` | 用于组织初始化或从外部系统同步新部门。 |
| [获取部门详情](/docs/08-API使用手册/API参考/组织与部门/获取部门详情) | `GET` | `/iam/api/orgs/{org_id}` | 适合在更新前做详情核验，或用于单条问题排查。 |
| [修改部门](/docs/08-API使用手册/API参考/组织与部门/修改部门) | `PATCH` | `/iam/api/orgs/{org_id}` | 用于调整组织名称、属性或结构信息。 |
| [获取下级部门](/docs/08-API使用手册/API参考/组织与部门/获取下级部门) | `GET` | `/iam/api/orgs/{org_id}/subs` | 适合按树结构逐层展开组织数据。 |
| [过滤部门](/docs/08-API使用手册/API参考/组织与部门/过滤部门) | `POST` | `/iam/api/orgs/search` | 适合定向查询部门而不是拉取整棵组织树。 |
| [删除部门](/docs/08-API使用手册/API参考/组织与部门/删除部门) | `DELETE` | `/iam/api/orgs/{org_id}` | 执行前建议确认下级部门与关联成员的处置策略。 |
