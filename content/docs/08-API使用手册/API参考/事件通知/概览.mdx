## 这组接口适合做什么

连接器回调、下游系统通知、事件驱动的数据同步。

<Callout>
本组共收录 **1** 个接口操作，推荐先阅读下面的调用顺序，再进入具体接口页。
</Callout>

## 推荐调用顺序

<Steps>
  <Step>

确认目标租户和连接器标识正确。

  </Step>
  <Step>

使用路径中的 apiKey 发起事件通知请求。

  </Step>
  <Step>

如果回调未生效，先核对连接器状态和事件日志。

  </Step>
</Steps>
