## 这组接口适合做什么

角色初始化、按应用查询角色、为用户批量授予角色。

<Callout>
本组共收录 **10** 个接口操作，推荐先阅读下面的调用顺序，再进入具体接口页。
</Callout>

## 推荐调用顺序

<Steps>
  <Step>

先查询角色列表或详情，确认目标角色是否已存在。

  </Step>
  <Step>

角色定义完成后，再为用户批量添加或移除角色。

  </Step>
  <Step>

按应用或按用户排查授权时，可使用两个角色查询接口辅助定位。

  </Step>
</Steps>

## 全部接口

| 接口 | 方法 | 路径 | 说明 |
| --- | --- | --- | --- |
| [查询角色列表](/docs/08-API使用手册/API参考/角色与标签/查询角色列表) | `POST` | `/iam/api/tags/search` | 适合角色排查、初始化校验和同步对账。 |
| [创建角色](/docs/08-API使用手册/API参考/角色与标签/创建角色) | `POST` | `/iam/api/tags` | 通常作为角色初始化的第一步。 |
| [获取角色详情](/docs/08-API使用手册/API参考/角色与标签/获取角色详情) | `GET` | `/iam/api/tags/{id}` | 适合在修改或排障前核对角色当前状态。 |
| [修改角色](/docs/08-API使用手册/API参考/角色与标签/修改角色) | `PUT` | `/iam/api/tags/{id}` | 适合调整角色名称、属性和配置。 |
| [删除角色](/docs/08-API使用手册/API参考/角色与标签/删除角色) | `DELETE` | `/iam/api/tags/{id}` | 删除前建议先确认角色是否仍被用户引用。 |
| [查看角色下的用户](/docs/08-API使用手册/API参考/角色与标签/查看角色下的用户) | `GET` | `/iam/api/tags/{id}/users` | 适合排查“角色绑定到了哪些人”。 |
| [给用户添加角色](/docs/08-API使用手册/API参考/角色与标签/给用户添加角色) | `POST` | `/iam/api/tags/{id}/users` | 适合批量授权和角色分配。 |
| [删除用户角色](/docs/08-API使用手册/API参考/角色与标签/删除用户角色) | `DELETE` | `/iam/api/tags/{id}/users` | 适合回收授权和批量解绑角色。 |
| [根据应用与用户获取角色列表](/docs/08-API使用手册/API参考/角色与标签/根据应用与用户获取角色列表) | `GET` | `/iam/api/tags/getTagList` | 适合校验用户在某个应用下的角色绑定结果。 |
| [获取用户角色信息](/docs/08-API使用手册/API参考/角色与标签/获取用户角色信息) | `GET` | `/iam/api/tags/getTagList/user` | 适合从用户视角排查授权结果。 |
