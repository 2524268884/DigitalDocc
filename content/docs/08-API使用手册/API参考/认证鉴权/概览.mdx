## 这组接口适合做什么

首次接入、联调前准备、Token 刷新策略确认。

<Callout>
本组共收录 **2** 个接口操作，推荐先阅读下面的调用顺序，再进入具体接口页。
</Callout>

## 推荐调用顺序

<Steps>
  <Step>

准备应用的 client_id 与 client_secret。

  </Step>
  <Step>

优先调用请求体方式的 Token 接口获取 access_token。

  </Step>
  <Step>

将 access_token 写入后续业务接口的 Authorization 头。

  </Step>
</Steps>
