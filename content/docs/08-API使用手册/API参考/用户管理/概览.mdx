## 这组接口适合做什么

账号同步、用户主数据维护、批量排查用户状态。

<Callout>
本组共收录 **8** 个接口操作，推荐先阅读下面的调用顺序，再进入具体接口页。
</Callout>

## 推荐调用顺序

<Steps>
  <Step>

联调阶段先用查询用户确认账号是否已存在。

  </Step>
  <Step>

新增账号走创建用户，已有账号的资料调整走修改用户。

  </Step>
  <Step>

状态变更、密码重置、逻辑删除分别使用对应的专用接口。

  </Step>
</Steps>

## 全部接口

| 接口 | 方法 | 路径 | 说明 |
| --- | --- | --- | --- |
| [查询用户](/docs/08-API使用手册/API参考/用户管理/查询用户) | `GET` | `/iam/api/users` | 建议作为用户联调的首条验证接口。 |
| [创建用户](/docs/08-API使用手册/API参考/用户管理/创建用户) | `POST` | `/iam/api/users` | 适合主数据同步、新用户开户和初始化导入。 |
| [过滤查询用户](/docs/08-API使用手册/API参考/用户管理/过滤查询用户) | `POST` | `/iam/api/users/search` | 适合批量排查用户属性或按条件做同步校验。 |
| [获取用户详情](/docs/08-API使用手册/API参考/用户管理/获取用户详情) | `GET` | `/iam/api/user` | 适合单账号核验、更新前校验和问题定位。 |
| [修改用户](/docs/08-API使用手册/API参考/用户管理/修改用户) | `PATCH` | `/iam/api/user` | 用于更新用户主数据而不是状态变更。 |
| [启用或禁用用户](/docs/08-API使用手册/API参考/用户管理/启用或禁用用户) | `PUT` | `/iam/api/users/status` | 适合账号冻结、恢复和批量状态治理场景。 |
| [修改用户密码](/docs/08-API使用手册/API参考/用户管理/修改用户密码) | `POST` | `/iam/api/users/password` | 适合首次初始化、人工重置或迁移后的密码矫正。 |
| [删除用户](/docs/08-API使用手册/API参考/用户管理/删除用户) | `POST` | `/iam/api/users/delete` | 执行前建议核对租户内关联关系和后续恢复策略。 |
