## 概述

数犀集成平台默认提供五种管理员角色：系统管理员、应用管理员、账号管理员、CLI应用角色、普通用户角色。以上默认角色不可被删除，但可以添加用户、编辑角色信息。

<Callout type="warn">
系统默认角色不可被删除，但可以添加用户、编辑角色信息。
</Callout>

## 默认角色说明

| 角色名称 | 权限范围 | 是否可删除 | 主要职责 |
|---------|---------|-----------|---------|
| 系统管理员 | 全部系统功能 | 否 | 管理整个系统 |
| 应用管理员 | 应用管理相关 | 否 | 应用上架、配置和授权 |
| 账号管理员 | 用户账号管理 | 否 | 用户创建、修改和授权 |
| CLI应用角色 | CLI命令操作 | 否 | 命令行接口操作 |
| 普通用户角色 | 基本应用访问 | 否 | 访问被授权的应用 |

## 操作步骤

### 步骤 1：进入管理员角色页面

在左侧导航栏选择 **权限中心** → **管理员角色**。

### 步骤 2：添加自定义角色

1. 点击「添加自定义角色」
2. 编辑角色名称、显示名称、描述
3. 指定该角色可控制的能力模块
4. 点击「确定」保存

### 步骤 3：添加管理员

1. 点击列表操作列中的「添加」
2. 在对应角色下添加管理员

### 步骤 4：编辑角色

- 点击列表操作列的「分配」：可编辑默认角色的角色信息，但不可修改权限组
- 点击列表操作列的「编辑」：可编辑自定义角色的角色信息，并修改权限组

### 步骤 5：其他操作

- **删除**：可删除自定义角色
- **查看**：查看管理员的详细信息
- **拷贝**：复制一个权限一样的管理员角色

## 用户权限级别

系统支持四种用户权限级别：

| 权限类型 | 可执行操作 | 限制说明 |
|---------|-----------|---------|
| 查看权限 | 查看和导出数据 | 不能修改数据或产生更新操作 |
| 新建权限 | 查看和创建数据 | 不能编辑已有数据，包括自己创建的数据 |
| 编辑权限 | 查看和修改数据 | 不能创建新数据 |
| 删除权限 | 查看和删除数据 | 不能编辑数据或创建新数据 |

### 权限详细说明

- **查看权限**：登录后只能查看和导出数据，不能进行任何数据的修改或者产生实际数据更新的操作。用户管理：只能查看列表用户列表，部门列表，搜索。列表左上角的操作菜单全部隐藏，包括创建、激活邀请、拖拽部门等所有操作都不可见。连接中心的上线/发布/下线等输入编辑权限；
- **新建权限**：登录后查看&创建，能查看菜单下的数据，能创建，但不能编辑已有数据，包括自己创建的数据，也不能编辑。以「应用管理」菜单为例，只能创建应用，从应用市场安装应用；
- **编辑权限**：登录后查看&编辑，能查看权限菜单下有权限的数据范围，能对数据进行修改，但不能创建新的数据；
- **删除权限**：登录后查看&删除，能对有权限查看数据范围进行删除操作，但不能编辑数据，也不能创建新的数据；

### 权限管理案例

场景描述：李四用户有 B 部门的查看权限，C 部门的所有增删改查的权限，A 部门的创建权限。张三在 ABC 三个部门中有不同的职位信息，1 人多职。

- 李四登录后，对张三进行用户属性内容的修改，可进行任意修改，因为李四有 C 部门的所有权限
- 李四登录后，修改张三 A 部门的主部门改成否，把 C 或 B 部门改成是，可进行修改
- 李四登录后，打开张三的详情页，把部门 A 给删除掉，把部门 B 进行部门职位、部门主管、部门工号编辑替换或者删除，都不可以
- 李四登录后，打开张三的详情页，把部门 C 给进行删除，或者编辑部门职位、部门主管、部门工号等信息，都可以进行操作，李四把张三的 C 部门删除后，在用户列表中只能查看张三信息，不能进行编辑删除操作
- 李四登录后，打开张三的详情页，张三把 A 部门作为主部门，同时部门工号写的 456，此时修改用户属性中的工号从 456 改为 123 并进行保存，可以修改保存
- 李四登录后，在用户列表页，选择 ABC 三个部门下的不同的用户，进行批量操作，启用、禁用、激活邀请、密码重置等功能，根据不同的权限给出不同的提示，批量操作中混杂 AB 两个部门的用户，此时只有 C 部门的用户可以操作成功，其他用户都需要给出 1 个统一的提示，无权限进行操作
- 李四登录后，在用户列表页，删除张三，张三会被永远删除

## 常见问题

<Accordions>
<Accordion title="系统默认角色可以删除吗？">

不可以。系统默认角色（系统管理员、应用管理员、账号管理员、CLI应用角色、普通用户角色）不可被删除。
</Accordion>
<Accordion title="「分配」和「编辑」有什么区别？">

点击「分配」可编辑默认角色的角色信息，但不可修改权限组；点击「编辑」可编辑自定义角色的角色信息，并修改权限组。
</Accordion>
</Accordions>
