## 概述

消息发送日志统一记录数犀集成平台发出的所有通知消息，分为三类：**短信**、**邮件**和**企业消息**（钉钉/飞书工作通知）。通过此模块，管理员可追踪消息发送状态，排查消息未收到等问题。

## 发送状态说明

| 状态 | 含义 |
|------|------|
| 发送中 | 消息已提交，等待通道处理 |
| 成功 | 消息已成功投递 |
| 失败 | 发送失败，可查看具体原因 |
| 无效用户 | 接收者不存在、已禁用，或接收信息（手机号/邮箱）无效 |
| 禁止发送 | 触发发送频率限制或黑名单规则 |

## 查看短信发送日志

<Steps>
<Step>
### 进入页面

在左侧导航栏选择**审计日志 > 消息发送日志**，切换到「短信发送日志」标签页。
</Step>
<Step>
### 设置筛选条件

| 筛选项 | 说明 |
|--------|------|
| 接收者 | 输入手机号精确搜索 |
| 模板类型 | 短信验证码 / 激活邀请 / 密码修改通知 / 密码重置通知 / 管理员重置密码通知 / 密码过期提醒 |
| 时间范围 | 自定义开始和结束时间 |
| 发送状态 | 发送中 / 成功 / 无效用户 / 禁止发送 / 失败 |

</Step>
<Step>
### 查看结果并导出

点击「搜索」查看结果，点击「导出」将当前筛选结果导出为 Excel 文件。
</Step>
</Steps>

## 常见问题

<Accordions>
<Accordion title="用户反馈没有收到验证码短信，如何排查？">
1. 在短信发送日志中按手机号搜索
2. 查看该条记录的发送状态：
   - **失败**：查看失败原因，可能是短信网关配置问题
   - **无效用户**：检查用户手机号是否正确绑定
   - **禁止发送**：检查是否触发了频率限制（短时间内多次请求）
   - **发送中**：等待一段时间后刷新查看
3. 若状态为「成功」但用户未收到，可能是运营商拦截，建议用户检查拦截短信箱
</Accordion>
<Accordion title="企业消息显示「无效用户」是什么原因？">
「无效用户」表示以下几种情况之一：
- 接收者在数犀集成平台不存在或已被停用
- 用户未绑定钉钉/飞书账号
- 用户的钉钉/飞书账号与集成平台账号未正确关联
</Accordion>
<Accordion title="日志保存多长时间？">
系统默认保存最近 **180 天**的消息发送日志。如需调整，请在系统设置中修改保存期限。
</Accordion>
</Accordions>
