## 概述

用户变更日志记录数犀集成平台中每位用户的完整变更历史，包括通过管理后台、通讻录集成、OpenAPI 发起的新增、修改、删除操作，是数据审计和问题溯源的核心工具。

## 日志字段说明

| 字段 | 说明 |
|------|------|
| 用户 ID | 用户唯一标识 |
| 用户名 | 用户账号 |
| 变更类型 | 新增 / 修改 / 删除 |
| 变更状态 | 成功 / 失败 |
| 操作者 | 执行变更的管理员账号或「通讻录集成」 |
| 来源 | 后台操作 / Open API / 集成名称 |
| 变更时间 | 变更发生的具体时间 |

## 查看变更详情

点击日志列表中某行左侧的展开按钮，可查看该次变更的详细信息：

| 变更类型 & 状态 | 展示内容 |
|--------------|---------|
| 新增成功 | 全部非空用户属性（字段名-字段值） |
| 修改成功 | 变更字段的修改前 / 修改后对比表格 |
| 删除成功 | 删除类型（如：直接删除） |
| 任意失败 | 失败原因说明 |

## 筛选日志

<Steps>
<Step>
### 设置时间范围

在页面顶部使用日期范围筛选器，选择要查询的变更时间段。
</Step>
<Step>
### 使用高级筛选

点击「高级筛选」按钮，在弹窗中配置以下条件：

| 筛选项 | 可选值 |
|--------|-------|
| 更新来源 | 全部 / 后台操作 / 通讻录集成名称（含已删除）|
| 操作状态 | 全部 / 新增 / 修改 / 删除 |
| 用户名 | 输入关键词 |
| 姓名 | 输入关键词（关联当前用户名）|
| 手机号 | 精确匹配 |
| 邮箱 | 精确匹配 |

</Step>
<Step>
### 应用或重置条件

点击「确定」应用筛选条件，点击「重置」清除所有条件。

<Callout type="info">
设置筛选条件后，「高级筛选」按钮会高亮显示，提示当前有活动筛选。
</Callout>
</Step>
</Steps>

## 排序与导出

**排序**：点击列表头部的「最后更新时间」列，可切换升序 / 降序排列。

**导出**：

| 场景 | 操作 |
|------|------|
| 导出全量数据 | 不设筛选条件，点击「导出全部数据」|
| 导出筛选结果 | 设置筛选条件后，点击「导出筛选数据（N 条）」|

<Callout type="warn">
筛选数据导出限制最多 **2 万行**，超出时请缩小时间范围或增加筛选条件分批导出。
</Callout>

## 常见问题

<Accordions>
<Accordion title="如何区分是谁触发了某次用户变更？">
查看「操作者」和「来源」字段：
- 操作者显示具体管理员账号名，说明是后台手动操作
- 操作者显示「通讻录集成」，说明是自动化同步触发
- 来源为「Open API」，说明是通过接口调用触发
</Accordion>
<Accordion title="为什么某次修改看不到变更前的数据？">
修改成功的记录会展示前后值对比表格。若看不到，可能该次操作实际未发生字段变更（如传入了相同的值），或属于批量操作中的部分记录，建议结合接口调用日志排查。
</Accordion>
</Accordions>
