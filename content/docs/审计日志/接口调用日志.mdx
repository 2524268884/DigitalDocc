## 概述

接口调用日志记录数犀集成平台所有开放平台接口及通讻录集成时的 API 调用情况，帮助管理员监控数据推送状态、定位同步异常，是 IT 运维的核心排障工具。

## 日志字段说明

| 字段 | 说明 |
|------|------|
| 调用时间 | 接口发起调用的时间 |
| 日志 ID | 调用唯一标识 |
| 来源系统 | 发起调用的上游系统或集成名称 |
| 接口类型 | 用户/部门/角色的新增、更新、删除等 |
| 耗时 | 接口响应时间（ms） |
| 状态码 | HTTP 状态码 |
| 状态 | 成功 / 失败 |

点击每行左侧的展开按钮，可查看完整的**请求报文**和**返回报文**。

## 筛选日志

<Steps>
<Step>
### 设置时间范围

页面顶部日期筛选器默认显示**今日**数据，可自定义查询时间段。
</Step>
<Step>
### 使用高级筛选

点击「高级筛选」按钮，配置以下条件：

| 筛选项 | 可选值 |
|--------|-------|
| 调用来源 / 同步系统 | 全部 / 具体集成或同步名称（含已删除）|
| 调用状态 | 全部 / 成功 / 失败 |
| 接口类型 | 用户新增 / 更新 / 删除、部门新增 / 更新 / 删除、角色新增 / 更新 / 删除 |
| 请求内容搜索 | 关键词全文匹配请求报文 |
| 响应内容搜索 | 关键词全文匹配响应报文 |

</Step>
<Step>
### 应用或重置

点击「确定」应用筛选，点击「重置」清除所有条件。
</Step>
</Steps>

<Callout type="info">
**排查技巧**：对于集成同步失败问题，可先按「调用状态 = 失败」筛选，再展开具体记录查看返回报文中的错误信息，通常能直接定位问题原因。
</Callout>

## 排序与导出

**排序**：点击「接口调用时间」列标题，切换升序 / 降序。

**导出**：

| 场景 | 操作 |
|------|------|
| 导出全量数据 | 不设筛选，点击「导出全部数据」|
| 导出筛选结果 | 设置筛选后，点击「导出筛选数据（N 条）」|

## 常见问题

<Accordions>
<Accordion title="通讻录集成同步后看不到接口调用日志？">
检查以下几点：
1. 筛选「来源系统」是否选择了对应的集成名称
2. 时间范围是否覆盖了同步执行的时间（默认今日，注意同步任务可能是昨日执行的）
3. 同步任务是否实际执行成功（到通讻录集成页面查看任务状态）
</Accordion>
<Accordion title="接口返回状态码 403 是什么原因？">
403 通常表示**权限不足**。可能原因：
- M2M 应用的字段权限未正确配置，目标字段不在允许操作列表中
- Token 对应的应用未被授权操作该接口
- 请参考 [M2M 接口调用权限配置](/docs/应用中心/M2M接口授权/接口调用权限配置说明)
</Accordion>
<Accordion title="日志中的「请求报文」看不到敏感字段数据？">
系统对手机号、邮箱等敏感字段进行了脱敏处理，日志中仅展示掩码后的值，这是正常的安全保护机制。
</Accordion>
</Accordions>
