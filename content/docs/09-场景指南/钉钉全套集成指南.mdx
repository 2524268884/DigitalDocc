## 概述

> 本文介绍如何在专属集成平台完成钉钉的全套集成配置，实现通讯录同步、认证源配置、单点登录和消息推送。

通过本指南，您将完成以下四个核心场景的完整配置：

| 场景 | 说明 | 配置位置 |
|------|------|----------|
| 通讯录集成 | 将钉钉用户同步到专属集成平台 | 用户中心 > 通讯录集成 |
| 认证源配置 | 使用钉钉账号登录专属集成平台 | 用户中心 > 认证源管理 |
| 单点登录 | 通过钉钉实现应用 SSO | 应用中心 > 应用管理 |
| 消息推送 | 向钉钉发送通知消息 | 参数配置 > 企业消息配置 |

## 前提条件

- 已开通专属集成平台管理员账号
- 已注册钉钉企业账号并开通企业内部应用
- 拥有钉钉应用管理权限
- 准备测试用户账号

## 第一部分：钉钉开放平台配置

### 步骤 1.1：创建钉钉应用

1. 登录[钉钉开放平台](https://open-dev.dingtalk.com/)
2. 进入「应用开发」>「企业内部开发」
3. 点击「创建应用」
4. 选择应用类型（建议选择「H5微应用」或「企业内部应用」）
5. 填写应用信息：
   - 应用名称：如「专属集成平台」
   - 应用描述：企业内部集成应用
6. 点击「创建」

### 步骤 1.2：获取应用凭证

1. 在应用详情页，点击「基础信息」
2. 记录以下信息：
   - `AppKey`：应用的唯一标识
   - `AppSecret`：应用的身份密钥
3. 点击「查看」获取完整密钥

### 步骤 1.3：配置权限

1. 进入「权限管理」
2. 开通以下必要权限：
   - `成员信息读权限` - 读取用户基本信息
   - `部门信息读权限` - 读取组织架构
   - `企业群通知` - 发送工作通知
3. 点击「申请开通」

### 步骤 1.4：配置回调地址

1. 进入「开发管理」>「消息订阅」
2. 配置服务器地址（SAAS平台提供）
3. 配置事件订阅：
   - `用户进入应用`
   - `用户信息变更`

## 第二部分：通讯录集成配置

### 步骤 2.1：创建通讯录集成任务

1. 登录专属集成平台，进入 **用户中心 > 通讯录集成**
2. 点击 **「新建通讯录集成」**
3. 选择身份源类型：「钉钉」
4. 填写连接信息：
   - AppKey：填入钉钉应用的 AppKey
   - AppSecret：填入钉钉应用的 AppSecret
5. 点击「测试连接」确认连接成功

### 步骤 2.2：配置字段映射

1. 在字段映射配置页面
2. 配置钉钉字段到平台字段的对应关系：

| 钉钉字段 | 平台字段 | 映射方式 |
|----------|----------|----------|
| userid | username | 创建且更新 |
| name | display_name | 创建且更新 |
| mobile | phone_number | 创建且更新 |
| email | email | 创建且更新 |
| department | department | 创建且更新 |

3. 点击「下一步」

### 步骤 2.3：配置同步策略

1. 设置同步频率（建议首次同步选择「立即执行」）
2. 配置冲突处理策略：
   - 「最新优先」或「身份源优先」
3. 点击「保存并启动」

### 步骤 2.4：验证同步结果

1. 进入「通讯录集成」列表
2. 查看同步任务状态
3. 点击「同步日志」查看详细信息
4. 进入「用户管理」验证用户是否已同步

## 第三部分：认证源配置

### 步骤 3.1：创建钉钉认证源

1. 进入 **用户中心 > 认证源管理**
2. 点击 **「新建身份源」**
3. 选择「企业认证源」>「钉钉」
4. 填写认证源信息：
   - 名称：如「钉钉认证源」
   - AppKey：填入钉钉应用的 AppKey
   - AppSecret：填入钉钉应用的 AppSecret
5. 点击「保存」

### 步骤 3.2：配置账号关联

1. 在认证源详情页，点击「账号关联」
2. 配置映射关系：
   - 钉钉 userid → 平台 username
   - 钉钉手机号 → 平台 phone_number
3. 配置 JIT（Just-In-Time）设置：
   - 开启「无匹配用户自动创建」
   - 选择默认部门
4. 点击「保存」

### 步骤 3.3：测试认证

1. 打开新的浏览器窗口
2. 访问专属集成平台登录页
3. 点击「使用钉钉账号登录」
4. 使用测试账号完成登录
5. 验证是否成功登录

## 第四部分：单点登录配置

### 步骤 4.1：创建 SSO 应用

1. 进入 **应用中心 > 应用管理**
2. 选择已创建的应用或新建应用
3. 点击「编辑」
4. 进入「单点配置」标签

### 步骤 4.2：启用 SSO

1. 启用「单点登录」开关
2. 选择认证协议（建议选择 OIDC）
3. 配置钉钉 SSO 信息：
   - AppKey：填入钉钉 AppKey
   - AppSecret：填入钉钉 AppSecret
4. 配置回调地址（钉钉应用中配置）

### 步骤 4.3：配置应用权限

1. 进入「应用授权」标签
2. 选择可访问用户范围：
   - 「全部成员」
   - 「指定部门」
   - 「指定角色」
3. 点击「保存」

### 步骤 4.4：测试 SSO

1. 使用已同步的钉钉用户账号
2. 在应用登录页点击「钉钉 SSO」
3. 完成钉钉授权
4. 验证是否成功登录目标应用

## 第五部分：消息推送配置

### 步骤 5.1：配置企业消息

1. 进入 **参数配置 > 品牌设置 > 企业消息配置**
2. 启用「企业消息服务」
3. 选择「钉钉工作通知」
4. 填入钉钉应用的 AppKey 和 AppSecret
5. 点击「保存」

### 步骤 5.2：配置消息模板

1. 进入「消息模板」标签
2. 配置需要的通知类型：
   - 密码修改通知
   - 密码重置通知
   - 待办提醒
3. 自定义模板内容（可选）

### 步骤 5.3：测试消息发送

1. 进入「测试发送」功能
2. 选择接收人（使用测试账号）
3. 发送测试消息
4. 验证是否在钉钉中收到消息

## 验证清单

完成以上配置后，使用以下清单验证：

| 验证项 | 验证方法 | 预期结果 |
|--------|----------|----------|
| 钉钉用户同步 | 查看用户列表 | 用户来自钉钉 |
| 钉钉账号登录 | 使用钉钉账号登录平台 | 登录成功 |
| 应用 SSO | 从应用发起钉钉登录 | 跳转并登录成功 |
| 消息推送 | 触发密码修改等事件 | 钉钉收到通知 |

## 常见问题

<Accordions>
<Accordion title="Q1：通讯录同步失败怎么排查？">
**排查步骤**：
1. 检查 AppKey 和 AppSecret 是否正确
2. 检查钉钉应用权限是否开通
3. 查看同步日志的具体错误信息
4. 确认网络连接是否正常
</Accordion>
<Accordion title="Q2：SSO 跳转失败怎么解决？">
**排查步骤**：
1. 确认回调地址配置正确
2. 检查浏览器是否允许弹窗
3. 确认用户已在应用授权范围内
4. 查看浏览器控制台错误信息
</Accordion>
<Accordion title="Q3：消息推送收不到怎么检查？">
**排查步骤**：
1. 确认企业消息服务已启用
2. 检查用户手机号是否正确
3. 确认钉钉应用已开通「发送消息」权限
4. 查看消息发送日志
</Accordion>
