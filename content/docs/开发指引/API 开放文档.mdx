## 快速开始

### 第一步：获取 Access Token

所有 API 调用都需要先获取 access_token：

```bash
curl -X POST 'https://{your-tenant}.idaas.example.com/iam/token' \
  -H 'Content-Type: application/x-www-form-urlencoded' \
  -H 'Authorization: Basic base64({client_id}:{client_secret})' \
  -d 'grant_type=client_credentials'
```

响应示例：

```json
{
  "access_token": "48709172-514c-40df-9f40-2ee97245b642",
  "token_type": "bearer",
  "expires_in": 599,
  "scope": "address phone openid profile offline_access email"
}
```

### 第二步：调用 API

获取 token 后，在请求头中携带：

```bash
curl -X GET 'https://{your-tenant}.idaas.example.com/api/v1/users' \
  -H 'Authorization: Bearer {access_token}'
```

### 第三步：处理响应

API 统一响应格式：

```json
{
  "result": true,
  "error": "0",
  "error_description": "SUCCESS",
  "data": {
    // 业务数据
  }
}
```

## 认证方式

| 方式 | 适用场景 | 说明 |
|------|----------|------|
| **Basic Auth** | 服务端调用 | 使用 client_id:client_secret 编码后传输 |
| **Bearer Token** | API 调用 | 使用获取的 access_token |
| **M2M** | 机器对机器 | 应用级别的数据访问权限控制 |

### M2M 权限说明

M2M 应用除了 API 操作权限外，还受到**组织架构数据范围**限制：

- 只能访问授权范围内的部门/用户数据
- 支持「允许」和「拒绝」规则
- 支持字段级访问控制（V5.8.1+）

详细配置：[M2M 应用数据访问权限](../集成平台API开放文档.mdx#m2m-应用数据访问权限)

## 常见问题

**Q: token 过期了怎么办？**
A: 重新调用获取 token 接口。建议在本地缓存 token，过期前主动刷新。

**Q: API 返回权限不足？**
A: 检查：1) token 是否有效；2) M2M 应用是否配置了对应数据范围；3) 字段级权限是否配置。

**Q: 如何查看 API 调用日志？**
A: 进入「审计日志 > 接口日志」，可查看所有 API 调用记录。
