## 概述

系统初始化会指定系统管理员、账号管理员、应用管理员。系统内置管理员帐号不可修改。同时也可在各个角色下添加用户为该角色管理员。

## 管理员类型对比

| 管理员类型 | 权限范围 | 是否可删除 | 主要职责 |
|---------------|------------|---------------|------------|
| 系统管理员 | 全系统管理权限 | 否 | 系统配置、整体管理 |
| 账号管理员 | 用户账号相关权限 | 否 | 用户创建、权限分配 |
| 应用管理员 | 应用相关权限 | 否 | 应用管理、配置、授权 |
| 自定义角色管理员 | 根据角色定义的权限 | 是 | 特定功能管理 |

## 操作步骤

1. 在搜索框输入用户名，从搜索结果中选择后进行添加。
2. 自行添加的管理员可被删除。
3. 自定义的角色将出现在「用户自定义」模块，点击操作列添加按钮，在对应角色下添加管理员。
4. 角色的创建详见[管理员角色](/docs/04-权限中心/管理员角色)

## 最佳实践

1. **遵循最小权限原则**：为管理员分配完成工作所需的最小权限集
2. **定义清晰的角色**：明确各管理员角色的职责范围，避免权限重叠
3. **定期审核管理员账号**：定期检查是否有不再需要的管理员账号
4. **记录变更操作**：重要的管理操作应留有记录，便于追溯

## 安全建议

1. **多因素认证**：为管理员账号启用多因素认证，增强安全性
2. **定期轮换密码**：要求管理员定期更换密码
3. **权限分离**：确保关键操作需要多人协作完成，避免权力过度集中
4. **活动监控**：监控管理员账号的活动，及时发现异常行为
