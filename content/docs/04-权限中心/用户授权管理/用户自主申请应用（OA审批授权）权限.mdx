## 概述

用户（尤其是企业外部用户）临时要访问某个应用，申请权限，之前的方式可能是单独找管理员，沟通申请或者是填写申请单，比较麻烦，管理员的工作量也比较大。现在可以通过自助门户进行自助申请，申请审批还将关联钉钉的OA审批，方便高效还可以留档追溯。

## 自助申请流程

```text
┌───────────────┐     ┌───────────────┐     ┌───────────────┐
│  进入工作台   │────>│  申请应用权限 │────>│  填写申请信息 │
└───────────────┘     └───────────────┘     └───────┬───────┘
                                                    │
                                                    ▼
┌───────────────┐                         ┌───────────────┐
│  权限生效     │<────────────────────────│  管理员审批   │
└───────────────┘                         └───────────────┘
```

## 自助申请优势

| 优势 | 说明 |
|------|------|
| 简化申请流程 | 用户可直接在门户提交申请，无需额外沟通 |
| 审批流程标准化 | 统一的审批流程，保证审批规范 |
| 自动记录存档 | 所有申请和审批记录自动保存，便于追溯 |
| 减轻管理员负担 | 管理员只需处理审批，无需手动授权 |
| 与OA系统集成 | 与企业现有OA审批流程无缝集成 |

## 申请参数说明

| 参数 | 说明 | 是否必填 |
|------|------|------------|
| 申请原因 | 说明申请应用权限的业务需求 | 是 |
| 使用时间 | 预计需要使用该应用的时间段 | 是 |
| 权限范围 | 申请的具体权限范围（如果应用支持） | 否 |
| 紧急程度 | 权限申请的紧急程度 | 否 |

## 最佳实践

1. **明确填写申请理由**：详细说明业务场景和必要性，提高审批通过率
2. **合理设置权限期限**：根据实际需求设置合适的权限期限，避免长期闲置权限
3. **分级审批流程**：针对不同重要程度的应用，设置不同级别的审批流程
4. **定期权限复查**：管理员应定期检查已授予的临时权限，及时回收过期权限
