## 概述

基于组织架构和用户授权是一种直接通过企业组织结构和具体用户进行授权的方式。它可以分为两块：组织架构授权可以理解成部门授权或用户组授权；用户授权是指定用户进行授权，可以是多个或单个用户。如果想授权全部人员，可开启"允许所有人使用"选项。

## 授权选择决策流程
```
┌───────────────────────┐
│ 是否需要全员访问？    │
└──────────┬────────────┘
           │
           ▼
┌───────────────────────┐     ┌───────────────────────┐
│ 是 → 开启"允许所有人  │     │ 否 → 继续选择授权方式 │
│      使用"选项        │     └──────────┬────────────┘
└───────────────────────┘                │
                                         ▼
                          ┌───────────────────────────────┐
                          │ 组织授权、用户授权或标签授权？│
                          └──┬───────────────┬────────────┘
                             │               │
             ┌───────────────▼───┐  ┌────────▼───────────┐
             │ 按组织架构授权    │  │ 按具体用户授权     │
             └───────────────────┘  └────────────────────┘
```

## 授权方式

### 允许所有人使用
- 开启后，该应用所有人均可使用
- 关闭后，可选择组织授权、用户授权、标签授权三种方式进行管理授权范围

### 组织授权
- 勾选对应的用户组，用户组内的成员将拥有访问该应用的权限
- 新添加的用户也将集成该应用的权限；如用户从该组移除，权限将被收回
- 勾选父用户组，子用户组继承父组权限

### 用户授权
- 直接在输入框内进行搜索添加用户，支持使用用户名、姓名、手机号、邮箱字段进行搜索
- 添加用户后该用户拥有应用权限

## 各授权方式比较

| 授权方式 | 优点 | 缺点 | 适用场景 |
|------------|------|------|------------|
| 全员授权 | 配置简单，覆盖全面 | 无法精细控制权限 | 全员应用（如企业OA） |
| 组织授权 | 批量授权高效，自动同步组织变化 | 无法针对个人灵活授权 | 部门专用应用 |
| 用户授权 | 精确控制到个人级别 | 管理成本高，人员变动需手动调整 | 特殊岗位人员专用应用 |

## 最佳实践
1. **部门级别批量授权**：对于同一部门需要相同权限的情况，优先使用组织授权
2. **特殊角色人员授权**：对于跨部门但角色相同的人员，可使用用户授权
3. **混合授权策略**：可同时使用组织授权和用户授权，灵活管理不同场景

## 典型应用场景

### 场景一：财务系统访问权限

| 授权方法 | 具体设置 | 效果 |
|------------|------------|------|
| 组织授权 | 勾选财务部门 | 财务部所有成员可访问，新加入财务部的人员自动获得权限 |
| 用户授权 | 添加财务总监、CFO | 特定高管可以访问财务数据 |

### 场景二：销售管理系统

| 授权方法 | 具体设置 | 效果 |
|------------|------------|------|
| 组织授权 | 勾选销售部门 | 所有销售人员可使用系统记录客户和销售数据 |
| 用户授权 | 添加销售总监、区域经理 | 管理层可查看汇总报表和所有区域数据 |

### 场景三：企业知识库

| 授权方法 | 具体设置 | 效果 |
|------------|------------|------|
| 全员授权 | 开启"允许所有人使用" | 企业全员可访问通用知识库内容 |
| 组织授权 | 对特定部门开放特定栏目 | 专业内容只对相关部门开放 |

## 注意事项
1. **定期权限审计**：定期检查组织架构授权，确保随组织变动保持权限合理性
2. **权限优先级**：当用户同时通过组织授权和用户授权获得权限时，遵循权限合并原则
3. **异动人员处理**：关注人员调动情况，确保权限及时调整
4. **临时权限管理**：对于临时需要的权限，考虑使用时效性授权机制
