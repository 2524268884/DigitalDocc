## 概述

系统默认管理员角色为系统管理员、应用管理员、帐号管理员、CLI应用角色、普通用户角色五个。以上默认角色不可被删除。但可以添加用户、编辑角色信息。

### 默认角色权限对比

| 角色名称 | 权限范围 | 是否可删除 | 主要职责 |
|------------|------------|---------------|------------|
| 系统管理员 | 全部系统功能 | 否 | 管理整个系统 |
| 应用管理员 | 应用管理相关 | 否 | 应用上架、配置和授权 |
| 帐号管理员 | 用户账号管理 | 否 | 用户创建、修改和授权 |
| CLI应用角色 | CLI命令操作 | 否 | 命令行接口操作 |
| 普通用户角色 | 基本应用访问 | 否 | 访问被授权的应用 |

## 用户权限级别说明

| 权限类型 | 可执行操作 | 限制说明 |
|------------|---------------|------------|
| **查看权限** | 查看和导出数据 | 不能修改数据或产生更新操作 |
| **新建权限** | 查看和创建数据 | 不能编辑已有数据，包括自己创建的数据 |
| **编辑权限** | 查看和修改数据 | 不能创建新数据 |
| **删除权限** | 查看和删除数据 | 不能编辑数据或创建新数据 |

### 详细说明
1. **查看用户权限**：登录后只能查看和导出数据，不能进行任何数据的修改或者产生实际数据更新的操作。用户管理：只能查看列表用户列表，部门列表，搜索。列表左上角的操作菜单全部隐藏，包括创建、激活邀请、拖拽部门等所有操作都不可见。连接中心的上线/发布/下线等输入编辑权限；
2. **新建用户权限**：登录后查看&创建，能查看菜单下的数据，能创建，但不能编辑已有数据，包括自己创建的数据，也不能编辑。以"应用管理"菜单为例，只能创建应用，从应用市场安装应用；
3. **编辑用户权限**：登录后查看&编辑，能查看权限菜单下有权限的数据范围，能对数据进行修改，但不能创建新的数据；
4. **删除用户权限**：登录后查看&删除，能对有权限查看数据范围进行删除操作，但不能编辑数据，也不能创建新的数据；

## 权限管理案例

### 场景描述
- 李四用户有B部门的查看权限，C部门的所有增删改查的权限，A部门的创建权限
- 张三在ABC个部门中有不同的职位信息，1人多职

### 操作结果
1. 用户李四登录后，对张三进行用户属性内容的修改，可进行任意修改，因为李四有C部门的所有权限；
2. 用户李四登录后，修改张三A部门的主部门改成否，把C或B部门改成是，可进行修改；
3. 用户李四登录后，打开张三的详情页，把部门A给删除掉，把部门B进行部门职位、部门主管、部门工号编辑替换或者删除，都不可以；
4. 用户李四登录后，打开张三的详情页，把部门C给进行删除，或者编辑部门职位、部门主管、部门工号等信息，都可以进行操作，李四把张三的C部门删除后，在用户列表中只能查看张三信息，不能进行编辑删除操作；
5. 用户李四登录后，打开张三的详情页，张三把A部门作为主部门，同时部门工号写的456，此时修改用户属性中的工号从456改为123并进行保存，可以修改保存；
6. 用户李四登录后，在用户列表页，选择ABC三个部门下的不同的用户，进行批量操作，启用、禁用、激活邀请、密码重置等功能，根据不同的权限给出不同的提示，批量操作中混杂AB两个部门的用户，此时只有C部门的用户可以操作成功，其他用户都需要给出1个统一的提示，无权限进行操作；
7. 用户李四登录后，在用户列表页，删除张三，张三会被永远删除。
