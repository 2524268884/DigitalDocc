## **一、功能概述**

统一凭证管理用于对连接流中使用的各类授权凭证进行集中管理。

通过该能力，租户管理员可以在统一页面中完成凭证的创建、修改、删除与查看，并将凭证作为连接流和身份自动化的**唯一授权来源**进行复用，从而避免多处配置、难以维护的问题。

![image.png](/images/凭证管理/a8b53cc8-b23f-4419-843c-6c04eeea20b5.png)



## **二、适用场景**

统一凭证管理适用于以下场景：
- 多条连接流使用同一应用的授权信息
- 需要集中查看和管理租户下所有连接流凭证
- 凭证需要统一替换，避免逐个流程修改
- 管理员需要明确凭证被哪些连接流或身份自动化使用

常见使用背景包括：
- API 集成类连接流（如业务系统、第三方平台）
- 身份自动化、通讯录集成与同步场景
- 企业对凭证安全和可控性要求较高的场景

## **三、统一凭证管理模块说明**

### **3.1 模块能力**

统一凭证管理模块具备以下能力：
- 提供独立的「连接流凭证管理」页面
- 支持凭证的新增、修改、删除、查询
- 支持按应用维度对凭证进行集中管理
- 支持查看凭证被连接流和身份自动化的使用情况
- 与现有权限体系对接，支持凭证级权限控制（后续能力）

该模块仅负责凭证管理，不直接执行流程。

## **四、如何进入统一凭证管理**

### **4.1 入口位置**
1. 进入平台管理后台
2. 打开 **连接流相关功能区域**
3. 进入 **连接流凭证管理** 页面

该页面为独立模块，不与现有「授权管理」模块合并。

### **4.2 页面结构说明**

凭证管理页面以列表形式展示当前租户下所有连接流凭证，支持按应用名称进行筛选和搜索。

## **五、新建统一凭证**

### **5.1 新建入口**

在凭证管理页面中：
1. 点击「新建凭证」
2. 页面右侧弹出 **新建凭证抽屉**

### **5.2 新建凭证配置流程**

新建凭证的操作流程如下：
1. **选择应用**
    - 点击「选择应用」按钮
    - 弹出应用下拉选择框
    - 展示当前所有需要授权的连接器应用
    - 默认展示前 10 条，支持按应用名称搜索
    - 展示形式为：应用 icon \+ 应用名称
2. **确认应用**
    - 选择应用后，按钮区域回显应用信息
    - 提供「切换」按钮，可重新选择应用
3. **填写授权信息**
    - 系统根据所选应用，动态展示对应的授权字段
    - 每个字段展示：
        - 字段名称
        - 字段描述
        - 是否必填
        - 字段类型
    - 同时提供该应用对应的授权帮助文档跳转入口
4. **操作按钮**
    - 取消：放弃本次新建
    - 连接测试：使用当前凭证进行接口调用校验（本期暂不支持）
    - 保存：保存凭证信息

## **六、参数配置说明（重点）**

### **6.1 凭证字段配置方式**
- 凭证字段由应用定义
- 系统根据应用授权模型自动生成配置表单
- 用户仅需按字段要求填写对应信息
- 不支持自定义新增授权字段

### **6.4 限制说明**
- 新建凭证时必须先选择应用
- 凭证创建完成后，不允许修改应用类型
- 凭证校验逻辑将在保存时进行（部分能力后续支持）

## **七、凭证使用与输出说明**

### **7.1 凭证的消费方式**
- 凭证可被多个连接流复用
- 凭证也可被通讯录集成与身份自动化引用
- 连接流在运行时，将自动使用最新版本的凭证进行 API 调用

### **7.2 列表字段说明**

凭证列表中展示以下信息：

| **字段** | **说明** |
|----------|----------|
| 应用名称 | 凭证对应的应用（icon \+ 名称） |
| 凭证名称 | 用户自定义的凭证名称 |
| 关联连接流数量 | 当前凭证被多少连接流使用 |
| 关联身份自动化 | 当前凭证被多少身份自动化使用 |
| 更新时间 | 凭证最近一次更新时间 |
| 最后修改者 | 最后修改该凭证的用户 |

## **八、保存与生效逻辑**

### **8.1 凭证保存规则**
- 凭证保存后立即生效
- 新建凭证可被后续创建的连接流直接选择使用
- 凭证作为统一授权来源，不区分单一流程\\

### **8.2 凭证使用方式**

完成凭证创建后，在进行连接流配置时，选择凭证对应应用，即可使用刚刚创建的凭证

![image.png](/images/凭证管理/447b1ca5-ce62-4dd9-adc1-f12cf2c71b38.png)

### **8.3 变更规则**
- 修改凭证后：
    - 所有引用该凭证的连接流
    - 所有引用该凭证的身份自动化

将在后续执行时使用**最新的凭证信息**
- 不会影响已执行完成的历史流程

## **九、使用建议**
- 建议按「应用 \+ 用途」维度命名凭证，便于后期维护
- 对被多个流程复用的凭证，优先使用统一凭证管理创建
- 修改凭证前，建议确认其关联的连接流数量与业务影响
- 不建议频繁删除正在被使用的凭证，避免流程异常
