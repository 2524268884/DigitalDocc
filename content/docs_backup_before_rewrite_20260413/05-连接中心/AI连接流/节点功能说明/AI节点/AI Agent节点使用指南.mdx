## 概述

AI Agent 节点是专属集成平台提供的大语言模型（LLM）智能代理节点，能够根据用户指令和大模型能力，自主决策并调用各种工具完成复杂任务。与传统的规则式节点不同，AI Agent 可以理解自然语言输入、进行推理分析、规划执行步骤，并灵活调用所需工具。

## 适用场景

- **智能问答系统**：构建基于知识库的问答机器人
- **数据分析助手**：自动解析数据查询请求并生成分析结果
- **文档处理**：自动提取、总结、转换文档内容
- **客户服务**：构建智能客服，处理复杂咨询
- **任务自动化**：根据指令自动完成多步骤复杂任务

## 节点配置

### 基本配置

| 参数 | 说明 | 必填 |
|------|------|------|
| **节点名称** | 为节点设置一个描述性名称 | 是 |
| **模型选择** | 选择使用的大语言模型 | 是 |
| **工具绑定** | 配置 Agent 可以调用的工具/子节点 | 否 |

### 模型配置

#### 支持的模型

| 模型 | 提供商 | 特点 |
|------|--------|------|
| **DeepSeek** | 深度求索 | 性价比高，中文理解能力强 |
| **通义千问** | 阿里云 | 阿里系产品集成方便 |
| **GPT-4** | OpenAI | 通用能力强 |
| **Claude** | Anthropic | 长文本处理能力强 |

#### 模型参数

| 参数 | 说明 | 默认值 |
|------|------|--------|
| **Temperature** | 控制输出的随机性（0-1） | 0.7 |
| **Max Tokens** | 最大输出 token 数 | 2048 |
| **Top P** | 核采样概率 | 1.0 |

### 工具绑定

工具是 Agent 与外部世界交互的桥梁。通过绑定工具，Agent 可以调用各种子节点完成具体操作：

#### 内置工具

| 工具类型 | 说明 | 用途 |
|---------|------|------|
| **HTTP 请求** | 发送 HTTP 请求 | 调用外部 API |
| **数据库查询** | 执行数据库查询 | 获取业务数据 |
| **文件读写** | 读取或写入文件 | 处理文件数据 |

#### 配置工具权限

1. 在 Agent 节点配置中，点击 **「添加工具」**
2. 从可用工具列表中选择需要绑定的工具
3. 为每个工具设置调用权限和参数映射
4. 配置工具的执行超时时间

## 使用示例

### 示例：构建智能客服 Agent

#### 步骤一：准备知识库数据

1. 在知识库中上传产品文档、FAQ 等资料
2. 配置知识库的检索参数（返回条数、相似度阈值等）

#### 步骤二：创建 Agent

1. 拖拽「AI Agent」节点到画布
2. 选择模型（如 DeepSeek）
3. 配置系统提示词：
   ```
   你是一个专业的客服助手，负责回答用户关于产品的问题。
   - 如果用户问题在知识库中有明确答案，直接回答
   - 如果需要查询数据，使用工具调用相关接口
   - 如果遇到无法回答的问题，礼貌地转人工服务
   ```

#### 步骤三：绑定工具

1. 添加「HTTP 请求」工具
2. 配置知识库 API 的调用参数
3. 设置返回数据的解析方式

#### 步骤四：连接对话入口

1. 添加「Webhook 触发」或「AI 对话」节点作为入口
2. 将用户输入传递给 Agent 节点
3. 将 Agent 输出返回给用户

### 示例代码配置

```json
{
  "model": "deepseek",
  "system_prompt": "你是一个专业的客服助手...",
  "tools": [
    {
      "type": "http",
      "name": "query_knowledge_base",
      "endpoint": "https://api.example.com/knowledge/search",
      "method": "POST",
      "headers": {
        "Authorization": "Bearer ${knowledge_base_token}"
      }
    }
  ],
  "parameters": {
    "temperature": 0.7,
    "max_tokens": 2048
  }
}
```

## 输出说明

AI Agent 节点返回以下输出：

| 输出字段 | 类型 | 说明 |
|---------|------|------|
| **result** | string | Agent 的最终回复文本 |
| **tool_calls** | array | 调用的工具列表及其结果 |
| **reasoning** | string | Agent 的推理过程（可选） |
| **tokens_used** | number | 消耗的 token 数量 |

## 最佳实践

### 1. 精心设计系统提示词

系统提示词决定了 Agent 的行为模式和回答风格。建议包含：
- Agent 的角色定位
- 回答问题的方式和风格
- 边界条件和限制说明
- 必要时调用工具的指引

### 2. 合理绑定工具

- 只绑定必要的工具，避免 Agent 过度调用
- 为每个工具提供清晰的功能描述
- 设置工具调用的最大次数限制，防止无限循环

### 3. 设置安全防护

- 在系统提示词中明确禁止生成有害内容
- 对用户输入进行安全过滤
- 设置敏感词库和过滤规则

### 4. 控制成本

- 根据实际需求选择合适的模型
- 设置合理的 Max Tokens 限制
- 使用缓存减少重复调用

## 常见问题

### Q1：Agent 回答不准确或偏离主题

**可能原因**：
- 系统提示词不够明确
- 缺乏相关的工具支持
- Temperature 设置过高

**解决方案**：
- 优化系统提示词，增加更具体的行为指引
- 添加相关的知识库或 API 工具
- 降低 Temperature 值减少随机性

### Q2：工具调用失败

**可能原因**：
- 工具配置参数错误
- 外部服务不可用
- 工具执行超时

**解决方案**：
- 检查工具配置参数是否正确
- 添加错误处理和重试机制
- 增加工具调用的超时时间

### Q3：如何提高 Agent 的响应速度

**解决方案**：
- 选择响应更快的模型
- 减少 Agent 调用的工具数量
- 使用流式输出（Streaming）提升用户体验
- 合理使用缓存减少重复计算
