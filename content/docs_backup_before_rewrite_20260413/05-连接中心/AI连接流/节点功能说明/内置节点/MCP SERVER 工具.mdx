## **一、功能概述**

MCP（Model Context Protocol）用于将**平台已有应用能力**以「工具」的方式提供给 AI Agent 使用。

通过在画布中配置 **MCP SERVER 节点**，可以让大模型在合适的时机自动调用对应的业务动作（如查询、创建、更新等），实现 **AI \+ 业务系统的联动执行**   。

![image.png](/images/MCP SERVER 工具/620962f8-ac06-4a58-a262-49b91ed202bb.png)

## **二、适用场景**
- AI Agent 在对话过程中需要调用真实业务系统
- 希望由大模型自主判断「什么时候用哪个工具」
- 一个 Agent 需要组合多个业务动作（查询 → 判断 → 执行）

## **三、MCP SERVER 节点说明**

### **3.1 节点能力**
- 每一个 MCP 节点对应 **一个后端 MCP SERVER**
- 一个 MCP 节点下 **可以配置多个动作**
- MCP 节点只作为 **工具集合**，不直接参与流程判断  。

## **四、如何插入 MCP SERVER 节点**

### **4.1 插入入口**
1. 在画布中点击 **「插入工具节点」**
2. 弹出工具选择抽屉
3. 一级分类中选择 **MCP**

### **4.2 选择 MCP 类型**

进入 MCP 二级菜单后，可看到两个选项：
- **内置 MCP**
    - 使用平台已接入的应用与动作
    - 选择后自动生成 MCP SERVER 节点
- **自定义 MCP**
    - 通过 SSE / Studio 对接第三方 MCP Server
    - 当前版本暂不开放   

![image.png](/images/MCP SERVER 工具/4b50cc61-cef6-4389-8c10-ee2b2fe975cc.png)

## **五、为 MCP 节点添加动作**
1. 在 MCP SERVER 节点中点击 **「插入动作」**
2. 弹出应用选择抽屉
3. 选择需要接入的应用

## **六、参数配置说明（重点）**

### **6.1 参数输入方式**

根据所选动作，系统会自动生成对应的参数输入组件。

### **6.2 AI 生成模式**
- 每个参数输入框右侧提供 **「AI 生成」按钮**
- 点击后：
    - 参数值由 AI 从用户对话中自动提取
    - 输入框进入 **AI 生成状态**
    - 不允许手动输入或变量绑定

### **6.3 取消 AI 生成**
- AI 生成状态下，右侧显示「取消」按钮
- 点击后恢复为普通输入模式

### **6.4 限制说明**
- **JSON 类型参数暂不支持 AI 生成模式**

## **七、节点输出说明**

### **7.1 输出行为**
- MCP 节点 **不支持单独测试运行**
- 当绑定该 MCP 的 AI Agent 节点运行，并成功调用工具时：
    - MCP 节点输出区域会展示 **接口返回的 JSON 数据**

## **八、保存与生效逻辑（非常重要）**

### **8.1 MCP 节点保存规则**
- **每一个 MCP 节点 = 一个 MCP SERVER**
- 点击保存时，系统会：
    1. 根据当前配置自动生成 MCP SERVER
    2. 将该 MCP SERVER 与当前 AI Agent 绑定

### **8.2 动作变更规则**
- 若在 MCP 节点下新增 / 修改动作
- 在动作配置保存时：
    - 后端会对 **对应 MCP SERVER 进行更新**
    - 不会新建 Server，而是增量更新  。



## **九、使用建议（产品经验）**
- 动作描述一定要写「业务语义」，而不是技术字段
- 参数能用 AI 生成的，优先使用 AI 生成
- 一个 MCP 节点建议控制在 **同一业务域**
- 不建议把完全无关的动作堆在同一个 MCP 下
