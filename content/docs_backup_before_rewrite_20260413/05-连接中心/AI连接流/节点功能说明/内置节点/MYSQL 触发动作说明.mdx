## **一、功能概述**

MySQL「当有数据新增时」触发动作，用于在 MySQL 数据表中出现新增数据时，自动触发连接流执行。

该触发动作通过定时轮询数据库表，并结合唯一识别字段与排序字段进行增量判断，在检测到新数据时，将新增记录按顺序推送至后续流程节点中使用。

## **二、适用场景**

该触发动作适用于以下场景：
- 数据存储在 MySQL 数据库中，且需要实时或准实时感知新增数据
- 希望在数据新增后自动触发流程，而非通过定时任务全量拉取
- 不希望通过「定时触发 \+ 查询节点」的方式手动编排流程
- 常见于邮件、日志、业务事件等持续写入型数据表

典型示例：
- 客户将邮件数据写入 MySQL 表
- 系统需要在有新邮件入库时自动触发后续处理流程
- 每条新增记录按顺序单独触发流程执行

## **三、MySQL 触发节点说明**

### **3.1 节点能力**

MySQL 触发动作具备以下能力：
- 作为 **触发节点** 使用，而非普通动作节点
- 定时轮询指定的 MySQL 数据表
- 基于字段游标判断是否存在新增数据
- 支持多条新增记录依次触发多次流程
- 无新增数据时，不触发流程

该节点仅在检测到新增数据时触发流程，不参与数据加工处理。

## **四、如何插入 MySQL 触发节点**

### **4.1 插入入口**
1. 在创建连接流时，选择 **触发器类型流程**
2. 点击「选择触发器」
3. 在应用列表中选择 **MySQL 数据库**
4. 选择触发动作 **「当 MySQL 中有数据新增时」**

### **4.2 选择 MySQL 触发类型**

当前 MySQL 触发节点仅支持一种触发类型：
- **当 MySQL 中有数据新增时**

插入后即可进入节点配置。

## **五、为 MySQL 触发节点添加动作**

MySQL 触发节点插入后即为可配置状态，无需额外添加子动作。

该节点作为流程起点存在，不支持前置节点。

## **六、参数配置说明（重点）**

### **6.1 参数输入方式**

MySQL 触发节点支持以下配置字段：

| **参数名称** | **类型** | **是否必填** | **说明** |
|----------------|----------|----------------|----------|
| 数据表名称 | text | 是 | 需要监听的数据表名称 |
| 唯一识别字段（unique） | text | 是 | 用于数据排重的唯一字段，如 ID |
| 排序字段（sort） | text | 是 | 用于判断新增或更新的排序字段 |

### **6.2 排序字段说明**
- 排序字段用于判断数据是否为新增或更新
- 支持的字段类型：
    - 数值类型
    - 日期时间类型
- 排序规则：
    - 数值字段：由大到小
    - 日期字段：由近到远

### **6.4 限制说明**
- 第一次启用触发器时，不会触发流程
- 首次执行仅用于记录当前数据表的游标位置
- 触发频率为 **每分钟一次轮询**
- 仅在查询到新增数据时才会触发流程

## **七、节点输出说明**

### **7.1 输出行为**
- 当查询到新增数据时：
    - 每一条新增记录将 **按时间顺序** 单独触发一次流程
    - 后续节点接收到的是单条新增记录数据
- 当查询结果为空时：
    - 不触发流程

### **7.2 默认状态提示**
- 无新增数据时，流程不会被触发
- 定时轮询本身不会产生流程执行记录

## **八、保存与生效逻辑（非常重要）**

### **8.1 节点保存规则**
- 触发节点配置保存并启用后，开始生效
- 系统会在后台记录当前游标位置（sort \+ unique）
- 游标信息用于后续增量判断

### **8.2 变更规则**

触发逻辑遵循以下规则：
1. **首次查询**
    - 记录当前最新一条数据的 (sort, unique) 作为游标
    - 不触发流程
2. **后续查询**
    - 查询条件为：
```
WHERE sort > last_sort 
OR (sort = last_sort AND unique > last_unique)
ORDER BY sort ASC, unique ASC
```
3. **游标更新**
    - 查询成功后，将游标更新为本次查询结果的最后一条记录
4. **多条数据处理**
    - 若一次查询中存在多条新增数据
    - 系统将按顺序依次触发多次流程

## **九、使用建议（产品经验）**
- 唯一识别字段建议使用自增 ID 或全局唯一字段
- 排序字段建议使用创建时间或递增时间戳
- 不建议在高频写入但无业务价值的表上使用该触发器
- 若下游流程较重，需评估新增数据量对流程执行压力的影响
