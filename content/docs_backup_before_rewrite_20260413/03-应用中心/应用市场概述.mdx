应用市场是专属集成平台的重要组成部分，提供2000\+预集成的企业应用，帮助企业快速实现单点登录和身份认证集成。本文档详细介绍应用市场的功能特点、使用方法和最佳实践。

## 功能概述

应用市场具有以下核心功能：
- **丰富的应用库**：提供2000\+主流企业应用，涵盖协同办公、人力资源、财务管理等多个领域
- **预配置集成**：应用已预先配置单点登录参数，减少集成工作量
- **快速搜索**：通过应用名称、类别、协议类型等多维度搜索
- **详细文档**：每个应用提供详细的集成说明和配置指南
- **分类管理**：支持自定义应用分类，满足企业特定需求

## 应用添加流程

### 从应用市场添加应用
1. 进入应用中心 \> 应用市场
2. 浏览或搜索需要的应用
3. 点击应用卡片，查看应用详情
4. 点击"添加应用"按钮
5. 配置应用基本信息和单点登录参数
6. 点击"保存"完成应用添加

![image.png](/images/应用市场概述/7e64eb48-2cdf-4812-8f72-53970183848a.png)

### 应用搜索技巧

应用市场支持多种搜索方式：
- **关键字搜索**：通过应用名称、描述等进行模糊搜索
- **分类筛选**：按业务类别筛选应用
- **协议筛选**：按认证协议类型（SAML、OIDC、CAS等）筛选
- **热门排序**：查看使用最多的热门应用
- **最新上架**：查看最新加入应用市场的应用

## 应用配置

从应用市场添加的应用通常需要进行以下配置：

### 基本信息配置
1. 设置应用显示名称
2. 上传或使用默认应用图标
3. 选择应用分类
4. 添加应用描述
5. 配置应用首页URL

![image.png](/images/应用市场概述/1e7cb94d-e6c0-4db3-8075-8c30a5b06c4d.png)

### 单点登录配置

根据应用支持的协议类型，配置相应的单点登录参数：

#### SAML应用配置
1. 复制应用市场提供的SP实体ID、ACS URL等参数到应用配置中
2. 下载IdP元数据文件，上传到应用配置中
3. 配置用户属性映射
4. 测试单点登录功能

#### OIDC应用配置
1. 复制client*id、client*secret等参数到应用配置中
2. 配置授权回调URL
3. 设置作用域和声明
4. 测试单点登录功能

### 应用授权配置

配置哪些用户可以访问该应用：
1. 设置授权范围（全部用户、指定组织、指定角色或指定用户）
2. 配置应用权限（如管理员角色、普通用户角色等）
3. 设置应用可见性

## 应用分类管理

企业可以根据自身业务需求自定义应用分类：

### 创建应用分类
1. 进入应用中心 \> 应用分类管理
2. 点击"新建分类"按钮
3. 输入分类名称和描述
4. 设置分类图标（可选）
5. 点击"保存"完成创建

### 调整分类顺序
1. 进入应用中心 \> 应用分类管理
2. 通过拖拽调整分类顺序
3. 点击"保存排序"按钮保存更改

### 编辑或删除分类
1. 进入应用中心 \> 应用分类管理
2. 点击分类右侧的"编辑"或"删除"按钮
3. 根据提示完成操作

## 应用上架与定制

对于市场中没有的应用，企业可以请求上架或自定义应用：

### 应用上架请求
1. 在应用市场页面点击"应用上架请求"按钮
2. 填写应用基本信息和认证协议类型
3. 提交申请，等待平台审核和配置
4. 审核通过后，应用将出现在应用市场中

### 自定义应用创建

如果应用市场中没有所需应用，可以自行创建：
1. 进入应用中心 \> 应用管理
2. 点击"新建应用"按钮
3. 选择"自定义应用"
4. 填写应用信息并配置单点登录参数
5. 点击"保存"完成创建

## 应用市场最佳实践

### 应用选型建议
- **优先使用应用市场**：应用市场中的应用已预配置，可以节省集成时间
- **评估协议兼容性**：选择与企业认证基础设施兼容的协议类型
- **考虑安全要求**：评估应用的安全特性，如支持多因子认证、会话控制等
- **关注用户体验**：选择登录流程简单、用户友好的应用

### 分类管理建议
- **基于业务划分**：根据企业业务部门或功能划分应用分类
- **控制分类数量**：避免过多分类导致导航复杂
- **使用统一命名**：采用一致的命名规范，如"XX系统"、"XX工具"
- **定期整理归类**：随着应用数量增加，定期审核和调整分类

### 批量部署策略
- **分批次实施**：按业务重要性或复杂度分批实施应用集成
- **创建模板**：为相似应用创建配置模板，提高部署效率
- **自动化脚本**：使用API和脚本实现应用批量配置
- **充分测试**：在生产环境部署前，在测试环境充分验证

## 常见问题

### 应用集成问题

**问题：应用市场中的应用配置失败**

解决方法：
1. 检查配置参数是否正确复制
2. 验证URL和端点是否可访问
3. 查看应用提供商的文档，确认配置步骤
4. 联系技术支持获取帮助

**问题：无法找到特定应用**

解决方法：
1. 尝试不同的搜索关键词
2. 检查应用的其他名称或别名
3. 浏览相关分类查找
4. 提交应用上架请求

### 认证协议问题

**问题：不确定选择哪种认证协议**

建议：
1. SAML：适合传统Web应用，广泛支持，适合企业级应用
2. OIDC：适合现代Web和移动应用，提供更丰富的用户信息
3. OAuth 2.0：适合API授权场景，专注于授权而非认证
4. 表单代理：适合无法改造的传统应用

**问题：应用不支持标准SSO协议**

解决方法：
1. 使用密码代填功能
2. 考虑表单代理或HTTP头代理方式
3. 与应用提供商沟通，请求添加SSO支持
4. 评估使用替代产品

### 用户体验问题

**问题：用户找不到应用**

解决方法：
1. 优化应用分类和命名
2. 实现应用搜索功能
3. 创建常用应用快捷方式
4. 提供应用使用指南

**问题：应用访问速度慢**

解决方法：
1. 检查网络连接和带宽
2. 评估应用服务器性能
3. 考虑使用CDN加速
4. 优化单点登录流程
