## 1\. 概述

本指南详细说明如何在专属集成平台中配置用友EHR连接器，实现将EHR系统中统一维护的组织架构和用户数据同步至平台，并可进一步同步至钉钉等应用，确保企业内各应用系统的身份数据一致性。

通过EHR集成，您可以获得以下好处：
- 无需重复维护用户信息，直接使用EHR中的权威数据
- 自动同步组织架构变更，确保组织结构实时更新
- 用户入职、离职等状态变更可自动同步至各业务系统
- 减少管理人员的工作量，提高数据准确性
- 支持增量同步，减少系统资源消耗

## 2\. 前提条件

在开始配置前，请确保满足以下条件：

### 2.1 EHR系统准备
- 拥有用友EHR开发者后台的管理员权限
- 已获取以下EHR系统接口信息：
    - 服务器地址
    - 部门token
    - 组织token
    - 用户token
    - 根组织ID

### 2.2 平台准备
- 拥有专属集成平台的管理员权限
- 了解组织架构的基本配置需求

### 2.3 网络环境
- 确保专属集成平台能够访问用友EHR的API接口
- 如有防火墙限制，需开放相关访问权限

## 3\. 配置步骤

### 3.1 创建EHR连接器
1. 登录专属集成平台
2. 在左侧导航栏选择「**用户中心 \> 通讯录集成**」
3. 点击「**新建连接器**」按钮
4. 在连接器类型选择界面，选择「**EHR**」身份源

![image.png](/images/通讯录集成-用友EHR/aad5ba4e-040a-44af-9af4-b032971ae0ce.png)



### 3.2 配置基本参数

根据实际情况，填写以下连接参数：

![image.png](/images/通讯录集成-用友EHR/2b4ecb47-b0c5-4085-a3ca-19ed28f034c2.png)



| **参数名称** | **说明** | **配置建议** |
|----------------|----------|----------------|
| 名称 | 自定义连接器名称 | 使用有意义的描述性名称，如"总部用友EHR" |
| 服务器地址 | 用友EHR API服务器地址 | 使用EHR提供的正式环境API地址 |
| 部门token | 用于获取部门信息的访问令牌 | 从EHR开发者后台获取 |
| 组织token | 用于获取组织信息的访问令牌 | 从EHR开发者后台获取 |
| 用户token | 用于获取用户信息的访问令牌 | 从EHR开发者后台获取 |
| 根组织ID | 指定同步的组织范围 | 取EHR对应的部门ID，同步全部部门可填写"0" |

### 3.3 配置同步策略

![image.png](/images/通讯录集成-用友EHR/0de03864-dfed-437b-a489-32a795228576.png)

| **参数名称** | **说明** | **配置建议** |
|----------------|----------|----------------|
| **同步方式** | 决定数据同步模式 |  |
|  | **自动增量**：只同步变更数据，显示同步周期，删除用户阈值不展示 | 生产环境推荐，减少资源消耗 |
|  | **自动全量**：同步所有数据，显示同步周期，删除用户阈值展示 | 适用于定期完整校验数据 |
|  | **手动全量**：手动触发同步，不显示同步周期，删除用户阈值展示 | 适用于初始配置和测试阶段 |
| **用户同步周期** | 自动同步执行频率 |  |
|  | 支持手动、30min、1小时、6小时、12小时、24小时 | 根据数据更新频率选择，通常6小时较为合适 |
|  | 可指定具体执行时间 | 建议选择非工作高峰期执行 |
| **同步部门** | 是否同步组织架构 | 建议启用，保持组织架构完整性 |
| **根部门挂载** | 组织结构导入方式 |  |
|  | **在选择节点下创建**：在指定部门下创建根节点 | 适合保持独立的组织结构 |
|  | **与选择节点合并**：与指定部门合并，指定部门作为根 | 适合与现有结构整合 |
| **删除用户阈值** | 防止意外大规模删除的保护机制 |  |
|  | 阈值 =（平台已回收数据和本次回收数据的差异值 / 已回收数据）\* 100% | 建议设置10%-15%，根据组织规模调整 |
|  | 超过阈值时系统将暂停删除操作 | 防止因配置错误导致的大规模数据丢失 |
| **账号关联** | EHR与平台账号的映射关系 |  |
|  | 建议使用"EHR手机号与专属集成平台手机号"映射 | 手机号通常是最稳定的关联字段 |
| **测试账号** | 用于验证配置的测试账号 | 填写已知存在于EHR中的用户手机号 |

### 3.5 启动同步流程
1. 完成配置后，点击「保存并同步」按钮启动同步
2. 系统将开始从用友EHR获取组织架构和用户数据
3. 同步完成后，可在「用户中心 \> 用户管理」查看导入的用户数据

### 3.6 同步至下游应用

配置完成EHR到平台的同步后，您可以进一步将数据同步至钉钉等下游应用：
1. 导航至「应用集成 \> 通讯录同步」
2. 选择目标应用（如钉钉）
3. 配置相应的同步参数
4. 启动同步流程

## 4\. 配置注意事项

### 4.1 数据映射注意事项
- **手机号格式**：确保EHR中的手机号格式统一，建议使用纯数字格式
- **必填字段**：确保平台必填字段（通常为用户名、手机号、邮箱）有对应的EHR数据源
- **扩展字段**：对于EHR中的特殊扩展字段，需在平台中先创建对应的自定义属性
- **数据转换**：特殊格式数据（如日期）可能需要设置转换规则确保格式一致

### 4.2 组织结构注意事项
- **部门名称唯一性**：确保同级部门名称在EHR中保持唯一，避免同步冲突
- **部门层级限制**：注意控制部门层级深度，过深的层级可能影响管理效率
- **特殊部门处理**：对于"默认组"等特殊名称的部门，注意可能的系统限制
- **组织变更影响**：大规模组织变更可能触发删除阈值保护，需提前规划

### 4.3 同步策略注意事项
- **初始同步**：首次配置建议选择非工作时间进行全量同步
- **增量同步限制**：增量同步依赖EHR系统的变更记录机制，确认其可靠性
- **同步频率**：根据业务变更频率设置合理的同步周期，避免不必要的系统负载
- **阈值设置**：根据组织规模和变动频率，合理设置删除用户阈值

## 5\. 常见问题与故障排除

### 5.1 连接问题
1. **无法连接EHR服务器**
    - **可能原因**：服务器地址错误、网络连接问题、防火墙限制
    - **解决方法**：
        - 验证服务器地址是否正确
        - 检查网络连接状态
        - 确认防火墙是否允许访问
2. **Token认证失败**
    - **可能原因**：Token已过期、Token权限不足、Token格式错误
    - **解决方法**：
        - 重新获取有效的Token
        - 确认Token具有足够的权限
        - 检查Token格式是否正确

### 5.2 同步问题
1. **无数据同步**
    - **可能原因**：根组织ID配置错误、API返回为空、权限不足
    - **解决方法**：
        - 检查根组织ID是否正确
        - 使用API测试工具验证返回结果
        - 确认Token权限范围
2. **部分用户未同步**
    - **可能原因**：用户数据不完整、账号关联规则匹配失败
    - **解决方法**：
        - 检查EHR中用户数据完整性
        - 调整账号关联规则
        - 验证关键字段（如手机号）是否存在并格式正确
3. **同步后组织结构错误**
    - **可能原因**：根部门挂载方式配置不当、EHR组织结构问题
    - **解决方法**：
        - 调整根部门挂载方式
        - 检查EHR中的组织层级关系
        - 验证部门ID和父部门ID的对应关系

### 5.3 常见错误解答
1. **Q: 同步时提示"部门同名冲突"怎么处理？****A:** 这是因为同级部门中存在同名部门。解决方法：在EHR中修改部门名称确保同级唯一，或调整组织结构避免冲突。
2. **Q: 为什么有些用户显示"手机号必填"错误？****A:** 平台要求手机号为必填字段，但同步的用户数据中缺少手机号。解决方法：在EHR中补充缺失的手机号信息，或调整属性映射确保手机号正确同步。
3. **Q: 组织架构同步不完整怎么解决？****A:** 可能是根组织ID设置问题或部分部门数据缺失。解决方法：检查根组织ID设置，确认是否涵盖所有需要同步的部门；检查EHR中组织架构的完整性。
4. **Q: 同步触发删除阈值保护如何处理？****A:** 这是系统的安全机制，防止大规模意外删除。解决方法：确认大量删除操作是否符合预期，如属于正常业务变更，可临时调整阈值或在待删除列表中手动确认删除。

## 6\. 最佳实践
1. **初始配置建议**
    - 先在测试环境完成配置并验证，再应用到生产环境
    - 首次同步选择业务低峰期执行，减少对业务的影响
    - 先同步小范围用户进行测试，确认无误后再同步全部数据
2. **数据质量管理**
    - 定期检查EHR中的数据质量，确保关键字段完整性
    - 建立数据规范，统一字段格式（如手机号、邮箱格式）
    - 对特殊字符和非标准格式数据进行预处理
3. **同步策略优化**
    - 新系统建议先使用全量同步建立基础数据，后续改用增量同步
    - 配置合理的同步频率，避免过于频繁同步造成系统负载
    - 关键业务变更期（如大规模组织调整）临时增加同步频率
4. **安全与合规**
    - 定期更新API Token，增强安全性
    - 遵循最小权限原则，只获取必要的数据
    - 建立同步日志审计机制，跟踪重要数据变更

### 7.2 常见错误代码解析

| 错误代码 | 错误描述 | 可能原因 | 解决方案 |
|------------|------------|------------|------------|
| 401 | 未授权 | Token无效或过期 | 更新Token |
| 403 | 禁止访问 | 权限不足 | 申请更高权限的Token |
