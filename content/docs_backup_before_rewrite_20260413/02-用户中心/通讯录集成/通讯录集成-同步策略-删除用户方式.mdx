## 1. 概述

通讯录集成的同步策略支持两种上游用户删除处理方式：**常规删除**和**用户停用**。本文档详细介绍这两种方式的概念、配置方法及适用场景。

![image.png](/images/通讯录集成-同步策略-删除用户方式/d8939f1b-aa81-4d5d-a3b6-119df1558f5e.png)

## 2. 删除阈值配置

### 2.1 阈值设置

在配置删除用户处理方式前，首先需要设置删除阈值：

| 阈值设置 | 说明 | 适用场景 |
|---------|------|---------|
| **设置为 0** | 同步时不执行任何删除操作 | 保守策略，避免意外删除 |
| **设置为正数（如 100）** | 超过阈值时暂停删除，需人工确认 | 生产环境推荐，防止大规模误删 |
| **设置为 -1** | 不设删除上限，全部自动处理 | 测试环境或明确业务需求 |

### 2.2 阈值计算逻辑

```
如果 (数据源删除用户数 > 删除阈值) 则：
    连接器暂停执行，等待人工确认
否则：
    按配置的处理方式执行删除
```

**示例**：
- 阈值设置为 100
- 数据源本次同步删除 150 个用户
- 结果：暂停执行，需管理员确认后再处理

## 3. 删除处理方式

### 3.1 常规删除

常规删除是指将用户在专属集成平台中完全移除，包括用户账号和相关信息。

**特点**：
- 用户账号被完全删除
- 用户无法再登录任何已集成应用
- 下游系统中的用户数据也会被同步删除
- 操作不可逆，需谨慎使用

**适用场景**：
- 用户已完全离职
- 测试环境清理测试账号
- 确定用户无需再访问任何系统

### 3.2 用户停用

用户停用是指将用户账号设置为禁用状态，而非物理删除。

**特点**：
- 用户账号保留但无法登录
- 用户的历史操作记录保留
- 下游系统可选择同步停用状态
- 可随时重新启用用户

**适用场景**：
- 员工暂时离开（如休假、借调）
- 需保留用户历史数据用于审计
- 希望保留用户账号以便将来快速恢复

**停用后的用户状态**：
- 无法登录任何已集成应用
- 历史操作记录保留
- 可由管理员手动删除或重新启用

## 4. 操作流程

### 4.1 常规删除流程

当数据源删除用户数在阈值范围内时，执行常规删除：

1. **查看待删除用户列表**
   
   系统会展示即将被删除的用户列表：
   
   ![image.png](/images/通讯录集成-同步策略-删除用户方式/e6992c35-9f8d-467e-b518-8e2e3699e784.png)

2. **确认删除范围**
   - 支持查看每个用户的详细信息
   - 确认删除影响范围

3. **执行删除操作**
   - **单个删除**：点击操作列的「删除」按钮
   - **批量删除**：勾选多个用户，点击「批量删除」按钮

4. **下游同步删除**
   
   确认后将对选中用户以及下游用户全部删除：
   - 专属集成平台中的用户被删除
   - 所有已集成应用中的账号被同步删除
   - 操作不可逆

### 4.2 用户停用流程

当数据源删除用户数在阈值范围内时，执行用户停用：

1. **查看待停用用户列表**
   
   系统会展示即将被停用的用户列表：
   
   ![image.png](/images/通讯录集成-同步策略-删除用户方式/04497233-6ca1-4736-b457-370bc2c75c63.png)

2. **确认停用范围**
   - 支持查看每个用户的详细信息
   - 确认停用影响范围

3. **执行停用操作**
   - **单个停用**：点击操作列的「停用」按钮
   - **批量停用**：勾选多个用户，点击「批量停用」按钮

4. **下游同步停用**
   
   确认后将对选中的用户全部停用：
   - 用户无法登录专属集成平台
   - 下游应用可选择同步停用状态
   - 用户可随时被重新启用

## 5. 下游系统同步行为

### 5.1 下游删除/停用同步

删除或停用操作会根据下游系统的配置，决定是否同步执行：

| 下游配置 | 删除同步 | 停用同步 |
|---------|---------|---------|
| **同步删除** | 下游用户被物理删除 | 下游用户被物理删除 |
| **同步停用** | 下游用户被停用 | 下游用户被停用 |
| **不同步** | 仅在专属集成平台本地删除/停用 | 仅在专属集成平台本地删除/停用 |

### 5.2 同步策略配置

在下游系统配置中，可以单独设置同步策略：

- **严格跟随源**：下游完全跟随专属集成平台的删除/停用操作
- **仅本地操作**：下游保持独立操作，不受专属集成平台影响
- **仅同步删除**：仅在专属集成平台执行删除时同步

## 6. 最佳实践

### 6.1 生产环境建议

| 建议 | 说明 |
|------|------|
| **设置删除阈值** | 生产环境建议设置 5-10 的阈值，防止大规模误删 |
| **优先使用停用** | 对于不确定的用户，优先使用停用而非删除 |
| **定期清理停用用户** | 建立停用用户定期清理机制（如 6 个月） |
| **保留审计记录** | 所有删除/停用操作都应记录审计日志 |

### 6.2 阈值计算参考

```
建议阈值 = (总用户数 × 1%)
```

| 总用户数 | 建议阈值 |
|---------|---------|
| 1000 | 10 |
| 5000 | 50 |
| 10000 | 100 |
| 50000 | 500 |

### 6.3 回滚方案

**常规删除的恢复方式**：
- 从备份中恢复用户数据
- 重新创建用户账号
- 手动恢复下游系统用户

**停用用户的恢复方式**：
- 在用户管理中直接「启用」用户
- 下游系统会自动同步启用状态

## 7. 常见问题

### Q1：删除阈值设置为 -1 有风险吗？

**风险说明**：设置为 -1 后，所有删除操作将自动执行，无人工确认环节。

**建议**：
- 仅在测试环境使用
- 或在确认业务需求后使用，同时做好数据备份

### Q2：用户被删除后可以恢复吗？

**常规删除**：不可以，操作不可逆。

**停用用户**：可以，只需在用户管理中重新启用该用户即可。

### Q3：下游系统没有同步删除用户？

**可能原因**：
- 下游配置了「不同步」策略
- 下游系统不支持同步删除

**解决方案**：
- 检查下游同步配置
- 手动在下游系统中执行相应操作

### Q4：如何批量处理大量待删除用户？

**操作步骤**：
1. 勾选页面底部的「全选」或分页选择
2. 点击「批量删除」或「批量停用」
3. 确认影响范围后点击「确定」

**注意事项**：批量操作不可逆，请谨慎确认。
