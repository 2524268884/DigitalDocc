## 1\. 概述

本指南详细说明如何在专属集成平台中配置致远OA通讯录集成，实现致远OA系统中统一维护的组织架构和用户数据自动同步至专属集成平台，并可进一步同步至下游应用（如钉钉）。通过此集成，企业可以保持内部各应用系统的数据一致性，简化用户身份管理，提高企业数据治理水平。

### 1.1 集成价值
- **数据统一管理**：以致远OA作为组织架构和用户信息的主数据源，确保企业内部身份数据一致性
- **流程自动化**：自动化同步流程，减少手动维护工作，降低人为错误风险
- **高效协同**：打通致远OA与其他业务系统，实现跨系统身份信息共享
- **管理简化**：集中管理用户生命周期，简化账号创建、变更和注销流程
- **降低成本**：减少重复的身份管理工作，降低IT运维成本

### 1.2 适用场景
- **多系统身份打通**：企业同时使用致远OA和其他业务系统，需要统一管理用户身份
- **组织变更自动同步**：企业组织架构频繁调整，需要变更自动同步到各系统
- **员工生命周期管理**：员工入职、转岗、离职等状态变更需自动同步至相关系统
- **身份认证中心构建**：构建以致远OA为基础的企业统一身份认证体系

## 2\. 前提条件

在开始配置前，请确保满足以下条件：

### 2.1 账号与权限要求
- **致远OA系统要求**：
    - 拥有致远OA系统的管理员账号
    - 具备致远OA开发者后台的访问权限
    - 具有REST API相关的配置权限
- **专属集成平台要求**：
    - 拥有专属集成平台的管理员账号
    - 具备通讯录集成模块的操作权限

### 2.2 系统与网络环境
- 确保致远OA系统的REST服务正常运行
- 确保专属集成平台能够通过网络访问致远OA的REST服务
- 如有防火墙，需开放相关端口，允许两系统之间的数据传输

### 2.3 信息收集清单

在配置前，建议收集以下信息：

| 所需信息 | 说明 | 获取方式 |
|------------|------|------------|
| 单位ID | 致远OA系统的单位标识 | 致远OA系统管理员可提供 |
| REST服务器地址 | 致远OA REST API的访问地址 | 通常格式为：http(s)://\[致远OA服务器地址\]/seeyon/rest |
| REST账号 | 用于调用REST API的账号 | 致远OA系统管理员配置并提供 |
| REST密码 | REST账号对应的密码 | 致远OA系统管理员配置并提供 |
| 根组织ID | 需要同步的组织架构根节点ID | 根节点ID，全部部门填写"0" |
| 组织架构信息 | 了解致远OA的组织架构特点 | 查看致远OA系统的组织管理模块 |

## 3\. 配置步骤

### 3.1 创建致远OA通讯录连接器
1. 使用管理员账号登录专属集成平台
2. 在左侧导航栏选择「用户中心 → 通讯录集成」
3. 点击页面右上角的「添加连接器」按钮
4. 在连接器类型选择界面，找到并选择「致远OA」选项

![image.png](/images/通讯录集成-致远/01d8dcec-06f1-4f90-be10-c6456420112b.png)

### 3.2 配置基本连接信息

在连接器配置页面，填写以下基本信息：

| 参数名称 | 说明 | 填写指南 |
|------------|------|------------|
| 名称 | 连接器显示名称 | 自定义名称，建议使用描述性名称，如"致远OA-总部" |
| 单位ID | 致远OA的单位标识 | 填写对应的单位ID，通常为数字 |
| REST服务器地址 | 致远OA REST API地址 | 格式为：http(s)://\[服务器地址\]/seeyon/rest |
| REST账号 | 调用REST API的账号 | 填写有权限调用REST API的账号名 |
| REST密码 | REST账号的密码 | 填写对应的密码 |
| 根组织ID | 需同步的组织根节点ID | 填写部门ID，全部部门填写"0" |

**填写建议**：
- **名称**：建议使用有业务含义的名称，方便后续管理和识别
- **REST服务器地址**：确保地址格式正确，包含协议(http/https)、服务器地址和路径
- **账号密码**：使用专用账号，避免使用个人账号，确保账号有足够权限

### 3.3 配置同步策略

在同步策略配置区域，设置以下参数：

![image.png](/images/通讯录集成-致远/2907f0b1-64a6-4140-b979-4b2734074734.png)

| 参数名称 | 说明 | 建议配置 | 适用场景 |
|------------|------|------------|------------|
| 同步方式 | 数据同步策略 | 初次配置建议"手动全量同步"验证后改为"自动增量同步" | 初次配置验证用"手动全量"日常运行用"自动增量" |
| 同步周期 | 自动同步的频率 | 根据数据变更频率选择通常建议6小时 | 变更频繁选较短周期(30min-1h)变更不频繁选较长周期(12h-24h) |
| 开始时间 | 自动同步起始时间 | 业务低峰期，如凌晨 | 避免业务高峰期执行同步 |
| 同步部门 | 是否同步组织架构 | 建议勾选 | 需要完整组织架构时勾选仅需用户数据时不勾选 |
| 根部门挂载 | 组织架构同步方式 | 视情况选择 | "在选择节点下创建"：作为独立组织"与选择节点合并"：合并到现有组织 |
| 删除用户阈值 | 限制删除比例 | 小型企业(\2000)：5-10% | 详见下方"删除用户阈值说明" |
| 账号关联 | 用户映射关系 | "致远OA手机号与专属集成平台手机号" | 标准配置，可根据实际情况调整 |
| 测试账号 | 用于验证配置 | 填写致远OA中存在的手机号 | 验证连接配置是否正确 |

**关于删除用户阈值的详细说明**：

删除用户阈值是专属集成平台的安全机制，用于防止因配置错误或数据异常导致的大规模用户误删除。当单次同步中待删除用户的比例超过此阈值时，这些用户会被标记为"待删除"状态，需管理员确认后才执行实际删除操作。

**计算公式**：
```
删除用户比例 = (本次待删除用户数 / 已同步用户总数) × 100%
```

**阈值设置值及其含义**：
- **1**：当删除1个用户时，不执行删除操作
- **2**：当删除1个用户时执行删除；当删除2个及以上用户时，不执行删除
- **3**：当删除2个用户时执行删除；当删除3个及以上用户时，不执行删除
- **0**：同步时不删除任何用户，适用于仅需添加和更新用户的场景
- **-1**：不设置删除上限，无论删除多少用户都会同步执行删除（谨慎使用）

### 3.4 配置属性映射

完成基本配置后，需要设置致远OA与专属集成平台之间的属性映射关系：
1. 导航至「用户中心 → 用户属性」菜单
2. 选择致远OA连接器对应的属性配置
3. 设置用户属性映射关系

**用户属性映射建议**：

| 致远OA属性 | 专属集成平台属性 | 映射方式 | 说明 |
|--------------|------------------|------------|------|
| id | sub | 创建且更新 | 用户唯一标识 |
| login\_name | username | 创建且更新 | 登录名 |
| name | name | 创建且更新 | 用户姓名 |
| mobile | phone\_number | 创建且更新 | 手机号码 |
| email | email | 创建且更新 | 电子邮箱 |
| dept\_id | primary*org*id | 创建且更新 | 部门ID |
| enabled | active | 创建且更新 | 用户状态 |
| position | title | 创建且更新 | 职位名称 |

**部门属性映射建议**：

| 致远OA属性 | 专属集成平台属性 | 映射方式 | 说明 |
|--------------|------------------|------------|------|
| id | connector*org*id | 创建且更新 | 部门唯一标识 |
| name | name | 创建且更新 | 部门名称 |
| parent\_id | parentRefId | 创建且更新 | 父部门ID |
| sort\_id | seq | 创建且更新 | 部门排序值 |
| code | code | 创建且更新 | 部门编码 |

**属性映射注意事项**：
- 确保必要字段正确映射，特别是用户和部门的唯一标识字段
- 对于重要字段，建议使用"创建且更新"映射方式
- 如致远OA中有自定义字段，可根据需要进行映射

## 4\. 验证与测试

完成配置后，需要进行全面的测试和验证，确保集成正常工作：

### 4.1 连接测试
1. 在连接器配置页面底部，找到"测试账号"输入区域
2. 输入一个确认存在于致远OA系统中的手机号
3. 点击「测试」按钮
4. 系统将尝试连接致远OA的REST API并验证配置有效性
5. 测试成功将显示用户基本信息
6. 测试失败将显示错误信息，根据提示进行问题排查

### 4.2 执行同步测试
1. 完成所有配置后，点击「保存」按钮创建连接器
2. 在连接器列表页，找到新创建的致远OA连接器
3. 点击「立即同步」按钮触发一次手动同步
4. 观察同步进度和状态，等待同步完成
5. 同步完成后，查看同步结果统计：
    - 成功同步的用户数量
    - 成功同步的部门数量
    - 失败记录数量及原因

### 4.3 数据验证
1. 导航至「用户管理 → 用户列表」页面
2. 使用筛选条件，选择致远OA连接器作为认证源
3. 验证同步的用户数据：
    - 抽查用户信息是否完整准确
    - 确认用户与部门的关联关系是否正确
    - 验证用户状态是否符合预期
4. 导航至「用户管理 → 组织管理」页面
5. 检查组织架构是否正确同步：
    - 验证部门层级结构是否正确
    - 确认部门名称和排序是否符合预期

## 5\. 常见问题与故障排除

### 5.1 连接问题

#### 5.1.1 无法连接致远OA REST服务

**可能原因**：
- REST服务器地址错误
- REST账号或密码错误
- 网络连接问题
- 致远OA REST服务未启用或异常

**解决方案**：
1. 验证REST服务器地址格式是否正确
2. 确认REST账号和密码是否正确
3. 检查网络连通性，确认可以访问致远OA服务器
4. 联系致远OA管理员确认REST服务状态

#### 5.1.2 连接超时

**可能原因**：
- 网络延迟高
- 致远OA服务器负载过重
- REST服务响应慢

**解决方案**：
1. 检查网络连接质量
2. 尝试在非高峰时段进行连接
3. 联系致远OA管理员检查服务器状态

### 5.2 数据同步问题

#### 5.2.1 部分用户或部门未同步

**可能原因**：
- 根组织ID设置不正确
- 用户或部门状态异常
- 权限不足，无法获取完整数据
- 属性映射配置不正确

**解决方案**：
1. 检查根组织ID配置，确保覆盖所需同步的范围
2. 在致远OA中确认用户和部门状态是否正常
3. 确认REST账号拥有足够的数据访问权限
4. 检查并修正属性映射配置

#### 6.2.2 组织架构层级错误

**可能原因**：
- 部门ID和父部门ID映射不正确
- 根部门挂载方式配置不适当
- 致远OA中特殊的组织架构设计

**解决方案**：
1. 检查部门属性映射，确保ID和父部门ID正确映射
2. 调整根部门挂载方式，选择合适的配置
3. 对特殊组织结构考虑使用自定义映射规则

### 5.3 权限与安全问题

#### 5.3.1 REST API访问权限不足

**可能原因**：
- REST账号权限配置不足
- 单位安全策略限制
- API版本不兼容

**解决方案**：
1. 联系致远OA管理员，确认并调整REST账号权限
2. 检查致远OA的安全策略设置
3. 确认REST API版本与致远OA系统版本的兼容性

#### 5.3.2 数据同步安全风险

**问题**：大量数据变更可能带来安全风险

**防范措施**：
1. 合理设置删除用户阈值，防止误删
2. 定期检查同步日志，监控异常变更
3. 建立数据备份机制，确保可恢复
4. 实施变更管理流程，重大变更前先评估影响

## 6\. 最佳实践

### 6.1 初始配置建议
1. **分步实施**：
    - 从小范围测试开始，如先同步部分部门
    - 验证配置无误后，再扩大同步范围
    - 完成小范围测试后，再进行全量同步
2. **关键字段映射**：
    - 确保唯一标识字段(ID)正确映射
    - 用户名、姓名、手机号等核心字段使用"创建和更新"模式
    - 确保部门ID和父部门ID映射关系正确
3. **同步策略优化**：
    - 首次配置使用手动全量同步，验证数据准确性
    - 验证无误后，改为自动增量同步
    - 根据组织变更频率合理设置同步周期

### 6.2 运维管理建议
1. **定期监控**：
    - 定期检查同步日志，关注同步状态
    - 监控同步数据量变化，及时发现异常
    - 定期抽查同步数据的准确性
2. **变更管理**：
    - 致远OA系统组织架构发生重大变更前，提前评估影响
    - 大规模变更前调整删除用户阈值
    - 重要变更后执行全量同步，确保数据一致
3. **问题处理**：
    - 建立同步异常快速响应机制
    - 记录常见问题及解决方法
    - 与致远OA系统管理员保持沟通，及时了解系统变更

### 6.3 安全建议
1. **账号安全**：
    - 为集成专门创建REST API账号，避免使用个人账号
    - 定期更新REST账号密码
    - 给予账号最小必要权限
2. **数据保护**：
    - 合理设置删除用户阈值，防止误操作
    - 定期备份关键配置和数据
    - 实施数据变更审计机制
3. **应用权限隔离**：
    - 严格控制有权修改连接器配置的管理员范围
    - 建立配置变更审批流程
    - 定期审计权限设置，清理不必要权限
