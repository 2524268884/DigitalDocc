角色管理是专属集成平台中用于管理用户权限和分组的重要功能，通过角色可以实现用户的分类管理和批量授权，提高权限管理效率。

## 界面概览

![image.png](/images/角色管理/7948fc94-b626-438f-a3f6-940491a2223d.png)



角色管理界面主要由应用列表和角色列表两部分组成，提供丰富的角色创建和管理功能。

## 应用管理

应用是角色的最上层分类，用于对不同业务系统或场景的角色进行归类管理。每个角色必须归属于一个应用。

### 创建应用

![image.png](/images/角色管理/d43fb1cd-299d-46ef-a59b-d3f10325d319.png)


1. 点击【添加】按钮
2. 填写应用名称、Code和描述信息
3. 点击【确定】完成创建

### 编辑应用

修改应用的基本信息。
1. 在应用列表中选择目标应用
2. 点击【编辑】按钮
3. 修改应用的名称、Code或描述
4. 点击【确定】保存修改

### 删除应用

移除不再使用的应用。
1. 在应用列表中选择目标应用
2. 点击【删除】按钮
3. 确认删除操作
> **注意**：如果应用下存在角色，则无法直接删除，需要先删除所有关联角色。系统默认应用不支持删除。   

## 角色管理

### 创建角色

#### 创建静态角色

静态角色是通过手动指定用户成员的角色类型。

![image.png](/images/角色管理/260c61dd-2007-418e-9a5f-758a975da766.png)

![image.png](/images/角色管理/a880a7f2-31eb-450c-a170-03978a82c4c3.png)


1. 选择所属应用
2. 点击【新建】按钮
3. 填写角色名称、Code和描述信息
4. 选择角色类型为"静态角色"
5. 点击【下一步】
6. 从用户列表中选择成员（支持通过用户名、姓名、邮箱、手机号搜索）
7. 点击【确定】完成创建

#### 创建动态角色

动态角色是通过规则自动匹配用户的角色类型，当用户属性变化时，角色成员会自动更新。

![image.png](/images/角色管理/95cf25c4-67f4-41b7-828c-3fede5aba955.png)

![image.png](/images/角色管理/142a1f56-08af-4657-8b81-713d0b386409.png)


1. 选择所属应用
2. 点击【新建】按钮
3. 填写角色名称、Code和描述信息
4. 选择角色类型为"动态角色"
5. 点击【下一步】
6. 设置匹配规则：
    - 分类选择：用户
    - 属性选择：系统所有的用户属性和扩展属性
    - 条件选择：支持多种条件运算符
        - 等于(=)
        - 不等于(!=)
        - 大于(\>)
        - 大于等于(\>=)
        - 小于(\<)
        - 小于等于(\<=)
        - 开头等于(start with)
        - 结尾等于(end with)
        - 包含(contains)
        - 有值(present/has value)
    - 输入匹配值
7. 可添加多条匹配规则，用户和部门的匹配规则之间是AND关系
8. 点击【确定】完成创建

### 角色组管理

角色组用于对角色进行进一步分类，一个角色只能属于一个角色组，角色组只能属于特定应用。

![image.png](/images/角色管理/ad10b85f-3c27-449f-9505-19ad47eab9af.png)


1. 在角色界面中点击【新建角色组】按钮
2. 填写角色组名称
3. 点击【确定】完成创建
4. 创建或编辑角色时，可以选择所属角色组

### 角色黑名单

对于动态角色，可以设置黑名单排除特定用户。

![image.png](/images/角色管理/2affb1a5-00c6-4e1d-8bc8-3257609d749a.png)
1. 在角色列表中选择动态角色
2. 在用户信息区域可以配置黑名单
3. 点击【添加用户】选择需要排除的用户
4. 黑名单中的用户即使满足动态匹配条件也不会被加入该角色

### 编辑角色

修改角色的基本信息和成员。

![image.png](/images/角色管理/31abf4b8-f0bd-48bd-be49-7f46e930f2af.png)
1. 在角色列表中选择目标角色
2. 点击【编辑】按钮
3. 修改角色的名称、Code或描述
4. 对于静态角色，可以修改角色成员
5. 对于动态角色，可以修改匹配规则
6. 点击【确定】保存修改

### 删除角色

移除不再使用的角色。
1. 在角色列表中选择目标角色
2. 点击【删除】按钮
3. 确认删除操作

## 角色查询

### 角色搜索

快速查找角色。

![image.png](/images/角色管理/ecd3ef17-934b-4bb4-b72f-55d23243509b.png)
1. 在角色列表上方的搜索框中输入角色名称或Code

根据应用筛选角色。
1. 在应用列表中选择特定应用
2. 系统仅显示该应用下的角色

## 通讯录集成与角色同步

系统支持通过通讯录集成从外部系统（如钉钉、企业微信等）同步角色结构。
1. 外部系统的角色会映射为专属集成平台中的应用和角色
2. 可以为每个外部系统创建对应的应用（如"钉钉"）
3. 外部系统的角色组会映射为平台中的角色组
4. 映射关系可以在通讯录集成模块中配置

## 角色的核心应用场景

角色是专属集成平台中非常核心的概念，在多个关键场景中发挥重要作用：

### 单点登录应用授权(RBAC)

在应用授权管理中，平台支持基于角色的访问控制(RBAC)，具体实现方式：
1. 在应用详情页的"授权范围"选项卡中，选择"基于角色授权(RBAC)"
2. 从已创建的角色列表中选择需要授权的角色
3. 被授权角色中的所有用户将获得访问该应用的权限
4. 当用户加入或离开角色时，其应用访问权限会自动更新

RBAC授权适用于按职能角色管理访问权限的场景，例如：
- 将"财务人员"角色授权给财务系统应用
- 将"HR专员"角色授权给人力资源系统应用
- 将"销售主管"角色授权给CRM系统应用

相比直接授权给个人或部门，基于角色授权具有更高的灵活性和管理效率。

### 基于属性的访问控制(ABAC)

平台支持基于用户属性的访问控制(ABAC)，主要通过动态角色实现：
1. 创建动态角色，设置属性匹配规则（如职位="经理"）
2. 在应用授权管理中选择该动态角色
3. 系统会根据用户属性自动计算符合条件的用户并授予权限

ABAC适用于需要根据用户特定属性（如职级、地区、职能等）灵活控制访问权限的场景。

### 通讯录同步集成

角色在通讯录同步过程中扮演重要角色：
1. **上游集成**：从外部系统（如钉钉）同步组织架构时，外部系统的角色会映射为平台中的角色
    - 例如，钉钉中的角色会同步为平台中对应的"钉钉"应用下的角色
2. **下游同步**：将平台用户同步到下游系统时，可基于角色筛选同步范围
    - 例如，只同步"ERP用户"角色的成员到ERP系统
3. **角色权限映射**：将平台角色映射到目标系统的权限组或角色
    - 实现不同系统间的权限模型转换和对接

### 审批流程与业务规则

角色在工作流程中的应用：
1. **审批流程**：基于角色设置审批人或审批路径
    - 例如，将"部门经理"角色设置为一级审批人，"财务主管"角色设置为二级审批人
2. **业务规则**：在连接流中基于角色配置业务规则
    - 例如，只有"合同管理员"角色的用户才能触发特定合同集成流程

## 常见问题

### 静态角色和动态角色有什么区别？
- **静态角色**：成员由管理员手动指定，不会自动更新。适合小范围、相对固定的用户分组。
- **动态角色**：根据规则自动匹配用户，当用户属性变化时自动更新成员。适合大规模、需要自动化管理的场景。

### 为什么需要应用和角色组两级分类？
- **应用**：是为了支持多系统集成场景，将不同业务系统的角色分开管理。例如，当集成钉钉的角色到平台时，可以创建一个"钉钉"应用，该应用下的所有角色都是从钉钉同步而来。
- **角色组**：提供应用内的二级分类，使角色管理更加精细。一个角色只能属于一个角色组，且角色组仅在其所属应用下有效。

### 为什么有些用户满足条件但没有加入动态角色？

可能的原因包括：
- 该用户被加入了角色黑名单
- 用户属性值格式不完全匹配(如有空格、大小写差异等)
- 系统正在进行角色成员计算，尚未完成更新

### 如何高效管理大量角色？
1. 充分利用应用和角色组进行分类管理
2. 对于会根据用户属性变化的角色，优先使用动态角色
3. 利用通讯录集成功能，从企业已有系统中同步角色结构
4. 使用动态角色的黑名单功能处理特殊情况
