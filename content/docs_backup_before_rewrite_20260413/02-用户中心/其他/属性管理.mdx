用户属性管理是专属集成平台的核心功能，用于定义和管理系统中各实体（用户、部门、角色）的属性字段，支持基础属性和扩展属性的灵活配置，并实现与外部系统的属性映射。

## 界面概览

![image.png](/images/属性管理/c1f3e965-29a5-48ef-a429-705847312088.png)

用户属性管理界面按照不同认证源和实体类型组织，提供全面的属性配置和映射功能。

## 属性管理范围

系统支持以下类型的属性管理：
1. **平台属性**：专属集成平台的核心属性管理
    - 用户属性：用户基础属性和扩展属性
    - 部门属性：组织机构相关属性
    - 角色属性：角色相关属性
2. **通讯录集成源属性**：从外部系统集成到平台的属性
    - 外部系统创建通讯录集成时，自动创建对应的属性管理
    - 支持与平台属性的映射配置
3. **通讯录同步源属性**：平台向外部系统同步的属性
    - 创建通讯录同步应用时，自动创建对应的属性管理
    - 支持与平台属性的映射配置
4. **认证源属性**：外部认证系统的属性
    - 创建认证源应用时，自动创建对应的属性管理
    - 支持与平台属性的映射配置

## 平台属性管理

### 属性查看与筛选

![image.png](/images/属性管理/bcae49dc-8c34-455c-ac33-e6fa118da22f.png)



属性列表展示所有配置的属性信息，包括：
- 属性名称：系统内部使用的属性标识
- 显示名称：界面上显示的属性名称
- 数据类型：属性的数据类型（字符串、时间戳、数字等）
- 属性类型：基础属性或扩展属性
- 特性设置：必填、唯一性等

支持按实体类型（用户、部门、角色）切换查看不同类型的属性列表。

### 添加扩展属性

为不同实体类型添加自定义扩展属性。

![image.png](/images/属性管理/9d9f8b38-67e0-4db8-b6b2-f2bf1806f65e.png)


1. 选择实体类型标签页（用户、部门、角色）
2. 点击【新增属性】按钮
3. 填写属性信息：
    - 属性名称：系统标识，建议使用英文
    - 显示名称：界面显示的名称
    - 数据类型：选择文本、时间或数字
4. 配置属性特性：
    - 是否必填：创建或编辑实体时是否必须提供该属性值
    - 全局唯一：属性值在所有实体中是否必须唯一
    - 个人中心展示：是否在用户服务台的个人中心页面显示
    - 允许用户修改：是否允许用户在个人中心自行修改该属性
5. 填写字段描述（可选）
6. 点击【确定】完成创建

### 编辑属性

修改属性的配置信息。

![image.png](/images/属性管理/0df026b9-db16-44d8-972e-b7ebc1d00715.png)
1. 在属性列表中找到目标属性
2. 点击【编辑】按钮
3. 修改属性配置
    - 基础属性：仅支持修改特性设置，不支持修改名称和数据类型
    - 扩展属性：支持修改所有信息
4. 点击【确定】保存更改

### 删除属性

移除不需要的扩展属性。
1. 在属性列表中找到目标扩展属性
2. 点击【删除】按钮
3. 确认删除操作
> **注意**：基础属性不支持删除。删除扩展属性后，相关数据将无法恢复。   

## 属性映射机制

属性映射是专属集成平台的核心能力，用于在不同系统间建立属性关联关系，实现数据同步和转换。

### 映射类型

系统支持多种映射场景：
1. **上游映射**：外部系统 → 专属集成平台
    - 通讯录集成源属性 → 平台属性
    - 认证源属性 → 平台属性（用于JIT创建用户）
2. **下游映射**：专属集成平台 → 外部系统
    - 平台属性 → 通讯录同步源属性

### 映射配置

![image.png](/images/属性管理/c7187509-eb14-466c-9cbe-f0ddd53df359.png)
1. 在用户属性管理中选择相应的外部源标签页
2. 配置源属性和目标属性的对应关系
3. 设置映射策略：
    - **仅创建**：仅在创建实体时映射该属性
    - **创建且更新**：创建和更新实体时都映射该属性
    - **不映射**：不同步该属性
4. 支持使用表达式处理或转换属性值
5. 保存映射配置

### 表达式支持

#### **表达式使用场景**

属性映射支持使用Express Language表达式进行数据处理：
1. 基本转换：`${源属性}` 提取源属性值
2. 条件判断：`${源属性 == 'value' ? '值1' : '值2'}`
3. 格式转换：`${format(源属性, 'yyyy-MM-dd')}`
4. 字符串操作：`${substring(源属性, 0, 5)}`

表达式可对同步的具体属性通过 JavaScript 的表达式和运算符以及内置函数以及正则表达式等进行简单的属性转换和属性值的简单逻辑处理，核心是对当前属性通过 JavaScript 实现转换或条件以及数据值的逻辑处理。

#### **表达式版本支持**

详细的 JavaScript 使用参考如下链接了解如何使用 JavaScript 。

[https://developer.mozilla.org/zh-CN/docs/Web/JavaScript/Reference/Operators](https://developer.mozilla.org/zh-CN/docs/Web/JavaScript/Reference/Operators)

JavaScript 的版本支持：
- GraalVM支持ECMAScript 2022标准
- 支持大部分现代JavaScript特性，包括ES6\+功能

#### **常用表达式示例**

**1、**  格式化或转换数据类型示例
```javascript
// 电话号码格式化
user.mobile.replace("+86", "+86-")
user.phone_number.replace("+86-", "+86")

// 提取域名前的用户名
user.email.split('@')[0]

// 转换为大写展示
user.employee_id && user.employee_id.toUpperCase()

// 字段拼接
user.first_name + " " + user.last_name

```

**2**、 处理用户和组织之间的关系示例
```javascript
// 用户与经理关系
{id: user.manager_id, name: user.manager_name}

// 用户与部门关系
user.departments && user.departments.map(dept => ({id: dept.id, is_primary: dept.is_primary}))

// 获取主管理员
org.admin_users && org.admin_users.find(user => user.is_primary)

// 获取用户所有角色
user.roles && user.roles.map(role => role.name).join(',')

```

**3、**  从复杂对象中提取特定信息示例
```javascript
// 提取自定义属性
user.attributes && user.attributes.find(attr => attr.name === "员工编号")?.value

// 提取特定扩展字段
user.extension_fields && user.extension_fields.skill_level

// 提取多层嵌套对象
user.profile?.contact_info?.emergency_contact?.phone

// 提取数组中特定条件项
user.certificates && user.certificates.filter(cert => cert.status === "valid").map(cert => cert.name)

```

**4、**  日期格式化和操作示例
```javascript
// 日期转字符串
user.hire_date && String(user.hire_date)

// 计算服务年限
user.hire_date && Math.floor((new Date() - new Date(user.hire_date)) / (365*24*60*60*1000))

// 获取当前日期
(function() { return new Date().toISOString().split('T')[0]; })()

// 格式化显示日期
user.birth_date && new Date(user.birth_date).toLocaleDateString('zh-CN')

```



**5、**  将代码转换为可读文本示例
```javascript
// 在职状态转换
user.status === "1" ? '在职' : '离职'

// 账号状态转换
user.account_status === "0" ? '激活' : user.account_status === "1" ? '锁定' : '禁用'

// 性别代码转换
user.gender === "M" ? '男' : user.gender === "F" ? '女' : '未知'

// 员工类型转换
user.employee_type === "FT" ? '全职' : user.employee_type === "PT" ? '兼职' : '合同工'

```

**6、**  基于条件判断返回不同结果示例
```javascript
// 根据部门返回不同角色
user.department_id === "HR001" ? 'HR专员' : user.department_id === "IT001" ? 'IT支持' : '普通员工'

// 根据级别返回不同权限
user.level > 7 ? 'manager' : user.level > 4 ? 'senior' : 'junior'

// 处理可能为空的数据
user.employee_id || user.login_name || user.email

// 复杂条件判断
user.is_manager && user.reports_count > 5 ? '高级经理' : user.is_manager ? '经理' : '普通员工'

```

**7、**  处理列表或数组数据示例
```javascript
// 提取ID列表
org.sub_departments && org.sub_departments.map(dept => dept.id)

// 过滤并转换
user.groups && user.groups.filter(g => g.status === "active").map(g => g.name)

// 数组转字符串
user.skills && user.skills.join(', ')

// 查找主要项
user.positions && user.positions.find(pos => pos.is_primary === true)

```

**8、**  提供默认值或常量示例
```javascript
// 默认语言
user.language || "ZH-CN"

// 默认时区
user.timezone || "Asia/Shanghai"

// 条件默认值
org.description ? org.description : org.name + "部门"

// 复杂默认逻辑
user.display_name || user.full_name || (user.first_name + " " + user.last_name) || user.email

```

**9、**  下游根部门映射示例
```javascript
//这里面的1858377061035466753表示与钉钉根部门映射的本地部门ID
org.id=='1858377061035466753'?1:org.connector_org_id 

//也可以用部门名称来做映射，要求是这个名称需唯一，比如：
org.name=='中国XX有限公司'?1:org.connector_org_id

```

## 属性特性管理

### 全局唯一性

控制属性值在系统中的唯一性约束。
1. 在属性配置中启用"全局唯一"选项
2. 系统将确保该属性值在所有实体中唯一
3. 用户ID、用户名、手机号、邮箱、身份证号等基础属性默认为全局唯一

### 必填设置

控制属性是否为必填字段。
1. 在属性配置中启用"必填"选项
2. 创建或编辑实体时必须提供该属性的值

### 个人中心展示与编辑权限

控制用户自助服务的属性可见性和编辑权限。
1. "个人中心展示"：属性是否在用户服务台的个人信息页面显示
2. "允许用户修改"：用户是否可以在个人中心自行修改该属性值
3. 允许用户修改的属性必须设置为个人中心展示

## 属性使用场景

### 动态角色定义

用户扩展属性可用于定义动态角色的匹配规则，实现基于属性的访问控制(ABAC)。

### 通讯录数据同步

属性映射确保在不同系统间正确同步数据，保持数据一致性。

### 单点登录用户匹配

认证源属性映射用于在单点登录过程中正确识别和关联用户。

### 用户信息自助管理

配置了"允许用户修改"的属性，用户可在服务台自行更新，减轻管理负担。

## 最佳实践
1. **设计合理的属性结构**：在添加扩展属性前，先规划好属性的用途和数据类型
2. **使用规范的属性命名**：遵循一致的命名规则，如`department_code`、`employee_type`
3. **合理设置属性特性**：根据业务需求配置必填、唯一性等特性
4. **定期审核属性配置**：删除不再使用的扩展属性，优化属性结构
5. **谨慎使用表达式**：使用复杂表达式时进行充分测试，确保数据正确转换

## 常见问题

### 为什么基础属性不能删除或修改数据类型？

基础属性是系统核心功能所依赖的字段，修改其数据类型或删除可能导致系统功能异常。

### 新增的扩展属性在哪些地方可以使用？

扩展属性可用于以下场景：
- 用户/部门管理界面的创建和编辑表单
- 角色管理中的动态角色匹配规则
- 通讯录集成和同步的属性映射
- 连接流中的数据处理
- 用户服务台的个人信息页面（如配置了展示）

### 外部系统的属性如何映射到平台的扩展属性？
1. 首先在平台中创建相应的扩展属性
2. 在通讯录集成或认证源的属性映射中，选择源系统属性和对应的平台扩展属性
3. 设置合适的映射策略（仅创建/创建且更新）
4. 保存配置后，系统将根据映射关系同步数据

### 如何批量更新用户的扩展属性？

可通过以下方式批量更新：
1. 使用Excel批量导入功能，在模板中包含扩展属性列
2. 通过通讯录集成同步，配置相应的属性映射关系
3. 使用连接中心的批量数据处理能力
