## 1\. 概述

本文档旨在指导用户如何在专属集成平台中配置使用OAuth2协议的应用作为身份源，使用OAuth2协议应用账号登录专属集成平台。通过配置OAuth2协议认证源，您可以实现以下功能：
- **统一身份登录**：用户可以使用支持OAuth2协议的第三方应用账号直接登录企业系统
- **标准协议支持**：基于业界标准OAuth2协议，实现与多种第三方应用的快速集成
- **安全授权机制**：利用OAuth2的授权流程，保障用户数据安全和隐私
- **无缝单点登录**：用户只需登录一次即可访问所有已授权应用，提升用户体验
- **灵活扩展集成**：可集成各类支持OAuth2的应用和服务，满足多样化认证需求

### 1.1 适用场景
- **多样化身份源集成**：需要集成多种第三方身份源的企业
- **自定义OAuth2应用对接**：企业内部已有基于OAuth2协议的应用系统
- **第三方平台身份复用**：利用用户已有的第三方平台账号快速接入
- **标准化身份管理**：建立基于开放标准的统一身份管理体系
- **SaaS应用快速接入**：需要与多个SaaS应用实现身份互通的场景

## 2\. 前置条件
- 已在OAuth2协议应用中创建应用并配置相应权限
- 拥有专属集成平台（专属集成平台平台）的管理员权限
- 确保网络环境可正常访问OAuth2协议应用服务
- 准备一个测试用的OAuth2协议应用账号，用于验证配置

## 3\. 配置操作步骤

### 3.1 添加OAuth2协议认证源
1. 登录专属集成平台，在上方导航栏选择 "应用中心 \> 企业认证源"，单击 "添加身份源" 选择 "OAuth2标准协议" 身份源

### 3.2 基本信息配置

自定义身份源名称，参考界面提示填写相关参数，配置完成后单击 "确定" 保存。

![image.png](https://alidocs.oss-cn-zhangjiakou.aliyuncs.com/res/Q35O87AjLebWq9Vb/img/2818f231-f967-4efc-bf38-b8a130ec7cf4.png)

| **参数** | **说明** |
|----------|----------|
| 名称 | 自定义认证源名称，建议使用有意义的命名，如"\[第三方应用名\]-OAuth2-SSO" |
| 回调地址 | 系统自动生成的回调地址，需将此地址填写在OAuth2应用的设置中 |
| 用户id映射 | 将OAuth2应用身份源中的用户标识映射到平台的用户标识，建立两者之间的关联关系 |
| 客户端id | 从OAuth2应用获取的客户端标识符 |
| 客户端密钥 | 与客户端id配套的密钥，用于身份验证 |
| 端点地址 | 包括授权端点地址、token端点地址、用户信息端点地址，根据OAuth2应用的实际URL填写 |
| 连接范围 | 建议选择openid、profile、address、phone、email、offline\_access，以获取足够的用户信息 |

### 3.3 属性准备与管理

在配置账号关联映射前，您需要了解OAuth2认证源相关的属性管理机制。这些属性是实现用户身份映射和JIT自动创建的基础。

#### 3.3.1 属性管理基础

导航至"**用户中心**" \> "**属性管理**"，您可以查看和管理系统中的所有用户属性
1. 在属性列表中，您可以看到系统预设属性以及自定义添加的属性
    - 系统预设属性包括：用户名、姓名、电子邮箱等基础字段
    - 自定义属性可根据业务需求添加
2. 属性分类：
    - **基础属性**：系统核心属性，如用户ID、用户名等
    - **扩展属性**：额外信息属性，如头像、昵称等
    - **认证源属性**：与特定认证源相关的属性，如OAuth2应用特有的用户标识

#### 3.3.2 添加新属性

当OAuth2应用提供的字段在系统中没有对应属性时，您需要先添加相应属性：
1. 在属性管理页面，点击"**新增属性**"按钮
2. 填写属性信息：
    - **属性名称**：系统内唯一标识符，如"oauth2*user*id"
    - **显示名称**：在界面上显示的名称，如"OAuth2用户ID"
    - **数据类型**：选择适当的类型，如字符串、数字等
    - **属性类型**：选择相应分类，认证源属性建议选择"基础属性"
    - ![image.png](https://alidocs.oss-cn-zhangjiakou.aliyuncs.com/res/oJGq75KVXQ41rlAK/img/f1b0cd96-8d86-4e8c-91fb-c1168e308ffc.png)

3.点击"保存"完成添加
> **重要提示**：新添加的属性会立即可用于认证源映射配置，无需重启系统。   

### 3.4 账号关联配置

账号关联配置是定义OAuth2应用账号与平台内用户账号的映射关系，决定了用户如何被识别和关联。如没有特殊要求，默认配置即可。

#### 3.4.1 映射流程设置
1. 从左侧下拉菜单中选择OAuth2应用账号属性
    - 常用选项包括：user\_id、sub、email、name等
2. 中间箭头表示映射方向（OAuth2应用 → 平台）
3. 从右侧下拉菜单中选择系统对应的用户属性
    - 常用选项包括：用户名(username)、电子邮箱(email)、显示名称(display\_name)等
    - ![image.png](https://alidocs.oss-cn-zhangjiakou.aliyuncs.com/res/oJGq75KVXQ41rlAK/img/8680b207-8c08-4890-bfdf-a632ab2c6357.png)

4.选择映射方式：

**不映射**：左侧认证源的该属性不会映射给系统用户

**仅创建**：只在用户首次登录创建时映射，后续不更新

**创建且更新**：用户创建时映射，并在每次登录时更新该属性

![image.png](https://alidocs.oss-cn-zhangjiakou.aliyuncs.com/res/oJGq75KVXQ41rlAK/img/14f755d4-85a5-44d4-9f55-efbf89cf5190.png)

#### 3.4.2 唯一字段匹配

为了正确识别和关联用户，至少需要配置一个**唯一字段匹配**规则：
1. 选择OAuth2应用中的唯一标识字段，通常是sub、user\_id或email
2. 映射到系统中的唯一标识字段，如用户名(username)或电子邮箱(email)
3. 选择"创建且更新"模式
4. 系统会根据此映射在登录时查找匹配用户
> **技术说明**：唯一字段匹配是系统识别已有用户的依据。如果找不到匹配用户且JIT开启，则会创建新用户。   

#### 3.4.3 推荐的映射关系

| OAuth2属性 | 平台属性 | 映射方式 | 说明 |
|------------|------------|------------|------|
| sub | oauth2*user*id | 创建且更新 | OAuth2应用用户唯一标识 |
| email | email | 创建且更新 | 用户邮箱地址 |
| name | display\_name | 创建且更新 | 用户显示名称 |
| given\_name | first\_name | 创建且更新 | 用户名 |
| family\_name | last\_name | 创建且更新 | 用户姓 |
> **数据类型匹配**：确保源属性和目标属性的数据类型兼容。例如，数字型字段不应映射到日期型字段，否则可能导致数据异常。   

### 3.5 JIT配置详解

JIT (Just-in-Time) 配置允许用户首次通过OAuth2应用登录时，自动在系统中创建对应账号，是提高用户体验的关键功能。

#### 3.5.1 基本JIT设置
1. 找到"**JIT**"部分
2. 将开关切换为"**已开启**"状态
> **说明**：开启JIT后，当用户通过OAuth2应用认证登录时，若系统中找不到匹配的用户账号，会根据映射规则自动创建新用户。若关闭此配置，则会将认证中不在平台的用户显示为认证不通过。   

#### 3.5.2 更新策略配置

JIT更新策略决定了已存在用户的属性是否在后续登录中更新：
1. 配置"**更新已有用户的属性**"选项：
    - **关闭**：用户首次创建后，后续登录不会更新任何用户属性，保持用户信息稳定
    - **开启**：每次用户登录时，系统会根据映射规则更新用户属性，保持与OAuth2应用数据的同步
2. 更新策略选择考虑因素：
    - **数据主导权**：谁是用户信息的主要维护方？如果是OAuth2应用，建议开启更新
    - **变更频率**：用户属性变更的频率高低，频繁变更建议开启更新
    - **数据敏感性**：是否包含手动维护的敏感配置，敏感配置建议关闭自动更新
> **技巧**：如果您在OAuth2应用中维护了完整的用户信息并希望自动同步到系统，建议开启更新功能。   

#### 3.5.3 用户组配置

配置新用户通过JIT创建后的用户组分配：
- **指定用户组**：通过JIT创建的用户会在指定的用户组创建用户
- **不指定用户组**：通过JIT创建的用户会创建在默认组中
> **安全提示**：默认用户组的权限配置非常重要，建议遵循最小权限原则，确保自动创建的用户不会获得过高权限。   

### 3.6 完成配置

设置完成后单击"确定"保存所有配置。

## 5\. 测试与验证

### 5.1 基础功能测试

配置完成后，请按以下步骤验证认证源是否正常工作：
1. 退出当前管理员账号
2. 在登录页面查看是否出现"**使用\[OAuth2应用名称\]账号登录**"选项
3. 点击该选项，浏览器会重定向到OAuth2应用的登录页面
4. 使用测试账号完成登录
5. 验证是否成功登录并跳转到系统首页

### 5.2 用户映射测试

确认用户信息是否正确映射：
1. 使用管理员账号登录系统
2. 导航至"用户管理"页面
3. 查找刚才通过OAuth2应用登录的测试用户
4. 验证以下信息是否正确：
    - 用户基本信息（姓名、邮箱等）
    - 所属用户组（是否为JIT配置中指定的用户组）
    - 用户状态（是否为激活状态）
    - 用户属性（是否按照映射规则正确设置）

## 6\. 常见问题与解决方案

### 6.1 认证失败

**问题描述**：使用OAuth2应用账号登录时出现认证失败错误。

**可能原因**：
- 客户端ID或客户端密钥填写错误
- 回调地址未正确配置到OAuth2应用中
- 网络连接问题或防火墙限制
- OAuth2应用权限不足或配置有误

**解决方案**：
- 核对客户端ID和客户端密钥是否正确
- 确认回调地址已添加到OAuth2应用的设置中
- 检查网络连接，确保可以正常访问OAuth2应用服务
- 确认OAuth2应用已获得足够的权限和正确的配置

### 6.2 无法自动创建用户

**问题描述**：登录成功但系统未自动创建用户。

**可能原因**：
- JIT功能未开启
- 映射规则配置不正确或不完整
- 唯一字段匹配规则存在问题

**解决方案**：
- 确认JIT配置已开启
- 检查账号关联映射规则配置是否准确（至少需要配置一个唯一标识字段）
- 验证OAuth2应用返回的用户数据是否完整（必要字段是否有值）

### 6.3 用户信息不完整或不准确

**问题描述**：自动创建的用户缺少部分信息或信息不准确。

**可能原因**：
- 映射规则不完整或映射关系错误
- OAuth2应用返回的用户数据不完整
- 连接范围（scope）设置不足

**解决方案**：
- 确认连接范围选择了所有必要的选项（openid、profile、email等）
- 添加或修改账号关联映射规则
- 考虑启用"更新已有用户的属性"选项

## 7\. 安全建议
1. **使用HTTPS协议**：确保所有认证相关的交互都使用HTTPS协议，保护数据传输安全
2. **保护认证凭据**：妥善保管客户端ID和客户端密钥，避免泄露
3. **最小权限原则**：为OAuth2应用分配最小必要权限，仅获取必要的用户信息
4. **定期更新密钥**：定期更新客户端密钥，提高系统安全性
5. **监控异常活动**：设置异常登录检测，及时发现潜在安全风险
