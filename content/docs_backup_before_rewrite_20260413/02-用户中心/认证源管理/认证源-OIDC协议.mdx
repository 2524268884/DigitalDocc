## 1\. 概述

本文档旨在指导用户如何在专属集成平台中配置使用OIDC协议的应用作为身份源，使用OIDC协议应用账号登录专属集成平台。通过配置OIDC协议认证源，您可以实现以下功能：
- **统一身份登录**：用户可以使用支持OIDC协议的第三方应用账号直接登录企业系统
- **标准协议集成**：基于OpenID Connect标准，实现与多种支持OIDC的应用系统快速集成
- **安全认证机制**：利用OIDC的JWT令牌和ID令牌，确保身份验证的安全性和可靠性
- **无缝单点登录**：用户只需登录一次即可访问所有已授权应用，提升用户体验
- **用户信息交换**：标准化获取用户身份信息，简化用户信息同步流程

### 1.1 适用场景
- **企业身份统一管理**：需要将多种身份源统一纳入管理的企业
- **云原生应用集成**：与支持OIDC的云原生应用实现无缝集成
- **跨组织身份互通**：不同组织之间的身份系统互通和共享
- **多云环境身份管理**：跨多个云平台的统一身份认证
- **第三方应用快速接入**：需要快速接入支持OIDC的第三方应用和服务

## 2\. 前置条件
- 已在OIDC协议应用中创建应用并配置相应权限
- 拥有专属集成平台（专属集成平台平台）的管理员权限
- 确保网络环境可正常访问OIDC协议应用服务
- 准备一个测试用的OIDC协议应用账号，用于验证配置

## 3\. 配置操作步骤

### 3.1 添加OIDC协议认证源

登录专属集成平台，在上方导航栏选择 "应用中心 \> 企业认证源"，单击 "添加身份源" 选择 "OIDC标准协议" 身份源

### 3.2 基本信息配置

自定义身份源名称，参考界面提示填写相关参数，配置完成后单击 "确定" 保存。

![image.png](https://alidocs.oss-cn-zhangjiakou.aliyuncs.com/res/4j6OJdL93jjKn3p8/img/5098708c-ee45-46f5-8173-1fdff709a23b.png)

| **参数** | **说明** |
|----------|----------|
| 名称 | 自定义认证源名称，建议使用有意义的命名，如"\[第三方应用名\]-OIDC-SSO" |
| 用户id映射 | 将OIDC应用身份源中的用户标识映射到平台的用户标识，建立两者之间的关联关系 |
| ClientID | 从OIDC应用获取的客户端标识符 |
| ClientSecret | 与ClientID配套的密钥，用于身份验证 |
| RedirectURI | 系统自动生成的回调地址，需将此地址填写在OIDC应用的设置中 |
| 客户端认证方式 | 根据OIDC应用的要求选择Client Secret Basic或Client Secret Post |
| Issuer | OIDC身份提供者的标识符URL，Authorization Endpoint、Token Endpoint、JWKs Endpoint会自动生成 |
| Scope | 建议选择openid、profile、address、phone、email、offline\_access，以获取足够的用户信息 |
| 签名算法 | 选择RS256或HS256，需与OIDC服务提供方使用的算法一致 |

### 3.3 属性准备与管理

在配置账号关联映射前，您需要了解OIDC认证源相关的属性管理机制。这些属性是实现用户身份映射和JIT自动创建的基础。

#### 3.3.1 属性管理基础

导航至"**用户中心**" \> "**属性管理**"，您可以查看和管理系统中的所有用户属性
1. 在属性列表中，您可以看到系统预设属性以及自定义添加的属性
    - 系统预设属性包括：用户名、姓名、电子邮箱等基础字段
    - 自定义属性可根据业务需求添加
2. 属性分类：
    - **基础属性**：系统核心属性，如用户ID、用户名等
    - **扩展属性**：额外信息属性，如头像、昵称等
    - **认证源属性**：与特定认证源相关的属性，如OIDC应用特有的用户标识

#### 3.3.2 添加新属性

当OIDC应用提供的字段在系统中没有对应属性时，您需要先添加相应属性：
1. 在属性管理页面，点击"**新增属性**"按钮
2. 填写属性信息：
    - **属性名称**：系统内唯一标识符，如"oidc\_sub"
    - **显示名称**：在界面上显示的名称，如"OIDC主题标识符"
    - **数据类型**：选择适当的类型，如字符串、数字等
    - **属性类型**：选择相应分类，认证源属性建议选择"基础属性"
    - ![image.png](https://alidocs.oss-cn-zhangjiakou.aliyuncs.com/res/oJGq75KVXQ41rlAK/img/f1b0cd96-8d86-4e8c-91fb-c1168e308ffc.png)

3.点击"保存"完成添加
> **重要提示**：新添加的属性会立即可用于认证源映射配置，无需重启系统。   

### 3.4 账号关联配置

账号关联配置是定义OIDC应用账号与平台内用户账号的映射关系，决定了用户如何被识别和关联。如没有特殊要求，默认配置即可。

#### 3.4.1 映射流程设置
1. 从左侧下拉菜单中选择OIDC应用账号属性
    - 常用选项包括：sub、email、name、given*name、family*name等
2. 中间箭头表示映射方向（OIDC应用 → 平台）
3. 从右侧下拉菜单中选择系统对应的用户属性
    - 常用选项包括：用户名(username)、电子邮箱(email)、显示名称(display\_name)等
    - ![image.png](https://alidocs.oss-cn-zhangjiakou.aliyuncs.com/res/oJGq75KVXQ41rlAK/img/8680b207-8c08-4890-bfdf-a632ab2c6357.png)

4.选择映射方式：
    - **不映射**：左侧认证源的该属性不会映射给系统用户
    - **仅创建**：只在用户首次登录创建时映射，后续不更新
    - **创建且更新**：用户创建时映射，并在每次登录时更新该属性

![image.png](https://alidocs.oss-cn-zhangjiakou.aliyuncs.com/res/oJGq75KVXQ41rlAK/img/14f755d4-85a5-44d4-9f55-efbf89cf5190.png)

#### 3.4.2 唯一字段匹配

为了正确识别和关联用户，至少需要配置一个**唯一字段匹配**规则：
1. 选择OIDC应用中的唯一标识字段，通常是sub（主题标识符）
2. 映射到系统中的唯一标识字段，如用户名(username)或专用字段(oidc\_sub)
3. 选择"创建且更新"模式
4. 系统会根据此映射在登录时查找匹配用户
> **技术说明**：唯一字段匹配是系统识别已有用户的依据。如果找不到匹配用户且JIT开启，则会创建新用户。   

#### 3.4.3 推荐的映射关系

| OIDC属性 | 平台属性 | 映射方式 | 说明 |
|----------|------------|------------|------|
| sub | oidc\_sub | 创建且更新 | OIDC主题标识符，唯一标识用户 |
| email | email | 创建且更新 | 用户邮箱地址 |
| name | display\_name | 创建且更新 | 用户显示名称 |
| given\_name | first\_name | 创建且更新 | 用户名 |
| family\_name | last\_name | 创建且更新 | 用户姓 |
| phone\_number | phone\_number | 创建且更新 | 用户电话号码 |
> **数据类型匹配**：确保源属性和目标属性的数据类型兼容。例如，数字型字段不应映射到日期型字段，否则可能导致数据异常。   

### 3.5 JIT配置详解

JIT (Just-in-Time) 配置允许用户首次通过OIDC应用登录时，自动在系统中创建对应账号，是提高用户体验的关键功能。

#### 3.5.1 基本JIT设置
1. 找到"**JIT**"部分
2. 将开关切换为"**已开启**"状态
> **说明**：开启JIT后，当用户通过OIDC应用认证登录时，若系统中找不到匹配的用户账号，会根据映射规则自动创建新用户。若关闭此配置，则会将认证中不在平台的用户显示为认证不通过。   

#### 3.5.2 更新策略配置

JIT更新策略决定了已存在用户的属性是否在后续登录中更新：
1. 配置"**更新已有用户的属性**"选项：
    - **关闭**：用户首次创建后，后续登录不会更新任何用户属性，保持用户信息稳定
    - **开启**：每次用户登录时，系统会根据映射规则更新用户属性，保持与OIDC应用数据的同步
2. 更新策略选择考虑因素：
    - **数据主导权**：谁是用户信息的主要维护方？如果是OIDC应用，建议开启更新
    - **变更频率**：用户属性变更的频率高低，频繁变更建议开启更新
    - **数据敏感性**：是否包含手动维护的敏感配置，敏感配置建议关闭自动更新
> **技巧**：如果您在OIDC应用中维护了完整的用户信息并希望自动同步到系统，建议开启更新功能。   

#### 3.5.3 用户组配置

配置新用户通过JIT创建后的用户组分配：
- **指定用户组**：通过JIT创建的用户会在指定的用户组创建用户
- **不指定用户组**：通过JIT创建的用户会创建在默认组中
> **安全提示**：默认用户组的权限配置非常重要，建议遵循最小权限原则，确保自动创建的用户不会获得过高权限。   

### 3.6 完成配置

设置完成后单击"确定"保存所有配置。

## 5\. 测试与验证

### 5.1 基础功能测试

配置完成后，请按以下步骤验证认证源是否正常工作：
1. 退出当前管理员账号
2. 在登录页面查看是否出现"**使用\[OIDC应用名称\]账号登录**"选项
3. 点击该选项，浏览器会重定向到OIDC应用的登录页面
4. 使用测试账号完成登录
5. 验证是否成功登录并跳转到系统首页

### 5.2 用户映射测试

确认用户信息是否正确映射：
1. 使用管理员账号登录系统
2. 导航至"用户管理"页面
3. 查找刚才通过OIDC应用登录的测试用户
4. 验证以下信息是否正确：
    - 用户基本信息（姓名、邮箱等）
    - 所属用户组（是否为JIT配置中指定的用户组）
    - 用户状态（是否为激活状态）
    - 用户属性（是否按照映射规则正确设置）

## 6\. 常见问题与解决方案

### 6.1 认证失败

**问题描述**：使用OIDC应用账号登录时出现认证失败错误。

**可能原因**：
- ClientID或ClientSecret填写错误
- RedirectURI未正确配置到OIDC应用中
- 网络连接问题或防火墙限制
- OIDC应用配置有误

**解决方案**：
- 核对ClientID和ClientSecret是否正确
- 确认RedirectURI已添加到OIDC应用的设置中
- 检查网络连接，确保可以正常访问OIDC应用服务
- 验证OIDC应用的Issue、签名算法等配置是否正确

### 6.2 无法自动创建用户

**问题描述**：登录成功但系统未自动创建用户。

**可能原因**：
- JIT功能未开启
- 映射规则配置不正确或不完整
- 唯一字段匹配规则存在问题

**解决方案**：
- 确认JIT配置已开启
- 检查账号关联映射规则配置是否准确（至少需要配置一个唯一标识字段）
- 验证OIDC应用返回的用户数据是否完整（必要字段是否有值）

### 6.3 用户信息不完整或不准确

**问题描述**：自动创建的用户缺少部分信息或信息不准确。

**可能原因**：
- 映射规则不完整或映射关系错误
- OIDC应用返回的用户数据不完整
- Scope范围设置不足

**解决方案**：
- 确认Scope选择了所有必要的选项（openid、profile、email等）
- 添加或修改账号关联映射规则
- 考虑启用"更新已有用户的属性"选项

### 6.4 签名验证失败

**问题描述**：登录过程中出现签名验证失败错误。

**可能原因**：
- 签名算法选择错误
- OIDC应用的密钥不匹配
- JWT令牌格式不正确

**解决方案**：
- 确认签名算法选择正确（RS256或HS256）
- 验证OIDC应用的密钥配置
- 检查OIDC应用的JWT令牌配置

## 7\. 安全建议
1. **使用HTTPS协议**：确保所有认证相关的交互都使用HTTPS协议，保护数据传输安全
2. **保护认证凭据**：妥善保管ClientID和ClientSecret，避免泄露
3. **最小权限原则**：为OIDC应用分配最小必要权限，仅获取必要的用户信息
4. **使用强加密算法**：优先选择RS256等非对称加密算法，提高安全性
5. **定期更新密钥**：定期更新ClientSecret，提高系统安全性
6. **监控异常活动**：设置异常登录检测，及时发现潜在安全风险

## 8\. 附录

### 8.1 术语表

| 术语 | 描述 |
|------|------|
| OIDC | OpenID Connect的缩写，基于OAuth 2.0的身份认证标准 |
| JWT | JSON Web Token，用于安全传递声明的开放标准 |
| ID Token | 包含用户身份信息的JWT令牌 |
| ClientID | 标识OIDC应用的唯一标识符 |
| ClientSecret | 与ClientID配套使用的密钥 |
| RedirectURI | 认证成功后OIDC应用重定向用户的地址 |
| Issuer | 身份提供者的标识符 |
| JIT | Just-in-Time的缩写，指用户首次登录时自动创建账号的功能 |
| Scope | 指定应用程序请求的权限范围 |
| RS256/HS256 | JWT签名使用的算法，RS256为非对称加密，HS256为对称加密 |
